Summary:

Adam Ivansky, the founder of Tiramisu Wallet, has proposed a Bitcoin Improvement Proposal (BIP) to structure the metadata of assets traded on TARO Protocol. The BIP for TARO can be found on Github, but it does not mention the format of metadata for the assets. Based on his observations, Ivansky suggests that the metadata should include fields such as ticker, asset name, description, image representation, and who minted the asset. He proposes that the metadata be structured as JSON stored as a string. The proposed structure includes fields such as ticker, type, description, data, hash_data, external_url, attributes, and minter_info.The Tiramisu wallet on testnet currently uses some of these JSON fields. Ivansky loosely based his proposal on OpenSea's metadata standards but included actual image data in the metadata. Since this data is not written directly to the blockchain, there is less incentive to make it small. Therefore, he requests feedback on his proposal from the mailing list. The agreement upon the format of asset metadata is necessary to trade NFTs, stablecoins, and synthetic assets.
