Summary:

The author of the post is experimenting with the bitcoin protocol using GNU guile scheme and has found inconsistencies with the documentation on https://en.bitcoin.it/wiki/Getblocktemplate regarding the creation of templates for the Merkle root. The author believes that a coinbase transaction should have the transaction data concatenated before creating the Merkle root, but getblocktemplate does not have a json cointbasetxn field, leaving the author unsure how to create a coinbase transaction without it. The author has provided links to a test template response data and a modified version of the merkle python reference script found on the wiki page, which enabled them to create a Merkle root hash but minus the coinbase transaction. Mike Rosset, a developer, posted a message on the bitcoin-dev mailing list seeking help with the inconsistencies he encountered while experimenting with Bitcoin using GNU Guile Scheme. He was unable to create a coinbase transaction without the json cointbasetxn field in getblocktemplate and requested assistance in acquiring the coinbase transaction and ensuring that his Python and Guile Merkle roots matched. Mike provided a test template response data and a modified version of the merkle python reference script found on the wiki page, which allowed him to replicate a Merkle root hash, except it did not include the coinbase transaction.
