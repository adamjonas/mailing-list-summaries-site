Summary:

The author proposes an idea for a hard wallet that ensures funds are unseizable through proof of creation. The concept is based on a 2-of-2 combination of private keys generated by the device and from a key file provided by the user. Both keys must be combined to spend from the resulting Bitcoin address. The device can be configured with random entropy separately from the key file to reduce the risk of malware accessing both. The author provides a well-defined concept and a link to a Github directory that details the idea. The necessary elements to execute the project are believed to already exist. The author also includes a PGP signature for security purposes. The author does not suggest the use of brainwallets, even for key files.
