Summary:

The email correspondence discusses various topics related to Bitcoin transactions and security. The first topic is about splitting multi-sig and multi-factor authentication into two separate categories, with wallets using multi-factor authentication before authorizing access to a wallet and before signing a transaction. The discussion also includes the suggestion that wallets should not create two keys on a single host or device, and provide a way to import external public keys which can be used as part of a P2SH address.Another conversation revolves around Bitcoin address re-use problems, proposing the solution of generating fresh addresses from public extended keys to enhance users' privacy and benefit non-merchant use of Payment Protocol. However, there is a challenge of how to push the extended public key down to the wallet itself, which could be facilitated by services or direct communication via PGP-encrypted files in Payment Protocol format through various channels.The debate over whether or not to sign every commit is also discussed extensively. While Linus Torvalds advises against signing each commit, Peter Todd argues that it is necessary for the Bitcoin community to use per-commit signatures due to the reliance on third-party sites like Github. Todd suggests focusing on the messaging layer and using OpenPGP and SSL certificate authorities rather than creating new identity systems that can increase the risk of MITM attacks. Finally, Drak mentions that Linus's recommendation for signing tags with `git tag -s` should probably be incorporated into the spec as a MUST.
