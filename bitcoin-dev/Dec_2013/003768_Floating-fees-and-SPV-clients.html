Summary:

In this email conversation, Gavin Andresen proposes the addition of an optional uint64 allowfee tag number to allow users to pay Bitcoin network transaction fees. A wallet implementation must not reduce the amount paid for fees more than allowfee, and transaction fees must be equal to or greater than the amount reduced. However, they discuss changing the name from "allow" to "min_fee" and suggest adding the value to the total value displayed to the user rather than reducing the output value. Additionally, there is a discussion about allowing users to pay more in fees if needed and shortening the BIP72 request code to save characters in QR codes. While some concern is expressed about senders not caring about confirmations, the consensus seems to be in favor of these proposed changes.
