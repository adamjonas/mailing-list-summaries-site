Summary:

Harry and David have suggested to the crypto group to add support for *secp256k1* in the browser Named Curve dictionary. This curve has been increasingly deployed in the last year, particularly in bitcoin and other cryptocurrencies. Currently, the API only exposes what exists in NSS, but adding support for this curve could benefit many people who already work with cryptocurrencies in the browser. The WebCryptoAPI's EcKeyGenParams dictionary's NamedCurve enum presently includes NIST recommended curves P-256, P-384, and P-521. For more information on the Koblitz curve used by bitcoin, a link to a bitcointalk.org forum post is provided.
