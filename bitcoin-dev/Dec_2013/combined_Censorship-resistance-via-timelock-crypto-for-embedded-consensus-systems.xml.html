Summary:

Embedded consensus systems like Mastercoin face a risk of censorship through the use of blacklists. However, steganography can be used to hide data in transactions using unused pubkeys or ECC math. Timelock cryptography is a solution to this problem, by hiding data in the transaction and setting a random, publicly-known value as the key. Efforts should be made to limit possible transactions to reduce computation required and consider how the data will be validated. Participants in the system should share keys to reduce effort. This scheme can also be used for censorship-resistant voting, providing strong resistance with strong consensus in a fixed amount of time.
