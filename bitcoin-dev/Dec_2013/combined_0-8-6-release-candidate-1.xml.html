Summary:

On December 9th, 2013, Bitcoin's development and release process were discussed. Drak pointed out that the bitcoin.org website was still pointing to version 0.8.5 for downloads and suggested it should be updated. Roy Badami responded by suggesting that perhaps version 0.8.6 had not yet been released or announced, and that it would make sense not to promote release candidates on bitcoin.org. However, it was then mentioned that version 0.8.6 had indeed been released and was being discussed on bitcointalk and reddit.In an email conversation between Gregory Maxwell and Drak, Drak claimed that Bitcoin version 0.8.6 was already available for download at sourceforge.net. However, Maxwell responded saying that it had not been officially released yet and was only queued for announcement. He further explained that they were waiting for another independent gitian build before sending out the announcement. Drak expressed concern about the release workflow, stating that there needed to be a better process in place as people were already downloading the 0.8.6 files and creating buzz around it within the community.On December 5th, 2013, Gavin Andresen sent an email to the Bitcoin-development mailing list requesting help in testing the 0.8.6 release candidate 1, which was available for download from SourceForge. Warren had previously found a tiny error in 0.8.6rc1 where the leveldb subtree merge was done incorrectly, leaving an errant db/ directory in the base of bitcoin instead of src/leveldb. After testing macos leveldb parts for six days, there have been no complaints of new corruption from OMG and LTC users. Therefore, it is time to release version 0.8.6. However, on December 8, 2013, Warren Togami Jr. found a tiny error in version 0.8.6rc1. The leveldb subtree merge was done incorrectly, leaving an errant db/ directory in the base of bitcoin instead of src/leveldb. Gavin Andresen noted that this bug was not a showstopper and given that there have been hundreds of downloads with no reports of insanity, they decided to tag v0.8.6 and ship it.There were also discussions about the need for an official announcement regarding Bitcoin's development and release process. Wladimir confirmed that a particular post on Bitcointalk.org was indeed an official announcement. However, Drak suggested that no files should be placed online unless they were the official release to avoid any potential issues, while Wladimir argued that once files are uploaded, they become the official release. Jeff Garzik confirmed an official announcement on Bitcointalk, and Roy Badami suggested announcements should be posted on both the mailing list and Bitcointalk.org.Finally, there were suggestions to add a list of places to announce new releases to the release process to avoid confusion and ensure proper communication within the community. It is necessary to ensure that users are able to access the latest version of the software, and the responsibility of maintaining and updating the bitcoin.org website was unclear at the time.
