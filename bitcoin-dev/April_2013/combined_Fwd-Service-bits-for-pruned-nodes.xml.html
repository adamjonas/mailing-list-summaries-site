The development of a roadmap for block downloading is underway and one idea being considered is the implementation of an HTTP/HTTPS protocol for block downloading. This new protocol could not only improve accessibility by allowing web proxies to cache blocks, but also provide the ability for resumeable downloads. If implemented, the overall experience of block downloading would be enhanced and made more user-friendly.
