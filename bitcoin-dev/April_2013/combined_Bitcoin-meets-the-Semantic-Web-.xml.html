Summary:

In April of 2013, Harald Schilly and Melvin Carvalho discussed the creation of a "vocabulary" for Bitcoin. Schilly suggested searching existing databases such as OKFN for established vocabularies for payments instead of reinventing it. He provided links to various payment protocols like schema.org's PaymentMethod and GoodRelations' PayPal ontology. While Carvalho was familiar with most of this work and planned to reuse as much as possible, some terms would be Bitcoin specific. During this discussion, Carvalho suggested using bitcoin.org to host schemas for Bitcoin, but noted that it lacked https and Github does not allow for setting mime types. Another participant in the conversation suggested using a subdomain such as schema.bitcoin.org for effective management and security of the files. Melvin Carvalho is currently involved in transferring crypto currencies like Bitcoin to the semantic web in order to make pages machine-readable, allowing for innovation and dissemination of knowledge to a wider audience. The first step towards this goal is to create a vocabulary specific to Bitcoin, similar to a dictionary, that can be written in a machine-readable format known as RDF. Melvin has made a start on creating the vocabulary which is hosted at https://w3id.org/cc. The model will be something like URI -> crypto-currency-address -> bitcoin-address, and those at w3id.org have offered to use their permanent identifier switchboard, then redirect to a locked-down vocabulary. Feedback or help is welcome as Melvin extends this over time.
