Summary:

Jule.Adka has proposed a new approach to address scalability and privacy issues associated with Bitcoin transactions. The proposal involves the use of Mekelized Output Set and the p2mos (pay to Mekelized Output Set) standard, which only specifies a Markle root instead of listing all the output set. This reduces the amount of data created through transactions and solves scalability problems. Additionally, this approach solves privacy issues by keeping information about the outputs private until they are spent. The proposal saves space in blocks and makes transactions smaller, decreasing the data used for SPV nodes. However, it requires more coordination between wallets and extra bandwidth for downloading the witness data. Retro compatibility is possible but requires implementation as a soft-fork. Old nodes that don’t follow the new consensus rule can accept this kind of transaction if it’s made as anyone can spend in the current consensus, but at a second spend, the node will interpret it as a double spend, hence invalidating it. The proposal requires more coordination between wallets, is a bit harder to compute for a validator and would require some extra bandwidth for downloading the witness data.
