The discussion on the Bitcoin-dev mailing list regarding the security of a proposal to use Galois Field (GF) multiplication to improve Bitcoin's signature scheme continues. Gregory Maxwell expressed concerns about the proposal's security against partial share leakage attacks. He provided an example to support his argument and suggested that using GF(2^256+n) might be a better alternative.
