Summary:

Aymeric Vitte asked about the presence of pubkey in a specific transaction scriptSig and was informed that it was present in the scriptPubKey of vout 0 of another transaction, which it is spending. He also highlighted the fact that pubkey is now present even for standard p2pkh transactions whereas this was not the case previously. However, he did not receive an answer regarding the reason behind this change. Артём Литвинович asked about the rationale behind having both public key and signature in Segwit witness. The question was whether it was done for security, performance, privacy or something else. It was explained that leaving out the public key would have saved 33 bytes per signature but it is slow to verify, incompatible with batch validation, doesn't save space if hashing isn't used, and is potentially patent encumbered.
