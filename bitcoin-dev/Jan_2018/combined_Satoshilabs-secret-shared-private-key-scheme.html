The Bitcoin developer community has been engaging in discussions on the security of cryptocurrency wallets over email. One topic of discussion was the use of "plausible deniability" schemes in devices like Trezor, which allow users to have multiple passphrases leading to different wallets, one of which is hidden. However, there are concerns about the weakness of the key derivation function used in the scheme and the potential for passphrases to be easily brute-forced, possibly revealing attempts to cover up funds. Another topic discussed was the proposed improvement of Bitcoin's signature scheme using Galois Field multiplication and the use of error-correcting codes over hash functions for checking integrity. There were also debates over the use of checksums and whether a stronger cryptographic function would be ideal. Some members suggested using blind host-delegated Key Derivation Functions (KDFs) that run on a user's computer instead of a hardware wallet to improve security. Discussions around the design and implementation of schemes such as BIP 39 and SLIP39, which provide security for brainwallets, were also held.Discussions on topics like Shamir Secret Sharing (SSS), entropy argument, and the lack of versioning in some proposed schemes were also raised. The importance of ensuring the security and integrity of cryptographic schemes and measures was emphasized, as well as the need to consider potential vulnerabilities and attacks when developing new schemes.In another conversation, hardware wallet security was discussed, specifically regarding the strength of key derivation functions and the possibility of brute force attacks. The "plausible deniability" feature of Trezor wallets was also debated, with concerns being raised about the weakness of the key derivation function used in the feature. However, some argued that plausible deniability could be useful in certain situations where people might need to conceal information from border security or other authorities. Travelers were warned not to use plausible deniability schemes in situations where they could be jailed for lying to border security.The Shamir Secret Scheme Splitting was also discussed, with concerns about interoperability, versioning, weak key derivation function, and the 16-bit checksum based on sha2 that could allow for input errors. The SSS protocol needs better advice on generating additional coefficients randomly to ensure security. The current proposal for security measures against partial share leakage has been criticized as being insecure. Overall, the discussions highlight ongoing debates and concerns within the cryptography and security community, highlighting the importance of continued research and development in this field.
