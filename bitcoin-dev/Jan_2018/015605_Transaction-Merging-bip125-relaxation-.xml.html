Summary:

Rhavar via bitcoin-dev proposed an idea to allow users to merge multiple unconfirmed transactions by stripping extraneous inputs and change. However, this is currently not possible because of the bip125 rule which states that the replacement transaction pays an absolute fee of at least the sum paid by the original transactions. If the rule was removed, it would introduce DoS vectors and allow users to push others' transactions out of the mempool. Rhavar also faced difficulty in tracking merged transactions and suggested a possible solution to only consider a transaction replaceable if it has change. However, this solution is limited and complex. Peter Todd questioned Rhavar's statement that most transactions don't have change.
