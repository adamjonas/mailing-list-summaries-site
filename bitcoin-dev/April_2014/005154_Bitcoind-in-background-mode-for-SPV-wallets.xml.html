Summary:

The Bitcoin core can activate pruning to support a full node in 4GB or so, with enough space to store the UTXO of about 350MB and a few gigs for blocks to serve out. Users will not be hammered on space constraints as SPV clients of the border router will build their own archive and indices based on their interest of the chain, which means that the border router core does not need to store or process anything not needed for consensus. This makes the memory or disk footprint of the core as low as an optimal storage of UTXO. SPV is a sufficient API to a trusted node to build sophisticated features not offered by the core. Feedback from SPV wallet developers regarding how much disk usage agility they think is required is being sought.
