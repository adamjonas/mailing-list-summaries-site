Summary:

The use of Bitcoin addresses for identification and authentication purposes has been a topic of discussion in the Bitcoin community since 2014. While some have proposed using Bitcoin addresses as a persistent identity key, others question the potential risks and suggest alternative solutions such as SSL client certificates. The development of BitID has progressed with a working wallet prototype based on Android Bitcoin Wallet, which offers fields for decentralized 2FA and "pay as guest" checkout. However, some believe that using Bitcoin addresses for authentication may be confusing and potentially linking funds with identity is not ideal. Eric Larchevêque has written a draft BIP for an authentication protocol based on Bitcoin public address, which could improve the Bitcoin ecosystem by simplifying sign-ups and logins to services and applications. However, classical password authentication is insecure and can be solved with public key cryptography. While classical password authentication is insecure, public key cryptography can solve the issue, and managing private keys securely is already being addressed in the Bitcoin ecosystem. Eric suggests using client certificates instead of Bitcoin addresses since they leave no trace in the blockchain and are already widely supported by browsers and web servers. However, the use of client certs is not more widespread due to issues such as poor UI for managing them. In an email conversation, Mike Hearn expresses doubt that Bitcoin addresses can replace passwords as a means of authentication, while Eric argues that two-factor authentication (2FA) does not remove the need for a password but adds another layer of security. The discussion revolves around making the sign-up and login process for Bitcoin ecosystem websites and apps more user-friendly, with the author emphasizing that wallets are a crucial part of this equation since everyone has one and should have already taken steps to secure their keys. Additionally, using a Bitcoin address for identification purposes on a Bitcoin-related service makes sense.There have been discussions among Bitcoin developers about the use of Bitcoin addresses for authentication. While some believe it could simplify the process and improve the Bitcoin ecosystem, others argue that client-side certificates are a better solution due to issues with safely storing and backing up secrets associated with using Bitcoin addresses. One proposed solution is an authentication protocol based on a user's Bitcoin public address, which would allow them to prove they control a specific Bitcoin address by signing a challenge. The protocol aims to simplify sign-ups and logins to services and applications, but some argue that SSL client certificates already solve the problem. The use of TREZOR for web authentication purposes has also been discussed, with suggestions to avoid using Bitcoin addresses directly and to generate private/public keys from the Bitcoin seed/private key to prevent leaking Bitcoin-related data to websites. Overall, while finding a solution that is better than a password but just as convenient would be fantastic, Bitcoin addresses are unlikely to be such a solution. It is important to consider the unique qualities and value of a product before making assumptions about its reception, and to avoid mixing payment and identification to prevent confusion between the two. Ultimately, the decision to use a cryptographic token over a password is a matter of personal preference and security measures already in place for Bitcoin wallets can be used for authentication purposes. In the meantime, proposing bounties for wallets to support a new protocol natively may be an effective way to encourage adoption.
