Summary:

The atomic cross-chain system requires hash-locked outputs, meaning a user must provide x corresponding to hash(x) to spend an output. One participant is required to provide the secret number, which then allows the other participant to use that secret number to spend an output on the other chain, linking the two chains together. This proposal suggests adding this as an add-on to standard transactions by checking the hash and then running the transaction regularly. This script form can be embedded in a P2SH output to gain all benefits, and if there is agreement, a reference implementation will be coded.
