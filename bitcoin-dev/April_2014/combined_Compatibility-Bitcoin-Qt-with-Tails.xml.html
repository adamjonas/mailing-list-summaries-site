Summary:

In April 2014, Kristov Atlas reported an issue with a binary built by Wladimir where the GUI segfaulted after choosing the default datadir. Wladimir fixed the issue and made a new test build available for download. On May 2, 2014, Kristov confirmed that a dev binary runs smoothly in Tails v1.0, which was tested by him. The usual Linux binary would be compatible when incorporated into the next release of Bitcoin Core. However, they may add a binary linked to Qt 5.x for newer distributions at some point in time. In an email exchange between Wladimir and Kristov, Wladimir mentioned that he has modified the gitian build to work with Qt 4.6 instead of Qt 4.8. In another email conversation between Kristov Atlas and Warren Togami Jr., Kristov shared that he had modified the gitian build to use Qt 4.6 instead of 4.8, with the intention of making it compatible with Tails/Debian Squeeze. He provided a link to a test build of master with the updated gitian descriptor which should work on Debian Squeeze / Tails Linux. The context is a sample terminal output for the latest Tails operating system version 0.23. The user attempts to run the bitcoin-qt command but receives errors, including "Bus::open: Can not get ibus-daemon's address," "IBusInputContext::createInputContext: no connection to ibus-daemon," and "sendto: Operation not permitted." Additionally, the user receives a "Segmentation fault" error when attempting to run the command with a proxy.
