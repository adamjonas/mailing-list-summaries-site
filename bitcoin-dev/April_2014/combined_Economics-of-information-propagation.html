One suggestion is to use headers-first propagation to determine the primacy of a block based on the time the block header is received, which is faster and more efficient than the current system. However, there are concerns that this method would disadvantage small miners who earn no profit while waiting for the information to validate the block and update their UTXO sets.Other suggestions include distributing partial proofs of work and 'priming' memory pools with transactions, allowing peers to request the list of transaction hashes that make up a block, and using alternative units of account like "credits" or "ubits".
