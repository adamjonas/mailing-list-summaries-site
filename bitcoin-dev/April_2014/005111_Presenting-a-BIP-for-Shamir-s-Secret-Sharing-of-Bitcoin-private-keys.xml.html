Summary:

In an email exchange on April 7th, 2014, Nikita Schmidt and another individual debated the use of big integer arithmetic in implementing a Bitcoin Improvement Proposal (BIP). The other individual argued that big integer arithmetic would not be necessary due to the simple maths in GF(2^8), as demonstrated by code from libblkmaker. Schmidt did not disagree entirely but noted that uniform processing of secrets of any size could be a selling point for using GF(2^8).
