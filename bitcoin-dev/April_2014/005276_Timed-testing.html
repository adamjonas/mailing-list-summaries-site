Summary:

Jorge Timón, a Bitcoin developer, has shared his implementation of a new testing mode that produces blocks periodically. The code is available on GitHub and can be used to run simulations with different mining policies and stress tests for performance bottlenecks. Timón aims to implement the concept of private chains described in Freimarkets for hardfork or double-spend attack simulations without calculating any pow, and he seeks feedback from Bitcoin developers on how invasive this would be to Bitcoin's source code. He believes that if done properly, it could result in more readable code.
