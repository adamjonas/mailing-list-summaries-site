Summary:

Ross Nicoll, in an email to the Bitcoin-development mailing list, expressed concern over the MIME types in BIP0071 not being IANA registered and unlikely to be accepted as they are. The latest RFC on media type registration restricts what can go in the default "application/" namespace and requires it to be an ISO standard or similar. Nicoll suggested registering a Bitcoin vendor namespace with IANA and allocating MIME types such as "application/vnd.bitcoin.payment.request", "application/vnd.bitcoin.payment.payment", and "application/vnd.bitcoin.payment.ack" as a solution. However, others in the thread expressed doubt that this would work as Bitcoin is not a vendor and the chance of a string collision is low.During the discussion on Bitcoin's registration as a vendor, Mike Hearn expressed doubts that it would work since Bitcoin is not a vendor. Nevertheless, according to RFC 6838, the Bitcoin Foundation or Gavin himself could register a vendor tree for Bitcoin. If the foundation were to register, obtaining the subtree named vdn.bitcoin would be easy. Despite this, there was doubt over whether time should be spent on the matter since the chances of a string collision are low. The current mime types were deemed sufficient. Ultimately, registering as a vendor appeared to be more about following proper procedures than anything else.
