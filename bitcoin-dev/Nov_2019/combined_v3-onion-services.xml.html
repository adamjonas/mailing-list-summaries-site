Summary:

There is ongoing effort to upgrade the Bitcoin P2P protocol to support other address types, including onion v3. However, Bitcoin Core currently supports Tor's v2 hidden service but not v3. A developer named Mr. Lee Chiffre is creating a new Bitcoin node that will use v3 hidden service instead of v2 and is looking for support from the Bitcoin-dev mailing list to know if any current node software supports v3 onion addresses. Carl Dong, another developer, has been working on an implementation of addrv2 (BIP-155) which includes support for I2P and Torv3 address.Bitcoin users who value privacy and anonymity often turn to using Tor hidden services as a workaround. However, there are ongoing efforts to upgrade the Bitcoin P2P protocol to support other address types, including onion v3. The current Bitcoin client core allows for the use of tor hidden service and supports v2 hidden service. To improve upon this, progress is being made towards creating a new bitcoin node that will use v3 hidden service instead of v2. This upgrade will enhance the privacy and security of Bitcoin transactions. Additionally, efforts are underway to support longer addresses used by i2p and tor v3.The author of a post on the Bitcoin-dev mailing list questions the rationale for a Bitcoin node to be hidden using Rendezvous (RDV) points like hidden services, suggesting there may be another way to achieve this goal rather than mimicking the Tor network for RDV points with .onion addresses. The author proposes a separate network using the Tor protocol over TCP, WebSockets, and WebRTC, making it possible for browsers to relay the traffic. Blockchain Commons has incorporated v3 tor authentication to control a full node remotely using their Bitcoin Standup project, which is currently available only for macOS but will soon be launched on other platforms.The Tor team encourages active participation from Tor nodes, preferably exit, middle and guard nodes rather than just client nodes. It is possible to use both v2 and v3 onion services simultaneously via the same torrc file by setting HiddenServiceVersion parameter after every HiddenServiceDir parameter.Overall, these upgrades to the Bitcoin P2P protocol will provide users with greater options for protecting their privacy and maintaining anonymity while using the cryptocurrency. The use of v3 ephemeral service is not necessary for node connectivity, but allows for hidden listening so that other nodes can connect privately without using the public internet or exposing the node's public IP or ports. Efforts are being made to upgrade the Bitcoin P2P protocol to support onion v3 and I2P addresses. The author argues that using the Tor network for Bitcoin is a bad idea for security and performance reasons and proposes a separate network using the Tor protocol over TCP, WebSockets, and WebRTC. The author does not think that the Tor network is designed and adapted to support Bitcoin nodes.
