Summary:

Salvatore Ingala, a Bitcoin developer, has proposed a new language called "wallet policies" to address challenges related to implementing descriptors and miniscript support in hardware wallets. The proposed solution for the security and UX challenges of scripts in hardware wallets is a registration flow for wallet policy. Wallet policy includes a wallet descriptor template and a vector of key information items, which can be used to obtain a 1-dimensional descriptor for receive and change addresses. Restrictions on supported descriptors have been put in place to simplify implementation while capturing common use cases.The proposed language aims to minimize the size of the information displayed on-screen for better user experience while ensuring security. Wallet policies propose a registration flow to register a wallet policy into the hardware wallet before it is used for the first time. This helps guarantee that the user knows precisely what "policy" is being used to spend the funds, making it impossible for an attacker to surreptitiously modify the policy and steal or burn the user's funds.The proposed language targets only a stricter subset of the output descriptors language and attempts to generalize the approach already used for single-signature accounts or multisignature setups. The document outlines a standard for output script descriptors used to derive addresses and scripts from wallet descriptor templates and provides examples of wallet descriptor templates for various use cases. Examples of wallet descriptor templates provided include native segwit account, taproot BIP86 account, native segwit 2-of-3, and miniscript for "1 of 2 equally likely keys." Additional rules, implementation guidelines, and examples are outlined with references to Output Script Descriptors, BIPs, and Bitcoin Secure Multisig Setup.The author of the wallet policy proposal provided additional practical examples and clarifications based on feedback received. They demonstrated that it is easy to convert from wallet policies to descriptors and back, arguing that aliases are better left out of descriptors in real-world usage. The post also provides several examples of miniscript policies, along with the corresponding miniscript descriptors, wallet templates, and vectors of keys that comprise the complete "wallet policy. "In discussions about the Bitcoin proposal, the author suggests that the language of descriptors should only represent either a single output or a list of outputs with the `/*` syntax. They propose keeping restrictions and extensions separated rather than changing descriptors. The author also discusses key aliasing and suggests it can be better built on top of wallet policies rather than within the language itself.The email exchange discussed optimizations for implementing descriptors on signing devices, proposing three key optimizations. The email also addressed challenges in implementing complex scripts in hardware wallets, including security and UX concerns, and proposed a solution in the form of a registration flow for wallet policies in the hardware wallet. Furthermore, the conversation touched on the feasibility of incorporating Musig2 descriptors into hwws, with some stating that it is possible as long as there is either enough state or a reliable RNG present.Overall, the proposed solution of wallet policies addresses the security concerns and user experience challenges when integrating descriptors and miniscript support in hardware wallets. By minimizing the amount of information displayed on-screen while ensuring security, wallet policies can benefit the UX of software wallets using descriptors/miniscript for multisignature or other complex locking conditions.
