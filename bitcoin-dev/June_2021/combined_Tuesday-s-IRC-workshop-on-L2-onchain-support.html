Summary:

The Bitcoin community recently held a workshop to discuss various topics related to L2 protocols. One of the main objectives was to strengthen the security assumption of transaction propagation and inclusion in a time-sensitive mined block. The participants discussed fee-sensitive timelocks, transaction relay overlay networks, and mining pools exposing public APIs to push transactions directly. However, it was acknowledged that there will never be perfect security guarantees on network transaction fee rates or transaction relay.The impact of changes to Bitcoin Core on L2 protocols was also discussed during the workshop. Chris_Stewart_5 proposed a nightly bitcoind build to give L2 developers a way to write regression tests against the latest builds of bitcoind. Additionally, a proposal for full Replace-by-Fee (RBF) in a future version of Bitcoin Core was posted, but concerns were raised about mempool partitions and its impact on businesses relying on zero confirmations.The Bitcoin community also discussed fee-sensitive timelocks and the potential challenges of obtaining the current network fee rate and feeding it into the script. Discussions revolved around miners' profit incentive, mining pools exposing public APIs to push transactions directly, and the impact of changes to Bitcoin Core on L2 protocols.Full replace-by-fee (RBF) proposals were also discussed, with ariard proposing full RBF in a future version of Bitcoin Core. However, BlueMatt pointed out that even with full RBF, it is trivial to create mempool partitions. The next meeting will focus on fee bumping and package relay advancements in Bitcoin Core. glozow has been working on these topics, and their progress will be discussed. The discussion log and additional notes from the workshop can be found in the L2 zoology repo set up by ariard.
