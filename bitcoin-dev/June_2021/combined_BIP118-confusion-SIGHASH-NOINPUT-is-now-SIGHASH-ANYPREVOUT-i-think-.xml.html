Summary:

During a discussion on the GitHub platform, Ryan Grant pointed out that there is confusion among people who are unaware of the recent updates to the BIP118 draft. The draft has undergone significant changes since May 2019, including a name change to SIGHASH_ANYPREVOUT. Details of these changes can be found in the "Revisions" section of the internal heading. However, the version in bitcoin/bips remained unmodified until it received a minor fix two months ago, still using SIGHASH_NOINPUT terminology.Ryan suggests that with taproot activating, the preferred method to ease lightning network client state requirements will now definitely be SIGHASH_ANYPREVOUT. Despite this, the updated draft for BIP118 is not being considered by a wider audience and it remains unclear why that is the case. The links provided on the GitHub platform offer further details regarding the revisions made to the draft.
