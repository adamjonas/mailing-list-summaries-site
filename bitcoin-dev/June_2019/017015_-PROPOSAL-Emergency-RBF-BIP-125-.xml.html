Summary:

In a discussion on the bitcoin-dev mailing list, Rusty Russell proposed a new "emergency RBF" rule for Bitcoin transactions. This rule would allow users to replace an unconfirmed transaction with a new one that has a higher fee and thus be more likely to be included in a block. However, there were concerns raised about the possibility of spamming the mempool by repeatedly using this feature to push transactions down the queue, effectively griefing other nodes. Russell O'Connor pointed out that it may be possible for someone to alternate RBFing two transactions to push each other down the queue, and suggested that the proposal could be amended to mitigate this issue. Rusty Russell agreed that this could be annoying for the recipient, but was unsure how practical it would be to execute across multiple non-direct peers.
