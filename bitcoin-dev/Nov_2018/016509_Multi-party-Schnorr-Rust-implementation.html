Summary:

The email is regarding the search for non-interactive threshold signatures that are more suitable for air-gapped use cases. The author, Omer Shlomovits, has been working with collaborators on providing Rust reference implementations to existing multi-party schemes for Schnorr signatures. These include aggregated signatures, accountable signatures and threshold signatures. The project can be found on Github. The protocol aims to be bip-schnorr compliant in a configuration of a single party. Omer welcomes any questions, suggestions and pull requests. Links to relevant papers and BIPs are provided.
