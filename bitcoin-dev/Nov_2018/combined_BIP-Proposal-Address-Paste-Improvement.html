Summary:

The Bitcoin-dev mailing list recently discussed the insecurity of copying Bitcoin addresses to the clipboard. Malware can easily replace any address with its own, potentially resulting in stolen Bitcoins if the user does not manually compare addresses. The community is seeking solutions that balance convenience and security.Suggestions for authentication include 2fa, visual fingerprints, and signing the destination address with a known key. Mechanisms for transferring an address from a web page to the wallet address input field include QR code scanning and similar mechanisms on desktop OSes. However, in many cases, addresses are presented as text, which users have to copy. This may incentivize users to use copypaste functionality extensively, raising concerns about crypto hijackers.For well-known entities, the BIP70 payment protocol offers authentication via certificates. However, it does not work for the "the person in front of you is the only trust anchor you have" use case. BIP75 and BIP47 may help, but the number of wallets that support these protocols is small. This may be due in part to the relative complexity of these protocols.To improve the user experience when copying and pasting addresses, the Address Paste Improvement BIP has been proposed on Github, with an implementation soon to follow in Wasabi Wallet. The community is deciding whether this needs to be a BIP or not based on how much they value UX. Overall, the community is discussing how to balance convenience with security when transferring Bitcoin addresses.
