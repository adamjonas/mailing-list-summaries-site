Summary:

On the Bitcoin-dev mailing list, a user has requested for non-interactive threshold signatures suitable for air-gapped use cases. However, candidates for such signatures do not seem to exist. The best solution proposed in response to the inquiry is taking the DKG from GG18 without Paillier and the DLog PoK, and using it for the threshold Schnorr DKG and ephemeral key distributed generation.A workaround was suggested by "batching" signing requests. This process involves four phases and can be almost as good as true non-interactivity if the signing hardware is capable of securely storing and updating a few kB of state. Links to relevant papers and implementations are provided for reference.The author of the email on the Bitcoin-dev mailing list has been working on Rust reference implementations with collaborators to provide multi-party schemes for Schnorr signatures. These include aggregated signatures, accountable signatures, and threshold signatures. The project aims to be bip-schnorr compliant if run in a single party configuration. The author welcomes questions, suggestions, and pull requests and provides a link to the GitHub project.
