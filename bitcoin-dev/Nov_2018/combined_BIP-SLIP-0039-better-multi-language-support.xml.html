Summary:

The Bitcoin-dev mailing list recently had discussions about BIP39 recovery and mnemonics in different languages. While the BIP-0039 seed generation process depends on the language chosen for the mnemonic sentence, the underlying entropy is still the same when comparing mnemonics generated across different languages sourced from the same initial entropy. The proposed solution is to have the mnemonic code represent the underlying entropy or pre-master secret, allowing for the same seed/secret to be displayed in any language. Additionally, the seed/secret could be represented in decimal numbers to support users who have difficulty with words in other languages.There are different options to address the issue of two mnemonics generated with the same entropy producing different BIP39 seeds depending on the language chosen. One option is to have the wallet recognize the language and show the corresponding index value per wordlist, and reverse-calculate the entropy and then re-map it to the language selected. Another option is to revise how the BIP39 seed is generated by hashing the entropy instead of the words. The BIP39 spec could also add greater detail regarding standards/best-practices for computing checksum values.In a message exchange between Neill Miller and Jonathan Underwood, it was mentioned that Electrum mnemonics are not based on BIP39, which allows it to handle recovery differently. However, Electrum handles BIP39 recovery as specified by the BIP and allows for restoring random strings with a warning that can be ignored if desired.In another bitcoin-dev post, Steven Hatzakis suggested revising the process for generating BIP39 seed, proposing that the initial entropy be fed into the PBKDF2 function instead of the words. Weiji proposed incorporating Shamir's Secret Sharing into the design to maintain language independence and enable mnemonics in multiple languages simultaneously. This will make it easier to figure out which word was handprinted incorrectly.
