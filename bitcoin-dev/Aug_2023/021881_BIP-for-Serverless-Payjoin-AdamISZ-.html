Summary:

The email discusses the interest in standardizing a protocol. The sender points out a concern regarding the use of base64 encoding for the psk in the URI. They mention that special characters in base64 need to be URL-encoded, which affects readability and adds complexity if a human needs to extract the psk from the URI. Instead, they suggest using [base64url](https://datatracker.ietf.org/doc/html/rfc4648section-5), which is modified to be more suitable for this purpose.
