Summary:

The email discusses the implementation of a proof-of-work requirement in a public key on an open relay server protocol. The purpose of this addition is to make it more difficult for spammers or scammers to roll new keys and gain access to the network. It is mentioned that embedding information in a public key is not challenging, but there is a limit to how much can be embedded, especially if the goal is to allow mobile devices to generate a new key within a few minutes.
