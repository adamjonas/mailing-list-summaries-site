Summary:

A proposed solution for securely sharing private keys and sensitive information is a modified SSS scheme with two key features. The first feature is that secret restoration is impossible without corresponding signatures. The second feature is that share[i] belongs to privateKey[i] only, meaning it cannot be signed with a different private key. This approach can be useful for sharing sensitive information at the application level or as an m-of-n multi-signature. Further details can be found in the attached PDF file named musig-esss.v1.pdf.Additionally, a working example of this concept in node.js is available on the author's GitHub repository at https://github.com/ega-forever/esss. The author expresses gratitude in advance for any review of the proposal.
