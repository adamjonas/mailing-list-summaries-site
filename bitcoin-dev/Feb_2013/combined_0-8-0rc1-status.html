Summary:

On February 8, 2013, Gavin Andresen wrote about the Windows builds with every compile and discovered that the -frandom-seed flag was not being passed down to the leveldb build. Luke responded that this issue was already mostly-fixed in pull #2243 submitted nine days ago. The Linux builds of version 0.8.0rc1 are good, but the Windows builds vary with every compile and need further attention. The OSX build is in pretty good shape but still requires some work. The author has conducted test builds on FreeBSD using gcc and clang to build both the qt gui and command line daemon. In a conversation on Feb 9, 2013, Gavin Andresen discussed the need to fix a non-determinism problem with Windows before the final release or rc2. He suggested pulling #2286 and creating a patch to address the issue. Pieter Wuille suggests that #2243 should fix the non-determinism problem in Windows. The plan is to announce 0.8.0rc1 with the available binaries and perform more testing before the final release. A patch will be created to fix the non-determinism issue in Windows before the final release or rc2. Gavin Andresen plans to sign the windows setup.exe and send out an rc1 announcement the next day.
