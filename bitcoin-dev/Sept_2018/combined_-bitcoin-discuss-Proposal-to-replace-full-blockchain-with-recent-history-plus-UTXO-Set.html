Summary:

A proposal to replace the full blockchain with recent history plus UTXO set has been discussed on the Bitcoin mailing list. The idea suggests that if a UTXO block set type existed, then every so often, a bunch of blocks containing just the validated UTXO set to a given height could be appended onto the valid chain and nodes would be free to drop all preceding blocks. This could save a lot of storage and make it easier to run a full node. However, this has consensus implications, and there are bugs (some of which are fixed in later commits to that repository). The proposal suggests relieving full nodes of storing the entire blockchain by using a UTXO set that is recognizable by humans at a certain block height. This would save storage and make it easier to run a full node. A utxo block set type could be appended onto the valid chain every so often, containing just the validated utxo set to a given height, say 100,000 blocks below the current blockheight, and necessary header data. Nodes would be free to drop all preceding blocks, but many may still desire to download the full blockchain. Additionally, the proposal mentions using the correct Merkle root for block 542322 instead of 542324 to see the correct Game of Life play out and make the apron image. The author offers $50 of Bitcoin to anyone who can tell them how to get a UTXO set from the Bitcoin client and send them the SHA256 hash of that set. The aim is to make a recognizable checkpoint for the Bitcoin blockchain, making it easier for more people to start running full nodes in the future. The email also includes links to an implementation of UTXO loading and a script for the open-source Game-of-Life software Golly. A video was shared showing the game playing out into an apron-shaped image. Overall, this proposal seeks to streamline the storage and operation of full nodes, while also offering incentives for community members to participate in its implementation and testing.
