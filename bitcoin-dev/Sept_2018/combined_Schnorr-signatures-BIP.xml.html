Summary:

Erik Aronesty, a developer, expressed his concern about the security of Bitcoin's multisig system in an email. He suggested that a redistributable threshold system would be more secure and efficient. However, Erik stated that he was not aware of any "non-redistributable multisig" proposed for Bitcoin. Moreover, Erik responded to feedback by modifying the protocol to make it work better.Gregory Maxwell, another developer, made a minor design preference by suggesting that the message should come after the nonce instead of before it. On the other hand, Russell O'Connor proposed changing the order of inputs to hash so that SHA256 expander could be partially prefilled for a fixed public key. A draft specification of the signature scheme itself was proposed as a first step towards standardizing the signature scheme. However, this proposal does not concern consensus rules, aggregation, or any other integration into Bitcoin.Pieter Wuille, a prominent Bitcoin Core developer, proposed changes to the Schnorr multi-signature (MuSig) scheme. The proposed changes would make verification faster and more efficient by avoiding negating a point or a coefficient. The Schnorr signature scheme is said to be simpler and more efficient than the currently used ECDSA signature scheme. In a mailing list post, Wuille proposed changing the verification equation from "Let R = sG - eP" to "Let R = sG + eP". This proposal removes one negation operation from the cost of doing verification.
