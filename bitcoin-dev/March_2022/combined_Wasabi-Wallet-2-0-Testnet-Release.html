Summary:

Wasabi Wallet 2.0 has been released for public preview on testnet, providing users with effortless privacy through its Tor integration and BIP158 block filters or the packaged Bitcoin full node. The upgrade includes a new Wabisabi coinjoin implementation that replaces the ZeroLink coinjoin coordination protocol, enabling anonymous DoS protection for centrally coordinated coinjoins without relying on equal amount outputs.The coordination also enables a more sophisticated amount decomposition resulting in "changeless" coinjoins, reducing mining fees paid and the time until the user's utxo set is private. To participate in the coinjoin, the minimum amount required is now 5000 sats, saving block space and reducing mining fees. Labeling is mandatory in Wasabi, so only contact labels are displayed for users to choose from as the utxo concept is abstracted away.The UX is simple: receive, wait, spend privately. Whenever a user wants to spend bitcoin to an address, the wallet automatically selects those private coins with sufficient sats, showing coin control to the user. Wasabi suggests adjusting the payment amount to avoid creating a change utxo, decreasing fees and improving future privacy. The release is currently available for testing on testnet, and users are encouraged to review the cryptography and coordination protocol and analyze the resulting transaction graph in the wild. However, several issues such as privacy, security, and others are not fixed and are shared in different GitHub issues or discussions.The bitcoin-dev mailing list may not be the best place to look for newbies as developers working on Wasabi mention that WW2 is not developed for power users. Wasabi Wallet does not have different types of addresses to use for a change, but Bitcoin Core recently made some related improvements which would improve privacy.
