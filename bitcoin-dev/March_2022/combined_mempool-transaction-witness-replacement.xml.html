Summary:

Larry Ruane's proposal to change the Bitcoin Core mempool would allow an existing transaction to be replaced by one with a smaller witness and higher feerate. This change is miner-incentive compatible, however, concerns around code complexity cost and transaction-relay DoS need to be considered. Witness replacement could defend against a mempool pinning attack, and Bip125 allows incoming transactions to replace conflicting ones if certain DoS-mitigation conditions are met. Witness-replacement is similar to RBF but differs in that it doesn't require evicting mempool descendant transactions and signaling isn't required since outputs can't change. The proposal may be implemented as a per-node setting, but it's preferable for the node network to agree on relay policies. Question 2 highlights a potential DoS tradeoff between rebroadcasting slightly smaller witness transactions many times or not being able to replace a transaction with a beneficially-smaller one.
