Summary:

In a Bitcoin development mailing list, Dmitry Petukhov has proposed the creation of a BIP draft that specifies "path templates" for BIP32 paths. The purpose of these templates is to create a standard format for describing constraints for BIP32 paths. Dave Harding, another member of the mailing list, asked for clarification on how these path templates compare to key origin identification in output script descriptors and for a specific example of how they might be used. He also questioned whether they are intended for backups, multisig wallet coordination, or managing data between software transaction construction and hardware device signing.
