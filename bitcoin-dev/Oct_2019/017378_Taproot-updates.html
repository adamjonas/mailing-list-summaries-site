Summary:

Pieter, a developer of Bitcoin, has given an update on some changes made to the bip-schnorr/taproot/tapscript drafts following discussions on the bitcoin-dev mailing list. The changes include using 32-byte public keys instead of 33-byte ones, dropping P2SH support for more efficient chain usage and no gratuitous fungibility loss, increasing the limit on the depth of Merkle trees from 32 to 128, and dropping the 10000 byte script limit and 201 non-push opcode limit as they serve no purpose and complicate creation of Scripts. A lot of clarifications and rationales were added, and the reference implementation was updated on GitHub.
