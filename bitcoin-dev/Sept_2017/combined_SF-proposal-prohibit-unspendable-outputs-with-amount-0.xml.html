Summary:

The bitcoin-dev mailing list has been discussing the creation of 0-value outputs that can remain in the unspent transaction output (UTXO) indefinitely. There are concerns about spamming attacks on the UTXO and the necessity or use case of allowing spendable outputs with null amounts in them. Jorge Timón proposed requiring at least 1 satoshi per output to combat spam attacks, while others argue that this proposal may cause complexity for future upgrades.Peter Todd expressed his negative response to the proposal of dropping zero value outputs, stating that it is a needless loss of flexibility. However, he acknowledged that requiring a minimum value may become meaningful if the value of 1e-8 btc becomes high enough to reduce a miner's ability to spam up the network. Nonetheless, at present, the argument does not hold weight as the cost in lost fee income from the spam would greatly outweigh any potential benefits.The discussion also touches on timelocked transactions and their potential risks, particularly with regards to a soft fork impacting the validity of long-term locktime use. Additionally, there is a discussion about the implementation of Confidential Transactions (CT) as a softfork in Bitcoin, including the proposed tradeoff between extra outputs and blockchain space.One proposed solution for confidential transactions is the use of "shielded" outputs with unknown values, but spending these may result in extra outputs and consume significant blockchain space if shielded-to-shielded transactions are common. It was suggested that a change to the blinding scriptPubKey could allow for the use of 0 value outputs, and there was a proposal to create a Bitcoin Improvement Proposal with code to address the issue of spendable outputs with null amounts.
