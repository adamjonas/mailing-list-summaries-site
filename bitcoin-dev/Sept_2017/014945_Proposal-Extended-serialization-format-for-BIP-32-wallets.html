Summary:

On September 7th, 2017, Thomas Voegtlin via bitcoin-dev proposed a solution for wallets that do not wish to use BIP43. He suggested adding another byte field, OutputType, which would include 0x00 for P2PKH output type, 0x01 for P2WPKH-in-P2SH output type, and 0x02 for native Segwit output type. The question that arose was whether this field should only be present if depth==0x00 or at all times. Pavol "stick" Rusnak, the CTO of SatoshiLabs, asked for suggestions on this matter.
