Summary:

On September 22, 2017, Sergio Demian Lerner suggested on the bitcoin-dev forum that if variable size increases by only a few bytes, there are three possible solutions: allow signatures to be zero padded and abandon strict DER encoding; allow spare witness stack elements and remove the cleanstack rule, however empty stack elements must still be counted as 1 byte; or signers should loop the generation of signatures until they reach their maximum size. Alternatively, Pieter suggested getting rid of DER encoding altogether and switching to fixed size signatures.
