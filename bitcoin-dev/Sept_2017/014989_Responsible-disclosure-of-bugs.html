Summary:

There seems to be a need for clarification of the current policy regarding publication of vulnerabilities in Bitcoin. This is due to historical practices where such vulnerabilities were only made public after 80% of nodes had upgraded. It is believed that this is the general but not publicly stated policy, and there is a request for any core developer with better knowledge to correct this belief.
