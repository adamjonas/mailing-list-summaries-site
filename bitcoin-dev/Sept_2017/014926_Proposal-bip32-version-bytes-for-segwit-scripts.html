Summary:

A suggestion has been made by Pavol Rusnak, the CTO of SatoshiLabs on 05/09/17 19:03 in a Bitcoin-dev forum to use a child number field instead of a fingerprint. He believes it would be desirable to use the same seed for all different script formats. If he were designing the serialization format today, he would drop the fingerprint and expand child number to full BIP32 path. The benefit is that depth already exists, so it is known how long the BIP32 path would be. The proposal includes using 4 byte version bytes, 1 byte depth, depth * 4 bytes bip32 path, 32 bytes and 33 bytes.
