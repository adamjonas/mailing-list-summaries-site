Summary:

The proposal to add paper wallet functionality to bitcoin-core software through a new RPC call genExternalAddress is the main focus of discussion in this context. The goal is to eliminate the need for third-party tools and provide a more secure option for personal auditing of code. Dan Libby proposes and discusses this addition, starting with a single new RPC call genExternalAddress [type], which returns a new Bitcoin address and private key for receiving payments. This call would not be stored in the internal wallet and could be used externally, such as for paper wallets.While paper wallets are seen as a security risk by some, the proposed API provides enough functionality for any moderately skilled developer to create their own paper wallet creation script using any scripting language. However, there are concerns about the safety and security of paper wallets, and some experts suggest using BIP45 Multisig HD Wallets instead. Private keys should be generated and used on trusted, minimal, and offline hardware/os, while backups should be encrypted and have a footgun-safe restore process.Luke Dashjr and Jonas Schnelli agree that paper wallets are unsafe and insecure, and private keys should be generated and used offline on trusted hardware. Andrew Johnson raised concerns that exposing Paper Wallet generation through QT could compromise security if users generate paper wallets from online machines. He suggested that Bitcoin-Core be installed on a dedicated offline machine for key generation purposes. Despite concerns about encouraging users to generate paper wallets from online machines, the APIs proposed by Dan Libby have uses beyond paper wallets, and they can also be used by third-party wallets. The removal of dumpprivkey and dumpwallet RPCs from Bitcoin Core is being considered, especially if HD public key derivation forms become allowed. In conclusion, there is no existing solution that addresses all the concerns regarding private key generation and storage. However, improving the system's security should be a priority. The proposal to add paper wallet functionality in bitcoin-core software has been suggested and discussed by Dan Libby, but proposals like this should use bitcoin-core-dev or Github for submissions. The focus should be on working towards improving the security and safety of Bitcoin transactions while emphasizing the importance of trust.
