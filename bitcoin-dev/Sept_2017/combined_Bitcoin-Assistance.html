Summary:

On September 27, 2017, the bitcoin-dev mailing list saw a discussion about a license for an alternative to the modulo reduction. During this discussion, Cory Fields provided context on the licenses used in various files for Bitcoin Core project. Black Duck Software has also scanned the codebase for open source licenses, which was found to include projects with inconsistent licenses. A lawyer new to the list is seeking to understand the licensing of Bitcoin core and will be presenting in a webinar with Black Duck Software on Blockchain on September 28.The licensing of Bitcoin Core has been examined by Black Duck Software, who scanned the code for open source licenses. The scan revealed that the code base includes code licensed under a variety of licenses, such as Apache License 2.0, Boost Software License 1.0, and MIT License, among others. However, some of the licenses are incompatible with each other, such as code licensed under the Apache Software License version 2 and GPLv2, which cannot work together in certain situations.Cory provides context on the licenses used in various files for the Bitcoin Core project. The Apache License 2.0 is used by bindings that allow other applications to use libsecp256k1 from Java, while the Boost Software License 1.0 comes from tinyformat.h and is used by Bitcoin Core. BSD 2-clause "Simplified" License was not found in any files. BSD 3-clause "New" or "Revised" License comes from leveldb, which is database software used by Bitcoin Core.A discussion regarding the licensing of bitcoin's GUI SVG icons has been initiated by Cory Fields via bitcoin-dev. It was noted that all icons not created by the team are listed in contrib/debian/copyright and are public domain. Additionally, some icons listed as "MIT" could be a violation of CC BY-SA.On September 27, 2017, Tim Ruffing raised concerns on the bitcoin-dev mailing list about possible violations of CC BY-SA license in the old version of some icons listed as "License: MIT". This issue was addressed by the copyright holder who made changes to the licensing of the icons. The commit for the changes can be found on GitHub and it is recommended to use the "git log -p" command to locate specific changes in a file.In conclusion, the licensing of Bitcoin Core has been found to include a variety of licenses, some of which are incompatible with each other. Additionally, there have been concerns raised regarding the licensing of Bitcoin's GUI SVG icons, which have been addressed by the copyright holder through a re-licensing process. It is recommended to use the "git log -p" command to locate specific changes in a file.
