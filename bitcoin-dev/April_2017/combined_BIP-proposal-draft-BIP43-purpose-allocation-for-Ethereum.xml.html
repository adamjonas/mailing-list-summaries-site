Summary:

User Tom Zander expressed support for a proposal to create a BIP defining a logical hierarchy for deterministic wallets on the Ethereum blockchain in a recent email thread on the bitcoin-dev mailing list. The proposed BIP, called bip-nickjohnson-ethereum-purpose, is a particular application of BIP43 and defines two levels in the BIP32 path: purpose and subpurpose. Purpose is a constant set to the hardened value of the BIP number assigned to this BIP, while subpurpose is set to the EIP number specifying the remainder of the BIP32 derivation path. This proposed BIP aims to standardize hierarchical deterministic wallets on the Ethereum blockchain in a manner that is better suited to its unique requirements. Ethereum's specific needs make the BIP44 hierarchy unsuitable and result in inter-client incompatibility. The proposed hierarchy defines two levels in the BIP32 path: m / purpose' / subpurpose' / *. Apostrophe indicates that BIP32 hardened derivation is used at each level. Purpose is a constant set to the hardened value of the BIP number assigned to this BIP, indicating that the subtree of this node is used according to this specification. Subpurpose is set to the EIP number specifying the remainder of the BIP32 derivation path, allowing new Ethereum-focused applications of deterministic wallets without needing to interface with the BIP process.This BIP proposes a logical hierarchy for deterministic wallets on the Ethereum blockchain based on an algorithm described in BIP-0032 and purpose scheme described in BIP-0043. The proposal seeks to provide a path to standardize hierarchical deterministic wallets in a fashion better suited to Ethereum's unique requirements. References to BIP32 and BIP43 are provided.
