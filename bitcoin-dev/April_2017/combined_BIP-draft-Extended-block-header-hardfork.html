Summary:

In recent discussions on the bitcoin-dev mailing list, developers have proposed various changes to the Bitcoin protocol. One such proposal is the "spoonnet" hardfork, which would provide several requested features for Bitcoin businesses and ensure predictable resource use. The hardfork aims to introduce deterministic activation logic, strong and simple 2-way replay protection, wipe-out protection, and more. A reference implementation for consensus codes can be found on GitHub.Another proposal involves moving nHeight from the coinbase script to the coinbase locktime. While Russell O'Connor suggests that this change would be more natural, Tom Zander states that it would not require consensus and could be made at any time. However, it is important to note that any changes to the consensus rules of Bitcoin would require consensus among network participants.In a separate discussion on the same mailing list, developers also considered a proposed change to the Bitcoin protocol regarding the witness of the first input of the coinbase transaction. According to the proposal, the extended header must contain specific data, including bytes 0 to 3, and nHeight must be equal to the height of this block (signed little endian). There was a suggestion to move nHeight from the coinbase script to the coinbase locktime. However, Tom Zander pointed out that such a change would not require a consensus change. The current protocol operates under a consensus rule, as stated in BIP-0034, that determines block headers must contain a merkle root of all transactions in the block, including the coinbase transaction. Any changes to this rule would require consensus among network participants.During these discussions, there were also requests for clarification on certain rules enforced in the code of Bitcoin. James Hilliard provided links to relevant code and BIP-0034, which outlines the consensus rule in question. Jean-Paul suggested that those with knowledge gaps on Bitcoin should visit the Reddit forum, r/BitcoinBeginners, to learn more.
