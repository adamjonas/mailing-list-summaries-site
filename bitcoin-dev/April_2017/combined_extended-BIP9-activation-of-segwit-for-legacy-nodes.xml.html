Summary:

The proposed method for activating segwit involves using a date-dependent UASF fallback, with deployment rules requiring signaling for segwit using the backwards-compatible bit. After 95% of recent blocks signal for the alternate segwit deployment, more legacy nodes would understand and validate transactions using segregated witnesses. The proposal suggests an expiration time of five years.Activating segwit has proven to be more contentious than expected, but Shaolinfry believes that technical consensus is clear despite objections. It would be beneficial if validating nodes capable of supporting segwit could participate to their fullest capacity without further upgrades. Although segwit is backward compatible with old validating nodes, there are still distinct advantages to validating and generating segregated witness transactions.In order to activate BIP149-like segwit in November, it is suggested that a separate deployment be made for legacy and semi-legacy nodes, allowing semi-legacy nodes immediate participation in a successful segwit activation. This proposal does not include any deployment specifics but leaves it to future proposals. Without this proposal, BIP149ish requires debates, tests, and a courtesy timeout, making it slow and difficult to plan for activation.With this proposal, the courtesy begins as soon as it goes live, simplifying the debate on how BIP149ish should deploy, and helps make quick segwit activation possible if BIP141 and BIP148 do not succeed earlier. The realities of system administration on the Bitcoin network mean that upgrades occur slowly. The proposed deployment logic for segwit is provided in the email.
