Summary:

In a message posted on the bitcoin-dev mailing list, Tom Harding mentioned that a network with many nodes maintaining a transaction index allows for a certain type of light node applications that request peers to verify the existence and usage of Transaction Outputs (TXOs). However, this additional capability requires commitment structures like those proposed by Peter Todd in his stxo/txo commitment designs. These designs include delayed TXO commitments and other forms of commitment structures. For more information on these designs, visit https://petertodd.org/2016/delayed-txo-commitments.
