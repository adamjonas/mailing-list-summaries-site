Summary:

The writer proposes making the threshold configurable and not fixed in the soft-fork, along with other parameters such as window size and grace period. They suggest using a coinbase signaling feature to include parameter information for forks which a miner is signaling for coordination, despite the limited space in coinbase for embedding data on the large number of possible independent deployments.The proposal also aims to mitigate the risks associated with User Activated Soft Fork (UASF) by initiating a soft fork with more than 33% of the hashing power, reducing the likelihood of adverse miners orphaning non-empty blocks on the forking branch. The proposal suggests that miners signal the threshold at which they are willing to activate a soft fork, allowing for public availability of hashrate threshold information prior to the soft fork to help miners decide whether they should join the fork right after it has been activated.The use of a small set of standard thresholds is suggested, with a 4-bit system being able to encode 15 different thresholds plus zero. This allows for all thresholds between 25% and 95%, separated by 5%. Using 4 bits per soft-fork proposal leaves enough room to fit seven simultaneous proposals in version bits. There is a discussion regarding the use of coinbase and the potential for it to clash with other proposals, as it is not part of the block header. While some agree with the suggestion of using a small set of standard thresholds, others still prefer the coinbase idea, as it is more appealing than using up the BIP9 versionbits range for verbose signaling. BIP9 currently assumes that participants will coordinate to agree on what the bits mean in terms of change deployments. However, it would be easy to agree on a set of "standard" threshold levels and map those onto one byte. In the coinbase, pairs of bit numbers and bytes could be used, where byte values correspond to standardized deployment schedules that people find useful. This would allow for compact yet readable signaling, with a large enough space of values to minimize contention.
