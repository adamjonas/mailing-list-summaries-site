Summary:

Mike Hearn, in an email chain, responds to Krzysztof Okupski's attempt to describe the Bitcoin protocol in English and advises against calling it the "Bitcoin protocol spec" as it is not a formal specification. He warns that implementing the protocol based solely on this description would result in errors since important areas such as the SIGHASH_SINGLE bug and difficulty transitions are left unspecified. Despite this, he acknowledges its value as a resource for newbies and suggests linking it from the developer guide.In response to Ron's question about which C++ original version is the only spec that matters, no further information is provided.
