Summary:

A seasoned software developer named Caleb Roger Davis wants to contribute to Bitcoin for learning purposes by working on unit and/or other types of tests and understanding the Bitcoin code from top to bottom. Additionally, he wants to write a Bitcoin wallet in another language. Caleb requests information on tools and frameworks needed to work on tests, preferred IDE, full development stack, where to start looking at the Bitcoin core code and the wallet code, and if there is a separate area for core wallet development on Github and mailing lists. Felipe Micaroni Lalli expresses his interest in implementing Bitcoin in Clojure or Scheme and shares his PGP ID and BTC address. Caleb shows his desire to create a toolkit first to learn more about how it works before tackling writing Bitcoin in Clojure. All of the wallet code is in `src/wallet.cpp` and `src/walletdb.cpp`. If Caleb wants to write a Bitcoin wallet in another language, it's better to look at SPV wallets, for example, the bitcoinj-based ones. Caleb can refer to https://www.bitcoin.org/en/developer-guide to understand the Bitcoin code as much as possible from top to bottom. A Linux box (or VM) is the best development environment for working on Bitcoin Core, but getting started building on Windows or Mac is harder (but possible), and there is work in progress to make building the dependencies easier for those.
