Summary:

Pieter Wuille, a Bitcoin Core developer, has submitted a pull request to make changes to BIP 62, his anti-malleability proposal that is still in draft. The proposed changes would make two of the seven new rules mandatory in new blocks, even for old-style transactions. According to Wuille, these transactions are already non-standard since version 0.8.0 and have no use cases. The reason for this change is to remove the requirement for signature verification engines to be bug-for-bug compatible with OpenSSL. This would allow any implementation to support DER, making it easier to ensure strict DER compliance for signatures. Pieter has also reordered the rules, added more reference documentation about minimal pushes and number encodings, and clarified that extra consensus rules cannot prevent someone from creating outputs whose spending transactions will be malleable.Rule four may not be sufficiently specific without a definition. Pieter's proposed changes aim to address this issue and provide more clarity on the matter. Additionally, he clarified that while extra consensus rules cannot prevent someone from creating outputs whose spending transactions will be malleable, the scriptPubkey can still specify rules that are inherently malleable.
