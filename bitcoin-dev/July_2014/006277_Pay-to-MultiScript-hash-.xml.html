Summary:

The proposed method of using 20-byte hash to represent a permission group would incur additional costs in terms of chance of user error/application error and extra bytes for control flow. However, this method allows for hidden-till-spent permission groups, reduces blockchain bloat and enables repeated spends with groups without having to recall all participants. The author suggests an interesting way to push the sigs onto the stack and have implicit orders, then do expressions with their aliases, and then be able to assign "spending groups". This can be done using a little SAT scripting language with an eval. Doing boolean logic with multisig groups is quite possible, e.g. "group AND group", "group OR (group AND group)" etc. and is referred to as "multi-multisig".
