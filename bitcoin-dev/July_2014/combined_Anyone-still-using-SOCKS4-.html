Summary:

The Tor project has warned about SOCKS and DNS information leaks, recommending the use of Socks4A as a better alternative. Applications using socks5 only give Tor an IP address, while applications doing DNS resolves themselves may leak information. Wladimir proposed removing support for SOCKS4 in the next major release of Bitcoin Core due to it being outdated and not offering the same privacy benefits as SOCKS5. The removal would remove some untested code paths and improve privacy by allowing DNS redirection. In addition, SOCKS5 supports IPv6. Odinn Cyberguerrilla raised the question of whether SOCKS4 was designed to prevent information leaks, and Wladimir noted that SOCKS4a supports DNS lookups on the server, but is not supported by bitcoin core. He added that if someone wanted to contribute SOCKS4a support, it would be acceptable.
