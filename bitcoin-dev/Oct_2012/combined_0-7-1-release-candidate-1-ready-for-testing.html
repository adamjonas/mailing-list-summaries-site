Summary:

Bitcoin has released its version 0.7.1's first candidate, featuring minor bug fixes, new features, and dependency changes, including a boolean argument to the RPC 'stop' command that sets -detachdb to create standalone database .dat files before shutting down. Other changes include the -salvagewallet command-line option, which moves any existing wallet.dat to wallet.{timestamp}.dat and then attempts to salvage public/private keys and master encryption keys into a new wallet.dat. There were also changes in dependencies, including Qt 4.8.2 for Windows builds and openssl 1.0.1c. Better detection and handling of corrupt wallet.dat and blkindex.dat files was fixed, along with occasional crashes on exit, an uninitialized variable bug that could cause transactions to be reported out of order, and warnings about creating fresh wallet backups after wallet encryption.Gavin Andresen, one of the core developers behind Bitcoin, shared the details of the update. In an email thread, Andresen inquires about the progress on a release candidate QA sanity testing plan to which the respondent replies that they have come around to his way of thinking. They discuss testing and the process involved. It is revealed that everything that needs to be covered for a newbe or someone upgrading clients has been covered and is available in many formats at the moment. They talk about getting a technet license with the donations so far.In the same email thread, Jeff Garzik suggests a separate Bitcoin test list for discussion of testing, tools, ideas, and chatter. Arklan Uth Oslin also suggests the creation of a separate Bitcoin test list but is met with resistance from Garzik due to low traffic on the current bitcoin-devel list. The email thread also includes some quotes from Arklan. A member of the Bitcoin community has asked for a Bitcoin-test mailing list to be created for discussion around testing, tools, ideas, and chatter.
