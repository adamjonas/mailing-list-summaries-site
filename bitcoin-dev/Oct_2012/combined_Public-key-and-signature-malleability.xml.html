Summary:

OpenSSL, which is used by almost every full Bitcoin node, accepts non-standard encodings for public keys and signatures that fail to conform to SEC and DER specifications. To make the Bitcoin network rules more well-defined and prevent malleability attacks, Pieter Wuille has proposed strict rules for acceptable public keys and signature formats. The proposed rules are already followed by almost all clients and include specific byte requirements for compressed and uncompressed EC points, as well as MSB encoded integers for signatures. Sergio Lerner recently discovered a vulnerability in the S value of the signature, which can be resolved by requiring the lowest bit of the S value to always be even. This would require coordination as no client currently enforces this. Malleability can allow malicious relayers to modify transactions in-transit without invalidating them, leading to wallet software not dealing gracefully with modified versions of transactions accepted in a block. If the rules are adopted, they could first be enabled as checks similar to IsStandard(), making it hard for non-compliant data to get into blocks but still be accepted when they aren't standard. Finally, a BIP could be written and enforced as a network rule when no significant amount of non-standard transactions are seen anymore.
