Summary:

Fred Concklin, a Bitcoin developer from Trucoin, recently introduced himself to the Bitcoin community. He shared his plans to contribute to the project by structuring it into a format compliant with GNU autotools, which would help facilitate testing and bug reporting. Fred is also interested in daemonizing Bitcoin on boot and bringing it into general coding and implementation paradigms seen on GNU/Linux platforms.Fred has experience with automated builds and testing for GNU/Linux and BSD platforms, although he mainly spends his time working with high-level functional programming languages. Despite this, he is excited to work on Bitcoin and spur discussion on new blockchain distribution techniques and improvements.To begin his work on Bitcoin, Fred plans to focus on the autotools structure and come up with a standard for a Wallet protocol. Some work has already been done in this area, so Jay Weisskopf advised him to start there. Luke-Jr mentioned that Bitcoin already works fine in regards to daemonizing on boot and implementation on GNU/Linux platforms using standard tools.
