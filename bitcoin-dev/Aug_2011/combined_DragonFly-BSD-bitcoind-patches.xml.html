Summary:

Three patches were provided to enable bitcoind to build and operate on DragonFly BSD. The first patch tests for SO_NOSIGPIPE instead of assuming that all BSDs support it, as this assumption is incorrect for NetBSD, OpenBSD, and DragonFly. The second patch changes the signedness of pchMessageStart to 'unsigned char' in main.cpp and net.h since leaving it unsigned is unsafe for values > 0x80. Lastly, the third patch qualifies make_tuple with boost:: namespace in db.cpp because without qualification, std::make_tuple is preferred, which is incorrect.These patches were suggested by Venkatesh Srinivas in August 2011, and they were issued in git format-patch, making them applicable with git-am. Jeff Garzik approved the patches and created pull requests on Github.
