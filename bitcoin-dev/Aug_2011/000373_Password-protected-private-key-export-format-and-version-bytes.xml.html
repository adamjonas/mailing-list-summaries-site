Summary:

A revised description of the password-protected export format has been posted, which now includes multiple algorithms and a parameter byte for selecting the encryption parameters. The thread is available on bitcointalk.org, and unless there is an existing or other planned use for data class 32, it will be used for password-protected private keys. Sipa's description indicates that this would mean that a password-protected private key for bitcoin is type 32 and type 79 for the testnet. The proposal seems to have no objections thus far.
