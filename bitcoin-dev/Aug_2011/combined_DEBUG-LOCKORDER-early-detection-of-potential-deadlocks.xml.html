Summary:

The commit 865ed8a introduces a new compile-time #ifdef: DEBUG_LOCKORDER. When compiled with -DDEBUG_LOCKORDER, every time a lock is acquired by a CCriticalSection, a check is made to record the order of locks and complain if they are being acquired in an inconsistent order. In case potential deadlocks are detected, debug.log reports the previous and current lock order. Gavin Andresen mentions that he will work on eliminating the potential problems and create some RPC stress-tests that try to exercise all the code paths to early-detect other potential problems.
