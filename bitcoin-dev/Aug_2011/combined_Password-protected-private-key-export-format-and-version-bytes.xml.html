Summary:

Sam Revitch has revised the description of the password-protected export format. The new format includes multiple algorithms and a parameter byte for selecting encryption parameters. He is seeking to use data class 32 for password-protected private keys, as suggested by Sipa. This would make the password-protected private key for bitcoin type 32 and for the testnet, type 79. Sam's proposed format was posted on bitcointalk.org. It is intended as an interchange format and is not specifically integrated with wallet encryption. However, importing it to the official bitcoin client using a method like the import/export patch would require the user to provide the password of the private key in addition to the wallet password if the wallet is encrypted.Sam also made a request to use one of the data classes proposed by Pieter Wuille about a month ago. If this does not fit with the general bitcoin development agenda, that's okay, but he wants to explore the possibility of a protected export format for private keys.
