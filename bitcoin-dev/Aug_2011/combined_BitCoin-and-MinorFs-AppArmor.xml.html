Summary:

Rob Meijer, the creator of MinorFs, is proposing an updated version that could be used by Bitcoin on systems that support AppArmor. The current version provides private members to objects, but Meijer plans to lower the barrier to using MinorFs in the new version. He wants to provide facilities to pick a granularity for 'object' more suitable for most lines of development. The concept is complementary to wallet encryption, and it can protect sensitive user data like Bitcoin wallets from malware running under the same user ID.MinorFs is a set of user space file-systems on top of current UNIX/ACL permissions. It allows a migration path to one with a finer granularity level where sub systems delegate sub-trees to each other in a pass-by-reference way. However, there are some caveats to using MinorFs with Bitcoin. For instance, there is a need for a bash script for starting up bitcoin with $HOME set to the MinorFs home, and there can only be one bitcoin started and running at once.In the new version, Meijer thinks there would be at least a minor change to Bitcoin needed. Bitcoin would have to use a small library that provides a 'minorfs_getpwuid' function. This function will work like getpwuid on any system without an active MinorFs, and for any non-apparmor confined process, and on a system with MinorFs running, it should return a passwd structure with the home changed to the MinorFs home. All read and write operations will map directly to the underlying file-system.Meijer is seeking input from Bitcoin developers on what attributes from the current granularity level should be kept, dropped, or added to arrive at an 'id' that is the best fit for granularity of persistent private storage for Bitcoin. He wants to accommodate Bitcoin developer needs so that code to use MinorFs where available can be added to Bitcoin.Rob Meijer has also created LibMinorFs, a library for making the usage of MinorFs by applications like Bitcoin more convenient. It would allow Bitcoin to store things like the BitCoin purse in a private directory that is not available to other unrelated processes running under the same UID.Overall, Rob Meijer wants to create a new version of MinorFs that suits common development languages and practices. He believes this will make it easier for application developers who want their application to protect user data, such as Bitcoin wallets, from malware.
