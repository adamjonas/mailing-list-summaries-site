Summary:

The Bitcoin network is experiencing a major issue with a lack of connectable slots, which is causing a slowdown in peer bringup for Android apps. Out of 48 IPs returned by the DNS seeds, only 19 are actually usable, and six are closed. Currently, there is no address-saving functionality for last-used peers in the Android apps, which needs fixing.To resolve this issue, a custom DNS server that watches the network to find long-lived peers running the latest version could be helpful. The developers suggest sorting nodes by version, how long they've been observed to exist, and the last polling time to better manage connectivity. Matt Corallo suggested using https://github.com/TheBlueMatt/dnsseed but recommended cleaning it up or using a different DNS server that pulls directly from the database in a more dynamic way. Mike Hearn discussed the possibility of finding a friend to take on DNS seeding as a project, and Rick Wesson expressed interest in contributing to the project.In addition to the DNS seeder issue, bad nodes appearing in the DNS seeds are causing problems for Android apps.
