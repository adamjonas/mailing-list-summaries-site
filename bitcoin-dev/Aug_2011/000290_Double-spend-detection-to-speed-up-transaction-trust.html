Double spend detection to speed up transaction trust

Summary:

The scenario presented involves two scammers attempting to buy two Ferraris using only one set of BTC. The current solution is for vendors to wait for six confirmations before releasing goods, which can take a long time. However, the Bitcoin network is essentially honest and a valid transaction will eventually make it into the longest blockchain. The proposal is to broadcast a new "inv" type called "MSG_DOUBLESPEND" when a node drops a transaction so that vendors can operate their own policy on waiting for confirmations. This would also allow miners to lower the priority of any transactions flagged in this way. The transaction must be internally-valid to result in a MSG_DOUBLESPEND, making it difficult to DOS by someone without the appropriate private keys. In the future, Bitcoin payment processors could handle double-spend checks and offer insurance against unconfirmed transactions being invalidated.
