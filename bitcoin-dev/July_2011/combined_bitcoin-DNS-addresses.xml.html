Summary:

The discussion on the Bitcoin mailing list was focused on finding the best method for resolving Bitcoin addresses. DNS-based resolution was considered bad due to its potential vulnerability to MITM attacks, and DNSSEC was deemed unsuitable as a security measure for this purpose. Instead, HTTPS was suggested as a better alternative, which had several advantages over DNSSEC-based solutions without any significant drawbacks. One of the key benefits of HTTPS was the ease of giving different addresses to each client based on IP, which was not possible with DNS.Although DNS mapping doesn't prevent or deny anyone from leveraging HTTP(S) for similar mapping, it is designed for name-to-thing mapping and has no clear advantages over HTTPS. Additionally, using an HTTPS library is preferred due to its thorough security audits, whereas implementing a full resolver with root trust anchors and knowledge of root servers in Bitcoin would be necessary to secure it against a coffee-shop network MITMer, which does not seem like a good idea.Rick Wesson had suggested producing a patch in rpc.cpp to support DNS lookups, but Matt responded that URIs are much simpler and more useful in terms of being able to give different addresses and/or values per user. He suggests using a simple HTTPS server with addresses and client support instead. While Rick agrees on the need for simpler user experience, he also acknowledges the potential limitations of DNS-based resolving. DNSSEC is great if someone is running their own recursive name server, but it seems like way overkill to do name->address mapping. Moreover, DNS is not well-suited for transmitting something that will result in money transfer due to its vulnerability to MITM attacks. What's liked about the DNS is that it is frequently leveraged as a proxy for identity and http URIs are not. Bitcoin, if used in the DNS, might make DNSSEC more popular, which is a good thing, but Bitcoin is still tiny, so it's highly unlikely that it will make a difference to DNSSEC adoption. In conclusion, HTTPS offers several advantages over DNS and DNSSEC for resolving Bitcoin addresses, and it is a simpler and more secure solution for this purpose.
