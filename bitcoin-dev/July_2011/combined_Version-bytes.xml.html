Summary:

Pieter Wuille, in a discussion on IRC, proposed to standardize the version bytes used by bitcoin for different applications. He suggests using three components: network (realnet, testnet, alternate chains), data class (address, private key, master key, etc.), and version (real version per data class defined). Pieter recommends reserving bit 16 in the version byte for alternate chain. When bit 16 is set, everything is up to the network itself, and no further semantics are defined. In addition, he proposes that bits 1 be utilized for testnet, and if bit 1 is set, XOR the rest of the version number with 111. Otherwise, we could reset testnet and simply state odd=testnet, even=realnet. Finally, Pieter suggests using 128, 64, 32, and 8, 4, and 2 to define data classes and version numbers. For example, 0 is used for addresses, 128 for private keys, and 224 for extended data class, another data class byte follows.
