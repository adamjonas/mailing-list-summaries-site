Summary:

Stéphane is questioning the behavior of generation transactions in the listtransaction output, noting that they always appear twice with different account and category listings. He questions if this is intended behavior or if the second should be removed. Additionally, Stéphane asks why account "" is enforced in the first item and whether this is a wrong assumption. Furthermore, Stéphane mentions that {list,get}receivedby{address,account} commands never list or take into account generation transactions. He is unsure how the help message associated with those commands should be understood and whether these issues are related to problems encountered by E-wallet providers. In response to the issue, it is suggested that for coinbase transactions, one "generated" item per txout that corresponds to an owned address can be listed and accounts would be associated accordingly in the same way as for usual "received" amounts. This would simplify the code and make few people rely on these specific details.
