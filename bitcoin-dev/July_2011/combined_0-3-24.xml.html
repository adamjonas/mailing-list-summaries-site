Summary:

Bitcoin developers are considering the release of version 0.3.24, which will address several issues such as flood control limits and network partitioning. The proposed updates include sipa's flood limit fix, DNS seed enabled by default, and UPnP enabled by default for Windows/Mac and disabled by default for Linux. Matt Corallo has set up a reliable hosting for DNS seed which dynamically fills its hostname with random nodes known to be up and accepting connections.There is a concern about the flood control limits becoming an issue again and causing network partitioning. To avoid this, it is suggested to have more listening nodes to support the network. The release notes should clearly explain the changes to avoid confusion between bitcoin and bitcoind.Furthermore, there is a debate whether to cherry-pick or release current upstream with specific fixes. The community is divided on whether UPnP should be enabled or disabled by default. A forum poll showed people were divided on the matter. However, users who do not like UPnP can still disable listening.Gavin is called upon to give his acknowledgement on enabling UPnP by default on bitcoin and disabling it by default on bitcoind. Finally, it is recommended that enabling UPnP for the GUI is probably sufficient in terms of network health.
