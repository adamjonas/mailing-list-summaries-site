Summary:

In a discussion on the bitcoin-dev mailing list, a user asked whether it is necessary to consider that someone may have a timelocked big tx with a lost private key. Peter Todd responded with a suggestion to tell people not to do this. He proposed a fix for the quadratic CHECK(MULTI)SIG execution time issue by limiting only SignatureHash() to return true if the transaction size is less than or equal to 100KB. This allows for a future soft-fork that properly fixes the hashing issue without limiting all transactions or counting sigops. The advantage of this method is its ease of implementation and quick deployment in case of an attack by a major miner exploiting slow-to-propagate blocks. Todd also provided a link to related discussions.
