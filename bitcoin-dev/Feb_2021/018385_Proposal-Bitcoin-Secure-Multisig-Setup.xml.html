Proposal: Bitcoin Secure Multisig Setup

Summary:

A new Bitcoin Improvement Proposal (BIP) has been proposed for Bitcoin Secure Multisig Setup (BSMS) to securely set up multisig wallets across different vendors. The proposal seeks to mitigate the risk of tampering during the initial setup phase and define an interoperable multisig configuration format, while concerns regarding Signers' storage being tamper-proof and using the multisig configuration to generate and verify receive and change addresses are out of scope. The Coordinator initiates the multisig setup and determines what type of multisig is used and how many members and signatures are needed, while the Signer provides its XPUB to the Coordinator, verifies that its XPUB is included in the descriptor record and persists the descriptor record in its storage. The setup process involves two rounds, and security is enforced through a temporary secret token used to encrypt communication between Signers and the Coordinator and descriptor checksum and visual inspection. The proposal introduces two layers of protection, namely the secret token and descriptor checksum and visual inspection. Attackers can modify the membership set by changing an existing member, removing an existing member, or adding a new member, but participating Signers can detect the changes made to their membership and reject the final descriptor.
