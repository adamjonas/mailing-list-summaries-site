Summary:

The post discusses the vulnerability of OpenSSL on Bitcoin and a patch that has been developed to cope with the new OpenSSL versions. The patch is available on stable branches 0.8, 0.9, and rc branch 0.10, and links are provided for users to fetch, build, and test these patches. Users can clone branches of their choice and update their systems accordingly.In addition, users have been warned not to update OpenSSL 1.0.0p / 1.0.1k as it could lead to consensus forks and is incompatible with the Bitcoin system. Tests included with Bitcoin Core in the test_bitcoin utility already detect this condition and fail. The patches fix one symptom of the general problem: relying on software not designed or distributed for consensus use (in particular OpenSSL) for consensus-normative behavior. Therefore, a targeted soft-fork has been proposed to enforce strict DER compliance soon, utilizing a subset of BIP62. Adding a blockchain rule for strict DER will reduce the risk of consensus inconsistencies from alternative implementations of signature parsing or signature verification, simplify BIP62, and better isolate the cryptographic validation code from the consensus algorithm.The post also announces the Go Parallel Website, sponsored by Intel and developed in partnership with Slashdot Media, which is a hub for all things parallel software development. On January 10, 2015, Ivan Jelincic inquired about the vulnerability of openssl1.0.1j, to which a response was given that it is unaffected and concerns CVE-2014-8275, listed under major changes between OpenSSL 1.0.1j and OpenSSL 1.0.1k on the OpenSSL website news section.
