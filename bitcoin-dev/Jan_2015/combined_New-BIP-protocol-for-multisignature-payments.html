Summary:

In a Bitcoin-development mailing list, Martin Habovštiak proposed an extension of BIP70 to solve the security problems faced by servers holding large amounts of bitcoins. The proposal supports making multisig transactions dynamically, with the user providing their address. Mike Hearn acknowledged the idea but noted that a wallet supporting dispute mediation is necessary for its implementation.The thread also discusses the importance of pairing protocol specifications with implementations and the benefits of descriptive standards. The author requested feedback from the community before considering implementing the protocol in the future. They believe that implementing a protocol is much more work than simply outlining its procedures and specifications.Another person responded with a similar proposal involving sharing generic P2SH scripts and facilitating requesting of signatures. Creating a specialist wallet for dispute mediation was proposed, which could eventually be integrated into general-purpose wallets. The email thread also includes links to resources for parallel software development.The email thread discusses the implementation of a dispute mediation transaction protocol in bitcoinj based wallets. The author mentions that it is relatively easy to create one-off, cross-platform app-specific wallets for this purpose. The author offers to provide guidance on implementing this protocol and suggests that eventually, general-purpose wallets could integrate the protocol, UI ideas, and code.The author of the message is proposing an idea to solve security problems for servers holding large amounts of bitcoins such as exchanges and markets. The idea is an extension of BIP70 that supports making multisig transactions dynamically, with the user providing their address. The proposal can be found on Github. The author is looking for feedback on whether this is a good solution or if there is something better. They would like to see this or something similar implemented by wallets.
