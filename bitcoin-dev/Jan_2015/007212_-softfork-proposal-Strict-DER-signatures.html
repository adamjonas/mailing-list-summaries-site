Summary:

In January of 2015, Pieter Wuille proposed a softfork to make non-DER signatures illegal. He suggested this as a way to fix the issue of transaction malleability. The draft BIP text for this proposal can be found on Github. After receiving feedback on Bitcointalk, Gregory Maxwell assigned the proposal the BIP number 0066. Later that year, a pull request was created on Github to implement the strict DER verification behavior and the deployment specified in BIP66 for Bitcoin Core. This implementation required review and testing.
