Summary:

Bitcoin Classic recently released version 1.2.0, which includes new features, bug fixes, and performance improvements. The most notable change is the decentralized block size solution where node operators decide on the maximum size. However, concerns have been raised over the lack of clarity and safety of the release.Eric Lombrozo, a member of bitcoin-dev, accused the Classic version of being "dangerous and irresponsible" due to its incompatibility with the current Bitcoin network and its consensus rules. The release notes also lacked detail about changes made to the code, making it difficult for average users and developers to assess.Despite these concerns, Tom Zander of Bitcoin Classic clarified that the software only activates if 75% of miners adopt it. The goal of the release is to provide users with a high-quality validating node for a large set of use cases.The block size debate between Bitcoin Classic and SegWit continues, with Bitcoin Classic only changing block format while SegWit requires everyone to write code to support the new transaction format. While the activation of Bitcoin Classic happens when 75% of miners adopt it, Lombrozo criticized the lack of community-wide consensus for such a hard fork, making it dangerous and irresponsible.Documentation and communication are crucial when releasing hard fork software, as proper labeling and community-wide consensus prevent confusion and potential harm. Bitcoin software vendors should take great pains to document features and changes from version to version, allowing for re-implementation and acting as part of the consensus-building and peer-review process.During a contentious conversation on a Bitcoin developers' mailing list, concerns were raised about the number of full nodes and the possibility of someone controlling a majority of them. Lombrozo dismissed the concerns, stating that the level of control over a majority of full nodes is irrelevant as someone could easily spin up a bunch of nodes at a trivial cost. The discussion became heated as some members wanted to discuss new proposals while others wanted to revisit old ones. Lombrozo reminded the group of the purpose of the mailing list, which is to discuss new ideas and proposals.
