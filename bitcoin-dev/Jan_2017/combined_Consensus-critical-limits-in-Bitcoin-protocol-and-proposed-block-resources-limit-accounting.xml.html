Summary:

The Bitcoin protocol has two categories of critical limits - the script level limit and the output value limit. The script level limit includes restrictions such as size, nOpCount, stack plus alt-stack size, and stack push size. These limits do not affect the limits of other levels if exceeded. The output value limit states that any single output value must be greater than or equal to zero.These limits can be expanded through a softfork by adding more metrics, ensuring that the maximum value is always taken into account. It is important to keep these limits in mind when using the Bitcoin protocol to avoid exceeding them.
