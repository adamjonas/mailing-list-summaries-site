Summary:

The idea of an adaptive or automatic block size has been discussed, but concerns have been raised that it may not consider the optimal economic equilibrium for transaction fees and size. This approach could create unforeseen externalities for miners and users, as miners may mine small blocks to increase marginal fees, while users may push for larger block sizes causing network issues. To address these concerns, an auto-adjusting limit would require solid code with a social benefit model built-in to optimize benefits while minimizing costs. A proposal called Block75 aims to stabilize transaction fees at the level they were in May/June 2016 by keeping blocks at 75% full. The proposal adjusts the maximum block size based on the average percentage fullness of the last 2016 blocks and maintains blocks as small as possible, allows for growth, and keeps transaction fees at a similar level to May/June 2016. It is expected to alter the max block size to keep blocks 75% full while accounting for changes brought about by SegWit. However, this is just an estimate and alternate ways to model future behavior through the Block75 algorithm are welcomed.
