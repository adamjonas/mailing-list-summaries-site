Summary:

A new concept that offers secure, efficient and privacy compatible model for Bitcoin client has been introduced. The concept known as the BFD can replace BIP37 and be cached between clients without needing to be recomputed. It can also be used by normal pruned nodes to do re-scans locally of their wallet without needing to have the block data available to scan, or without reading the entire block chain from disk. However, a user on the Bitcoin-dev mailing list raised concerns about handling 0-conf/mempool filtering using BDF after it is deployed. The user believes that this issue is important because otherwise incoming funds can take several minutes to hours until they appear. They suggested that using BIP37 bloom filters just for mempool filtering would result in an unwanted privacy-setup. Meanwhile, the system where miners commit to potential transaction inclusion with fractional difficulty blocks is possible but there are some concerns. Although unconfirmed transaction display in a wallet may provide useful information, the user has very little to gain from this knowledge until the transaction is in a block. As such, the privacy trade-off may not be worth displaying unconfirmed transactions in wallets.
