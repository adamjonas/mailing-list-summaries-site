Summary:

In a discussion on the bitcoin-dev mailing list, Police Terror seeks clarity on the rules for transaction replacement in Bitcoin, specifically regarding valid sequence value ranges and changing outputs or adding inputs to pay higher fees. The response suggests referring to BIP125, which outlines the standard way to signal. A GitHub link is also provided for further information. Some miners mine full replace-by-fee, which has no limitations on nSequence. Peter Todd's implementation of this can be found on his GitHub page. While it is faster to use nSequence signaling specified in BIP-125, doing so is not mandatory as long as the replacement transaction can be sent to a full-RBF node. The user in this context attempts to get RBF working on a multisig input, setting the nSequence to 0, but the script fails to verify when using python-bitcoinlib. The user uses SignatureHash(...) method of signing instead of rpc.signrawtransaction(...). Greg Sanders explains that BIP125 is the standard way to signal, with a GitHub link for reference. Additionally, some miners mine full replace-by-fee, which has no limitations on nSequence. It is noted that in practice full-RBF works fairly well, and contact information for Peter Todd is provided on his GitHub page.
