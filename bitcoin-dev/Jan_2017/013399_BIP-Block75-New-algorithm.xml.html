Summary:

On January 2, 2017, t. khan via bitcoin-dev proposed a change to the maximum block size, not the block size itself. The proposal suggested that miners should have full control over the maximum block size since they ultimately decide on the actual created block size. The current protocol allows miners to set the block size but only up to a maximum of 1MB. The proposal keeps this property but questions why there needs to be a maximum block size in the protocol. Tom Zander, the author of the proposal, has a blog and vlog related to cryptocurrencies.
