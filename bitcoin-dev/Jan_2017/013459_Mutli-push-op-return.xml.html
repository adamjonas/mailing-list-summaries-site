Summary:

A proposal has been put forward to make op_return outputs with two pushdatas standard, with the same maximum data size. This proposal is aimed at facilitating tagging transactions so they can be returned by bloom filtering nodes. The use of op_return output script provides an easy way of fetching transactions where the tag matches a specific pattern. It appears that a good number of nodes/miners already accept such transactions as the first block was mined using this method.
