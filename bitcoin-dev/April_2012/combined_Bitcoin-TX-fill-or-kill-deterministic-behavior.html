Summary:

Bitcoin transactions have been a source of concern for developers due to their non-deterministic nature. To address this, several improvements have been proposed by Jeff Garzik, a bitcoin developer. One of his proposals is to have transactions expire if they fail to make it into a block within a certain period of time. This would make the transaction process more predictable and enable users to be notified if their transactions "expire" after N days.Jeff Garzik also suggested that merchants should handle fee calculations themselves instead of forcing customers to think about it. This would require a minor change in how fees are calculated inside the memory pool. However, there are concerns that users may exploit this behavior by creating transactions that they know will get stuck and expire, similar to canceling zero-conf transactions.Another proposal is to explore market-based and negotiated fee systems. However, Jeff believes that there are limited means to determine an ideal fee. The discussion also involves ideas for far more direct payee-payer communication than currently envisioned. Jeff suggests that these sorts of payments and direct communication should be done via a separate protocol than the Bitcoin P2P protocol, perhaps using an HTTP derivative to make life easier for mobile payments, firewalled scenarios, and client-merchant communications.The discussion also focused on exploring a market-based and negotiated fee system for transactions. One proposal is for transactions to be passed around outside the broadcast network, with transactions and their dependencies being broadcast only when there's a lack of trust between recipient and sender.There were also discussions regarding direct payee-to-payer communication, which was originally envisioned as the normal way to make payments. Andreas notes that Bitcoin Wallet for Android already supports directly sending transactions via NFC and QR-Code, with receiving such transactions handled the same way as if received via P2P.Overall, the proposed solutions aim to make Bitcoin transactions more deterministic, predictable, and user-friendly.
