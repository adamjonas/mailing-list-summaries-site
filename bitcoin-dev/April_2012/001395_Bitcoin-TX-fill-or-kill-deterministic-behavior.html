Summary:

In a forum discussion, Jeff Garzik raised concerns about the wallet client's direct payer-payer communication with merchants and suggested a separate protocol for such transactions. However, it was noted that Satoshi Nakamoto initially saw pay-to-IP-address as the primary payment method, with pay-to-address as a backup when the recipient was offline. The original Bitcoin design involved direct communication between participants, but things have evolved differently. Jeff Garzik believes that using a separate protocol on a separate port, possibly an HTTP derivative, would make life easier for mobile payments, firewalled scenarios, and client-merchant communications.
