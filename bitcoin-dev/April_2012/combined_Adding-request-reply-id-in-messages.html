Summary:

The email conversation discusses a proposal to add request/reply ID in all messages of the Bitcoin protocol. The idea is to make robust blockchain downloading easier. However, some members of the email thread argue against the change, stating that stateless protocols are easier to implement and prove correct. They also express concern about opening up nodes to exploitation if they start relying on external parties for state maintenance. Despite concerns, the discussion highlights the importance of ensuring the reliability and security of the Bitcoin network.In addition, Pieter Wuille suggests adding a "denied" message to indicate when a client is unwilling to answer and report transactions that are not accepted into the memory pool. This would help prevent denial-of-service attacks. Amir Taaki and Jeff Garzik discuss a proposed change to the protocol that would rely on the server to track a piece of information. The proposed change was more about distinguishing between replies and spontaneous notifications of the other peer, with all state still being tracked locally on the client side. Amir did not understand why Jeff thought the proposal would make the protocol more stateful as each reply was only the result of the current request and there was no new session information.Gavin Andresen finds the proposal to add request/reply ID fields reasonable and asks for others' opinions. However, Pieter Wuille points out that the Bitcoin P2P protocol is mostly stateless, which makes it more secure. He argues that waiting for "inv" messages and distinguishing between "getblocks" results and spontaneous "inv" notifications can be handled properly without adding request/reply ID fields. The email thread includes discussions on the cost of implementing any protocol change and the potential for opening up nodes to exploitation if they start relying on external parties for state maintenance. Finally, the context mentions a modification made to the behavior of making a second "getblocks" request, which can be found on the GitHub repository of the Bitcoin project.
