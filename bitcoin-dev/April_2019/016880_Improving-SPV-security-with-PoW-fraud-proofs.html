Summary:

The email exchange discusses the approach of making SPV clients download only S+1 instead of both S+1 and S+2. The weakness of this approach is that miners can make SPV clients do more work. However, mining a block which will never be accepted is expensive. SPV clients may not even learn about these splits because it requires someone to relay the split to them, and honest full nodes should not relay such splits. Having SPV clients slow down or become full nodes when a malicious miner with significant mining power is attempting to disrupt the network is probably a best case outcome. It requires UTXO commitments to be mandatory. Any service that can provide the actual UTXO set at each block would potentially be attackable by simply requesting enough past blocks.
