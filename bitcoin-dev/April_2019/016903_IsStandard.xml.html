Summary:

On April 27, 2019, Aymeric Vitte via bitcoin-dev asked about where to find information on standard and non-standard transactions without having to read the core code. Luke-Jr responded that the name "IsStandard" is misleading and has little to do with actual standards. It is simply one function that implements a node's policy and can vary from node to node without breaking consensus. Therefore, it is not even a possible subject for standards. Instead, Luke-Jr recommends focusing on documenting actual standards rather than IsStandard.
