Summary:

The discussion between Ruben and ZmnSCPxj revolves around the validity of UTXO sets and fraud proofs in blockchain technology. While BIP157 manages to function without a commitment, UTXO sets can only be validated by running the entire blockchain as they summarize data in the entire chain. Validation via BIP157 is possible by downloading the block in question, but validation with UTXO sets requires downloading the entire blockchain. Although trusting every peer you have is not omitting the proof is acceptable because it requires one honest peer, peers can be set up to allow hearing of all chains while denying proof of invalidity of some UTXO. This is precisely the "data unavailability claim" that shot down the previous fraud proofs. In combination with BIP157/158, it may be possible to validate UTXO sets if filters contain UTXO spends and a BIP158 filter was committed. However, if new information is added to be committed, it may increase the resource usage of fullnodes. Therefore, it is important to store those commitments in fullnodes to acquire the data being committed to after-the-fact.
