Summary:

Pieter has shared a BIP draft for changing the checksum in native segwit addresses for v1 and higher. The idea is to define a new encoding which is a tweaked variant of Bech32, called Bech32m, which replaces the corresponding section in BIP173. This prescribes using Bech32 for v0 witness addresses and Bech32m for other versions. The discussion on this topic can be found at https://lists.linuxfoundation.org/pipermail/bitcoin-dev/2020-December/018293.html. Pieter is seeking comments, suggestions and ideas from the community.
