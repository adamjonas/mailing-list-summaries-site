Summary:

Pieter Wuille, a Bitcoin developer, has proposed a BIP draft to change the checksum in native segwit addresses for v1 and higher. The proposal includes defining a new encoding called Bech32m, which is a tweaked variant of Bech32, and a new segwit address encoding that replaces the corresponding section in BIP173. The updated website includes error-locating JS decoder and demo, and several reference implementations (C, C++, Python, Javascript) have been updated to support Bech32m. A Bitcoin Core PR has also been opened, and updates have been made to the BIP draft. Nakagat, a Go developer, suggested using hash function instead of Bech32's algorithm for computing checksum, but Pieter argued that Bech32's algorithm performs better for detecting certain subsets of errors.
