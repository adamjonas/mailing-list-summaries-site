Summary:

The question posed on the Bitcoin-dev mailing list is about whether it is possible to modify a Taproot output without changing its address. The response explains that while the 32-byte public key contained in a P2TR address can be used for creating Schnorr signatures or revealing tapscript, modifying the spending conditions would require a commitment to be made. This is because the Taproot equation involves hashing P together with the message to create a commitment. Therefore, any attempt to change the message would result in a different commitment, and moving the on-chain coins would be required to modify the spending conditions. The original Taproot thread is also linked for reference in the response.
