Summary:

Pieter has recently submitted a pull request to re-write BIP32 based on feedback to make it more readable. The changes include replacing the terminology "public" vs "private" with "hardened" for the alternate derivation scheme, adding explicit type conversion functions, and using the suffix 'h' to indicate hardened instead of using the "' " suffix for two different meanings. CKDpriv and CKDpub are used for the derivation functions that operate on private and public keys. Pieter clarifies that these are only documentation changes and the semantics remain unchanged. The pull request is open for comments at https://github.com/bitcoin/bips/pull/12.
