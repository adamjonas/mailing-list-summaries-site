Summary:

The discussion in this context is about the inclusion of CPFP (Child Pays for Parent) in upstream, with BitPay wanting to see it included. However, various people disagree on edge case handling and implementation details, preventing its inclusion. Despite this, there are no fundamental objections to its inclusion.In January 2014, Wladimir responded to a message from Odinn Cyberguerrilla on GitHub regarding issues with the blockchain and microdonations. Wladimir requested that Odinn provide pull requests to address these issues and stated that if Odinn's organization had not contributed anything towards improving the issues, it should not come as a surprise that they did not make it into the Bitcoin codebase. In an email exchange dated January 19, 2014, Wladimir expressed concern over the lack of (auto)tests for pull #1647, which had received comments expressing a need for such tests. However, there was no response from Luke or anyone else regarding who would be responsible for addressing these comments. Wladimir then requested that someone give pull #1647 its final push by addressing the comments so that it could be integrated in the 0.9 release.The Bitcoin team is preparing for a 0.9rc1 release in February 2014, after a long period without major updates. Many new features and improvements have been added to the master branch, but there are still some open issues that need to be resolved before the release. The team encourages users to report any bugs or suggest additional features that should be included in this release. A link to the open issues on GitHub is provided.The conversation about merging CPFP in upstream involves disagreements between people regarding various edge case handling and implementation details, which is preventing the merger. Some proposals are controversial as they affect mining/consensus and Wladimir would like to see ACKs from more reviewers and testers before merging. Jeff Garzik, a Bitcoin core developer and open source evangelist, is also part of the conversation. The emails end with an advertisement for CenturyLink Cloud and a link to join the Bitcoin-development mailing list. Additionally, the email exchange between Odinn and Luke-Jr was light-hearted, with Odinn making a joke about the failure to address issues being "very bad and much news." Meanwhile, the Bitcoin team was preparing for a 0.9rc1 release in February 2014, which would include many improvements and new features. However, there were still open issues that needed to be resolved before the release, and they were requesting input from the community regarding any other issues or bugs that should be addressed.One of the open issues was with the master branch of Bitcoin, where leveldb renamed all .sst files to .ldb, causing the 0.8.x version of Bitcoin to think the index is corrupt. A workaround was to simply rename all the files back to .sst. Also, there was a discussion about merging CPfP (Child Pays for Parent) in upstream, but disagreements on edge case handling and implementation details were preventing its merge. However, there were no fundamental objections to merging the feature.The Bitcoin-development mailing list was discussing the importance of the Child-Pays-For-Parent (CPFP) feature in the 0.9 version of the blockchain.info wallet. Mark Friedenbach stressed its importance, as people have lost money without it. Jeff Garzik from BitPay also expressed his support for CPFP, but noted that disagreements on edge case handling and implementation details were preventing its merge. The message also mentioned that incoming transactions without fees are rebroadcasted to keep them in mempools at several nodes including bc.info.
