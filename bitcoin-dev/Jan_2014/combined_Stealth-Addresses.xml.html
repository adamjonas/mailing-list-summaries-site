Summary:

In a thread discussing payment to Q1 and discovery to Q2, Jeremy Spilman suggests using a 160-bit curve for d2/Q2 and e/P resulting in faster multiplication and reduced OP_RETURN size. However, there is concern that super-fast hand optimized code may not exist and the extra code required would make it slower in practice. In another thread, Ben Davenport suggests changing the name 'stealth address' to something more neutral like 'static address'. Alan Reiner suggests using ECDH to create multiple payment addresses without querying the payee. When Gavin Andresen requests an easy solution for non-geeks, Alan Reiner explains that stealth addresses are necessary for cases where real-time two-way communication is not available, such as donation addresses on static webpages or billboards.
