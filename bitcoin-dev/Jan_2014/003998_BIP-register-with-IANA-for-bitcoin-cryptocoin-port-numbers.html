Summary:

The discussion is about having one or two ports for Bitcoin, one for the main network and another for the test network. It is suggested that if conservation is an issue, the testnet port can be forgone as there is no logic to multiplexing them since they are entirely separate networks. In case the port is already in use, Bitcoin can run on any other available port and announce it. There isn't even an argument for convenience since most users don't specify nodes and can specify a port when necessary. The issue of alt coins reserving ports is not a concern for Bitcoin and it may be realistic for Bitcoin to allocate one or two ports for its use.
