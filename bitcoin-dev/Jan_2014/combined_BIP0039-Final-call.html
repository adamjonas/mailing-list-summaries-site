Summary:

The Bitcoin community has been discussing the implementation of a wordlist for a new standard called BIP39. The current draft no longer requires a specific wordlist, allowing each client to use its preferred set of words. However, there are concerns about users losing money due to fat-finger errors and the inability to verify the mnemonic passes its checksum. Peter Todd expressed concern over the security of the proposal and refused to approve it as a standard. In response, Slush clarified that Trezor and bitcoinj were going in this direction, which is of interest to 100% of clients interested in bip39.Additionally, MultiBit, a popular Bitcoin wallet, plans to implement BIP39 using the implementation present in Bitcoinj. Proper Unicode handling is also an important issue being addressed to prevent accessibility issues for users who may switch input methods or machines. To ensure accessibility for all users, MultiBit will allow users to select their language and locale. If necessary, the system will fallback to English. This approach will ensure that MultiBit's Bitcoin wallet remains accessible and user-friendly for users worldwide.The email thread discussing the implementation of BIP39 also includes discussions about the compatibility of different word lists used in the process of transforming a seed into mnemonic, the implementation of BIP39 client-side versus in Trezor firmware, and the use of brute force methods to generate a seed. There are concerns about the computational requirements of devices such as Trezor when using checksums that do not depend on the wordlist.In conclusion, the implementation of BIP39 and the use of a wordlist has been a topic of discussion within the Bitcoin community. While the proposal has been improved based on feedback from the community, there are still concerns about the security and accessibility of the proposed solution. MultiBit is taking steps to address proper Unicode handling and ensure accessibility for users, while also implementing BIP39 using the implementation present in Bitcoinj.
