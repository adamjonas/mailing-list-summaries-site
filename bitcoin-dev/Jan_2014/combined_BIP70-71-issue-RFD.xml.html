Summary:

This email thread discusses the issue of using BIP70/71 payment protocol in face-to-face payments. Andreas Schildbach, an experienced developer in the Bitcoin industry, encountered a problem with the protobuf format as they are not self-delimiting and can cause endless reading if the stream is undelimited. The Java protobuf API offers delimited I/O as a solution to this issue by writing the size of the message as a varint before writing the data using payment.writeDelimitedTo(os). Mike Hearn thinks that protobuf varint length prefixes should be used for binding the payment protocol to those transports but is unnecessary for all cases. Gavin Andresen suggests that message encoding, length, terminator, checksum or error correction should be included in the transport protocol. Both Schildbach and Hearn agree that adding these length prefixes is open for discussion. NFC NDEF and QR codes are not affected by this issue.However, it has been identified that HTTP request/responses being affected by broken proxies and other streams as opposed to files. This problem does not affect NFC NDEF and QR codes. As a result, it has been suggested that the delimiter can be skipped for these mediums. It is important to keep this in mind when dealing with HTTP requests and responses, as well as other forms of communication that involve streams rather than files. By taking this into consideration, it may be possible to prevent or mitigate any potential issues that could arise from this problem.In conclusion, the issue of using BIP70/71 payment protocol in face-to-face payments has raised concerns about the compatibility of protobufs and the need for length prefixes. Schildbach suggests applying the delimited I/O solution to all messages, while Hearn believes that length prefixes should only be added for specific cases. Andresen proposes including message encoding, length, terminator, checksum or error correction in the transport protocol. It is also important to note that HTTP request/responses via broken proxies and other streams are affected by this issue, while NFC NDEF and QR codes are not.
