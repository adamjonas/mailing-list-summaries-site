Upon further reflection, there is confusion regarding the use case of the payment protocol. Currently, a PaymentRequest contains the outputs that specify the addresses to send to. Reusing a PaymentRequest like this without using stealth addresses implies address reuse, which highlights the need for extending the protocol in a future version.
