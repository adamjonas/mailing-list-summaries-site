Summary:

A user on the bitcoin-dev mailing list is curious about the possibility of encrypting text data in a Bitcoin transaction so that only the receiver can read it, without exchanging information off the network. They are interested in exploring the use of sidechains or RSK for this type of encryption and have specific uses in mind for it. A response to the query suggests that theoretically, if the recipient address has already disclosed the public key or shared it with the sender alongside the address, then the sender could use the recipient's ECDSA key to encrypt the message with Elliptic Curve Cryptography (ECC). However, in practice, it is uncertain whether nodes would broadcast such a non-standard transaction. The original post invites others to share their ideas for potential uses of encrypted transactions.
