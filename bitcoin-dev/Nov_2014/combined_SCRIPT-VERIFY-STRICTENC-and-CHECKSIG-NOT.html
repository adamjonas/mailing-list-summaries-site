Summary:

On November 6, 2014, Peter Todd reported a flaw in the Bitcoin protocol stating that git head was accepting an invalid transaction into the mempool which spends the redeemScript. This could allow double spending attacks and increase the risk of fraud. While this vulnerability doesn't affect IsStandard() transactions, it allows fake transactions to fill up the mempool. The emails between Pieter Wuille and Peter Todd discuss the vulnerability of pubkey formats that are not recognized by the STRICTENC flag. They proposed two solutions: either make STRICTENC fail unrecognized pubkeys immediately or fail the script if the pubkey is non-standard and signature verification succeeds. The latter was implemented as a "least invasive" measure, as hybrid encoding for pubkeys still needs to be supported for a while.Pieter Wuille suggested adding a second validity check with the actual consensus rules. However, it was decided to implement the solution proposed by Peter Todd. It was also acknowledged that making this a consensus rule could make existing P2SH outputs/addresses unspendable. Meanwhile, Bitcoin-Ruby forked yet again, but no further information was provided on the topic.
