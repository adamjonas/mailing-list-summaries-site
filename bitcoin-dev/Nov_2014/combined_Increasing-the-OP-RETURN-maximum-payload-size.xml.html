Summary:

The discussion is regarding the use of OP_RETURN. The main concern with OP_RETURN is that it encourages people to use blockchain as a convenient transport channel. For Open Assets, they need to store a URL in the OP_RETURN output with some bytes of overhead, and 40 bytes are not enough for that purpose. Storing only a hash is not enough to address an HTTP or FTP resource. Most colored coin applications do not require everyone to know the data published. They just need to send external data directly to the receiver with transactions committing to it, which is sufficient. In this regard, Pieter suggests using solutions that do not require additional data. In the context of HD wallets, Chris Pacia mentions that if users wish to use stealth addresses with out-of-band communication, the benefits of HD would be lost, and they would have to make regular backups. However, Pieter explains that anything that lists previous transactions needs backups. The benefit of HD wallets is that you can separate secret data and public data, and you only need one safe backup of the master secret key.
