Summary:

In 2014, Francis GASCHET posted a query on the storage, RAM, and bandwidth requirements for setting up a full Bitcoin node. He expressed concern about the decreasing number of full nodes with appropriate resources in France and planned to set up a node to contribute to the network. Thomas Zander suggested checking out other people's opinions on the topic and directed him towards a stats script running on a particular node.Daniel F provided guidelines for storage, RAM, and bandwidth resources required to run a full node. The disk usage for blockchain data is currently around 30GB and may continue to grow slowly. He suggests allocating at least 50GB of disk space for a full node. CPU usage is expected to be minimal after syncing the blockchain. RAM usage may reach up to 1.5 GB after running for a few months. Bandwidth usage averages about 500GB of traffic per month, mostly outgoing.Jameson Lopp and Melvin Carvalho also shared their experiences and provided guidelines for running a full node based on statistics from their own nodes. Jameson Lopp's Bitcoin node system metrics charts show abnormal CPU spikes due to automated RPC calls to query the UTXO set. The bandwidth usage chart for his node can also be found on http://statoshi.info/#/dashboard/file/default.json?panelId=1&fullscreen. Melvin Carvalho provided similar guidelines based on statistics from his own node. CPU usage is not a problem once the blockchain is synced. Disk usage for blockchain data is currently around 30GB and may continue to grow slowly. RAM usage after a few months of operation is approximately 1.5GB. Bandwidth usage averages about 500GB of traffic per month, mostly outgoing.Gaschet had the capacity to host an HA server in a secured environment through his company's dual-homed internet access. He successfully set up his own node using Linux HA + dual homed Internet transit and his node is up and running on 5.56.40.1:8333.
