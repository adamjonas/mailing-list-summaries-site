Summary:

In a thread on November 6, 2014, Pieter Wuille and Peter Todd discussed the implementation of the STRICTENC flag in Bitcoin. The STRICTENC flag was designed to make pubkey formats that were not recognized by Bitcoin act as though they were invalid signatures, instead of failing the transaction. Todd found that this allowed mempool garbage transactions that would never be mined. Wuille agreed with Todd's suggestion to change the implementation of STRICTENC to either fail unrecognized pubkeys immediately or fail the script if the pubkey is non-standard and signature verification succeeds. However, Wuille noted that this change would make existing outputs difficult to spend, and it could not be made into a consensus rule as it may make existing P2SH outputs/addresses unspendable.
