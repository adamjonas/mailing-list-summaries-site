Summary:

The satoshi client v0.7 is set to undergo a change with the addition of a new command 'mempool'. This change will enable simplified payment verification (SPV) clients to access the memory pool and allow for diagnostics and miner download. Additionally, the "getdata" behavior will be extended to include any memory pool transaction, easing a previous restriction on the "getdata" behavior. While this proposal did not require a Bitcoin Improvement Proposal (BIP), Jeff Garzik offered to write one if needed. Jeff Garzik has proposed a small change to the protocol for the upcoming release of satoshi client v0.7. The change involves adding a 'mempool' P2P command and extending 'getdata' behavior. Wladimir fully agrees with the changes but suggests that a small BIP should be created for consistency and to document it for other client developers. This would ensure that the new changes are well-documented and consistent across all client developers.
