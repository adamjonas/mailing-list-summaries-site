Summary:

The current version of MSG_MEMTX has a higher Just Works value. It is better to do something unconditionally than have a response generated only under certain conditions for empty "inv". Unknown messages are ignored intentionally for expansion, as Alan notes. Unconditionally returning a response has little to do with feature probing/discovery, but it provides a clear and deterministic indication that processing is complete for each invocation.
