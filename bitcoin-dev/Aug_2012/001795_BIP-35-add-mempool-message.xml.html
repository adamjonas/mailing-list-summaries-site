Summary:

In this email exchange, Pieter Wuille and Jeff Garzik discuss the topic of nodes getting a memory pool refill at startup. They both agree that this is interesting for nodes to have. The conversation then moves on to the use of an "inv" message, which is always returned even if empty. Jeff questions the usefulness of this message as inv packets can always be sent, without requiring a response, and suggests sending a ping(nonce) after each P2P command instead. The conversation also touches upon the allocation of an extra service bit for the mempool-providing services. Although an nServices bit may be a better fit for this optional service, Jeff chose to be conservative due to scarce-resource concerns.
