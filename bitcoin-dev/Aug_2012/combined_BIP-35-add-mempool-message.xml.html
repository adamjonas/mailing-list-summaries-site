Summary:

Pieter expressed his disagreement with Jeff Garzik's proposal to use an empty "inv" message as a way of replying to the request for an empty mempool. He argued that Bitcoin P2P is not a request-response protocol and this approach would be hacking a request-response system on top of the asynchronous system. Instead, Pieter suggested using MSG_MEMTX for confirming the transmission of the mempool. In another email, Pieter provided feedback on BIP 0035 proposed by Jeff Garzik, which enables querying the memory pool of a node. While Pieter liked the idea of being able to query the memory pool, he questioned the usefulness of returning an empty "inv" message and suggested allocating an extra service bit for mempool-providing services to be discovered before connecting to them.
