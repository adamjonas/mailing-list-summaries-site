Summary:

Велеслав has proposed a draft BIP for a new transaction template that defines a unique transaction structure that can be used as a cryptographic pseudonym. The proposal is not expected to bring any consensus changes and is meant for the application layer of Bitcoin. The writer hopes that the community will find this proposal useful and provide a careful review. The first BIP within the project is available on Github, and the writer thanks the entire team that supported him in creating this proposal.Kalle raised a few comments and questions on the proposal, including why only one input is demanded, why exactly type p2wpkh on input is demanded and why verify "SIZE 32 EQUALVERIFY" on output 2. Additionally, Kalle questioned the requirement for a segwit version 0 change output, and asked for clarification on what is considered IsStandard rules and what is nym protocol rules in the specification section. Kalle also could not find information on what 1-byte-nym_version and 1-byte-nym_use are and how they are used.The proposal can be found on Github, and Велеслав thanked the team that supported him in creating this proposal.
