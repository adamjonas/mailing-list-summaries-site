Summary:

In a discussion about the separator for Bitcoin addresses, it was determined that the & symbol could be used and no delimiter would be needed. The returning value would either be an address or an array of addresses. It was suggested to name the parameter "address" instead of using a shortcut. On October 18, 2018, Clark Moody proposed making the system future-proof by passing a list of addresses to "addr" using the syntax bitcoin:[?addr=[address1,address2]].
