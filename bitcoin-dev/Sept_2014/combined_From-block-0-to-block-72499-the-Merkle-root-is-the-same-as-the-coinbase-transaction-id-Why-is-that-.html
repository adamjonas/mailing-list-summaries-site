Summary:

Peter Grigor, in a Bitcoin-development mailing list, explains that only certain transactions in the Bitcoin blockchain have associated Coinbase transactions and nothing else. The merkle root of a tree with just one item is the hash of that item. On September 20, 2014, Peter Grigor asked why the Merkle root is the same as the coinbase transaction ID from block 0 to block 72499. This is because the merkle tree algorithm works by pushing all the txids onto the vMerkleTree vector, which is initially blank. For most of the early blocks, there was just the coinbase transaction and no other transactions, so the for loop never executes because nSize is always equal to vtx.size() == 1.
