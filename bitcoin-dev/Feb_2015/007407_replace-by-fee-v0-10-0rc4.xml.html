Summary:

In order for the replace-by-fee patched core to remain a useful border router, it should only relay double spend if it actually replaces an earlier transaction. This is because replicating the replace logic in the proprietary stack would cause the mempool to become out of sync with that of the border router. The suggestion was made to use the getrawmempool RPC call to synchronize mempool contents.
