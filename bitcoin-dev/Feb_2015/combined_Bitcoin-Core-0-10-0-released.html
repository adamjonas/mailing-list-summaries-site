Summary:

Bitcoin has released two new versions of its software, Bitcoin Core 0.10.0 and 0.13.0, both of which include various changes, enhancements and bug fixes. The release notes detail the changes in the Bitcoin Core v0.10.0 release, including a new DNSseed policy and improvements to the LevelDB database. The release also includes contributions from a large number of developers and translators, including Andreas Schildbach, Gavin Andresen, Peter Todd, and many others. Bitcoin Core version 0.10 includes faster synchronization for the downloading of blocks and headers, improved signing security, watch-only wallet support, and relaxed standard script rules for P2SH redemption scripts. It also introduces new command-line options, block and transaction handling, mining and relay policy, as well as BIP 66.Version 0.13.0 introduces separate flushing modes, a locking mechanism, P2P protocol and network code changes, validation changes, build system changes, wallet changes, GUI changes, and test changes. Both updates are expected to provide improved stability and functionality for users going forward. Bitcoin Core version 0.10.0 represents a significant step forward for the Bitcoin network, improving both its reliability and usability. Users can expect to see improvements in performance, security, and overall functionality. Additionally, there is support for multi-wallets, allowing users to manage multiple wallets within the same installation. There are several notable changes that come with Bitcoin Core version 0.10.0, including an upgrade of the OpenSSL library to version 1.0.1m, which includes fixes for the Heartbleed bug. Other changes include enhancements to the peer-to-peer networking protocol, such as better handling of incoming connections, and improvements to the user interface. A full list of changes and updates can be found on the Bitcoin website. Overall, these updates are expected to provide improved stability and functionality for users going forward.
