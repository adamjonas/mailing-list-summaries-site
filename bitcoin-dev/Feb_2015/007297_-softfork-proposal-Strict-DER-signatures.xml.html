Summary:

On January 25, 2015, Pieter proposed an upper bound for R/S length in IsStandard and STRICTDER. Gregory Maxwell agreed to fix it at some point. To make signatures with too-long R or S values non-standard, a patch can be added to version 0.10 or BIP66's DERSIG can be modified. The latter is considered a simpler alternative as it requires only a one-line change and is obviously softforking. Pieter asked if anyone opposed this modification at this stage.
