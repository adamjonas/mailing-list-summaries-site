Summary:

A discussion on storing large and small data using witness and OP_RETURN respectively was held by Aymeric Vitte on February 5, 2023. It was suggested that if one wants to store something big and can afford several transactions in their design, they should use witness. Whereas, if one wants to store small things like signatures, addresses hashes, and some metadata and their design does not make several transactions easy, then they should use OP_RETURN. However, the question arises of how to move forward with several OP_RETURN and no size limit. The answer lies in the fact that OpReturn isn't in UTXO space and doesn't even take advantage of the witness discount, so it clearly has minimal impact. Therefore, there's no reason to micromanage exactly how people use it. They should be allowed to decide for themselves with the fee market. This is similar to how artificial limits were not put on Taproot.
