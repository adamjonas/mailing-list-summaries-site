Summary:

The Bitcoin-dev mailing list recently discussed the storage of NFT content as witness data using the ordinal scheme. There are concerns about the future disk usage of unpruned nodes and the practicality of allowing unlimited storage. Robert Dickinson suggested imposing a size limit similar to OP_RETURN for such inscriptions. Another proposal was linking a sat to a legal construct for proof of ownership in the real world. However, storing the actual property itself on the blockchain seemed nonsensical. Members also raised concerns about potentially malicious content being stored on the main chain.Andrew Poelstra expressed concerns that allowing NFTs and other non-currency data on the Bitcoin chain could entangle the fee market with pump&dump markets, undermining legitimate use cases and preventing new technology like LN from gaining a strong foothold. Despite this, Aymeric Vitte proposed a bitcoin-based NFT system that can reference millions of low-value NFTs in one transaction. While his system is centralized, he believes it is a good solution compared to existing ones.Members proposed various solutions, including using an opt-out feature or extension blocks like Litecoin, continuously educating people, and charging a premium for storing images in the blockchain. However, there were doubts about the feasibility of implementing these solutions. Some members believed that finding a way to impose a size limit for inscriptions would be beneficial, while others argued that there is no sensible way to prevent people from storing arbitrary data in witnesses without breaking legitimate use cases.In 2017, multisig transactions were made cheaper and more efficient. However, there are still concerns about the potential storage of malicious content on the main chain. The author proposes a "local bitcoin" that is compatible with Bitcoin code but does not store anything in the main Bitcoin chain, making it an alternative to Ethereum and a suitable option for low-value NFTs such as loyalty programs and discount coupons. The author provides proposals for a Bitcoin-based NFT system and links to their CV, LinkedIn profile, GitHub repositories, and various projects including a Universal Coin Swap system, a bitcoin NFT system, and Bitcoin transactions made simple.As block space becomes limited, there are concerns that soft fork opcode tech improvement proposals may not be accepted by the community, potentially impacting the adoption of useful technologies such as schnorr sigs for anonymity and cheap multisig purposes. Implementation of vault tech would also be highly helpful in ensuring secure storage of assets. Overall, the discussion on the Bitcoin-dev mailing list revolves around the potential risks and benefits of storing NFTs and other non-currency data on the Bitcoin chain, with various proposals being suggested to address these concerns.
