Summary:

In a bitcoin-dev mailing list post, Alice Wonder expressed her frustration with testing errors in Bitcoin Core updates. She almost didn't update to version 0.13.0 because of Python errors in the test suite, which were fixed by following instructions on bitcointalk. However, version 0.13.1 came with new Python errors and she decided not to update at all until the test suite works properly in her CentOS environment. Douglas Roark requested more information about the issues and offered help, suggesting that posting an issue on Github would be useful.Alice explained that the LANG environment variable was the main cause of the problem and she had to set it to "en_US.utf8" before running the test suite for version 0.13.0. However, she noticed that this issue was fixed in version 0.13.1. The errors encountered in version 0.13.1 arose while running the "bitcoin-util-test.py" file, which led to a traceback error. The error message shows that the test failed because it was unable to find the file "./test/data/blanktx.json", resulting in an Error 1. Alice also mentioned that she builds in a clean chroot build environment to avoid linking to non-standard libraries and that normally, the LANG environment variable does not affect software compilation. The error message displayed is from version 0.13.1 of Bitcoin. Specifically, running the "bitcoin-util-test.py" file caused the error. The traceback shows that the issue is with the "bctest.py" file. In particular, the line of code that reads the output data from a file is causing a FileNotFoundError due to the specified file not being found in the directory. This issue was previously known on the master branch. If users encounter similar issues on the current 0.13.2 release candidate, they are advised to report it on the Bitcoin GitHub issues page and include the version of CentOS they are running. Despite these issues, Alice reported that version 0.13.0 worked fine once the LANG variable was set to "en_US.utf8".
