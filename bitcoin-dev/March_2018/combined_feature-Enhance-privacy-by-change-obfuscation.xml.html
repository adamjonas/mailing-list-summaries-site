Summary:

There is a new option available in Electrum v3.0.6, which enables multiple change addresses, but it is currently off by default. In response to the proposal of enhancing privacy through change obfuscation, Eric Voskuil corrected a common misperception regarding the cost of unspent outputs. He stated that there is no necessary additional cost to the network from the count of unspent outputs and that this perception arises from an implementation detail of particular node software.Bitcoin Core has introduced an 'Enhanced privacy' option for transaction creation, which enhances privacy by change obfuscation. The feature allows the user to randomly distribute change across up to twenty output addresses (minimum five), as long as each output is not dust. It is suggested that the total random number of addresses to distribute to should be limited by change amount. Additional inputs can be selected if available to increase change, although this may eventually result in a decrease in obfuscation in some cases when the outputs are spent. However, the transaction cost will be higher for the initial spend with the change due to increased outputs and possibly for later spending the change depending on the future spend amount(s) and the number of inputs required.Although transaction linkage is still possible, it is far more difficult to guess what was retained by the owner of the originating utxo's unless the new change outputs are spent together in the same transaction. Overall, the 'Enhanced privacy' option provides an improved level of privacy for Bitcoin Core users. Damian Williamson proposed adding an "Enhanced privacy" option for transaction creation to randomly distribute change across up to twenty output addresses, but Evan Klitzke argued that it would be expensive for the network due to the bloat in UTXO size, which everyone would have to pay for. Moreover, it doesn't seem private as the wallet will have to rejoin those inputs in future transactions, resulting in a high transaction fee for the user. Thus, despite understanding the additional cost, Klitzke did not support the idea.In summary, while there is a suggestion to add an "Enhanced privacy" option for transaction creation to randomly distribute change across multiple output addresses, it may not be as private as intended due to the high transaction fees for users and network bloat in UTXO size. However, Bitcoin Core has already introduced an 'Enhanced privacy' option for transaction creation that enhances privacy by change obfuscation, allowing users to randomly distribute change across multiple output addresses and provides an improved level of privacy for Bitcoin Core users. Additionally, Eric Voskuil corrected a misperception regarding the cost of unspent outputs, stating that there is no necessary additional cost to the network from the count of unspent outputs.
