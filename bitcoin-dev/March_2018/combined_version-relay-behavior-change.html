Summary:

On the Bitcoin-dev mailing list, Eric Voskuil raised concerns about nodes running /Satoshi:0.15.0/ and later versions of Bitcoin Core that no longer appear to be following the version.relay=false flag (BIP37). Andrew Chow responded, suggesting that these nodes may not actually be forked from Bitcoin Core, but rather fake nodes that do not follow the protocol or perform verification. Despite reviewing relevant Core sources and syncing against a Core node locally, Voskuil still suspects this behavior is related to Core, as it is very common and all user agents are consistent. However, he does not see any issues with Core sources or his own flag and version settings. It is unclear whether this behavior is intentional or a bug, and specific details on how to reproduce the issue have been requested. The email thread discusses dropping peers for protocol noncompliance as the best solution.
