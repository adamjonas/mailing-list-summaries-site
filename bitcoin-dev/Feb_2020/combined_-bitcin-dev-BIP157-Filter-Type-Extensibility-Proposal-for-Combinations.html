Summary:

BIP157, a Bitcoin Improvement Proposal, has defined the "Filter Types" section which allows for future extensibility and reduction of filter sizes. This proposal enables full nodes to generate one filter per block per filter type supported. Each type is identified by a one byte code that specifies the contents and serialization format of the filter. While it provides a way to extend to multiple filter types, it does not provide a way to extend to filter type combinations. However, it is possible to achieve this by defining the filter types with powers of two. The idea was originally detailed in a GitHub issue and can be proposed to the mailing list for consensus before an update to the BIP is made.
