Summary:

Segwit changes the way transactions are serialized and excludes signatures from the TXID to fix the transaction malleability issue. The normalised TxID approach in BIP140 can also solve this issue using a softfork without changing the definition of TxID. However, it has the downside of increasing UTXO overhead and not providing incentives for controlling the size of the UTXO set. Segwit, on the other hand, has additional benefits outlined in BIP141 and is more suitable for a new cryptocurrency. The format of segwit does not separate signatures from the rest of the transaction data but excludes them from the TXID. There are different ways to hash a transaction with different SIGHASH flags, making it impossible to compute the Transaction ID the same way as the hash for signing the transaction. Changing the definition of TxID is a hardfork change, requiring everyone to upgrade or causing a chain split.
