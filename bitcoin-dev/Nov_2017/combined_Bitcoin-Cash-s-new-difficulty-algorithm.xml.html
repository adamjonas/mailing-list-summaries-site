Summary:

Bitcoin Cash is set to implement a new difficulty algorithm with their hard fork on November 13th. The proposed algorithm is regarded as excellent because it followed the development mailing list's guidelines for non-contentious changes and follows the "simplest is best" route. Bitcoin may also need to hard fork to use a similar algorithm if necessary. One of the contenders to the current algorithm is Degnr8's D622 difficulty algorithm, which gives a linearly higher weight to the more recent timestamps. Scott Roberts recommends that Bitcoin consider using Bitcoin Cash's new algorithm and making it N=50 instead of 144. The current averaging window for Bitcoin's difficulty algorithm is probably significantly too long (N=144).In an email sent on November 2, 2017, Scott Roberts praised the code of a group that he had previously criticized for their adversarial nature. Despite his reservations, Roberts presented the code as a viable solution to a threat facing bitcoin. He expressed concern that mentioning the word "Cash" in relation to the code would turn the discussion into a political issue.The bitcoin-dev mailing list received a post from Scott Roberts, recommending the implementation of a new difficulty algorithm for Bitcoin. Gregory Maxwell responded to this by stating that the mailing list was meant for Bitcoin development and not for reviewing proposals from adversarial competing systems. Maxwell criticized the proposal as being from an adversarial competing system and stated that it is off-topic for the mailing list. The discussion ended with a reminder that the mailing list is intended for Bitcoin development, and any off-topic posts should be avoided.There is some concern that changes to the DAA (difficulty adjustment algorithm) for Bitcoin Cash (BCH) may have implications for Bitcoin (BTC), and developers are advised to watch what happens as BCH goes live with a much better algorithm, in case BTC needs to hard fork for the same reason and needs a similar fix. Overall, the discussion highlights the need for a more efficient difficulty algorithm for both Bitcoin and Bitcoin Cash, and suggests several potential solutions. However, there is also concern about off-topic posts on the development mailing list and the politicization of technical discussions. Developers are recommended to closely monitor the situation and consider implementing a new algorithm if necessary.
