Summary:

Simplicity is a proposal for a new blockchain programming language that aims to be more secure and efficient than existing languages. One of the key features of Simplicity is the use of "jets", which are precompiled functions that can be called directly from Simplicity code, rather than being interpreted. The aim is to identify certain popular Simplicity expressions and allow the interpreter to skip over them and directly evaluate the function using specialized C or assembly code. When the interpreter encounters an expression with a jet, it can look for it in a predefined jets dictionary and find the binary for a precompiled, formally proven implementation of a function that is equivalent to the original Simplicity code. Jets are briefly discussed in section 3.4 of the Simplicity proposal document. For instance, when the interpreter encounters the ECDSA verification expression, it can directly call into libsecp instead of continuing with interpreted Simplicity.Jets can help make Simplicity programs lighter on CPU usage and replicate the behavior of complex chunks of Simplicity code without introducing side effects. The proposal suggests that discounted jets will be explicitly labeled as jets in the commitment, and a Merkle path from the root to an explicit jet node can also be provided. In addition, when Simplicity code for a jet is defined as "consensus" in a softfork, it mitigates the need for it to be provided within every script output. By formal verification, it would be ensured that they match.Hampus Sjöberg asked on a mailing list whether the Simplicity code for a jet still needs to be publicly available/visible in the blockchain and if there is any way to mitigate this. Russell O'Connor replied that the plan is that discounted jets will be labeled as such in the commitment, and a Merkle path can be provided from the root to an explicit jet that isn't among the finite number of known discounted jets, it will also be considered a jet.Jets are expected to increase the efficiency of the blockchain language while ensuring its security. In summary, Simplicity proposes the use of "jets" in its blockchain programming language that allows for recognition of popular Simplicity expressions and direct evaluation of functions using specialized C or assembly code instead of interpreted Simplicity. The plan is that discounted jets will be labeled as such in the commitment, and a Merkle path from the root to an explicit jet node can also be provided.
