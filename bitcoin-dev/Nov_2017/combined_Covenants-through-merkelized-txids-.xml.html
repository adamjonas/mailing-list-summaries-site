Summary:

The author proposes an alternative to jl2012's BIPZZZ (OP_PUSHTXDATA[1]) by using OP_CHECKSIGFROMSTACK, which was implemented by Russell for covenants. The proposal suggests using OP_TXMERKLEVERIFY to compare the top stack element against a merkle tree of the current transaction. TXMERKLE is constructed using a combination of nVersion | nLockTime | fee, inputs & outputs. Inputs and outputs are calculated using a combination of inputmerkle and outputmerkle. The input is calculated using prevoutpoint, nSequence | inputAmount while the output is calculated using nValue, scriptPubkey.The author notes that there are many variants of this proposal, but using OP_CAT would make it easier to test a particular tx property. Furthermore, having a dedicated OP_MERKLE would simplify the process even further. If there is ever a hard fork and merklized TXIDs are added, this method could also be used to inspect the tx being spent.
