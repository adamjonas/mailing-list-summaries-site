Summary:

Johnson Lau has proposed to make OP_CODESEPARATOR and FindAndDelete in non-segwit scripts non-standard. The proposal can be found at https://github.com/bitcoin/bitcoin/pull/11423. The author argues that FindAndDelete() is one of the most useless and complicated functions in the script language, which is omitted from segwit but still needs to be supported in non-segwit scripts. Disabling both guarantees that scriptCode serialized inside SignatureHash() must be constant. Non-segwit scripts also use a FindAndDelete()-like function to remove OP_CODESEPARATOR from scriptCode. However, in BIP143, only executed OP_CODESEPARATOR are removed so it doesn’t have the FindAndDelete()-like function. OP_CODESEPARATOR in segwit scripts are useful for Tumblebit so it is not disabled in this proposal. A softfork could be used to remove FindAndDelete() and OP_CODESEPARATOR from non-segwit scripts and completely remove FindAndDelete() from the consensus code later by whitelisting all blocks before the softfork block. The first step is to make them non-standard in the next release.Mark Friedenbach disagrees with this proposal because it has the potential of destroying value and is something that we have only done in the past either because we didn’t understand forks and best practices very well, or because the features (now disabled) were fundamentally insecure and resulted in other people’s coins being vulnerable. He thinks that making these features non-standard may be a good idea to further discourage their use.
