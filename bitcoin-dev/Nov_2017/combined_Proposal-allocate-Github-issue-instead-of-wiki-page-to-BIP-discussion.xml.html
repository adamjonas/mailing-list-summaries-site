Summary:

The author of the post suggests using Github issues for each Bitcoin Improvement Proposal (BIP) in order to make the commenting process smoother. Currently, each BIP has a wiki page for discussion, but leaving comments requires editing the page. However, there are concerns about moving discussion away from the mailing list and confusion between pull requests and discussion issues. To prevent this, the issue could be temporarily locked. If people agree with the idea, a change to the current BIP commenting process would need to be proposed on Github. Sjors Provoost, a member of the bitcoin-dev mailing list, suggested enabling Github issues for BIPs.
