Summary:

A newcomer to the Bitcoin-dev mailing list has suggested a DNS-like decentralized mapping service for cryptocurrency addresses. The idea is to simplify the process of using long wallet addresses by providing a human-readable address that can be converted to a wallet address with network confirmations. Several members of the community have suggested resources for further research, including Namecoin, Blockstack, and WalletNames.com.Bitcoin enthusiasts are discussing the possibility of creating a DNS-like decentralized mapping service for crypto addresses, which would provide users with a more user-friendly address format. Instead of long wallet addresses or QR codes, this system would allow for human-readable addresses that can be easily converted to wallet addresses by the software. Using an additional service like DNS introduces an additional availability risk, but there is no additional privacy risk providing each mapped name or address was only used once to send/receive one payment unless something personally identifiable like an email address was used. Some members of the community see the addition of DNS as a potential security risk, while others find it an interesting draft that could be turned into something useful and leveraged with BIP32 to allow payment requests to be generated that automatically pointed payees to the correct branch.Douglas Roark has suggested leveraging BIP-47 to associate payment codes with email via DNS and getting rid of the announcement transaction in BIP-47 and establishing a shared secret out of bounds. The sender could then email the recipient the information they need to retrieve funds, and the first transaction could have a time-locked refund in case the payment code is stale. DNSSEC may have some issues, but it remains the easiest way to bootstrap identity to a common, reasonably secure standard.Namecoin, the original altcoin, attempted to build a blockchain-based decentralized DNS system similar to what is being proposed. However, it did not gain widespread adoption. Mandar Mulherkar suggested that such a system could provide users with a human-readable address for cryptocurrency transactions, instead of long wallet addresses or QR codes. In response, Lucas Clemente Vella suggested that OpenAlias.org could be a solution to this issue. Another responder recommended using WalletNames.com, which provides users with a human-readable name that can be mapped to their cryptocurrency address.The community is also discussing the possibility of a store-and-forward server that would allow users to use an email address as their account instead of a single Bitcoin address or xpubkey.
