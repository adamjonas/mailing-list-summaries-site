Summary:

A new soft fork called CSV (CheckSequenceVerify) will be deployed using BIP 9 version bits, according to a message that has been announced. This deployment will activate three BIPs as consensus rules - Relative lock-time using consensus-enforced sequence numbers, CHECKSEQUENCEVERIFY, and Median time-past as endpoint for lock-time calculations. The deployment information has been updated with start and end times for both mainnet and testnet. Bitcoin Core will release the 0.11.3 and 0.12.1 software which implements these soft forks in the near future.
