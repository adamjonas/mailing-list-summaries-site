Summary:

A recent issue in the Bitcoin Core repository has sparked a discussion about the vulnerability reporting process. A developer raised concerns that vulnerabilities should be reported privately, even if they work only in debug mode, to avoid exploitation. Another developer argued that not all bug reports and investigations can go through a small funnel, stressing the importance of considering the potential impact of any vulnerability that gets exploited. The email exchange highlighted the trade-offs between wider collaboration on resolving issues versus keeping knowledge of the issue within a smaller group. The developers also discussed the need for better documentation and guidance on what class of bug reports should go through the vulnerability reporting process. The issue raised by the developer has since been assigned CVE-2023-33297.
