Summary:

Luke Dashjr proposed a new opcode(OP_CHECKBLOCKATHEIGHT) for the Bitcoin scripting system via bitcoin-dev on Sep 23, 2016. The purpose of this opcode is to handle the reissuing of bitcoin transactions in case the coins they spend have been double-spent or conflicted. This BIP can be found on Luke's Github page.A user requested an explanation of a use case scenario where this opcode would come in handy. While the user understood the proposal, they couldn't see how the attack could benefit the attacker or cause any harm to the attacked party.
