Summary:

A new Bitcoin Improvement Proposal (BIP) has been introduced to address the issue of transaction malleability related to OP_IF and OP_NOTIF in pay-to-witness-script-hash (P2WSH) scripts. These are flow control codes that decide the program flow based on whether the top stack value is true or false. However, this behavior creates a vulnerability where a third party can replace a true (false) stack item with any other true (false) value without invalidating the transaction.To prevent this, proposed changes have been specified in the BIP for Bitcoin script validity rules. The new rules apply only to P2WSH scripts described in BIP141 and are deployed using BIP9 after segregated witness (BIP141) is activated. The argument for OP_IF and OP_NOTIF in P2WSH must be exactly an empty vector or 0x01, or the script evaluation fails immediately.This is a softfork on top of BIP141 and is enforced as a relay policy by the reference client since the first release of BIP141 (v0.13.1). It is important for users to avoid creating P2WSH scripts that are incompatible with this BIP to prevent risks of fund loss. An OP_0NOTEQUAL may be used before OP_IF or OP_NOTIF to imitate the original behavior.The proposed changes are not applied to non-segregated witness scripts to ensure that OP_IF and OP_NOTIF transactions created before the introduction of this BIP will still be accepted by the network. More information and discussion about this topic can be found at https://lists.linuxfoundation.org/pipermail/bitcoin-dev/2016-August/013036.html.
