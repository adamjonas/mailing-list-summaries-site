Summary:

A proposed solution called "Deterministic Entropy From BIP32 Keychains" has been presented to the bitcoin-dev community to address the issue of using too many seeds and backups due to various wallet implementations and security models. The proposal suggests deriving entropy from a hierarchical deterministic (HD) keychain path to deterministically create initial entropy for generating keychain mnemonics and seeds. This way, one root key or mnemonic can derive any variety of different root keychains in whatever format is required. HMAC-SHA512 has been chosen as the function to process the resulting private key because it may have better compatibility in embedded devices as it's already required in devices supporting BIP32.The proposed standard would allow deriving all child derivation schemes depending on the user's preference or hardware signing devices. Devices that store the HD seed as a BIP39 mnemonic, Electrum seed, or BIP32 root key would all be able to implement this standard. The proposal could potentially be split into multiple discrete BIPs but has been avoided to reduce complexity. Test cases have been provided for 128-bit, 256-bit, WIF for Bitcoin Core, and 25 word Monero mnemonic.In an email exchange, other proposals and solutions related to Bitcoin wallet security and seed backups are discussed. One proposal suggests turning any child derived xprv key into a new set valid seed words that could be used by a wallet or other devices, but Christopher Allen disagrees with this proposal and suggests finding standard ways to protect the entropy seed using airgap solutions. Blockchain Commons is working on standards for airgap solutions for storing and signing from offline keys.Additionally, the context provides information about BIP32 and BIP39 standards for hierarchical deterministic wallets and their references. Wallets require separate 12 or 24 word mnemonics, which is five times the complexity to store than one. A proposed extension/standard for "seed management" aims to solve the problem of too many backups due to various wallet implementations and security models. The proposal details a way to deterministically derive the initial entropy used for new root keychains using a single BIP32 style "master root key". The proposal could be a practical solution for people managing multiple seeds for various wallets in a bitcoin business setting or even power users. This proposal could potentially be split into multiple discrete BIPs, but it has been avoided to reduce complexity. The references for this document are BIP32 and BIP39, and the document is dual-licensed under the Open Publication License and BSD 2-clause license.Lastly, an email addresses a problem with using multiple wallets with different levels of security. The issue lies with BIP39, which requires separate 12 or 24 word mnemonics for each wallet, resulting in more complexity in storing backup information and difficulty in adding new geographical locations for backup. The proposed solution involves having one BIP39 mnemonic as their "master root key" and creating child BIP39 seeds deterministically from there. Peter D. Gray, the founder of Coinkite, expressed his approval and recognition of the proposed solution.
