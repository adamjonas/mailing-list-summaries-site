Summary:

The author is starting to work on an implementation of the statechains concept, with particular interest in using the protocol enable the change of ownership (novation) of an individual position in an active discreet log contract (DLC) without an on-chain transaction and without needing the cooperation of the counterparty. The protocol as outlined by Ruben requires features not currently available in Bitcoin, like SIGHASH_NOINPUT, and it is uncertain when this will be added. Therefore, they are looking at variants that would work with current Bitcoin functionality. There are two main modifications they are looking at: Instead of an eltoo-based backup/refund transaction, they propose using a decrementing nLocktime for backup transactions as the output changes hands. The second modification is to use the Decker-Wattenhofer decrementing-nSequence, which although has a limit on the number of updates, does not have a limit on the time that a UTXO is locked in this mechanism. It even allows chaining multiple such mechanisms, which resets all further stages in case of a unilateral close. This multiplies the total number of updates your statechain can have. As the first stage decrements, more stages can be added dependent on it, keeping a total maximum time that a unilateral close will resolve. The original Decker-Wattenhofer paper "Duplex Micropayment Channels" has prettier graphics.
