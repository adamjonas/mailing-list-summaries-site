Summary:

The Bitcoin Core community has recently been discussing concerns and changes related to the project's website and release signing process. One issue is that users searching for "download bitcoin core" may come across the outdated website bitcoin.org, which lacks version 22.0 and can lead to confusion. To resolve this, a user suggests removing download links from bitcoin.org and directing users to the correct website, bitcoincore.org. It is important to note that there is only one website which represents Bitcoin Core full node implementation, and it can be downloaded from https://bitcoincore.org. The GPG signature verification for Bitcoin Core version 22 and later has changed, with two main changes to the verification process that users need to be aware of. However, some users have expressed concern about these changes and expressed distrust in the GPG network. In response to these concerns, Owen Gunden suggested building one's own release and verifying the tree hash while trusting signed annotated tags. Pieter Wuille corrected Owen Gunden's assumption that Wladimir is no longer signing Bitcoin Core releases, stating that Wladimir has provided attestations on the 22.0 release, and there is no separate special release key anymore. Additionally, the discussion involved the need for more people from different countries to take up important roles in Bitcoin, but there are concerns about funding individuals from under-represented countries due to threats of attacks on their funding.One suggestion to improve decentralization is by having more people use alternative full node implementations like Bitcoin Knots and having independent contributors and maintainers who do not rely on funding sources. However, funding is still a major issue, and it is difficult to fund individuals from under-represented countries due to threats of attacks on their funding. Despite these challenges, it is important to have more people from different countries contribute to Bitcoin's success.Overall, while there are valid concerns regarding security, it appears that the changes to the website and signing process are part of long-standing practices within the Bitcoin Core community. It is important to follow the instructions provided by Bitcoin Core for verifying downloads and keeping up-to-date with any changes or updates on the official website.
