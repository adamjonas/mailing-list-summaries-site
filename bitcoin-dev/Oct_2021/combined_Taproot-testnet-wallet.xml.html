Summary:

On the bitcoin-dev forum on October 15th, Anthony Towns raised a discussion about whether testnet faucets would accept bech32m addresses. Some faucets, including Bitcoin Faucet, do accept such addresses but require a bech32 checksum instead of bech32m.Andreas Schildbach is trying to complete bitcoinj's implementation for sending to taproot addresses. To do so, he needs a testnet wallet that can receive to P2TR and spend back. However, despite their efforts, they have been unable to obtain a taproot address from Bitcoin Core 22.0. They are requesting someone to send them a testnet wallet with at least one taproot address for testing purposes, and they don't care about the safety of their testnet coins.Bitcoin Core users can create a taproot-capable wallet by following several steps, which include creating a descriptor wallet with descriptors=true, importing a taproot descriptor using a bip32 tprv private key, and getting a new address with addresstype=bech32m. Pieter has provided a taproot address for testing purposes: tb1p84x2ryuyfevgnlpnxt9f39gm7r68gwtvllxqe5w2n5ru00s9aquslzggwq. If testnet coins are sent there, Pieter has requested an email address to return them.It should be noted that this process also works with testnet, but it is unclear if any testnet faucets will accept bech32m addresses directly. Once taproot has been activated on mainnet and PR#22364 is merged, users will only need to do the first and last steps.
