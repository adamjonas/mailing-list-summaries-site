Summary:

On the bitcoin-dev forum, Giacomo Caironi raised concerns about the documentation of Taproot signature message. He found that Segwit signature messaging had better documentation than Taproot. Caironi suggested creating a separate BIP for Taproot signature message like Segwit. However, Pieter Wuille pointed out that there is no separate BIP for the signature message function as the message function is different for BIP341 and BIP342.Wuille suggested improving existing documentation instead of creating a new BIP. Caironi also highlighted the issue of test vectors for Taproot having no documentation and not being atomic, making it difficult to debug signature messages that don't pass validation. The existing tests are intended for verifying an implementation against and have much higher coverage than the segwit tests had but they aren't useful as documentation.Caironi proposed writing test cases for Taproot if the idea is accepted. Wuille encouraged Caironi to write test vectors that are more aimed at helping debugging issues, such as for certain P2TR scriptPubKeys, derived from certain internal keys and script trees. Wuille stated that he is happy to help integrate such vectors in Bitcoin Core and the BIP(s).In response to Caironi's concerns, a developer named Riccardo Casatta has written the Rust implementation of Bitcoin signature messages and created some test vectors to double-check. These test vectors can be found on GitHub. Meanwhile, Caironi has worked on a Python implementation of Bitcoin signature messages and noticed that there is better documentation for Segwit signature message than Taproot.He suggests creating a different BIP only for Taproot signature message instead of defining it in BIP 341 and keeping the test vectors in a different BIP. Caironi is willing to write the test cases for Taproot if his idea is accepted. Wuille welcomed Caironi's offer to write test cases and offered to help integrate them in Bitcoin Core and the BIPs. Caironi apologized for any mistakes in his language as English is not his first language, and this is his first contribution to Bitcoin.
