Summary:

The recently merged "obfuscation" patch for Bitcoin obfuscates the content of the "chainstate" LevelDB by XOR'ing it against a random key. The patch author is questioning whether or not the obfuscation key should be applied when iterating over the database, specifically in CCoinsViewDB::GetStats. The issue has been confirmed as a bug and a fix has been filed on GitHub. James O'Beirne confirmed the bug and will be writing tests to prevent future regressions. Daniel reported the bug and expressed gratitude for the quick fix, planning to submit a patch in case it was an oversight. The patch author suspects that the implementation of GetStats may be dead code and will investigate further. Users of CLevelDBWrapper who go directly for the iterator run the risk of dealing with obfuscated data, which should be remedied.
