In October 2015, concerns were raised by Jonathan Toomim and other members of the Bitcoin community about anomalous memory usage scaling. They reported that several running bitcoind processes were using over 3GB of RAM despite mempool seeming to be under control. The issue appeared to happen with both Bitcoin Core 0.10.1 and Bitcoin XT 0.11B. Possible explanations for the effect include leaks, caches, and fragmentation. Debugging tools were not ideal to investigate the issue, but a build with tcmalloc may be useful in investigating heap statistics.Valgrind did not reveal any promising results, but the mempool growth can be mitigated by setting `-mintxfee`. One user suggested using a build with tcmalloc to investigate heap statistics. There were also concerns about misleading information from tools like top when measuring memory usage on Linux.
