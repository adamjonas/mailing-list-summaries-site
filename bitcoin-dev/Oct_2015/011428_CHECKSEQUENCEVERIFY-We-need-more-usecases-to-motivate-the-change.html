Summary:

The nSequence for future expansion has been covered in the BIP68 specification section. For transactions with version >= 2, if the MSB of nSequence is unset, it is interpreted as relative locktime. If the MSB is set, the remaining 31 bits are free for repurposing in the future. The BIP112 text has been updated to include more use cases.
