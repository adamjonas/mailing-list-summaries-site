Summary:

A software developer, Byron Gibson, asked for information about using simulators like Shadow and BTCSim to test proposed changes to Bitcoin. Both simulators are free and open-source software tools used to simulate and test blockchain protocols. Shadow is designed to simulate large-scale networks of nodes, while BTCSim is built to test the behavior of Bitcoin nodes in different scenarios. Byron showed interest in collaborating with others who have been using simulators to test proposed changes on Bitcoin, but he wanted to select experiments that benefit from a more controlled environment. In response, Nina K shared her experience with running Shadow with Tor, stating its limitations in accurately collecting statistics. Andrew Miller acknowledged the importance of simulators but noted Shadow's limitations. The conversation took place on the bitcoin-dev mailing list in 2015.
