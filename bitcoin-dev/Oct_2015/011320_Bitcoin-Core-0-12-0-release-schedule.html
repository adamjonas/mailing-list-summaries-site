Summary:

In a discussion on the bitcoin-dev mailing list in 2015, Wladimir J. van der Laan, Bitcoin Core's lead maintainer, explained that the versioning of Bitcoin software does not follow the Semantic Versioning (SemVer) specification. He stated that the "feature freeze" for version 0.12 meant that only critical bug fixes would be included, and that postponing the release for consensus changes was not ideal. Van der Laan also noted that decoupling consensus changes from major releases had been discussed to avoid release date drift. The conversation ended with the suggestion of creating a 0.12.1 release if necessary.
