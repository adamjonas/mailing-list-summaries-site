Deprecating Bitcoin Core's regtest-specific `setgenerate` behaviour

    Author: Pieter Wuille 2015-04-12 14:26:12
    Published on: 2015-04-12T14:26:12+00:00

  • Click here to read original discussion on the bitcoin-dev mailing list

Summary:

Bitcoin Core developers are planning to deprecate the special meaning of `setgenerate` RPC call for -regtest and replace it with a separate RPC call `generate`. This change is not related to any production functionality, and the developers are generally conservative in changing RPC behavior. The old behavior will be switched unless there is any software or user who needs compatibility with it. The bitcoin.org developer documentation will need to be updated accordingly.
