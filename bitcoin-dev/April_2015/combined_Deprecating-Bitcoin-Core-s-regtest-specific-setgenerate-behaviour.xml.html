Summary:

Bitcoin Core developers are planning to deprecate the special meaning of `setgenerate` RPC call for -regtest and replace it with a separate RPC call `generate`. This change is not related to any production functionality, and the developers are generally conservative in changing RPC behavior. The old behavior will be switched unless there is any software or user who needs compatibility with it. Pieter Wuille, a developer at Bitcoin Core, has announced the deprecation of the `setgenerate` RPC call's special meaning for -regtest. Instead, it will be replaced with a separate RPC call `generate`. Pieter is asking if there are any software or users who would need compatibility with the old behavior. The bitcoin.org developer documentation will need to be updated accordingly. Sean, a Java RPC client and Groovy/Spock functional tests writer, agrees that a separate `generate` or `generateblocks` call is a great idea. However, he points out that keeping tests compatible with the current stable and unstable versions of Bitcoin Core (and the Omni Core fork) would require writing code that checks the RPC server version.
