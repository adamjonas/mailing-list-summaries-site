Summary:

On April 29, 2015, an email was sent to the Bitcoin-development mailing list inquiring about a Python script that can generate op codes from the hash of locking and unlocking transactions. The purpose of this script is not mentioned in the message. However, Richard Moore, the founder of Genetic Mistakes Software Inc., recommended a Python library called pycoind that can be used for script parsing and execution. The library can be imported using "import pycoind" and can be found on GitHub at https://github.com/ricmoo/pycoind. Moore also provided an example of how to use the library, which involved importing pycoind and using the Tokenizer function. The example outputted the op codes for a given hex string. Jeff Garzik, a Bitcoin core developer and open source evangelist, signed off on the email chain. In addition, another email by Braun Brelin requested a good python script that takes the hash of the locking and unlocking tx scripts and outputs the actual op codes. It is unclear why this is needed. Lastly, the email included an advertisement for server and application monitoring software.
