Summary:

The Python-ecdsa module has recently implemented RFC 6979, which proposes a deterministic algorithm to calculate the 'k' value for signatures from private key and signed data. This feature is expected to enable unit testing of code using ECDSA signatures and black box testing of third-party software. The implementation can be found on slush's fork of python-ecdsa on GitHub, and a pull request is currently pending approval by the author of python-ecdsa. For further reference, the full RFC 6979 document can be accessed on tools.ietf.org.
