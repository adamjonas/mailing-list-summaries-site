Summary:

The email thread discusses the possibility of embedding data in the blockchain as a hash, which has been made possible by services like Proof of Existence and sx-embed-addr. The author proposes creating a new payment type called "embed_hash" to support this feature. This unspendable payment type would include a script with a single 20 byte push data, offering optimization possibilities and avoiding indexing Bitcoin addresses used for data indexing. The idea is to separate this class of Bitcoin usage from normal Bitcoin addresses to improve the core Bitcoin payments system.In September 2013, a proposal was suggested on the Bitcoin-development mailing list to create a new payment type that would be unspendable and could be used to support embedding data in the blockchain as a hash.
