Summary:

Bitcoin-qt / Bitcoind nodes have a problem with coming back up after a restart and report an error message that says "*** coin database inconsistencies found" and prompts to rebuild the block database. Reindexing will not solve the issue, so users should say no to this prompt. To work around the problem, users need to specify the command-line or configuration file argument "-checklevel=2" to Bitcoind or Bitcoin-qt. This problem is caused by 0.8.0 and only affects local nodes, but it requires a software update to fix completely. The problem will persist until no more than 288 blocks after 256818 unless another trigger transaction is added to the blockchain. A patch has been developed, but developers need to review it before applying it. Users can refer to https://bitcointalk.org/index.php?topic=290922.msg3117923#msg3117923 for instructions on performing the workaround in Windows.
