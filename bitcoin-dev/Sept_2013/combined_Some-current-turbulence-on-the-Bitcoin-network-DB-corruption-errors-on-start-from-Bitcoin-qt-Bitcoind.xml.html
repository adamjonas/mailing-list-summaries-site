Summary:

Bitcoin-qt/Bitcoind nodes are facing a problem which prompts users to rebuild the block database. It is caused by 0.8.0 and only local, but a software update will be required to fix it completely. Users can use the command-line or configuration file argument -checklevel=2 as a workaround. The issue will persist until no more than 288 blocks after 256818 unless another trigger transaction is added to the blockchain. Gregory Maxwell has provided a patch for review at https://github.com/bitcoin/bitcoin/pull/2982 which should not be applied until other developers have reviewed it. A useful write-up on how to implement the workaround is available on BitcoinTalk forum, but it is specific to the Windows operating system and caution should be exercised when making changes to settings.
