Summary:

Jeremie Dubois-Lacoste, the Director of Belgian Bitcoin Association and a Post-Doctoral Researcher at Université Libre de Bruxelles, is inquiring about the availability of a patch he intends to implement. He aims to add an optional parameter to "-rescan" that would enable the rescan to start from a given point in time provided by the user. The proposed feature would work similarly to the wallet birthday mechanism; however, the user would give the starting point at launch time on the command line. An alternative option could be to provide a timestamp instead of a block height. Everything before the specified block number or timestamp would be ignored during the wallet rescan.In his search for information, Jeremie is seeking feedback on whether this feature already exists to avoid duplicating efforts. This new feature would allow users to perform faster and more efficient wallet rescans as they can explicitly provide the starting point, saving time and resources.
