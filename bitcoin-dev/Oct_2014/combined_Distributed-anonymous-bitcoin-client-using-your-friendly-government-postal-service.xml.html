Summary:

When making a transaction with two outputs, one output is for the payee and the other is for the fee given to the person transmitting the transaction. The transaction must be signed and printed along with the private key controlling the address of output B. It should then be sent to unknown greedy individuals through mail. Namecoin allows for mixing outputs for name registration with pay to hash outputs, which can be used for anonymous publishing through posts. If the transaction is a 40-byte tweet, output A can be created in Bitcoin scripted with op_return.
