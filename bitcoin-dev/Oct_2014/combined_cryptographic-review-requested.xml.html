Summary:

A proposal has been made for a standardized ECIES scheme for Bitcoin communication, including a change to BIP44 to add identity keys for structured wallets. Mem Wallet suggested creating a new BIP for describing the HD paths used for ECIES, which is available on http://memwallet.info/btcmssgs.html. The implementation allows for secure messaging using a Bitcoin wallet without other software and enables proof of ownership privately between address holders without requiring secure communication channels. It uses CompactSize format and assumes familiarity with the Bitcoin compact signature format, employing HMAC_SHA256 for mac and PBKDF2 for KDF. Random nonces from a cryptographically strong DRBG are compliant, but deterministic nonces should only be used for signatures unless high-quality random bytes are available. In an email conversation on Oct 22, 2014, Chris D'Costa expressed concerns about knowing for certain that the public key received to encrypt a message is not from a MITM (Man-In-The-Middle) attacker. Pavol Rusnak replied, asking if this was the same problem with PGP.
