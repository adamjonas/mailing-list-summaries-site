Summary:

Bitcoinj has released version 0.12, after 8 months of work. This release includes hierarchical and deterministic (HD) wallets that use BIP32 specification, support for mnemonic codes (BIP 39), Tor integration using the Orchid library, support for multisig wallets, faster and deterministic ECDSA, many API improvements, and upgrades to the included GUI wallet. The new HD wallets are a major privacy enhancement, as change and receive addresses are no longer being reused. The library is used by several Android and desktop wallets, academic research projects, blockchain.info, Circle, Lighthouse, CryptoCorp, biteasy, BlueMatt's relay network, bitpos, and countless alt coin wallets.The author of BitcoinJ has explained that stealth addresses and SPV do not mix well, so there are no plans to support them. Instead, a description of how to do something similar with the payment protocol has been written. The author will be focusing on fully launching Lighthouse and crowdfunding for decentralisation/crypto related projects rather than working on major feature work on bitcoinj. If the author were to tackle a big project, it would be refactoring the wallet so it doesn't store transactions directly anymore, just unspent outputs. This is because the top user reported problem with the current design is people hitting scalability limits.Mike Hearn, a developer of Bitcoin, announced the release of version 0.12 of bitcoinj library on October 3, 2014.
