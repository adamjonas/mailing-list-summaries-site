Summary:

Ami Jr. has created his own branch of Bitcoin 0.9.3 including watch-only and several patches to clean it up from the master branch, but it is not heavily tested. Sipa's address index branch may be more suitable for an use case where information about any address is required, and jmcorgan continued to maintain an up-to-date version. DexX7 has his own branch for balance queries with a few other RPC calls, and he recommends reviewing the changes first before considering using it. Additionally, Bitcoin Core can now read the balance of any public address via RPC call or after importing the balance, which was previously only available through the QT interface. However, a watching wallet still needs to be synchronized with the chain in the same way as any other wallet.Overall, Flavien wants to know the current status of watch-only addresses in Bitcoin Core and whether they are usable, documentation on how to add them through RPC, and the ETA for a release candidate as it moves towards version 0.10. Wladimir confirms that watch-only addresses have been merged into Bitcoin Core, and provides information on the "importaddress" RPC call. He also mentions that a release candidate is expected in a few months, probably by the end of the year. However, it's not recommended to use the master version without at least a little bit of development/debugging experience of yourself. Warren Togami Jr. has built nightly builds until around June, but there are no nightly builds available at this time.
