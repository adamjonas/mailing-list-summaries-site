Summary:

Pieter Wuille has submitted a pull request (#649) which allows the use of compressed public keys in Bitcoin. These keys are 33 bytes long and contain only the X coordinate of the point, while the value of the Y-coordinate is reconstructed upon use. This requires some CPU but is much cheaper than verifying a signature. The implementation chooses only one key at generation time, as one private key now corresponds to two public keys and thus two different addresses. Compressed public keys are supported by default by OpenSSL and all alternative implementations that use OpenSSL should support these keys just fine as well. However, it is unclear whether all client implementations support them. Additionally, there is a need to figure out how to represent secrets for compressed pubkeys, and send-to-pubkey transactions are untested for now. Pieter provides an address corresponding to a compressed public key to help in testing.
