Summary:

Namecoin is a peer-to-peer generic name/value datastore system that is not limited to .bit usage. The system is designed to map things to names in a fully decentralized way, making it the perfect starting point to map names to other things such as a public bitcoin address or a URL. However, using other systems like DNS and HTTP(S) as the first entry point will not provide you with all the advantages of Namecoin. Constructing a zero-trust (or nearly zero trust) Namecoin resolver without having a copy of the ever-growing complete Namecoin block chain can be challenging. The Bitcoin lite node mechanism will not work because a peer could return stale records or no-result. There are some EC crypto tricks to generate a temporary address that requires interaction but not necessarily directly with the entity hosting the wallet. The wallet-hosting entity has a private key x, with public key X. The address-generating entity knows X, and generates a fresh private key y for each transaction.
