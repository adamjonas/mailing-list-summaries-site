Summary:

The context discusses the correct implementation of segwit. It is explained that empty scriptSig doesn’t imply segwit input if the previous scriptPubKey is OP_1, which does not allow witness. Similarly, a scriptSig looking like a spend of P2SH-segwit doesn’t imply segwit input if the previous scriptPubKey is empty. The validators have no way to tell whether an input is segwit-enabled or not until they look up the UTXO set. The use of “00” means "this input has no witness stack element”. You need this even for non segwit inputs because there is no way to tell whether an input is segwit-enabled or not until you look up the UTXO, which might not be always available. The spec requires an empty element on the stack for an input with no witness stack element. If all inputs have no stack element, the spec requires the non-segwit serialization.
