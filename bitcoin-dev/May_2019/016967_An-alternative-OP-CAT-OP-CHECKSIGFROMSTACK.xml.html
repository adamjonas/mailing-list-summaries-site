ZmnSCPxj proposes a way to achieve a more interesting Turing-complete system even without `OP_PUBKEYTWEAK`. With the help of `OP_CHECKSIGFROMSTACK`, he suggests that introspection into the transaction being spent can be leveraged to acquire the previous state data. The SCRIPT can validate the execution of one iteration of the interpreter loop, and MAST can compress branches not taken, reducing the number of operations required at each iteration. Though creating this by hand may be difficult, it should be doable with an army of lower-level cognition agents.
