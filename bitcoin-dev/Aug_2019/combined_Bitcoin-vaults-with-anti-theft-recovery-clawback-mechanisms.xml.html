A proposal for implementing Bitcoin vaults without the need for soft-forks or software upgrades has been suggested. The scheme binds both users and attackers to use a public observation and delay period before a weakly-secured hot key is allowed to arbitrarily spend coins. This is called pre-signed vaults. The security of this scheme is enforced by pre-signing transactions and deleting private keys. Bishop notes that having a vault construction might go a long way to discourage would-be attackers.
