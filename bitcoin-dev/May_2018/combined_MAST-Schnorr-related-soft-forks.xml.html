Summary:

Bitcoin developers are proposing several upgrades to the cryptocurrency system, with the aim of improving fungibility by making different features indistinguishable. These proposals include introducing a soft fork for Merkelized Abstract Syntax Trees (MAST) in SegWit v0, new SIGHASH modes, and Schnorr p2pk(h) addresses and taproot+mast addresses in SegWit v1.In addition, there are plans for graftroot in SegWit v2, which requires Schnorr and allows delegation of authorization to spend an output already on the blockchain. Interactive Signature Aggregation in SegWit v2 is another proposal that enables signers to interactively collaborate when constructing a transaction. Non-interactive sig aggregation in SegWit v3 is also being considered.The developers have categorized their ideas into five categories: Schnorr CHECKSIG, MAST, Taproot, Graftroot, and Interactive Signature Aggregation. However, there are concerns about certain features like graftroot, and aj has suggested delaying its implementation until non-interactive sig aggregation is possible.It is important to note that developing all these features together could delay the implementation of some improvements. Interested parties can subscribe to the bitcoin-dev mailing list for more information on these proposed upgrades.
