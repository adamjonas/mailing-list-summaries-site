Summary:

The writer is discussing the theoretical problem of being unable to use OP_TRUE to CPFP a tx if someone spends it with a low-fee non-RBF transaction. They acknowledge that it would be difficult, but possible. Despite this risk, the writer believes that the benefits of using OP_TRUE outweigh the potential downside. They suggest that using OP_CSV with a relative locktime of 0 could be a solution to enforce RBF on spending transactions.
