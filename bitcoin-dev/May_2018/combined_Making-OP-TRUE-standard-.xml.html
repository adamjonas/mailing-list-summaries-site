Summary:

The Lightning Network is experiencing difficulties predicting future fees. To resolve this issue, Rusty Russell suggested adding a 546 satoshi OP_TRUE output in commitment transactions to use minimal fees and rely on Child Pays for Parent (CPFP) to offset CSV delays on outputs. However, due to a raw 'OP_TRUE' being non-standard, it must be P2SH-ed at the moment. Johnson Lau suggested using "ANYONECANPAY" approach instead, which would allow more inputs for fees without asking for any protocol change. He also recommended having a more flexible SIGHASH system in the long-term to add more inputs and outputs easily. Peter Todd discussed that a zero-fee tx with exactly one OP_TRUE output won't get relayed even with CPFP, due to the fact that package-based transaction relaying has not yet been implemented. There were also discussions about UTXO pollution and a CPFP anchor softfork. Finally, there was a question about downsides of using P2WSH instead of P2SH.
