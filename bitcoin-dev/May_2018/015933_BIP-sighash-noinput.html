Summary:

The email discusses a technical point regarding SIGHASH_NOINPUT. The author suggests that it would be technically simpler to have a boolean specifying whether inputs listed must be all of them or a subset of everything. They also suggest a similar boolean should be made for outputs. The author notes that one could always use SIGHASH_SUBSET and not specify any inputs and achieve the same effect as SIGHASH_NOINPUT. The email then requests BIP editors to assign a BIP number to the SIGHASH_NOINPUT proposal. The author has implemented a hashing implementation in Bitcoin Core, although they do not think it will sail through review and cannot test it now because there is no groundwork on witness V1 scripts. The email concludes with a note that fork coins have used sighash 0x40 as FORKID, which does not conflict with this proposal since the proposal only applies to segwit transactions, which the fork coins have explicitly disabled.
