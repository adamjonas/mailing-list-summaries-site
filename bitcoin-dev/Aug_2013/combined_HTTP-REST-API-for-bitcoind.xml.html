Summary:

A proposal has been made to implement an HTTP REST API for "bitcoind," which would provide a decentralized block explorer capability, enabling easy external access to transaction/address/block indices that are maintained. The goal of the HTTP REST interface is to access unauthenticated, public blockchain information, with no plan to add wallet interfacing/manipulation via this API. In discussions between developers, the possibility of exposing the UXTO set of a given address was discussed as being useful for Simplified Payment Verification (SPV) wallets to swipe a previously unknown private key such as a paper wallet. However, there were concerns about maintaining indexes and security risks associated with relying on unauthenticated data.There have also been proposals for reducing the blockchain storage requirement for SPV clients, including the partial UTXO mode, where users would only fetch blocks that contain transactions needed to verify their balances. However, concerns about security and resource usage remain.Pieter Wuille clarified that the blockchain serves as a historical record for other nodes and wallet rescans, while the UTXO set is the state needed to validate blocks and transactions. The importance of UTXO commitments and maintaining a headers-only blockchain was also highlighted.Finally, debates occurred around whether multiple linear brute force searches of the blockchain are carried out for every new transaction or if transactions are only recorded if they were in a block, and just their presence indicates they're valid.
