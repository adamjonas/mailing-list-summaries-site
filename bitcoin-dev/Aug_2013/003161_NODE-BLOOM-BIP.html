Summary:

In an email conversation from August 2013, Mike Hearn argued that the Bloom filtering feature in Bitcoin should not be optional, as it is not a lossless function and making every possible feature optional is not smart design. He compared it to making re-sharing optional in BitTorrent and expecting users to never upload data to peers that needed it. This argument suggests that certain features should be mandatory for optimal functionality.
