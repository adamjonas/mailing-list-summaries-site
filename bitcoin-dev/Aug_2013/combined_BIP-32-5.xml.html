Summary:

In August 2013, Gregory Maxwell suggested creating a BIP32 addendum to make signing-related recommendations. He highlighted blockchain.info's mywallet as another DSA duplicated K disaster occurring. The author proposes removing the "CS-PRNG" component from signing, which adds no value and detracts from deterministic signing benefits. However, the CS-PRNG is necessary for generating the root master key of an HD wallet, creating vulnerability if a single invocation is weak. The author suggests cumulative entropy collection over time through multiple sources to mitigate this risk and using smartphone sensors to provide additional entropy for increased protection.The author proposes a BIP32 addendum for specific deterministic recommendations for DSA derandomization procedures. Derandomized DSA has become industry practice as it eliminates hazards present otherwise. Maliciously modified hardware wallets could leak key material via signatures, even without producing insecure K values. Deterministic signatures make such misbehavior practically discoverable. The author recommends implementing a procedure for using only even S values in signatures, eliminating a source of mutability in transactions. A new BIP that includes the above recommendations would be implemented if reviewed positively.Bitcoin developer Mike Hearn questioned the necessity of hardware wallets in an email conversation from August 2013, expressing mistrust in both hardware wallets and computers. The discussion also touched on the lack of multisig support in the first version of TREZOR and the challenges of implementing P2SH support for sending payments through Android wallets.
