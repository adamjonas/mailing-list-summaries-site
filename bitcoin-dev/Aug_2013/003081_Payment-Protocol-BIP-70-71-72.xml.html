Summary:

Gavin Andresen, a Bitcoin developer, has updated the BIP 72 spec at https://en.bitcoin.it/wiki/BIP_0072 to make the bitcoin address optional. If the "request" parameter is provided and backwards compatibility is not required, then the bitcoin address portion of the URI may be omitted (the URI will be of the form: bitcoin:?request=... ). The spec already said what should happen if both request and address/amount/etc were given: "it should ignore the bitcoin address/amount/label/message in the URI and instead fetch a PaymentRequest message and then follow the payment protocol". This upgrade path gives us a smooth, clear upgrade path.
