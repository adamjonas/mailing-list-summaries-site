Summary:

The email thread begins with a discussion about implementing an auto-updater in Hive, a wallet app, for usability purposes. One member suggests using a server behind a Tor hidden service that outputs a checksum of the update package, but others point out that this method is not safe enough and recommend digital signing instead. To ensure better security, a distributor public key can be hardcoded in the software, and client software will only trust signed data from that key. The private key for that data is kept offline. When executing an upgrade, the new checksums are signed offline, and the signed checksum is uploaded to the distribution server.Another member suggests committing to the blockchain to ensure making a release available for download is public. This would involve each person on the signing list creating a transaction with a special form from a specific pubkey that commits to the digest of the binaries, and the auto-update code refusing to update unless it sees that special transaction with a sufficient number of confirmations. Developers would then be unable to make a special release for a specific target without letting the world know they did so, even under coercion.The potential danger of auto-updating being easy to target individuals is discussed, and anonymous downloads are suggested, but they tend to be fragile with regard to deanonymization attacks. Gregory Maxwell suggests using quorum signatures and timed quarantine with negative signatures to ensure security. Peter Todd suggests extending gitian with quorum signatures and timed quarantine with negative signatures.It is noted that Bitcoin-QT lacks an auto-updater due to security reasons, but it is unclear if this decision has been revisited. In addition, the discussion also veers off-topic when someone shares information about getting SQL databases under version control. In conclusion, the email thread emphasizes the importance of digital signing over just relying on a checksum for package authentication. The thread ends with links to the SQL database version control and Bitcoin-development mailing list.
