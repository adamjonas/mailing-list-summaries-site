Summary:

The conversation is about setting up BIND forwarding for DNS seeds for the bitcoin network. The discussion includes setting up different zones to forward requests to separate DNS seed daemons running on different non-public IPs or two different ports on the same IP, while also taking into consideration that dnsseed data rotates through a lot of addresses if available. Further, there is a problem with testnet DNS seed itself, as it only returns one IP address, whereas the mainnet seed is working fine. The DNS seeds can be reached through node.alexykot.me and are available on their own ports. However, currently bitcoinj cannot deal with nonstandard ports.
