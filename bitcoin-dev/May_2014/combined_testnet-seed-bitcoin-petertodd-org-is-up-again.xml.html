Summary:

In May 2014, Andreas Schildbach suggested teaching the seeder or plain bitcoind how to write a zone file and then use matured DNS servers to serve this zone. However, this approach was previously used, and it produced fairly bad load imbalances, especially since some resolvers only pass on a single result. Matt's implementation is doing this approach, which Pieter thought was a good thing as it provided more diversity. Additionally, Robert McKay suggested an option to the seeder for non-matching requests, forward to another DNS server at IP-PORT, so you could cascade them. On debugging his DNS settings, Alex Kotenko approached Pieter for help as he had set up DNS seeds himself but couldn't figure out why nslookup testnet-seed.alexykot.me failed. Robert McKay thought that the problem was with his suggestion to use bind forwarding. Bind strips off the authoritative answer bit in the reply, causing the recursor to go into a loop chasing the authority server. He wasn't sure if this could be fixed without hacking the bind src.
