Summary:

The discussion on the Bitcoin-development mailing list is about setting up DNS seeds for Bitcoin. The conversation includes suggestions for using matured DNS servers to serve zonefiles and forwarding non-matching requests to other DNS servers to cascade them. There is also a suggestion to write a zone file and use bind to serve it, which involves running a couple of `dig` scripts that every minute 'dig' the bitcoin-seed and write the results to two zone files that BIND can then serve. However, this solution has a problem as bind is stripping off the authoritative answer bit in the reply. Therefore, the recursor goes into a loop chasing the authority server, which again returns a non-authoritative answer with itself as the authority again. Alex Kotenko reported a problem with his DNS settings while trying to configure DNS seeds himself. He could reach his nameserver and get dnsseed response if he queried it directly but when he tried to nslookup testnet-seed.alexykot.me, it failed.
