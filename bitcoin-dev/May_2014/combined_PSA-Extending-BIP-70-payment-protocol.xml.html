Summary:

Jeff Garzik, a Bitcoin core developer and open source evangelist at BitPay Inc., suggests that payment protocol extensions should be added to a new draft BIP instead of continuously updating BIP 70. He recommends following the IETF model where new RFCs are created as protocols are extended and once published, an IETF RFC is considered static. Garzik believes it is acceptable to break this "static" rule for significant BIP bugs or harmless maintenance of links-to-resources.
