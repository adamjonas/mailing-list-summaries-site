Summary:

In May 29, 2014, Toshi Morita reported a bug he found while running `bitcoind` under `valgrind` to the Bitcoin-development mailing list. The uninitialized variable in question was located in the CWallet class and caused a conditional jump or move depending on uninitialised value(s). This occurred because `nTimeFirstKey` was not initialized when the wallet was instantiated. Toshi provided a fix for the issue in his fork of the code, which involved initializing `nTimeFirstKey` to 0. Mark Friedenbach suggested that Toshi submit a pull request on Github, where it will likely be merged quickly.Further analysis revealed that the error occurred due to `nTimeFirstKey` not being initialized in the source code for bitcoin, as indicated by an error message when running `bitcoind` under `valgrind`. This was identified using Valgrind-3.8.1 and LibVEX, with the code located in the wallet.cpp file, specifically on line 63. The error can be fixed by initializing `nTimeFirstKey` or ensuring it is given a value before use.In addition to reporting the bug, Toshi also included an advertisement for Restlet, a web API service, in his email to the Bitcoin-development mailing list.
