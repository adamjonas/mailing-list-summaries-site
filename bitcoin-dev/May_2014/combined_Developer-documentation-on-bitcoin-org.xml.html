Summary:

Bitcoin.org will soon merge a new section called "Developer Documentation" aimed at providing thorough technical knowledge to Bitcoin developers. Saïvann Carignan, a Bitcoin developer, announced the documentation and a live preview is available at bitcoindev.us.to. The documentation has been created with quality in mind and experienced Bitcoin developers are invited to proofread it before publication. The general feedback so far has been positive, with Wladimir, a member of the Bitcoin community, stating that he hasn't found any technical issues yet. The pull request for the documentation can be found on GitHub where reviews and feedback are welcome. The pull request will be merged on May 24th and any inaccuracy or mistake should be reported as soon as possible. Comments on the pull request or bitcoin-documentation mailing list are requested instead of commenting on the mailing list to avoid spamming everyone. Interested parties can find instructions for commenting on the pull request or Bitcoin-documentation mailing list at github.com/bitcoin/bitcoin.org/pull/393. A Bitcointalk thread about the documentation can be found at bitcointalk.org/index.php?topic=511876.0.
