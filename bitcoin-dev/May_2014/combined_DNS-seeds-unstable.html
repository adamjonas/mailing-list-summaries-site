Summary:

The Bitcoin DNS seed infrastructure has been facing instability issues due to a custom DNS implementation that is not fully compatible and has had bugs in the past. The issue was brought up by various individuals, including Alex Kotenko, who suggested setting up BIND forwarding requests to two dnsseeds running on separate ports. However, Jeff Garzik argued against this idea, stating that using the bind/zone-xfer system would result in fewer total addresses going through to the clients. Andreas Schildbach raised concerns about the reliability of DNS seed and suggested maintaining a quality infrastructure as the first step towards addressing the issue.There have been reports of SERVFAIL errors with some of the seeds, such as dnsseed.bitcoin.dashjr.org and testnet-seed.bitcoin.petertodd.org, when tested by different ISPs. Even large ISPs like British Telecom and Telefonica are unable to access these servers. The current state of Bitcoin DNS seeds shows that some are okay while others have SERVFAIL issues, one is tried multiple ISPs and one only returns one node. Gregory Maxwell suggests that relying exclusively on DNS seeds increases isolation vulnerabilities and that clients should be robust regarding DNS lookups.Despite the issues, there are efforts to improve the situation. Robert McKay suggested configuring bind as a DNS forwarder in a zone context, which can forward different zones to different dnsseed daemons running on different non-public IPs or different ports on the same IP. There are also pull requests outstanding to include hard-coded seed peers and getaddr sourced IPs, which will provide more backup paths once they are merged in. Additionally, the writer mentions that their android wallet is working "OK" despite some temporary availability problems with seeds.The discussion in the email chain is about setting up BIND forwarding for DNS seeds for the bitcoin network. There are issues with testnet DNS seed as it only returns one IP address, whereas the mainnet seed is working fine. The seeds can be reached through node.alexykot.me and are available on their own ports. However, currently bitcoinj cannot deal with nonstandard ports. Suggestions were made to include more seeders and enable zone transfers so that existing seeders can spread their records to multiple servers running standard DNS servers like bind etc. There are also discussions about configuring BIND as a DNS forwarder and forwarding requests to different ports on 127.0.0.1 for each DNS seed instance.Andreas Schildbach reported on the unstable state of Bitcoin DNS seed infrastructure and its bugs, such as a case-sensitive match for the domain name. Telefonica, one of Europe's largest ISPs, is unable to communicate with Peter Todd's server. Meanwhile, Alex Kotenko faced issues with Andreas' testnet Bitcoin Wallet due to a failure in peer discovery. Michael Wozniak suggested setting up two daemons listening local on different ports and using a separate daemon or normal DNS server that proxies incoming queries to either domain.
