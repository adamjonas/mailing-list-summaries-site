Tamas Blummer asks Eric Lombrozo to enumerate or point to arguments that discourage the use of a key both for signing and for derivation of a deeper level of the hierarchy. In response, Eric Lombrozo mentions that he has submitted a BIP proposal that addresses the issue of needing to predefine HD wallet structures and not being able to arbitrarily nest deeper levels. The proposal can be found on GitHub (https://github.com/bitcoin/bips/pull/242). Eric does not directly answer Tamas' question about discouraging the use of a key for both signing and deriving a deeper level of the hierarchy.
