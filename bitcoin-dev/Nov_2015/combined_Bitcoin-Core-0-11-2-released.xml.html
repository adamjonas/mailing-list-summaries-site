Summary:

Bitcoin Core version 0.11.2 has been released, bringing several bug fixes, the BIP65 (CLTV) consensus change, and relay policy preparation for BIP113. The release includes changes related to the BIP65 soft fork, which aims to redefine the existing OP_NOP2 opcode as OP_CHECKLOCKTIMEVERIFY (CLTV), enabling the creation of unspendable transaction outputs until a specified point in the future.Other notable changes include the implementation of BIP113, which enforces the rule that blocks must not contain transactions with locktimes more than two hours in the future. The update also includes a fix for the corrupted UTXO database on unclean shutdowns in Windows. Users are advised to subtract one hour from their transaction locktimes to allow them to be included in mempools at approximately the expected time. The new release was made possible by contributions from over a dozen developers, including Alex Morcos, Gregory Maxwell, and Wladimir J. van der Laan. The issue tracker at Github can be used to report bugs. However, there are warnings for users looking to downgrade back to previous versions due to incompatibility issues caused by changes to block files and databases.In addition to discussing the Bitcoin Core update, the context also touches on other related topics. Kleopatra's invalid signature issue is highlighted, suggesting that something went wrong with the email system. Dave reported that he was unable to verify a new key on Windows with Kleopatra. However, the SHA256SUMS.asc file downloaded from the magnet link posted in the email verifies fine using the new key.The context provides information about PGP encryption as a widely used tool for secure communication in many industries. It also cautions against copy/pasting from web pages when verifying gpg keys for Bitcoin Core binary release, recommending verifying the binary release signing key instead.
