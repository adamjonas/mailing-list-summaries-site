Summary:

Bitcoin developers have been discussing the need for standardized hardware wallet protocols to improve user experience and security. The lack of standardization in desktop and mobile wallets interacting with hardware devices has resulted in proprietary code being added by wallet vendors for non-standardized interfaces. This has led to concerns over blindly signing transactions, resulting in potential problems. To address this missing part of the ecosystem, Jonas Schnelli proposed a BIP titled “Detached Signing” that describes a generic interface for wallets to interact with signing-devices, resulting in better user experience, less critical code, and simpler adaption for various signing-devices.There are concerns raised over the use of smart-cards in hardware wallets that are used to store bitcoins, which could lead to problems. Peter Todd recommends using hardware security modules (HSMs) instead of smart-cards to prevent discrepancies between the actual bitcoin history on the blockchain and what the hardware wallet thinks is happening.The discussion also revolves around the usefulness of a signing protocol that would benefit not only hardware wallets but also websites that want to request public keys/signatures. While some believe that building a standard on top of the URI protocol is a limitation, others believe that driving any hardware wallet from the wallet itself or from a third-party daemon implementing a URL scheme would provide better device interoperability and easier maintenance paths for wallets.There is also a debate about the URI scheme versus the stdio/pipe approach for hardware wallets. According to one opinion, the URI scheme doesn't require wallet app level configuration, unlike the stdio/pipe approach that needs to configure details about the used hardware wallet. However, another person points out that hardware wallet vendors do most of the work on talking between the computer/mobile and the wallet on a lower level of communication using protocols like Google's ProtoBuf.Additionally, there has been ongoing discussion among Bitcoin developers regarding the need for a standardized hardware wallet interface. Some have proposed driving the signing workflow through wallet software, while others suggest high-level APIs that implement hardware-specific protocols and transports as plugins.In August 2016, a bitcoin-dev thread discussed the need for a standard in new forms of transactions such as SegWit and Lightning. Jonas Schnelli proposed two approaches to communicate with signing-devices app but suggested that the pipe approach could be dropped because major platforms support the URI approach. However, Luke-Jr disagreed and suggested that hardware protocol standardization would eliminate the need for plugin installation. A step towards creating a standard for new forms of transactions was taken when Rusnak agreed to the idea. He believed that the existing transactions were "ugly" and that a new standard should be created.Overall, the proposals could provide improved security measures for Bitcoin Core users and make a whole spectrum of attacks less likely and more straightforward for typical users to guard against. The lack of a standard for desktop and mobile wallets to interact with hardware devices will ultimately reduce the end-user experience. It is suggested that a third-party library defining high-level API and implementing hardware-specific protocols and transports as plugins would be more likely.
