On August 16, 2016, Johnson Lau proposed a new BIP on bitcoin-dev to address OP_IF and OP_NOTIF malleability in P2WSH. The proposal can be viewed on GitHub along with a pull request to Bitcoin's repository. However, Luke questioned the need for this since SegWit transactions are already non-malleable due to skipping the witness data in calculating the transaction id.
