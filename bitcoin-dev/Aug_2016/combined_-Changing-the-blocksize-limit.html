Summary:

The current block size limit in Bitcoin is denominated in bytes, which can create issues for transactions with many inputs that become larger and require higher fees to be included in a block. Proposals to change the block size limit have been met with resistance, but Chris Priest, a Bitcoin developer, has proposed changing the block size limit to a block output limit. This would incentivize microtransactions and decrease the unspent transaction output (UTXO) set, addressing the UTXO bloat problem. Priest's new proposal, called BIP131, achieves the same effect as his initial proposal of changing the block size limit denominated in outputs without needing a database index. Miners would choose transactions based on maximum fee per output, essentially making it free to include an input to a transaction. The proposed block output limit should be set at roughly the amount of outputs found in 1MB blocks today to minimize controversy. However, restrictions are necessary to maintain a level playing field between all miners, and serialized size needs to be limited to prevent potential denial-of-service vectors. Despite arguments against keeping usage restricted, restrictions are still necessary in some cases.
