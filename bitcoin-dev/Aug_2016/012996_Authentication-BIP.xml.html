Summary:

The proposed BIP suggests that each peer can configure multiple identity-keys for each network interface including IPv4, IPv6 and tor. The author of the BIP recommends using different identity-keys for each network interface to avoid linking a tor address to an IPvX address. The identity-public-key(s) can be shared over a different channel with other node-operators or non-validating clients to grant authorized access. The author has updated the relevant part of the BIP and the changes can be viewed on GitHub.
