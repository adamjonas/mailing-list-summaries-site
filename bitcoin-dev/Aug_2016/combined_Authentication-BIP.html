Summary:

The proposed BIP aims to provide protection against MITM attacks and fingerprinting by allowing peers to configure multiple identity-keys for each network interface including IPv4, IPv6, and tor. The table generated from this configuration will inform the client of the server ID to expect, ensuring that the client knows the identity of the server it is connecting to. This protects privacy as changing IPs would not provide any privacy if the server announced itself.To avoid linking a tor address to an IPvX address, the author recommends using different identity-keys for each network interface. These identity-public-keys can be shared over a different channel with other node-operators or non-validating clients to grant authorized access. There is no wildcard option provided, however, nodes can have as many listening identities as they want.The purpose of the table is to identify which server ID to expect, but the design aims to preserve privacy even if IP addresses change. To prevent MITM attacks, the client must know the identity of the server it is expecting. Nodes should be able to have multiple identity-keys per network interface and different identity-keys should be used for each network interface.Jonas Schnelli has proposed an authentication scheme to allow MITM detection and rejection in conjunction with BIP151, which requires node operators to build trusted connections for various reasons. The proposed authentication BIP uses ECDSA and secrets will never be transmitted. Each peer that supports p2p authentication must provide two users editable "databases" and the authentication state must be kept until the encryption/connection terminates. Only one authentication process is allowed per connection.The author has updated relevant parts of the BIP which can be viewed on GitHub.
