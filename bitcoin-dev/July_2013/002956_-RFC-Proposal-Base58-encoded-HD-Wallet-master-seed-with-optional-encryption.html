Summary:

The author, JP, updated a proposal by adding a 2 byte 'weeks since 2013-01-01' field and updating the prefixes, ranges and test vectors. The updated proposal can be found on bitcointalk.org. However, Mike Hearn pointed out that it is not usable for SPV wallets unless it has a birthday in it. To solve this issue, he suggested adding a UNIX time as an extra 4 bytes or using a uint16 that represents "days since birth of this specification" to save a few characters.
