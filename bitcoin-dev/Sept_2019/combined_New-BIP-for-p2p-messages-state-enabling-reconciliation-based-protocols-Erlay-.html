Summary:

A new BIP draft has been released that outlines low-level specifications for a reconciliation-based transaction announcement protocol. The proposed spec includes salted short transaction IDs and introduces wtxid-based truncated transaction IDs that could improve efficiency in transaction announcement protocols. If agreed upon, this specification would allow for the integration of more bandwidth-efficient relay protocols. However, an extra round has been added to prevent transmitting duplicate transactions, where two parties explicitly map 32-bit short IDs to 128-bit truncated IDs. The draft provides all the necessary background information and specifies all the messages required for an efficient reconciliation-based protocol, along with new state variables necessary for its operation.Rusty provided feedback on the draft, questioning the use of "requirements" in reference to data and suggesting that actors have requirements instead. He also requested clarification on how to encode q, a coefficient used to estimate set difference, and whether it's one or two sets of truncated IDs per peer. Rusty recommended removing the phrase "To the best of our knowledge," from a statement about PinSketch being more bandwidth efficient than IBLT since he found IBLT to be worse after implementing and experimenting with it.Overall, the proposed specification has the potential to significantly improve efficiency in transaction announcement protocols, making it an important development for the cryptocurrency community. Interested parties are encouraged to read and review the draft thoroughly.
