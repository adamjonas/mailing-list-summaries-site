Summary:

In this email conversation, Gavin Andresen discusses the need for detailed test scripts and plans, especially for boring tasks. He suggests using GUI::Test package for Perl to automate tasks. The idea behind having specific test cases is to see what tests have not been run. He also expresses his concern about laying proper foundations and selecting the right tools for the job. Gavin believes that third-party software can be beneficial and checking out other software that has been recommended could be helpful as well. However, he acknowledges the challenge of changing processes once the team is used to them. Gavin offers to open up his VPS and set up a fake git project to illustrate his point.
