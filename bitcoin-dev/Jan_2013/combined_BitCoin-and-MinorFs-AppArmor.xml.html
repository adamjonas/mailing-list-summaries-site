Summary:

Rob Meijer, the creator of MinorFs, a filesystem that provides private storage to objects in a least authority-based set of filesystems, has proposed an updated version of MinorFs that could be used by Bitcoin on systems that support AppArmor. The existing MinorFs provides a pseudo-persistent process that private members provide to objects and encapsulation of variables that still can be delegated by the object that encapsulates them. However, it has a hard-fixed granularity level that can be cumbersome for some applications like Bitcoin wallets.The new version of MinorFs aims to make it easier for application developers who want their application to protect user data (like bitcoin wallets) from malware running under the same UID as that user. Rob Meijer is seeking input from Bitcoin developers on what attributes should be kept, dropped, or added to arrive at an "id" that is the best fit for granularity of persistent private storage for Bitcoin. In the new version contemplated, there would be at least a minor change to bitcoin needed where bitcoin would have to use a small library that provides a 'minorfs_getpwuid' function.MinorFs would not help secure the wallet on a server or VPS because all read and write operations will map directly to the underlying file-system, which comes with the same lack of guarantee that any file-system comes with once the underlying media becomes flaky. It needs a bash script for starting up bitcoin with $HOME set to the MinorFs home, and proper backup procedures are currently unknown.Rob Meijer is rewriting MinorFs in Python to make it more accessible for application developers who want to protect user data from malware running under the same UID as the user. He has put down initial specs for a rewrite of MinorFs, and he thinks this would be the best option since he has moved most of his new development projects to Python or C++. He has also started work on a library called LibMinorFs to make the usage of MinorFs more convenient by applications like Bitcoin. The library and the filesystems are still far from being done, but Rob would like to know if Bitcoin developers would be open to using this library within Bitcoin for making Bitcoin MinorFs2 aware, allowing it to store things like the Bitcoin purse in a private directory that is not available to other unrelated processes running under the same uid.John Smith expressed interest in Rob's proposal and suggested that Ubuntu needs more strict privilege separation between applications on a least-authority basis. He also emphasized that wallet encryption is useful and necessary but no substitute for OS-level protection. There are questions about how MinorFs would help secure wallets on servers, guarantees against wallet corruption, and proper backup procedures. Despite receiving little interest, Rob Meijer has put down initial specs for a rewrite of MinorFs for future matches with Bitcoin.
