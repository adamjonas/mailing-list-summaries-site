Summary:

The writer proposes using covenants without consensus changes by requiring a specific signature. They suggest using P2PK and legacy signatures but note that Schnorr signatures will be more flexible and allow more use cases. The proposal is to use P2WSH with a redeemScript including OP_CHECKSIG, which would pre-commit to a particular transaction depending on the SIGHASH flags of the fixedSignature. The post-COVID congestion control application proposed for OP_CTV could work in the absence of OP_CTV. A purported OP_CHECKSIGHASHVERIFY which accepts a SIGHASH flag and a hash, and checks that the sighash of the transaction is equal to the hash, would be more efficient and would not differ much from OP_CTV. This technique should work with Tapscripts inside Taproot, but the fixedPubKey cannot be the same as the internal Taproot key.
