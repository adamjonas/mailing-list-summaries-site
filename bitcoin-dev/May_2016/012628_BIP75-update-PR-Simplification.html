Summary:

BIP75 has undergone significant changes which are aimed at simplifying the process. Instead of introducing encrypted versions of all BIP70 messages, a generic EncryptedProtocolMessage type has been defined which acts as a wrapper and enables encryption for all existing BIP70 messages. This reduces the number of new messages being defined and makes it easier to add new message types in the future. Additionally, AES-GCM has been chosen over AES-CBC, thus eliminating the need for the verification hash. The pull request for these changes can be accessed on GitHub. All comments and feedback are welcome.
