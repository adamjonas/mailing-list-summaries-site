Summary:

The Bitcoin Improvement Proposal (BIP) 75 has undergone significant changes to simplify the process. The changes involve defining a generic EncryptedProtocolMessage type that acts as a wrapper and enables encryption for all existing BIP70 messages, rather than introducing encrypted versions of all BIP70 messages. This reduces the number of new messages being defined and makes it easier to add new message types in the future. The pull request for these changes can be accessed on GitHub.In addition, AES-GCM has been chosen over AES-CBC, which eliminates the need for the verification hash. These changes are expected to improve the efficiency and security of BIP75. All comments and feedback are welcome on the GitHub page.
