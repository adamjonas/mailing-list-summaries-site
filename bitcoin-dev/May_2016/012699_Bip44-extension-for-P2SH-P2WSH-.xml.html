Summary:

The proposed scheme discussed in the email chain involves using the chain index to indicate the type of address instead of overloading the account index, making it independent of the number of accounts. The scheme is compatible with BIP44 and BIP43 purpose 0, or any other BIP43 purpose/layout. Aaron Voisine, co-founder and CEO of breadwallet, suggests using 0x40000000 as the bit field flag to specify witness addresses since 0x80000000 is already used to specify hardened derivation. However, Pavol Rusnak questions the advantage of this optimization and reminds that they are discussing multiple-accounts per wallet layout rather than one-account-per-wallet design.
