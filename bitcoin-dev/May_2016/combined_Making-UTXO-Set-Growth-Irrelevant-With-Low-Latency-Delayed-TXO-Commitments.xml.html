Summary:

Bitcoin's long-term decentralization is threatened by unbounded Unspent Transaction Output (UTXO) growth. To address this issue, Peter Todd proposed TXO commitments that involve using Merkle Mountain Ranges (MMRs) to allow new items to be cheaply appended to the tree with minimal storage requirements. However, implementing TXO commitments in Bitcoin raises issues with the security model by allowing miners to profitably mine transactions without validating prior history.TXO commitments propose a merkle tree committing to the state of all transaction outputs, both spent and unspent, providing a method of compactly proving the current state of an output. Implementing UTXO commitments without delaying commitment can impact small miners' orphan rates, making block validation slow. The proposed high-performance/low-latency delayed commitment full-node implementation requires storing the UTXO set, STXO set, TXO journal, and TXO MMR list. There are challenges in implementing the scheme, including determining the minimum age that can be "assumed valid" and the tradeoff between political, social, and technical concerns. Further work is needed to optimize the TXO commitment scheme to avoid complexity and prioritize bitcoin value-transfer over other uses.Peter Todd explains that MMR proofs provided by transactions aren't proofs of how the MMR should be changed; they're just proofs that the MMR is in a certain state right now. Updating a leaf from unspent to spent in the TXO MMR tree incurs a log2(n) merkle path overhead, which is the only significant storage overhead compared to the status quo. Most information is shared between states, which allows simple reference counting for garbage collection.TXO commitments represent an improvement in Bitcoin's scalability and storage efficiency. However, there are still questions that need to be addressed, including how TXO commitments will interact with fraud proofs and whether incentivization schemes for miners can be used to discount or ignore TXO commitment proof sizes. Furthermore, there are proposals related to Bitcoin mining that require only the pruned UTXO set and 32 bytes per archived block for mining.Overall, the proposed TXO MMR scheme allows for efficient storage and retrieval of UTXOs. However, further work needs to be done to determine whether or not TXO commitments should be implemented. The issue of UTXO growth in Bitcoin and its threat to decentralization is a significant concern, with no current consensus rules limiting the growth of the UTXO set.
