Summary:

Gavin Andresen has created a transaction fuzzer on Github that aims to stress-test BIP 16. The tool can be used by using the send* RPC commands to obtain one or more transaction IDs and then running a script that repeatedly calls relayfuzzed with a nonce/txid while observing how the connected peer(s) react to all the fuzzy 'inv/tx' protocol messages. Gavin stated that there are many other useful features that could be added and welcomes patches. He also mentioned that adding block fuzzing is on his to-do list for the future.A sender interested in finding more information on this topic has inquired about the availability of a fuzzer for libbitcoin, as they plan to create one themselves if not. They want a simple fuzzer that can set the previous block hash, set the current target, start hashing, and connect and send to a specified host (i.e localhost). Although the sender believes that building one should be trivial, they still wish to ask for more details.
