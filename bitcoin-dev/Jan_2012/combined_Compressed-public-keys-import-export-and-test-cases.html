Summary:

Pieter, a developer, has proposed a new import/export format for private keys with compressed public keys on the Bitcoin development mailing list. The format includes a marker that specifies whether to use the corresponding compressed or uncompressed public key, as they have different addresses. The format for an uncompressed key is 0x80 + [32-byte secret] + [4 bytes of Hash() of previous 33 bytes], base58 encoded. For a compressed key, the format is 0x80 + [32-byte secret] + 0x01 + [4 bytes of Hash() previous 34 bytes], base58 encoded.To test implementations, Pieter provided three cases, each including a secret (in hex), an uncompressed and compressed secret (in base58), a pubkey (in hex), and an address (in base58). The first case has a secret of 1111...1111 and an uncompressed and compressed address of 1MsHWS1BnwMc3tLE8G35UXsS58fKipzB7a and 1Q1pE5vPGEEMqRcVRMbtBK842Y6Pzo6nK9, respectively. The second case uses a secret of dddd...dddd and has an uncompressed and compressed address of 1JyMKvPHkrCQd8jQrqTR1rBsAd1VpRhTiE and 1NKRhS7iYUGTaAfaR5z8BueAJesqaTyc4a, respectively. The third and final case includes a secret of 47f7616ea6f9b923076625b4488115de1ef1187f760e65f89eb6f4f7ff04b012 and an uncompressed and compressed address of 1PM35qz2uwCDzcUJtiqDSudAaaLrWRw41L and 19ck9VKC6KjGxR9LJg4DNMRc45qFrJguvV, respectively.Pieter has invited comments or suggestions regarding this new format.
