In a message sent on January 29, 2012 at 12:10 AM, Amir Taaki discussed the size of compressed pubkeys. He noted that each pubkey is 33 bytes and when combined with 1 byte for N (n-of-m), 1 byte for the address version, and a 4 byte checksum, the total size becomes 72 bytes. However, to obtain an address from these bytes, base58 encoding is necessary which results in a 99 character long address.
