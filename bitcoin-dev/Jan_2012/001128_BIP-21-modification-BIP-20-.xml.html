Summary:

The discussion revolves around the standard of treating unknown URL parameters and the need to be open to future backward-compatible changes. The suggestion is to define the standard upfront, rather than refusing all unknown parameters, which can break older clients. A versioning scheme is proposed, where any BIP breaking backwards compatibility should increase the version number, and clients reject URLs with a higher version number. This approach is deemed to be the simplest and sufficient way to handle the issue. Ultimately, the standard defined in the BIP will be the agreed-upon standard.
