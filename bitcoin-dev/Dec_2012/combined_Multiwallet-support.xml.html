Summary:

A proposal has been made by Eric Lombrozo to enable watch-only addresses in bitcoin wallets, however, this will require disabling signing and privkey export operations for such addresses. To implement this feature, multiple wallet capabilities will need to be added so that upon creation, a wallet can be specified as a full signing wallet or a watch-only wallet. Exposing multiple wallets via RPC will be tricky since the existing RPC is not designed to support multiple wallets. As a solution, a main wallet which is always used as the default wallet can be implemented to avoid breaking compatibility with existing RPC calls. A separate set of calls will also be required that require a wallet name and passphrase if the wallet is encrypted.The developer also proposes adding the ability to specify multiple wallets with associated names and passphrases in the config file, which should be straightforward. The discussion then moves on to how rpcs that don't make sense should return an error instead of being deactivated and how sendtoaddress on a watching wallet should return an unsigned raw transaction and a wallet-specific message that tells you where to find the private key. It's mentioned that while compatibility should not be broken, it shouldn't get in the way of doing the feature right.Overall, the developer welcomes proposals or suggestions on how this should be done and emphasizes the importance of implementing this desirable feature without compromising compatibility.
