Summary:

Christopher Gilliard has proposed a Bitcoin Improvement Proposal (BIP) regarding signature formats when using Bitcoin private keys. The proposal includes actual code for verification, but not signing. It is similar to the implementation that Trezor implemented in a previous issue on Github.However, a commenter suggests that the proposal should have a more precise description of the signing/verification process and the "Background on ECDSA keys" section should focus on signatures instead of keys. Additionally, the purpose of the header is unclear and testing different possibilities is not a big deal. Aymeric Vitte also asked for a more precise description of the signing/verification process since it is documented nowhere.The reason for including data in the header is the same as compressed/uncompressed keys being included in the header, so that the type of key used for the signature can be known without having to test all options. The proposal suggests that if there is a header field in use, it seems reasonable to use it to specify which type of key is being used. Testing the four possible options is certainly a possibility; however, not every wallet implements something like this. If the header includes whether the key is compressed or not compressed, it seems logical to include all data about what type of key it is and not just this one type of information.The proposal includes actual code for verification but not for signing. It is in line with the implementation that Trezor has already done. However, Aymeric Vitte suggests that a more precise description of the signing/verification process should be added as it is currently undocumented. There are also questions about the purpose of the header field and the link between keys and signatures.The document could be useful to others since as far as Gilliard could tell, it was not specified anywhere. The developer invites others to review/test their PR and provide feedback. The steps for getting a BIP number assigned or any other process steps required are yet to be determined.In related news, the Wasabi wallet is almost finished with the development of an encryption manager, and the developer has requested the OP to review the proposed BIP. The BIP can be found on Github and was written in response to a Github issue regarding signature formats when using Bitcoin private keys.
