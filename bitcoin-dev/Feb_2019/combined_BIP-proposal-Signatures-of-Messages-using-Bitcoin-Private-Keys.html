Summary:

A proposed Bitcoin Improvement Proposal (BIP) has been written by Christopher Gilliard, addressing signature formats when using Bitcoin private keys. The proposal includes code for verification but lacks code for signing. Gilliard's proposal aligns with Trezor's implementation in a previous GitHub issue. Feedback on the proposal is welcomed and Gilliard is seeking guidance on how to assign a BIP number or any other necessary process steps. Aymeric Vitte suggests adding a more precise description of the signing/verification process, as it is currently undocumented. Vitte also questions the purpose of the header and its link to keys and signatures. The inclusion of compressed/uncompressed keys in the header serves to identify the type of key used for the signature, avoiding the need to test all options. Gilliard clarifies that the proposal documents existing practices rather than presenting an original invention. The document could be useful to others since this information was not previously specified.
