Summary:

A proposal to use satoshi as the base unit for Bitcoin has been suggested, but it has faced criticism from some members of the community. The idea was presented on the bitcoin-dev mailing list by Amine Chakak, and the proposal link was provided, but one user argued that if Satoshi Nakamoto had wanted the currency units named after him, he would have done so himself. Additionally, another user pointed out that the lightning network uses units of 10 picobitcoin rather than satoshis. Gregory Maxwell expressed his opposition to naming the smallest unit of bitcoin after Satoshi, stating that it is "creepy" and harmful to Bitcoin. However, another participant in the discussion disagreed, arguing that having a common term for small amounts is important for improving user experience. They believe that using satoshis as a unit is more likely to achieve this goal than scientific notations. One potential issue with using satoshis as a unit is that it may sound strange or even humorous to native Japanese speakers.Regarding the proposal to formalize the symbol for satoshi, there is some disagreement about whether it has sufficient support to become a standard. The suggestion is made to wait for something to emerge that already has widespread usage, such as a symbol used by popular wallets or service providers. Non-engineers are unlikely to adopt scientific notation or mili/nano/pico prefixes for money, and Bitcoin engineering blindness ignores evident cultural preference and pre-existing finance-related software. Tamas Blummer supports the BIP for bits as a base unit for Bitcoin, but Gregory Maxwell finds it creepy and harmful to Bitcoin to name currency units after Satoshi. In conclusion, a BIP for satoshi as a base unit is not appropriate.
