Summary:

In this email thread, ZmnSCPxj is responding to a previous message by Anthony Towns regarding the use of a boutique protocol versus PSBT for onchain wallet integration in UI. ZmnSCPxj agrees that widespread support for PSBT would be beneficial. Anthony suggests that PSBT would be sufficient for lightning node to generate a PSBT for a new channel and the wallet would fund the PSBT but not sign it until lightning creates update/settlement tx's for funding. This process is more convoluted than using NOINPUT addresses but avoids users passing them around or risking a user typo'ing the value and losing money immediately. The benefit of using PSBT is that the wallet can tweak the value if necessary and avoid ever needing to publish a NOINPUT sig or NOINPUT tagged output.
