Summary:

The maximum transaction size is increased by approximately 81 bytes, allowing nearly four MB transactions with Segwit transactions not counting witness data. The plain block size is defined as the serialized block size without witness programs. Deploying a modified BIP91 to activate Segwit replaces 'segsignal' with 'segwit2x.' If segwit2x activates at block N, then block N+12960 will activate a new weight limit of 8M, corresponding to a size of up to 8,000,000 bytes. Any transaction with a non-witness serialized size exceeding 1,000,000 is invalid. Larger block sizes are not likely to have a meaningful impact on fee pressure.
