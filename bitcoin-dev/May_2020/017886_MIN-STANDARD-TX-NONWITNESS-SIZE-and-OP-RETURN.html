Summary:

In a Bitcoin development discussion, Thomas Voegtlin asks why a 60-byte transaction (without witness) is rejected because it's lower than MIN_STANDARD_TX_NONWITNESS_SIZE, which is currently set at 82 bytes. In response to this query, ZmnSCPxj suggests that if they accept a 60-byte tx, then SHA-256 will pad it to 64 bytes, and it may still be possible to mount CVE-2017-12842 attack with 32-bits of work. Therefore, it would seem safer to avoid the area around length 64. It *might* be safe to accept 65-byte or larger, but this does not help Thomas' specific application, which uses 60 byte tx. Greg Sanders adds that the number was picked to protect againstÂ CVE-2017-12842 covertly, and provides a link to the update in the text that explicitly mentions this fact. The discussion suggests that lowering the value to 60 may pose security threats and risks, thus keeping the current value of 82 seems more feasible.
