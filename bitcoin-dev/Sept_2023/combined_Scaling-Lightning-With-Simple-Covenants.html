Summary:

The email discusses the tradeoff between trust and capital efficiency in the context of the Lightning Network. It addresses the potential "thundering herd" problem where a large number of dedicated users collude to congest the blockchain and steal funds from casual users. The proposed solution involves expiring timeout-trees only after a sufficient number of low-fee blocks have been added to the blockchain, thus safeguarding the funds of casual users. The email also suggests pairing dedicated users to fund hierarchical channels and using "short-cut" transactions and funding HTLCs from two possible sources to route payments. The sender proposes a change to the Bitcoin consensus rules to prevent dedicated users from colluding and stealing funds. The email highlights concerns about the impact on-chain if the scalable path of the Lightning Network fails and proposes using simple covenants enabled by CheckTemplateVerify (CTV) or AnyPrevOut (APO) to create Lightning channels for millions of casual users. By changing the consensus rules to allow simple covenants, a covenant tree can be created that guarantees the creation of Lightning channels for a large number of casual users without requiring their signatures. The email also suggests using the FFO-WF Lightning channel protocol to facilitate the rollover of casual users' funds without requiring their input. It recommends incorporating mechanisms like CTV or APO in Bitcoin's consensus rules to enable simple covenants and timeout-trees, improving the scalability of Lightning as a means of payment. Overall, the email proposes using off-chain transactions and covenants to improve the scalability of the Lightning Network, reducing the on-chain footprint and enabling various actions to be performed off-chain.
