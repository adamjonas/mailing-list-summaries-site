Summary:

On February 2nd, an open meeting was held on IRC to discuss Taproot activation. The meeting was previously announced on a Linux Foundation mailing list. The conversation log is available online for those interested in the discussion. Rusty Russell provided his takeaways from the meeting which included unanimous support for BIP 8 and overwhelming consensus that one year is the correct timeout value. There was also majority consensus for lockinontimeout false, though some individuals, including Luke Dashjr, opposed it. No decision was made on start time.Following this, a Taproot activation meeting was held on the ##taproot-activation Freenode channel. The agenda was posted in advance to the mailing list by BitcoinMechanic. The conversation log is available on gnusha.org. During the meeting, participants discussed an alternative release to Bitcoin Core with Speedy Trial (BIP 8, consistent use of block height) followed by BIP 8 (1 year, LOT=true) as the activation mechanism(s). However, some participants expressed concern over having "Bitcoin Core" in the name of the alternative release, suggesting that it would be confusing for users and give them the impression that it had been signed off by Bitcoin Core maintainers. The draft release notes for this alternative release are available on Google Docs, while the GitHub repo can be found on BitcoinActivation's GitHub page.To compare this to the most likely activation mechanism(s) in Bitcoin Core at this point, Core PR #21377 is currently being considered for merge, which proposes that the activation mechanism in Bitcoin Core will be Speedy Trial (BIP to be decided, mix of block height and MTP). A BIP PR for the Core release has also been opened with suggested finalized parameters. However, if these plans continue as is, Bitcoin Core and this alternative release won't be entirely compatible due to startheight and timeout being defined according to MTP and block height respectively. While they should both activate at the same block height (or not activate) in the majority of cases, there are unlikely edge cases where one activates and one doesn't.Chun Wang, co-founder of F2Pool with approximately 16% of global hash rate, has decided to support BIP 8 (false, 1 year) according to an update on taprootactivation.com provided by Alejandro De La Torre. In addition, two BIP 8 PRs were merged following the open meeting due to an observation from Jonas Nick that without them nodes could end up on the wrong chain. Bitcoin Core PR #19573 requires further review before it can be considered for merging.
