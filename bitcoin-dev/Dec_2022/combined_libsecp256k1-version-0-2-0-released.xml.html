Summary:

After a decade of development, the libsecp256k1 team has announced the first tagged release of version 0.2.0. Previously, development only had a master branch which caused confusion about API compatibility and stability. The new tagging system will provide clarity in future releases and clear documentation for any API breaks that may occur. The team is skipping version 0.1.0 as it does not uniquely identify a set of source files. Although binary releases will not be created, ABI compatibility issues will be taken into account for release notes and versioning. Furthermore, the release tag is signed with the PGP key of the lead developer, and suggestions are welcome for improving the release process in the future.
