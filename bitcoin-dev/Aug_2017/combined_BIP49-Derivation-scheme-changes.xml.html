Summary:

The Bitcoin development mailing list was the center of several discussions related to Segwit wallets and BIP proposals. One of the issues raised concerned the need to define a structure for Segwit wallet seed words, as they are incompatible with previous seed words. Electrum has already defined a structure that can be found in their documentation.Another issue concerned P2WPKH nested in BIP16 P2SH requiring a different version than P2WPKH. Pavol Rusnak suggested both proposals from Thomas Voegtlin and a user were fine, urging the group to decide soon. There was also discussion regarding proposed changes to the BIP32 serialization format, with suggestions for three types of keys instead of two due to the existence of two types of segwit output scripts. In addition, the writer suggests creating new wallets as segwit m/49' without asking the user anything, with the option to choose non-segwit from advanced options. XPUB derivation is another issue not addressed in the BIP yet, with no way to know whether an xpub is from m/44' or m/49'. Peter Todd also shared concerns about the security implications of using compact SPV proofs for timestamping in OpenTimestamps. Meanwhile, Shiva Sitamraju reached out to the Bitcoin development mailing list to propose changes to the BIP32 serialization format and differentiate segwit xpub/xprv, using letters z,y,z combined with pub/prv for three types of keys. Despite these discussions and proposals, timestamping remains a significant aspect of Bitcoin despite its vulnerability to malicious miners.
