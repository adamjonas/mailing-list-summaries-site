Summary:

The writer of the message wants to discuss changes in BIP49. They suggest breaking backwards compatibility because of issues that arise when recovering a wallet using seed words. The problem is that there is no difference in seed words between segwit/non-segwit, so the wallet would discover both m/44' and m/49' accounts. Asking the user to choose an account to operate on is not a good UI solution, and the wallet would need to constantly monitor both accounts for transactions. The writer suggests creating new wallets as segwit m/49' without asking the user anything, with the option to choose non-segwit from advanced options. Segwit wallet seed words would have a different format which encodes the information that the wallet is segwit. XPUB derivation is another issue not addressed in the BIP yet. Currently, there is no way to know whether an xpub is from m/44' or m/49'. This breaks things like importing xpub as a watch only wallet and displaying balance/generating merchant addresses. The writer is looking forward to hearing thoughts on these suggestions.
