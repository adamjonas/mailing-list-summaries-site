Summary:

A significant number of cryptocurrency products contain a JavaScript class named SecureRandom() that has deficient entropy collection and a PRNG, allowing key material to be recovered by a third party with medium complexity. The library attempts to collect entropy from window.crypto's CSPRNG, but due to a type error in a comparison, this function is silently stepped over without failing. Entropy is subsequently gathered from math.Random and a single execution of a medium resolution timer. The core of the RNG is an implementation of RC4 ("arcfour random"), which is publicly known to have biases of several bits, which are likely sufficient for a lattice solver to recover an ECDSA private key given a number of signatures.The vulnerability disclosure made regarding a pre-2013 version of jsbn, a JavaScript crypto library, may affect a significant number of past and current cryptocurrency products. Even with the latest version (v1.4) of jsbn, it still does not use high-quality entropy for Internet Explorer because getRandomValues is provided under window.msCrypto for that browser. Davies also notes that Chrome used RC4 for crypto.getRandomValues at one point in 2015.Necessary actions include identifying and moving all funds stored using SecureRandom(), rotating all key material generated by or that has come into contact with any piece of software using SecureRandom(), not writing cryptographic tools in non-type safe languages, and not taking the output of a CSPRNG and passing it through RC4. It should be noted that products that aren't vulnerable today due to moving to other libraries may be using old keys that have been previously compromised by usage of SecureRandom(). The issue with Internet Explorer is that it implements the Web Cryptography API under window.msCrypto instead of window.crypto, so unless msCrypto.getRandomValues is used, high-quality entropy will not have been used by any of these libraries under Internet Explorer. Jason Davies provided links to several repositories including Bitcoin transactions made simple, Zcash wallets made simple, and Bitcoin wallets made simple. Additionally, he also shared links to get the torrent dynamic blocklist, check the 10 M passwords list, anti-spies and private torrents, dynamic blocklist, Peersm, torrent-live, node-Tor, and GitHub pages.To prevent compromise of their keys, products using SecureRandom() should identify and move all funds stored, rotate all key material generated by or in contact with any software using SecureRandom(), not write cryptographic tools in non-type safe languages, and avoid passing the output of a CSPRNG through RC4. It is recommended that users take necessary actions to prevent compromise of their keys.
