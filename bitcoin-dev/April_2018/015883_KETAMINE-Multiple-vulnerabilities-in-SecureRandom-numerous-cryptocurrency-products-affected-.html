Summary:

The discussion in the given context is about the implementation of Web Cryptography API by Internet Explorer. The issue with Internet Explorer is that it implements the Web Cryptography API under window.msCrypto instead of window.crypto, so unless msCrypto.getRandomValues is used, high-quality entropy will not have been used by any of these libraries under Internet Explorer. This impacts jsbn only normally since all others from the time getRandomValues was available are supposed to implement both.
