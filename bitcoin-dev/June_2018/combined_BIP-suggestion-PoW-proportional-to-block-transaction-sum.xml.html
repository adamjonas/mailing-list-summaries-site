Summary:

Darren Weber, a Bitcoin developer, has proposed a new idea on the Bitcoin Github page to make it more difficult for malicious attackers to double-spend large amounts with a relatively brief majority of the network hashing power. He suggests introducing a hash workload that is proportional to the sum of transactions in a block, which could be linear or exponential. However, concerns have been raised regarding its implementation and exclusion of transactions based on economic value. Thomas Guyot-Sionnest notes that this proposal could have detrimental side-effects such as an increase in UTXO size for everybody and would require good consensus and a hard fork. Despite these concerns, Darren plans to follow-up with a better BIP proposal if there is value in the idea.
