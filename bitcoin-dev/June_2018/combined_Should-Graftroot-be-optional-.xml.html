Bitcoin developers are discussing the implementation of Graftroot and Taproot, which offer different trade-offs for spending paths in transactions. While Graftroot is more compact when dealing with multiple spending paths, Taproot allows for proving all the different spending paths. There are concerns about accidentally signing scripts and the potential for certain participants to remove others from the set of signers in a multisignature setting without Graftroot. Developers are also discussing the possibility of introducing graftroot-like delegation to all existing and new P2PK and P2PKH UTXOs with a soft fork.
