Summary:

The Bitcoin Core is working on a new protocol to replace the existing getblocktemplate and Stratum protocols used in Bitcoin mining pools. The new protocol aims to achieve three goals. Firstly, it aims to be more performant and robust for consensus changes while allowing for better optimization of work switching within Bitcoin Core. Secondly, it seeks to be more secure, simpler to implement, and better-defined than Stratum. Finally, it provides decentralization pressure by allowing users to select transactions for pool acceptance, thereby removing network-level centralization attacks. The proposal has undergone several rounds of feedback and provides VENDOR_MESSAGEs for extensibility instead of conflicting specifications from different vendors. The full BIP draft and implementations of the work-generation and pool/proxy parts can be found on GitHub. The whole stack has successfully mined numerous testnet blocks on different hardware types. Overall, the new protocol could significantly improve decentralization in Bitcoin mining.
