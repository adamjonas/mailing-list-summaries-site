Summary:

Peter Todd, a Bitcoin developer, responded to a discussion on the bitcoin-dev mailing list regarding the possibility of attaching work to low fee transactions as a way to compensate and prevent spam. However, this idea was deemed problematic due to the prevalence of ASICs. Instead, Todd proposed a hashcash-based alternative broadcast scheme that would rate-limit access to a broadcast network used by a small minority of nodes. This system would not be a consensus system, so it would be okay to change the PoW algorithm if someone were to build an ASIC for it. Additionally, it is acceptable for multiple versions of the scheme to run in parallel. Todd's proposal is sophisticated and consequential, but not infeasible.
