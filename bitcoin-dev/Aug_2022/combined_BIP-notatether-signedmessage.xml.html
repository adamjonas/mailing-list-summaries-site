Summary:

Bitcoin Core contributor Ali Sherief has proposed a new Bitcoin Improvement Proposal (BIP) called notatether-signedmessage. This BIP defines a step-by-step process for signing and verifying messages from various types of Bitcoin addresses, including legacy, native/nested segwit, and taproot addresses. However, the BIP does not define a new signature format itself, except in the case of Taproot, where a signature format is defined that has 1 byte header/recID, 64 bytes signature, and 32 bytes x coordinate of a public key.Ali plans to move the Taproot signature format to its own BIP as soon as possible. Comments regarding notatether-signedmessage can be left on the mailing list with BIP editors CC'd. Pavol Rusnak suggested that Ali should focus more on what he is adding (Taproot) while saying his BIP is an expansion of BIP137. He proposed that Ali move the algorithms (of BIP137) and stuff to Bitcoin Wiki and then convert the BIP to strictly a "Taproot message signing BIP".In an email sent on August 5th, 2022, Ali discussed the importance of advancing BIP322 as the first priority. However, fellow contributor Luke Dashjr expressed his opinion that there is no benefit to an additional message signing standard, especially one that does not address the problems with the current standard or BIP322. Ali admitted that the Taproot signature format should not be located inside this BIP but rather contained in a newer Standards Track BIP that supersedes BIP137. The proposal will only solve half of the problems, and the other half must be solved by getting other wallet implementations to implement this standard. It is a non-trivial task, and assigning a number to this BIP would help to get halfway to the finish line. Message signatures are highly relied upon in some places, such as mining pools and forums, and it is necessary to ensure as many wallets as possible use the specification in the BIP for message signing and verification, along with implementing the legacy and segwit address parts.In summary, Ali Sherief's notatether-signedmessage BIP defines a process for signing and verifying messages from various types of Bitcoin addresses. The BIP does not define a new signature format itself, except in the case of Taproot. Ali plans to move the Taproot signature format to its own BIP as soon as possible. Additionally, the importance of advancing BIP322 as the first priority has been discussed, but there are differing opinions on the need for an additional message signing standard.
