Summary:

The context revolves around the implementation of hierarchical deterministic (HD) wallets in Bitcoinj, which allows users to generate and receive payments into BIP32-style addresses. One proposed solution involves using PubKeyParent and HMAC(Multiplier, PubKeyParent) to calculate the destination address and provide proof of payment to a third party who has not seen the communication between payer and payee. The article also discusses the use of ECDSA signatures for type2 public keys and limitations of BIP32 when using multiple chains.A new address format is proposed that would allow for a "Wallet ID" and "One-time payment" code to be specified instead of the standard one-use Base58-Hash160 addresses. This new format enables parties with a persistent relationship to prove that payment addresses they provide each other are linked to a particular wallet, reducing exposure to Man-in-the-Middle (MitM) attacks without compromising privacy. However, there are concerns about the violation of P2SH goals and revealing more information than necessary.The discussion covers the use of PubKey and ChainCode for forming a persistent relationship between two parties using BIP32. Exchanging PubKey and ChainCode beforehand can help in transacting securely forever without exchanging any further information or reusing any addresses. The limitations of BIP32 in using multiple chains are also discussed, with suggestions on how to generate new receive chains from a watch-only wallet and adding minimum expectations about how a wallet should be 'probed' when imported.The article emphasizes the benefits of leaving the age of single address generation behind and implementing a mechanism for generating address chains that does not complicate the wallet. It highlights the importance of creating new receive chains from a watch-only wallet, probing wallets when imported, and using static public keys for persistent identity and security from MitM. Overall, the discussion provides valuable insights into the challenges and opportunities of implementing HD wallets in Bitcoinj.
