Summary:

The discussion revolves around the exchange of public keys for multisig transactions. The current method of displaying public keys, hex encoding, is not ideal as it results in longer strings and lacks 4-byte verification which raises security concerns. Thus, the author suggests using a standard way to encode public keys as base58-check addresses to make it easier and safer to display and exchange public keys. The author proposes using 0x37/0x38 as a prefix byte resulting in letter P for "Public", since these bytes are currently not being used for anything else. It is clarified that the author is referring to the actual public key, not the hash160 of it used for Bitcoin addresses. Although not usually used, it is needed for multisig transactions. Overall, this approach would serve the purpose of securely exchanging public keys while making it easier for users to display and share them.
