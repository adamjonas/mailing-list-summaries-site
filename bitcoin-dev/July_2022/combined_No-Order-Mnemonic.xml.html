Summary:

The bitcoin-dev mailing list recently discussed ways to improve the security and usability of mnemonic encoding for Bitcoin wallets. One suggestion was to reduce the entropy of a 12-word seed by removing its ordering, but concerns were raised about security. Sorting a seed alphabetically can reduce its entropy by approximately 29 bits, which is significant. However, James MacWhyte proposed choosing 11 random words and sorting them alphabetically before assigning a checksum, which would significantly reduce the entropy. Despite these challenges, the discussion concludes that any ordering is okay as long as the new words are in the same pool as the old words.Another proposal suggested encoding the BIP39 words in a mnemonic phrase without requiring users to remember the order. There are concerns about the limited combinations needed to guess the one true ordering even if the words are written down in a 3x4 grid. Various suggestions were made, such as swapping two positions or making a single deletion, but they become more difficult geometrically each time. Anton Shevchenko shared his Python implementation for a different mnemonic encoding that requires users to remember words but not their order. Additionally, a mathematical question was raised about whether it is possible to create an encoding using BIP-39 words as part of error correction if the order is lost or confused.Overall, the discussion touches on various ways to improve the current systems, including using unordered encodings or sorting randomly chosen words alphabetically before assigning a checksum. Several experts in the field participate in the discussion, sharing their insights and suggestions for improving the current systems.
