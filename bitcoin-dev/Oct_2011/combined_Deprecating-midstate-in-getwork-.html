Summary:

The email thread revolves around improving the efficiency and simplicity of the internal reference miner in Bitcoin. Nils Schneider proposes simplifying the miner by removing all dependencies on cryptopp, which is only used in one location. The plan is to instead use OpenSSL for all functions, except for calculating getwork "midstate," which is currently calculated using cryptopp. However, since midstate can be easily calculated from the blockheader, it is suggested that it should be removed to allow for cleaner code.John Smith expresses doubts about depending on two crypto libraries and suggests using OpenSSL instead. Stefan Thomas mentions that BitcoinJS uses OpenSSL to calculate midstate, while Laszlo provides a prototype for the OpenCL miner without eliminating redundant calculations. Jeff Garzik confirms that he has already field-tested a guess that midstate can be removed, although he warns that at least one miner (cpuminer) requires it.Schneider agrees that OpenSSL can be used to compute midstate but still deprecates midstate and hash1 anyway, stating that this would lead to cleaner code. He provides a pull request for marking midstate and hash1 as deprecated. The email exchange concludes with Schneider asking where such a change should be announced so all miners can be upgraded.Overall, the email thread focuses on improving the efficiency and simplicity of the internal reference miner in Bitcoin. The proposed changes include removing dependencies on cryptopp, deprecating midstate and hash1, and implementing SHA2 without relying on other crypto libraries. The discussion also touches on compatibility with miners and the need for announcing changes to ensure all miners can be upgraded. Links to relevant pull requests and code snippets are provided throughout the thread.
