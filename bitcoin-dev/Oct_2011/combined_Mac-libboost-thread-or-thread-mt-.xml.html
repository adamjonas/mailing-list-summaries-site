Summary:

Brian McQueen had an issue with his cmake btc buildsystem, where he had installed Boost via mac ports but it didn't have libboost_thread.a. He was unsure whether to modify the makefile or get a different version of Boost. Michael suggested changing the name to -mt, but it is unclear what this means for a threading library.Gavin Andresen updated src/makefile.osx and doc/build-osx.txt to assume that MacPorts versions of dependencies will be used and the -mt boost libraries will be used. He also modified makefile.unix and makefile.osx to auto-build dependencies using gcc's -MMD option. Douglas Huff suggested that Brian should install the newer 1.46 version of Boost instead of the couple of boost packages available in Macports. Eric Mockensturm recommended that Brian could statically link libz into crypto when he compiled crypto. It is strange that libz would be coming from the crypto libs. Additionally, Gavin Andresen had to add libz.a to the makefile.osx when compiling.
