Summary:

In an email exchange on the Bitcoin-development mailing list, members discussed various issues related to the compilation of crypto libraries and boost packages. Gavin Andresen made updates to the makefile for Mac OS X and the build instructions, assuming that the dependencies would be from MacPorts and the -mt boost libraries would be used. He also modified the makefile for Unix and Mac OS X to enable auto-building of dependencies using gcc's -MMD option. Douglas Huff suggested installing the newer 1.46 boost packages due to issues with the previous versions available on MacPorts.Brian McQueen encountered a problem with his boost installation via MacPorts as it did not include the required libboost_thread.a file. He reached out to Michael for advice on whether to modify the makefile or obtain a different version of boost. Michael responded that he had faced a similar issue with his cmake btc buildsystem and recommended changing the name to -mt.
