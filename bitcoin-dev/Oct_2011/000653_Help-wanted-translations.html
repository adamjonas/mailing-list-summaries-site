Summary:

The operator of the Eligius mining pool has announced that he will no longer be using bitcoind for the pool's primary operations, but rather will be writing custom software designed to more efficiently create work. Bitcoind will still be used for peering with external nodes and selecting valid transactions for blocks. This move is similar to what Steve is doing with poolserverj. The custom software will handle tasks such as incrementing the extraNonce and recalculating the merkle tree/root.
