Summary:

Gavin Andresen reported that there is no final 0.5 release out yet as he is "really paranoid about bitcoin binary builds" and the switch to Qt means a change in the way builds are done. Linux builds should be set while Windows builds are being difficult. Gavin Andresen suggested that they need a GCC expert to help debug the 'gitian' cross-compile. He also mentioned that Mac builds were slightly broken for the 0.4 release and need to fix it. On his wish list for builds, he thinks it would be nice to have a .pro file to compile bitcoind as maintaining N different makefiles is annoying and error-prone. In response, someone suggested that it would be better to add a GUI=0/1 option to the bitcoin-qt.pro until a nicer system is implemented. In addition, Gavin Andresen said that requiring qmake for the daemon build may not be the best option since it does not add anything for non-Qt software. The suggestion was made to stick with a makefile instead or create a bash script that wraps both the daemon and UI builds if possible and converts command line flags.
