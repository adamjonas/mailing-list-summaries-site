The Bitcoin Improvement Proposals (BIPs) are important and have been previously hosted on a wiki that has been unreliable in the past. A user is seeking information on how to contact MT to get access to BIPs that they have not finished copy-editing, and there is a new one to be added. It has been suggested that a git repository could be used as a backup for the BIPs, and there is agreement amongst users to use Github's dedicated wiki feature, which is git-backed and uses markdown syntax that automagically pushes git commits out to the website.In 2011, the Bitcoin-development mailing list hosted on sourceforge.net discussed the possibility of using a git repo or something more redundant for BIPs. Daniel F endorsed the use of Git, not necessarily as replacement, but at least as a backup. He also suggested using markdown and github pages.
