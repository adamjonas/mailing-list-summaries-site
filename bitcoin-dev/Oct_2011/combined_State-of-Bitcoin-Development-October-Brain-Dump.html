Summary:

Gavin Andresen, a prominent developer in the Bitcoin community, has provided updates on his current projects and priorities for Bitcoin development. He notes that Wladimir has joined the core dev team to help pull patches, particularly Qt-GUI related ones. Network stability and wallet security are a top priority, as well as improving the start-up experience for new users.Andresen also highlights his work on improving security vulnerabilities and implementing/experimenting with multi-signature transactions and new types of Bitcoin addresses. He also plans to write BIPs proposing OP_EVAL and standard multi-signature transactions. Additionally, he is working on cross-platform testing infrastructure and tightening block-time rules to fix potential attacks. In the long-term, he aims to rethink/rework transaction fees and create a non-profit organization to better support Bitcoin development.During a discussion on October 13, 2011, Gavin Andresen proposed two ideas to improve Bitcoin's security. The first one was to tighten block-time rules to prevent potential "timejacking" attacks, but this was deemed unnecessary as the existing rules were already strict enough for the two-week window. The second idea was to discourage blocks and transactions that are bad for the common good but beneficial for individual miners or nodes. However, this concept was deemed flawed and only potentially useful in extreme cases where a miner intentionally creates many 1 MB blocks of junk.In addition to these proposals, it is mentioned that the Tor Project will be paying core developers, testers, a PR person, and expenses such as the Jenkins nightly build server. However, no further details are provided on this topic.
