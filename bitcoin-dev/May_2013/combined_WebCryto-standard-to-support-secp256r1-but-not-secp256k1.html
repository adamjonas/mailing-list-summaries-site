Summary:

Melvin Carvalho has proposed adding Bitcoin's Koblitz curve to the native crypto browser support expected to arrive next year. The Web Crypto API currently includes three NIST recommended curves: P-256, P-384, and P-521. Melvin reached out to the chair of the crypto group for more information and was advised to email public-webcrypto-comments at w3.org with his use-cases. The goal of the proposed native crypto browser support is to expose the crypto code that already exists rather than having browser vendors write new code. However, an open issue remains regarding whether an "experimental" registry for identifiers for new curves not included in the core spec will be maintained. The chair of the crypto group suggested that if browsers don't currently support it, they may want to support it in the future given Bitcoin's growth. Anyone with a use case for ecdsa in the browser is encouraged to contact Melvin or write to the public list.
