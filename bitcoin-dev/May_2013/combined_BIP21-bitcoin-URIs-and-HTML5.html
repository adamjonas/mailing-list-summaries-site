Summary:

During a recent discussion on the BIP21 spec, issues were pointed out and discussed. Gavin Andresen suggested that fixing the bugs in place would be the best approach, rather than creating a new BIP. Gregory Maxwell agreed with this suggestion but emphasized that any corrections made should not be gratuitously incompatible. In addition to the discussion on the BIP21 spec, developers of the Chrome browser have made changes to whitelist bitcoin: URIs for web apps. However, Firefox does not use whitelisting at all, potentially discouraging developers from supporting the payment protocol. The goal now is to encourage web wallet developers to support the payment protocol by filing bugs against WebKit for Safari/iOS/misc other platforms and IE. It is uncertain whether Microsoft uses open bug trackers much.To address the issues with the BIP21 spec, a proposal was made to add a new meta-data item in the BIP header labeled "Corrected". This proposal was agreed upon by Gregory Maxwell. The aim is to improve the value proposition of the Web and encourage more web wallet developers to support the payment protocol.It was noted that HTML5 now allows web apps to register themselves for handling URI schemes, such as the bitcoin: URI that is already in use and being extended as part of the payment protocol. Bitcoin has been added to the whitelist of acceptable schemes in the specification for security reasons, and browser makers should notice the change soon. The IETF URL specs are being obsoleted by http://url.spec.whatwg.org/, at least for Web purposes. The IANA application will be improved if there is evidence of bitcoin being used.
