Summary:

In 2013, Bitcoin wallets were discussed regarding several minor security and functionality issues. Gavin Andresen suggested adding a "sweep private key" feature to avoid spending the entire wallet balance onto a new address to move BTC off a specific address. This method created more network traffic, higher fees, and damaged privacy as all funds ended up linked. Adam Back also pointed out three other issues that needed attention, including unlocking the wallet without displaying the password in the console screen, a button to import a private key, and a UX way to transfer BTC off a specific address.Pieter recommended a Python RPC client that would provide more features than the current RPC client built into bitcoind. Another issue discussed was adding a button to import a private key with an option to transfer it to another key, but Pieter was opposed to per-key fiddling in the GUI because it could cause misunderstandings about how wallets work. Pieter suggested entire-wallet export and import rather than individual keys.The need for a way to transfer BTC off a specific address without having to spend the entire wallet onto a new address was raised again. Pieter suggested adding this capability to coin control. At ZKS, pseudonyms were used to segregate online linkability for different purposes such as medical, casual browsing, social media, private, work, and true name. However, people still make mistakes with individual keys and accidentally link all their coin sources together. Clients do not have any privacy algorithm to minimize cross-linking of coin sources. To help people achieve better privacy, it would be useful to allow them to control which address the payment is from by creating multiple wallets for different unlinked purposes. Misbehavers could be blocked based on pseudonym in ZKS. Bitcoind's naive RPC client is not a good fit for many features, and it's better to provide a feature-rich Python RPC client. Per-key fiddling in the GUI should be avoided, and instead, entire-wallet export and import should be used. Coin control belongs in transferring BTC off a specific address rather than spending the entire wallet onto a new address, which creates more network traffic, bigger packets, higher fees, and damages privacy.
