In a Bitcoin-development mailing list, members discussed the ongoing block size debate and the need for a summary of positions and arguments. Mats Henricson expressed frustration with people rehashing the same arguments without reading previous discussions or offering alternatives. Gabe Appleton offered to help create a wiki page to condense the information if someone else had permission to create it. Ethan Heilman agreed that a summary would be helpful, as he did not have time to read all the emails generated by the debate. The conversation also touched on issues such as testnet4 using 8mb blocks, bitcoin XT's role in the scheme of things, and the need for a git repo to avoid exclusive-by-default wiki policies.
