Summary:

In a message from Martin, it has been proposed to Gavin that the doubling of the maximum message size limit be stopped at 32MB in order to avoid code failure in 2022. This proposal requires a fix for the message size limit and a second hard fork, which could later reinstate the current 8GB limit. Martin suggests that Gavin address this topic in the BIP, even if he disagrees with the proposal.
