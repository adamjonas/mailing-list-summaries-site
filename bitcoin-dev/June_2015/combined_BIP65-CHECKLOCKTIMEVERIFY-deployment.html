Summary:

Peter Todd, a co-author of the Version Bits BIP proposal, proposed on June 25, 2015, the deployment of Check Lock Time Verify (CLTV) due to its benefits to scalability solutions. He suggested using the existing median block version mechanism for CLTV and noted that there is no lack of consensus on this proposal. The thresholds would be similar to those used in BIP66.Todd also stated that if another soft-fork is proposed prior to BIP65 enforcement, they have the option of setting in motion yet another soft-fork as the median mechanism only requires forks to be serialized in sequence. The CLTV code has been extensively reviewed and tested, including running in production on Viacoin for six months.On June 25th, 2015, Peter Todd wrote on a Bitcoin Github thread about his plan to deploy CHECKLOCKTIMEVERIFY via the IsSuperMajority() mechanism. He opened a pull request which was considered the final step towards CLTV deployment on mainnet. Todd copied the logic and tests from the previous BIP66 (DERSIG) soft-fork line-by-line for ease of review, noting that any code review applicable to BIP66 should be applicable to BIP65. Once merged, he prepared a backport of the soft-fork logic for the v0.10.x branch as well.Pieter Todd proposed deploying CHECKLOCKTIMEVERIFY via the IsSuperMajority mechanism on June 28, 2015, which was the final step towards CLTV deployment on mainnet.
