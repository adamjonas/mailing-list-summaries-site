Summary:

The idea that a loud minority can make any change seem controversial has been mentioned before. However, it is important to note that pushing something through despite a well-informed minority with valid reasons can still be considered controversial. When implementing new software, the entire network must agree to the switch, and if there are prolonged and heated discussions on various forums, it is a clear indication that the change is controversial. As Greg Sanders suggests, we can recognize controversial changes when we see them.
