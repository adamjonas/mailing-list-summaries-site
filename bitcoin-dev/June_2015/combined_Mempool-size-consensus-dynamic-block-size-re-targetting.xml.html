Summary:

The context discusses a proposal for determining the appropriate block size limit for Bitcoin that takes real-time changes to the mempool into account. The proposal involves adding a mempool_size field to each full-node that broadcasts a new transaction, which would allow all nodes to quickly reach consensus on the current average/median/etc mempool size and agree on a suitable periodic blocksize "re-targetting." This approach would enable all full-nodes, not just miners, to vote with their transactions, resulting in truly global consensus. In a discussion, it was pointed out that anyone could lie about the mempool size, but there is no apparent benefit for a full-node to do so, and it is unlikely that the majority of transaction broadcasting full-nodes would engage in such behavior as it goes against their own interests.
