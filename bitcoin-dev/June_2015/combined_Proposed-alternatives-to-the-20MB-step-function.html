Summary:

The issue of block size limit in Bitcoin has been a topic of discussion for years, with various proposals being made. These proposals include Segregated Witness, Lightning Network, replace-by-fee, and building in UTXO set cost to tie fees to block size. However, these solutions come with their own set of challenges, such as network congestion and potential manipulation by mining companies.In a 2015 email exchange among Bitcoin developers, potential solutions for the issue of transaction fees were discussed. The group proposed dynamic maximum block size limits based on recent history, with an average over the last 144 blocks calculated at every block. Gavin Andresen suggested implementing a big increase that grows over time to the Bitcoin-XT project and asking for help lobbying companies to run it. Discussions also focused on the use of block size limits to enforce scarcity in Bitcoin transactions and the need for default policy implementations like replace-by-fee and child-pays-for-parent.Several ideas were raised for determining the hard block size limit of Bitcoin, including using bitcoin days destroyed, basing the limit on blockchain length or votes from miners. Discussions also emphasized the need to find a solution that can handle a higher volume of transactions without compromising security or efficiency while ensuring that the system can scale up gradually and organically.Mark Friedenbach proposed a dynamic block size limit controller that applies a cost to raising the block size to preserve decentralization and long-term stability features while still allowing for emergency relief within the confines of a self-supporting fee economy. The author proposes a 20mb limit as a reasonable compromise that should free up enough space for user growth while still being below VISA capacity. Ultimately, the code the majority of merchants, exchanges, and miners are running will determine consensus on the issue of block size. There were concerns about potential incentive edge cases and centralization pressures if miners had complete control over the max block size. Discussions also touched on infrastructure scaling risks and the delay in developing a constraint-based fee economy if the block size is raised without any feedback mechanism.
