Summary:

In a discussion about BIP32, James Poole asked for clarification on the meaning of "n" in the context of an algorithm. The answer to this question can be found on the Bitcoin wiki page for Secp256k1, which explains that "n" is the curve order. The purpose of this step is to keep the user on the curve. The function secp256k1_ec_privkey_tweak_add from libsecp256k1 handles this automatically, but if OpenSSL or some non-EC math library is used, it may have to be done manually.
