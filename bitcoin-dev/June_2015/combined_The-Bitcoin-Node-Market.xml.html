Summary:

In a 2015 Bitcoin-development mailing list, an idea was proposed to create a market-based approach to incentivize running full nodes on the Bitcoin network. This involves paying nodes for supplying data to the network, with payments based on the cost imposed on the node. The proposal aims to share the cost of running the network among those who use it but do not contribute by running a full node. Fees would develop for connecting to the network, which should cover the cost of running it. However, there are concerns that this system may negatively impact casual users and incentivize the use of centralized wallets. Aaron Voisine, CEO and co-founder of breadwallet.com, responded to the proposal suggesting that punishing users who use Simplified Payment Verification (SPV) wallets is not an ideal way to incentivize people to run full nodes. He argued that SPV wallet authors would be the ones providing the nodes required for mobile phones to connect if they want their wallets to function. Voisine suggested that paying for connectivity would not be more confusing or annoying than transaction fees, and proposed that some full nodes in the network offer paid connection slots. This would allow users who checked the "pay subscription fee" box in their wallet configuration to have an easier time connecting than those who did not.However, Luke Dashjr argued that running a full node is the "right" way to participate in the network, as it allows users to earn more bitcoins than they pay out. In response, Aaron Voisine suggested that if some full nodes started offering paid connection slots, users who opted for it would have an easier time connecting than those who didn't, similar to how paying transaction fees makes it faster and more probable for it to get mined. Breadwallet, a popular bitcoin wallet, plans to create its own full nodes to reduce the load on volunteer networks as its user base grows, and CEO Aaron Voisine hopes that running its own full nodes will mitigate negative impacts on the network. Some have raised concerns about whether Simplified Payment Verification (SPV) wallets would have to pay fees to connect to the Bitcoin network, which could discourage decentralized participation and incentivize the use of centralized wallets.
