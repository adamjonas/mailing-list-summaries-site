Summary:

A developer named Nakagawa has written a multisignature procedure using bip-schnorr and is requesting feedback on it. The code for the procedure is available on Github, along with the original documentation for Multisignatures and Threshold Signatures using bip-schnorr. However, Jonas Nick has pointed out that the writeup appears to be vulnerable to key cancellation attacks due to the aggregated public key being just the sum of public keys without proof of knowledge of individual secret keys. To solve this issue, Jonas suggests a secure key aggregation scheme described in the MuSig paper and provides a link to it. Nakagawa has written a new text and provided a link to it, requesting a review and feedback on it. On a related note, Nakagat posted a multisignature procedure using bip-schnorr to the bitcoin-dev mailing list on August 13, 2018, which also requested feedback on the implementation. The original implementation can be found on GitHub, along with information about Multisignatures and Threshold Signatures using bip-schnorr. However, while the method is cool, there are a lot of online steps and it's not a threshold system. A solution to this issue is using a shamir scheme, which isn't subject to birthday attacks. A link to a Medium article explaining this solution was provided.
