Summary:

The bitcoin-dev mailing list has been discussing the idea of having multiple testnets available for testing purposes. Currently, the testnet has over 1.4 million blocks and takes at least 48 hours to fully sync, which can be inconvenient for testing purposes. A user on the mailing list suggested a potential testnet reset in the next release. However, Peter Todd recommended having a larger blockchain for testing purposes instead of resetting the current testnet. He also suggested using regtest as an alternative for testing, as private blockchains can be easily set up and controlled.Another proposal was to have two testnets at the same time, with one having a smaller block size for faster testing. Ryan proposed implementing small blocks on testnet to simulate a fee-market-like situation. However, Gregory Maxwell suggested creating a signed blocks testnet with a structured reorg pattern as an alternative to the current testnet. This approach previously increased the size of blockindex objects in memory, which could be rectified by turning one of the fields into a union with a pointer to a look-aside block index used only in signed block testnet mode.Such a mode would serve as a middle ground between a regtest that never sees anything surprising and a full-on testnet where attempts to test against ordinary noise require coping with the universe being removed and replaced at the whim of some Cthulhuian blind idiot god. The idea of a signed blocks testnet with a predictable structured reorg pattern has been proposed in response to the suggestion of having two testnets with different block sizes.Although patches were previously developed for such a testnet in Bitcoin core, there were concerns about increased memory usage due to the larger size of blockindex objects. However, this could potentially be addressed by turning one of the fields (such as the merkel root) into a union of its normal value and a pointer to a look-aside block index used only in signed block testnet mode. Overall, the bitcoin-dev community is actively exploring various options to improve testing capabilities and efficiency.
