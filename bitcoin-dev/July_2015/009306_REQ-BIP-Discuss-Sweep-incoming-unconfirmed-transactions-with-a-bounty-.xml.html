Summary:

The conversation is regarding the implementation of a process BIP request to add functionality to Bitcoin-Core reference implementation. The feature called 'child pays for parent' allows users to boost incoming transactions. PR#1647 only addresses miner policy, and the BIP is addressing the user-facing side of this functionality. However, the sweep transaction UI points are out of scope for the BIP. Mark Friedenbach suggests talking with wallet authors, and if agreement can be found, writing a pull request. If accepted, the BIP could add flexibility into any future fee schedules.
