Summary:

Eric Lombrozo, a Bitcoin expert, has highlighted the trust issues that exist in the current network, which contradict Bitcoin's raison d'etre of trustlessness. The network relies on trust in validators, miners, relayers, explorer websites, and online wallets to function. This reliance on trust compromises the security model that Bitcoin was founded upon, which is based on preventing double spending without using a trusted third party. Currently, the security model holds for any user who has access to the complete blockchain but not for those who do not. To address this issue, there needs to be a mechanism via which honest nodes can prove the invalidity of blocks. In an email conversation, Eric Lombrozo and Justus Ranvier discuss the security issues related to Bitcoin. Lombrozo suggests that looking into his work might change Ranvier's perspective. However, Ranvier believes that striving for perfection can hinder progress towards achieving good security measures.Justus Ranvier also stresses the importance of good security rather than infinite security and argues that while trust has been introduced into some areas of the Bitcoin network, the security model still holds as long as users have access to the complete blockchain. However, there is a lack of a mechanism to prove the invalidity of blocks for honest nodes, which needs to be addressed to improve the security of SPV clients. The cost of validating blockchain has become too expensive, making compliance with the original security model impractical for most use cases. A new security model that takes these trust assumptions into account or solving the validation cost/bottleneck issue is necessary. Overall, while the current state of the network relies on trust, it does not align with the trustlessness that Bitcoin aims to achieve.
