Summary:

In an email exchange between Thomas Voegtlin and Justin W. Newton, the former suggests that the "_oa2" portion of the domain name should be hidden to make it easier to delegate just wallet names from a zone. He proposes that "_oa2" be used only when an intermediate level is needed and "_oa2_keys" be used for key listing. Voegtlin also provides an example of how this system could work with Bitcoin aliases being delegated to Netki.
