Summary:

The email conversations in this context revolve around various topics related to securing domain name aliases on Bitcoin, including the use of OpenAlias and DNSSEC. There are concerns about the security of DNSSEC, especially since private KSK must be secured and not kept on servers. The relevance of using DNSSEC in OpenAlias is also debated, with some arguing that it is necessary while others disagree. Netki, a blockchain startup, provides open source tools for developers to build blockchain applications, and they are working on DNSSEC integration with BIP70. They have released an Addressimo API server that provides unique HD wallet addresses or payment requests for every lookup, thus allowing a user to have a private, secure way to share a Wallet Name that can be used to send them any digital currency. However, critics argue that the system is not completely decentralized as it layers complexity on top of a convenience service, and Netki's service does not yet have DNSSEC support.Discussions have taken place about creating a common standard for DNS records holding Bitcoin addresses, with the idea initially proposed to provide a censorship-resistant option and allow individuals to register their own names at a low cost. Furthermore, the email exchange discusses the use of DNSSEC in payment requests along with its potential problems due to the large size of DNSSEC proof and 1024-bit RSA signature. The author suggests not including the proof in the request and only using the final signature, which is possible through DNSSEC.The conversation also covers zone delegation and creating new digital identities to ensure security and privacy while using various online services. In addition, there were discussions about creating a mechanism that eliminates the need for identity provider cooperation. The use of SSL certificates has been proposed to create lightweight payment requests if the final signing key is an EC key and if the payment request does not include the whole chain of certificates. However, using SSL certificates would require additional infrastructure to publish the chain of certificates.Finally, the email exchange discusses the use of OpenAlias, a domain name system that allows for easier cryptocurrency transactions. However, there was skepticism expressed about the potential success of OpenAlias and concerns about whether people would want to use a domain name under a new root as their identity. The discussion also explores other options such as compact cert+optimized BIP70 and using pastebin to store small bits of data. Overall, the Wallet Name Service has always been built on open standards, and extensions and modifications have been made as a result of conversations with industry participants to ensure that what they are building meets the needs of the community at large. They want to work with the community to find a common standard but there are some places that would cause hesitation in "just implementing" the Open Alias standard.
