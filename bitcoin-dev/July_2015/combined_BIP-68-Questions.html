Summary:

The code in BIP 68 involves comparing the input's nSequence with the transaction's nLockTime. If a transaction is already in the blockchain, there is no need for a relative locktime because the time of the previous transaction is known. However, if the transaction is yet to be added to the blockchain, it cannot be created since the value to set nLockTime to is unknown. Rusty Russell initially misunderstood the code and asked Mark for clarification. Mark ELI5'd him offlist and informed him that the code functions properly. Rusty expressed his gratitude towards Mark for helping him understand the code better and his happiness about the code functioning correctly.
