Summary:

The user is considering using a Jetson TK1, a GPU-powered development board, as a power-efficient full node for Bitcoin. They plan to run bitcoind on the Jetson as a node with high performance-per-watt, since a pure-CPU implementation of bitcoind will clog the CPU. The user believes that there are heavy-compute highly-parallel functions that could be outsourced to a GPU and may want to fork and/or contribute to them. However, they are seeking opinions on the idea/configuration before coding a complete mess, as they have more background on parallel programming than on the Bitcoin protocol.The processor load for signature verification involves a hash function call, large number maths, and an elliptic curve operation. The main task is proving that R = u1 * G + u2 * Q, where G is a constant and the rest are different per signature. Some of the core team have created a fast CPU implementation available at https://github.com/bitcoin/secp256k1.git. The longest step is the elliptic curve operation, which takes about 1ms per signature on normal hardware. However, optimized code can make the process faster. There has been discussion about batch verification of signatures, with the potential to compute 16 signatures together. However, it's believed that the benefit of using GPU would be lost due to communication bandwidth. GPU miners typically benefit from needing only minimal information to be sent to the GPU routine.
