Summary:

Pieter Wuille and Wladimir discussed the compatibility of softforks with Bitcoin Core major releases in a conversation on April 28th, 2015. Pieter suggested that since backports to older releases would be necessary anyway, there is no need for softforks to be bound to major releases. Wladimir agreed with this and stated that consensus changes are properties of the underlying system that need to be adapted to, rather than software features. He also noted that if these changes do not require extensive code modifications, they could be done in minor releases as well.
