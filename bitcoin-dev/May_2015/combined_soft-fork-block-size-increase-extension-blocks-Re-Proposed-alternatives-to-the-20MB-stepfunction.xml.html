Summary:

Bitcoin developers, Adam Back and Raystonn, had a conversation via email about scaling Bitcoin with subchains. Raystonn proposed a tree structure of blockchains to scale transactions instead of one 10MB chain and explains the advantages of having multiple 1MB chains. Gavin Andresen suggested implementing the proposed solution in Bitcoin Core wallet and creating UI mockups for user experience. He also questioned the complexity of the proposed solution and suggested opting for a simpler solution if possible. The extension block idea is a way to soft-fork an opt-in block-size increase, giving users a choice. However, there are risks involved, such as imposing a security downgrade on non-upgraded users who don't have the bandwidth to validate 10MB blocks. The implementation of the extension block is not as simple as a hard-fork but safer. It allows users to keep using 1MB blocks for higher decentralization or opt-in to 10MB blocks.
