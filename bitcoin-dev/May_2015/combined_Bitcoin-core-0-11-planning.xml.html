Summary:

Bitcoin Core version 0.11 is scheduled to release on July 1st, with a soft translation string freeze on May 1st and feature freeze on May 15th. Wladimir J. van der Laan proposed the schedule for Bitcoin Core version 0.11 in an email dated April 28, 2015. In the same email, he stated that the release would not be held up for features, and anything that does not make it to 0.11 will be postponed to the next release scheduled for the end of the year. In an email thread on the Bitcoin-development mailing list, Peter Todd suggested that minor releases could be used for softforks if they do not require large code changes. He recommended releasing a v0.12 with just a CLTV soft-fork as soon as the BIP66 softfork triggers, as he believes the main reason to accelerate CLTV is scalability. Pieter Wuille also suggested that softforks do not have to be bound to Bitcoin Core major releases and asked what the oldest version is that needs backports. He mentioned that if the code changes for absolute CLTV are small, they can easily be ported to any Bitcoin Core version.The proposed schedule for the release of version 0.11 has been shared with the community, with a deadline of May 1st set for the soft translation string freeze, opening Transifex translations for 0.11, and finalizing and closing translation for 0.9. The feature and string freeze will take place on May 15th, with the split off of the 0.11 branch, tag, and release of 0.11.0rc1 scheduled for June 1st. The aim is to release 0.11.0 final by July 1st.
