Summary:

Nicolas Dorier has proposed a solution to the block size problem in Bitcoin. The proposal aims to promote decentralization and relieve core developers from the role of central bankers. Dorier suggests that technical and decentralization considerations should be the only factors used in decision-making, not economic, political, or ideological principles. Block voting would be used where miners vote for the size of the block they want at the next deadline, and the median of the votes is implemented as the new block size limit. This solution addresses all objectives, including increasing network capacity and storage, as well as making Bitcoin what miners want. However, it remains uncertain how this would affect Bitcoin's core stability or whether a better statistical function is possible. Dorier calls on core developers to focus on their strengths and avoid decisions based solely on maximizing metrics.
