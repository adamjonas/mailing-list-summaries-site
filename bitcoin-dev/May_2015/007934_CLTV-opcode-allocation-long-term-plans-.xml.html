Summary:

Bitcoin developer Peter Todd has suggested a change to the way opcode OP_CLTV works in the Bitcoin protocol. In a message to the Bitcoin developers' mailing list, Todd proposed that OP_CLTV should work as follows: "the 1 selects absolute mode, and all others act as OP_NOP's". He added that a future relative CLTV could then be implemented as a soft-fork using "2 OP_CLTV". Another developer, Rusty Russell, responded, saying he would "mildly prefer" for the change to have the OP_NOP1 become OP_EXTENSION_PREFIX, with the next opcode defining which extended opcode it is.
