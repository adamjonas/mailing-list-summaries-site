Summary:

In an email exchange on May 9th, 2015, Gregory Maxwell provided an example of tx_size in the context of a soft fork implementation. The proposed formula was tx_size = MAX( real_size >> 1, real_size + 4*utxo_created_size - 3*utxo_consumed_size). Additionally, there were discussions around block size limits, with proposals for a 1 MB hard limit and a 900 kB soft limit. The adjusted UTXO size (U) was defined as S + 4 * outputs - 3 * inputs. A block would be considered valid if its size (S) did not exceed the specified limit.
