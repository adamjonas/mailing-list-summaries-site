Summary:

This email thread from 2015 discusses the benefits of using Replace-by-Fee (RBF) versus Child-Pays-For-Parent (CPFP) in Bitcoin transactions. The conversation was sparked by a concern raised by Danny Thorpe about the possibility of fraudulent payment reversals using RBF. Peter Todd argues that RBF is more cost-effective than CPFP, particularly for defragmenting outputs. He presents four case studies to demonstrate that RBF can save transaction fees ranging from 30% to 90%. In Case 1, Todd explains how RBF can be used to reduce transaction fees by 30% when defragmenting an output. In Case 2, he shows how paying multiple recipients from a multisig wallet can result in an 84% cost savings with RBF, compared to CPFP. In Case 3, when using a multisig wallet, defragmenting UTXOs can be expensive, but RBF can result in cost savings of 90%. In Case 4, for dust defragmentation, RBF can reduce transaction fees by 32% to 59%, or even more if defragmentation without RBF costs more than it saves.The email chain includes some contentious exchanges, with one member criticizing Todd and suggesting he should stick to serving his big bank clients rather than meddling in Bitcoin development. Additionally, there is a heated exchange between joliver and Peter Todd regarding ViaCoin and big banks. Overall, the discussion highlights the ongoing debates around Bitcoin development and the importance of finding solutions that balance security and cost efficiency.In this context, it is explained that users of Bitcoin Core can change transactions through the Replace-by-Fee (RBF) feature. The RBF feature allows users to modify parameters from the transaction structure, including changing the transaction ID. Todd, the person being questioned in this context, does not answer a specific question about this feature. However, he explains that standard RBF lets users change anything, while Full Sequence Replace-by-Fee (FSS RBF) lets users modify inputs and add outputs or increase the value of outputs.Additionally, Todd notes that any significant changes to mempool policy like RBF are unlikely to be incorporated into the Bitcoin Core v0.10.x branch due to its minor focus on bug fixes. However, he maintains a separate branch for standard RBF in v0.10.x and has been requested by a major miner to backport FSS RBF for v0.10.x as well. It is not mentioned why the miner requested this backport, nor is it explained what real use cases exist for RBF.
