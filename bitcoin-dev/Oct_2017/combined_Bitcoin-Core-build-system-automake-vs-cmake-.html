Summary:

Bitcoin Core uses Automake as its build system, which has been questioned by Kosta Zertsekel in an email who suggested that CMake is better and requested a relevant discussion or explanation for this decision. Jeffrey Paul defended the use of Automake stating that it is a standard and does not require additional installation on most systems. However, there have been discussions about replacing Automake with other advanced build systems such as Meson and Ninja, which were developed to replace Automake. Some companies and projects have already switched to these alternatives, including JetBrains and KDE Project.In a conversation on the bitcoin-dev mailing list, Kosta Zertsekel asked why automake is used instead of cmake, which he considers better. Jeffrey Paul responded that automake is the standard and requires no additional installation on most systems but asked what exact problem Zertsekel was trying to solve for bitcoin-core. Zertsekel provided quotes about CMake build tools and asked if there had been any discussion about choosing the best build system for Bitcoin Core. Thomas suggested that unless there is a specific problem that CMake can solve without causing other issues, there is no reason to change the current build system. He also encouraged Zertsekel to convert bitcoin-core to CMake and demonstrate its benefits before suggesting adoption by developers.
