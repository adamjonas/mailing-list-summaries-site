Summary:

In a recent email thread on the bitcoin-dev mailing list, Peter Todd suggested adding a new service bit, NODE_SEGWIT, and/or bumping the protocol version so that outgoing peers only connect to peers with segwit support. He also noted that future upgrades to add new block data would have a similar relaying problem all over again, and proposed some solutions. One proposal was to remove the restriction that the coinbase witness contain exactly one 32-byte value. Another proposal involved hashing the contents of the coinbase witness as a merkle tree and committing them in place of the current nonce commitment. This approach would allow commitments to be ordered arbitrarily and cleaned up with infrequent hard forks. The proposals were discussed in a PR on the segwit branch, where there was also a proposal to have the coinbase witness include the merkle path to the segwit data.
