Summary:

On the Bitcoin-dev mailing list, Ethan Heilman inquired about whether libsecp256k1 is used for all cryptographic functions in Bitcoin. Douglas Roark replied that while libsecp256k1 performs elliptic curve operations required by Bitcoin, OpenSSL is still used for other crypto functions. However, for consensus-critical functionality, OpenSSL will be replaced with libsecp256k1 in the upcoming version 0.12 of Bitcoin. In the meantime, OpenSSL is being used for getting random numbers to randomize the ECC signing context, AES256 encryption of private keys for wallets, and SSL/X.509 for BIP70 payment protocol for GUI. Suggestions are made to remove OpenSSL dependency for the first two purposes, but keep it for the BIP70 payment protocol. The plan is to make OpenSSL a dependency only if the GUI is enabled in the future, with most of the work already done but requiring careful testing and integration. The release notes for version 0.12 of Bitcoin mention the switch from OpenSSL to libsecp256k1 for signature validation. Additionally, it is noted that libsecp256k1 relies on an implementation of RFC 6979 rather than a PRNG, possibly for portability reasons. There are still some crypto functions not covered by libsecp256k1 at the API level.
