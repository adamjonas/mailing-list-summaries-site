Summary:

A Bitcoin developer named Michael Folkson has expressed interest in setting up a custom signet and raised some questions regarding its purpose. The email was in response to another user's post about setting up a custom signet for testing full RBF. Michael asked what could be tested on a custom signet that can't be tested on the default signet with Bitcoin Core 24.0, whether it is safe to run a custom signet node on the same machine as a default signet, mainnet or testnet full node, and how regularly the sole block signer on the custom signet is mining blocks.The original post by Alicexbt was about setting up a custom signet for testing full RBF and sharing details of the nodes where other users can connect to test. She shares an example configuration and a simple test case for testing the custom signet. The nodes can be connected using `addnode` with the given IP address or through i2p or onion nodes. Bitcoin addresses can be posted in the "full-rbf room" to show interest in testing. Alicexbt believes testing is important and encourages others to try the experiment. The -mempoolfullrbf configuration option is available on the recent Bitcoin Core 24.0 release for mainnet and default signet, but there may be advantages to testing on a smaller custom signet network starting from scratch. They are the sole block signer and are mining blocks regularly with default settings, running the custom signet node on the same machine as other nodes without issue. The email was sent using Proton Mail secure email. If successful, the experiment may proceed towards setting up an explorer.
