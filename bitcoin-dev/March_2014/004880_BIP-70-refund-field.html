Summary:

The speaker states that they have no objection to incremental development, but it will only be effective if it provides added benefits compared to current payment processor solutions. The suggested benefits include offering symmetrical payments, aggregating and netting multiple installments, and providing greater security by having a check not only on the payment address but also on the refund.In response to a question about whether the current payment protocol can handle salaries, Gavin Andresen replied that it does not. He suggests focusing on solving existing problems with the minimal payment protocol before attempting to solve all the possible issues at once.
