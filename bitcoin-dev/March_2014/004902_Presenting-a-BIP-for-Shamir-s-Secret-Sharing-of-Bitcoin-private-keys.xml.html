Summary:

In a discussion about Shamir's Secret Sharing Scheme, it was noted that multi-signature transactions offer more fine-grained control of security at the transaction level than Shamir does. With Shamir, a reconstituted private key can be used to control anything that key controls, necessitating policies such as "no key reuse." While initially impressive, the complexity and DIY factor of the scheme make it less appealing. Nonetheless, the value of an informational BIP on the topic is recognized. The discussion was held among Bitcoin developers and open source evangelists, including Jeff Garzik of BitPay, Inc.
