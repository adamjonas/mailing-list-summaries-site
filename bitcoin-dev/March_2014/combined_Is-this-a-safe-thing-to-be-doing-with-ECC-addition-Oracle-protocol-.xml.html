Summary:

On March 8th, 2014, a conversation thread was initiated by Alan Reiner to discuss the limitations of ECDSA and alternative methods. The discussion covered several topics such as ensuring public key integrity and the use of multisig as an alternative solution. Edmund Edgar, founder of Social Minds Inc and Reality Keys, participated in most of these discussions, sharing information about his company and its services. To ensure public key integrity, one suggestion was to insist on seeing a hash of the other party's public key before sharing one's own. The use of multisig was also proposed as an alternative solution. ECDSA has a limitation that there is no known protocol to create a signature with a+b without sending private keys, but this can be done with Schnorr signatures. Joel Kaartinen suggested that hashing the keys may be sufficient for the process.The security of EC Diffie-Hellman was also discussed, highlighting the importance of well-known, well-studied operations in a good cryptosystem. Alan Reiner gave an example of creating a keypair and giving someone minus the public key, which could only be detected after the private key had been abused. To avoid this situation, parties must exchange their public keys before combining them.In a separate email, Edmund Edgar introduced a new m-of-n contract implementation based on Reality Keys. Peter Todd suggested an effective way to use these keys for m-of-n contracts without non-standard transactions or refund transactions. However, Edmund Edgar expressed concern that Bob could intentionally weaken the resulting public key before producing it so that he could sign a transaction with it without needing to know the private key. The example script is available on GitHub and people can comment on whether it's a safe thing to do.Overall, the email threads discussed various aspects of public key exchange, contract implementation, and cryptosystem security. Edmund Edgar attempted to combine the public key of each party with the public key of the outcome they're representing to create a public key that goes into a 1/2 P2SH address for m-of-n contracts. However, he expressed concern about the security of this approach and welcomed comments from people who understand how it works.
