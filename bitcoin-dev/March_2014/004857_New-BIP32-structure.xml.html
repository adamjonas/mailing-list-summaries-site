Summary:

In an email exchange, Allen Piscitello questioned whether most coins have a unique magic number assigned to them already. Pavol Rusnak spoke out against the use of the magic number in BIP-32 design, calling it unfortunate and superfluous. The purpose of the magic number is to distinguish BIP-32 trees for various altcoins, but it doesn't make sense when storing various altcoins in the same tree using the proposed /m/cointype/reserved'/account'/change/n scheme. Rusnak would like to see the magic number removed from BIP-32 and instead use 0x0488B21E/0x0488ADE4 (xpub/xpriv) for identification purposes.
