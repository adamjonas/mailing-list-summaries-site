Summary:

The email thread from June 3, 2014, discusses the use of IsoDep protocol to authorize Bitcoin transactions on Android devices via NFC tags. Andreas Schildbach raises concerns about potential connection breaks when the phone is picked up and suggests using fallback options such as HTTPS or Bluetooth. The discussion turns to the possibility of dispatching different IsoDep applications to different apps, but conflicts between apps fighting for access to IsoDep endpoints are a concern. Mike Hearn suggests having a NDEF tag trigger the app to start an IsoDep protocol, but he is unsure if this is possible with the Android API. The email ends with advertisements for Perforce, a tool for version control, and a link to the Bitcoin-development mailing list.
