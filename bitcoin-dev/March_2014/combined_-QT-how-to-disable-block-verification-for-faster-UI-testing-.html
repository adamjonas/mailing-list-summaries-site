Summary:

Angel Leon and Wladimir had a conversation regarding database issues and UI development in Bitcoin-Qt. Angel was facing database corruption while opening the official binary, leading to reindexing of blocks on disk, which takes an entire day. Wladimir suggested having separate data directories and using -regtest to set up test scenarios quickly. He also cautioned against switching between 0.8.x and 0.9.x with the same directory and provided a link to the downgrading warning. Angel stated that he is not trying to test anything specific but wants to make the startup of the Qt client faster. When developing a user interface for Bitcoin-Qt, it's essential to consider the -checklevel parameter in the command line options. Passing 0 assumes little to no verification, which can lead to database corruption and reindexing of blocks on disk. This issue has occurred multiple times when opening the official binary for non-development use. Angel posted this context on Twitter at @gubatron, seeking suggestions from the development community to avoid these issues while still developing the UI.
