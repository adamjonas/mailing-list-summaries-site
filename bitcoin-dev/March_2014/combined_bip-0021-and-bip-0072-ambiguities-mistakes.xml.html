Summary:

On March 6, 2014, Mike Hearn requested a pull request from William regarding QR code escaping. William submitted the pull request and noted that the behavior of the bitcoin-qt client should now align better with the recommended practices outlined in the RFC. The de-escaped QR codes resulting from this adjustment are also reportedly much more aesthetically pleasing.The writer intends to create a parser for bip-0021 URIs with new bip-0072 payment parameters, aiming for complete accuracy. They suggest that any bitcoin URI scheme should use internet RFC 3986 to govern the general syntax for URIs instead of earlier RFCs. However, while bip-0072 explicitly refers to the obsolete RFC 1738, bip-0021 never mentions RFC 3986. Updating bip-0072 to refer to RFC 3986 would allow shorter and more readable bitcoin URIs as the old standard requires more escapes than necessary. The writer highlights that neither bip describes what to do with international characters. They recommend defining character encoding, and suggest adding UTF-8 as encoding into bip-0021 as it is recommended by RFC for all new URI schemes. Lastly, the writer notes an error in the bip-0021 BNF grammar, which never mentions the '&' separator between query elements. They seek clarification on the procedure for correcting these issues and suggest submitting a pull request with the changes.It is worth noting that the bitcoin-qt client already uses QUrl to parse bitcoin URIs based on RFC 3986, indicating that it may have already implemented these suggestions. However, it is unclear what project or codebase the conversation about pull requests is taking place in, as there is mention of an HTML attachment that is not related to the conversation.
