Summary:

The email thread discusses the benefits of a new specification over BIP 0038, highlighting multiple key derivation functions, seed lengths, explicit BIP 0032 support, creation date field and plausible deniability. The conversation then shifts to comparing the proposed specification to BIP 0039, which is for key import/export and does not deal with encryption or wallet creation date. It is suggested that BIP 0039 should not hold back improvements on BIP 0038. The discussion also covers the integration of the BIP39 word list into Bitcoinj, timestamping and checksums, entropy length options, password typo detection, user selectable KDF, and cross-wallet compatibility. The email thread concludes with links to a free book on graph databases and the Bitcoin-development mailing list.
