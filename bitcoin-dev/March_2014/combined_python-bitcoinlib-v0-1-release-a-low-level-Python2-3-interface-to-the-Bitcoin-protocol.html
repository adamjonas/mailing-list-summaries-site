Summary:

On March 15, 2014, Peter Todd, a Bitcoin core developer, released v0.1 of his Python2/3 library on Github. The library provides an easy interface to the bitcoin data structures and protocol with a low-level approach that focuses on providing tools to manipulate the internals of how Bitcoin works in a Pythonic way. It supports data-structures related to transactions, scripting, addresses, and keys, and includes Bitcoin Core RPC support. EvalScript(), VerifyScript(), and SignatureHash() are all functional and pass all the Bitcoin Core unittests, as well as some that are yet to be merged.Todd's library is signed by his PGP key, as well as every publicly pushed commit. However, he mentioned that the p2p network-related code is not in good shape and lacks unittests. Jeff Garzik, another Bitcoin core developer and open source evangelist, attributes the bulk of the code structure of Todd's fork to his implementation. Garzik responded that maintaining three client libs was a bit much for him.Drak suggested adding Peter with commit access since Todd's repo is top of the fork tree, indicating that the fork has been under more development and upstream has stalled. On March 15th, 2014, Peter Todd responded to Drak's suggestion by saying that people actually using his 'pythonize' code have been linking directly to his tree in things like documentation and build scripts, so the URL is probably not a problem. He also opened a pull-req on the bitcoin.org repo to change the URL.In addition, Todd's fork of python-bitcoinlib has been added as a dependency in ngccbase Colored Coin client, and there is now a rudimentary python-bitcoinlib package in Arch Linux.Lastly, the email thread included a call to action from O'Reilly Books to download their free book on Graph Databases.
