Summary:

A new proposal for deriving digital scarcity from bitcoin transactions has been put forward by PJ Fitzpatrick to Jakub Trnka. This method involves creating coins based on the hash closest to the non-zero portion of the block hash, with only one coin created per transaction regardless of its size. This creates n coins per block, and the initial coin supply and addresses can be fully determined by the existing blockchain, making them scarce as they can only be produced through transactions. Fitzpatrick suggests that overlaying scarcity and value on top of the bitcoin blockchain would incentivize people to transact more frequently, resulting in an equilibrium between paying transaction fees and mining new coins. Jakub Trnka supports this idea, stating that creating congestion on the bitcoin network would encourage people to borrow scarcity from bitcoin in a sidechain. There are several variations of this approach, such as creating computation puzzles from the previous block. Fitzpatrick plans to start a discussion about this proposal on bitcointalk and is seeking feedback on whether others have seen a similar method. Overall, the proposed method could effectively borrow scarcity from bitcoin in a sidechain and create a scarce initial coin supply and addresses.
