The email thread centers around whether MtGox-related posts should be allowed on the Bitcoin-development mailing list. Some members argue that the list is only for discussion of core bitcoin development and not for sites or software that use bitcoin, especially since MtGox's code is closed source. They suggest that MtGox-related topics should be discussed elsewhere. In one of the emails, Doug Huff reports a live trade matching bug on MtGox and recommends that the site should be taken down immediately until the issue is resolved or publicly shown to never execute. The bug could potentially affect market conditions if it is only a display bug. However, Mike Hearn and Gavin Andresen argue that security vulnerabilities should be kept private for a reasonable amount of time to allow for fixes to be made. Meanwhile, Jeff (unspecified) is suggested to create a new list specifically for discussing vendor security issues. On June 27, 2011, Doug Huff had reported the Live mtgox.com trade matching bug on the mailing list, but he agreed to stop posting Mt Gox-related information on the mailing list per Jeff's suggestion.
