Summary:

Mark Visser, a user of Bitcoin, has suggested allowing wallet.dat location to be specified in the preferences. This would enable users to physically disconnect their wallets by unplugging USB keys and prevent thefts. Jeff Garzik, a member of Bitcoin, stated that -datadir is used for specifying the wallet file path but all databases must be in the datadir because wallet.dat is a Berkeley db and a transactional environment is needed. Ted, another member, suggested using symlinks instead of specifying the wallet file path. He also recommended allowing users to open more than one wallet to transfer coins from one to the other easily. Jeff Garzik agreed that wallet crypto should be a top priority for the next version due to increased reports of stolen wallets.
