Summary:

The Bitcoin-development mailing list is discussing the possibility of merging the Qt GUI to improve the user interface of Bitcoin. Gavin Andresen supports replacing wxWidgets GUI with a Qt GUI for the reference client to avoid having more than one GUI. However, there is a concern that adding another GUI would require cleanup of RPC/GUI code changes in three places instead of two.Pieter Wuille suggests merging autotools and qtgui, but it has not been tested enough on native Windows and OSX platforms. John Smith's Qt GUI is ready for merging, and he is seeking information on including the autotools build system in his code. In an email exchange, Pieter expresses support for merging the autotools now unless there are still known issues with it.Luke-Jr notes that the Wallet protocol implementation does not need to touch the GUI code at all except when porting the GUI to use it. Jeff Garzik suggests calling the github/bitcoin/bitcoin.git the Satoshi Client if they go with Qt. Brendan Molloy suggests EditLive Enterprise, a content authoring tool.In one email exchange, John Smith announces that his Qt GUI is ready for merging. He asks about the current roadmap for including the autotools build system so that it will be possible to support multiple GUIs. Pieter expresses his support for merging the autotools now unless there are still known issues with it, such as mingw32 compatibility.Jaromil re-based his autotools branch against git master and included Mark's NOSIGPIPE/NOSIGNAL patch. People are requested to test this by compiling and working on several systems such as ubuntu, fedora(? ), mingw cross compile, osx. The discussion is about merging qt to improve the GUI of Bitcoin. It is mentioned that merging qt will enable real GUI improvements to be made sooner.The email ends with a promotion for EditLive Enterprise - a content authoring tool.
