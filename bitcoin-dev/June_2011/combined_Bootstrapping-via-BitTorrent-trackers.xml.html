Summary:

In June 2011, the Bitcoin development mailing list discussed different bootstrapping mechanisms to improve the decentralized P2P network of Bitcoin. One suggestion was to use HTTP-based trackers, similar to BitTorrent trackers, as they would be sporadically contacted by each client rather than keeping an IRC connection open at all times. Another proposal was to use BitTorrent tracker bootstrapping, given that there are many open trackers accepting and tracking any hash thrown at them. However, concerns were raised regarding the potential for Sybil attacks on DHTs and the creation of a vulnerable hot spot in the DHT.Christian Decker suggested using HTTP-based trackers like OpenBittorrent as a viable alternative to IRC bootstrapping. He argued that HTTP-based trackers have several advantages over IRC and DNS Seeds such as sporadic requests, better load balancing, and more peers. Jeff Garzik, however, disagreed, saying that DNS bootstrapping can be very robust and invalidated some of Christian's assumptions about DNS seeds. There was also a proposal to create a convention where bitcoind listens on a specific last octet of IPv4 address, such as .14 when possible, as a bootstrap method of last resort. But Christian deemed this impractical, leading to network partitions and issues with IPv6.Jeff Garzik proposed publishing build instructions for a simple DNS server build and script which feeds it with node data to hardcode a bunch of such DNS seeds into the client. This method of improving DNS bootstrapping using djbdns (tinydns) was considered great for scalability and stability. Trusted people running DNS seeds on stable connectivity will be required once the DNS software is implemented. Another proposal was to create an RPC in Bitcoin that returns fresh P2P node addresses from addr.dat.In response to Chris's question about why BitTorrent tracker bootstrapping was not adopted for Bitcoin, Jeff Garzik explained that it could be vulnerable to Sybil attacks on DHTs. Christian Decker reiterated the benefits of HTTP-based trackers over IRC and DNS Seeds, but Jeff disagreed, saying that DNS bootstrapping can be very robust. Vladimir Marchenko suggested improving DNS bootstrapping by publishing build instructions for a simple DNS server build and a script to feed it with node data, which anyone can host, and hardcoding seeds into the client would not be necessary.
