Summary:

Karl-Johan Alm has authored a Bitcoin Improvement Proposal (BIP) that outlines the Signet network, which uses signatures in addition to proof of work for block progress. This results in better coordination and robustness for longer-term testing scenarios involving multiple independent parties, even with unreliability. The aim of this test network is to be more suitable for integration testing by organizations such as exchanges or for testing next generation Layer-2 protocols like Eltoo or sidechain pegs. The specification describes the "signet" network, which takes an additional consensus parameter called the challenge (scriptPubKey). It is important to note that this specification is backwards compatible, meaning that existing software can use Signet without any issues.
