Summary:

Jonathan Underwood proposed a wording change to the Bitcoin Improvement Proposal (BIP) for the sighash field on July 9th, 2019. He presented an attack scenario where a hacker changes the sighashtype of a partially signed Bitcoin transaction (PSBT) to SIGHASH_ANYONECANPAY | SIGHASH_NONE and then adds inputs and outputs to send BTC to themselves. Underwood suggests adding a check for signers to ensure that the sighash type provided is acceptable to them and fail signing if it is not. In response to Underwood's proposal, Andrew Chow explained that this was the original intent of the sighash field; either the sighash is acceptable to the signer and the signer signs with it or they do not sign at all. Jonathan agreed with Andrew's explanation and said he would write the amendment and make a PR. Underwood proposes adding a wording change to the bullet list for signers to ensure that the sighash type provided is acceptable to them. He suggests that if a sighash type is provided, the signer must check if it is acceptable and fail signing if it is not. If no sighash type is provided, the signer should sign using SIGHASH_ALL but may sign with any sighash type they wish.Overall, the proposal aims to prevent a scenario where a hacker can steal BTC from a signer by changing the sighashtype of the input to SIGHASH_ANYONECANPAY | SIGHASH_NONE, changing the outputs to send to the hacker's address, and adding an input that they signed with SIGHASH_ALL.
