Summary:

Andreas Schildbach via bitcoin-dev clarified that BIP 157/158 is not an alternative to BIP 37, but they complement each other. Both can be used by wallets to save deterministic GCS filters in the same way as headers. This allows blocks to be re-scanned offline if necessary, such as when importing new keys. Bloom filters are useful for limiting mempool bandwidth and controlling the fraction of each block downloaded. While this doesn't matter much on the BTC chain, larger block chains may end up using both filter schemes for these reasons.
