Summary:

The email thread on the bitcoin-dev mailing list discusses various aspects of using OP_RETURN and Single Use Seals for creating and trading NFTs on Bitcoin. The possibility of using OP_RETURN for creating and trading NFTs on Bitcoin via Lightning is one of the main topics of discussion. Participants question whether it could be included in opening or closing transactions within a channel or network of channels, if it could change hands within a network of channels, and if it could be divisible.It is noted that OP_RETURNs do not have ownership according to the bitcoin network, but it is possible to define a protocol that associates an OP_RETURN with ownership. This could then be transferred via lightning by sending associated currency. However, this would require a protocol layer parallel to Bitcoin/Lightning that stores and reads all transactions involving the node's channels and OP_RETURN.Christian Moss and Peter Todd discuss the use of single-use seals for asset transfers on the Bitcoin ledger, which require an on-chain transaction to post proof of publication to the ledger. Batch processing can be used by adding the merkle root into the single transaction going into the ledger, thus allowing scalability. The need for a parallel protocol layer that stores and reads all Bitcoin transactions involving the node's channels as well as the ones with the OP_RETURN is highlighted.It has been discussed that using Lightning Network for NFTs on Bitcoin is not feasible due to liquidity issues, but Ruben Somsens' state chain protocol is suggested as a potential solution. The discussion also explores the possibility of defining a protocol for OP_RETURN ownership transfer via lightning and making OP_RETURN divisible. Overall, the email thread highlights the potential for using OP_RETURN and single-use seals for creating and trading NFTs on Bitcoin, but acknowledges the need for further development of protocols and parallel layers to ensure scalability and ownership transfer. Links to relevant projects and articles are provided throughout the discussion for further research.
