Summary:

Bitcoin developer Gloria Zhao has proposed a set of mempool/transaction relay policies to aid L2/contract protocols. The proposal aims to address issues with the previous Package Mempool Accept package RBF and Rule 3 pinning. The new policies include additional rules for V3 transactions and modifications to package RBF rules.For V3 transactions, there are specific rules in addition to existing standardness rules. A V3 transaction can be replaced even without signaling BIP125 replaceability if it meets other RBF rules regarding fees. Descendants of an unconfirmed V3 transaction must also be V3, allowing for "inherited" replaceability signaling and simplification of descendant limits. However, this rule does not apply to descendants of confirmed V3 transactions. An unconfirmed V3 transaction cannot have more than 1 descendant to prevent a pinning attack. Additionally, a V3 transaction with an unconfirmed V3 ancestor cannot exceed 1000 virtual bytes to control fee-bumping replacements and ensure funding for child transactions.Modifications to package RBF rules include allowing new unconfirmed inputs in a package, but the child transaction's feerate must be at least as high as the input's ancestor feerate. There is also a suggestion to make OP_TRUE a standard script type and allow a single dust-value output that is immediately spent by the package, facilitating easier design of anchor outputs.The proposal clarifies that V3 transactions are currently non-standard in Bitcoin Core, and any transactions that were standard before this policy change would still be standard afterwards. The proposal is open for review and feedback, with the aim of solving Rule 3 pinning and potentially eliminating CPFP carve-out.In summary, Gloria Zhao's proposal introduces mempool/transaction relay policies for L2/contract protocols, addressing issues with previous proposals. The proposal includes new rules for V3 transactions and modifications to package RBF rules. Feedback and review are encouraged to refine and implement these policies effectively.
