SegWit activation proposal with 2MB Hard Fork by next Block Halving on 2020

Summary:

The proposal aims to verify whether 50% of the Tx mined from the SegWit activation block to block 630000 are SegWit Tx. If this is confirmed, the legacy max block size would be increased to 2MB and the SegWit max block size to 8MB. The implementation of this proposal could be done in the next release of Bitcoin Core or require a restart of the SegWit activation period while it is implemented and released. The rationale behind this proposal is to minimize the chance of a chain split while adhering to the original bitcoin scaling roadmap. This proposal follows the guidelines set out in https://lists.linuxfoundation.org/pipermail/bitcoin-dev/2015-December/011865.html.
