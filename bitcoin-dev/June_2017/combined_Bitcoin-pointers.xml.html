Summary:

José Femenías Cañuelo is developing a specification for Bitcoin pointers that can be used to indicate transactions, inputs, outputs or internal items within them. He has made three proposals for these pointers, with each building on the one before. The first proposal is Canonical Pointers, which uses an example such as "btc at 170.1/179-631-520". The second proposal is Mnemonic Pointers, using words from the BIP39 dictionary instead of numbers. The third proposal is Domain Pointers, building on the Domain Name Registration system to encode a pointer to a Canonical Pointer, like "btc at example.com". The email author, Jonas, proposes the idea of Canonical Pointers and provides a link to a Google Docs document with more information. He also mentions a related proposal posted last month and links to it. The Bech32 encoded transaction position reference is mentioned, which allows for error detection/correction with just 14 characters. The email ends with a PGP-signed message from the author. José Femenías Cañuelo welcomes feedback from the community as he has not yet had any peer review.
