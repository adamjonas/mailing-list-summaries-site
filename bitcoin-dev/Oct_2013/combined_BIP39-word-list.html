Summary:

A discussion on the BIP39 wordlist has been ongoing, with various proposals and suggestions being shared among interested parties. One proposed solution is to use a mapping of similar characters already present in the source code to find a combination of characters that actually appears in the mapping when a user types a word not in the wordlist. An updated wordlist has been pushed, which is filtered to similar characters taken from a matrix provided by Gregory Maxwell in an old wordlist solver code. The goal is to avoid confusion when encoding public and private keys and prevent words that could look similar when written down.Certain letter pairs are now considered similar, including combinations such as 'a' & 'c', 'a' & 'e', 'a' & 'o', 'b' & 'd', and many others. Brooks Boyd suggested including similar words that refer to the same number in the wordlist, while Marek proposed an application-based solution using a mapping of similar characters. Examples were provided to illustrate both proposals.An email thread shared contains a code for a wordlist solver with a visual similarity matrix of 52x52 letters. The author suggests avoiding words that could look similar when written down while encoding public and private keys, giving examples such as 'public', 'private' and 'secret'. They also provided a search utility to identify words that only differ by specific letters. The code requires an English wordlist file which can be downloaded from a GitHub repository.Gregory Maxwell shared a link to a search utility he used on his word list visual.py. Pavol Rusnak responded by stating that posting links to tools on the internet is not helpful and sending pull requests with particular changesets with explanation is more effective. He also believes it's no longer wise to introduce changes to the word list, especially since there are 50 people with 51 different opinions on the topic. Additionally, the author included a promotional message for Intel webinars on Code for Performance.Finally, a new unit test called test_similarity() was pushed to Github to find similar words in the current wordlist. The test currently identifies approximately 90 similar pairs, and the wordlist will be updated the following day to pass this test. The proposed wordlist includes the possibility of multiple words on a single line, with the first word on the line being the "primary" or "real" word to be used. The discussion continues towards a final wordlist standard.
