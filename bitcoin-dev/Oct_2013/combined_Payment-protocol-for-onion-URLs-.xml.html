Summary:

In 2013, a concern was raised about the limitations of Bitcoin's Payment Protocol. Gregory Maxwell noted that hidden service sites were unable to make use of its full authentication capability due to the inability to obtain SSL certificates. He proposed a solution of packing a tor HS pubkey into a self-signed x509 certificate with the cn set to foooo.onion. Adam Back expressed concern over the use of X.509 in the protocol due to its susceptibility to corruption attacks and potential security issues. Instead, he suggested using bitcoin keys to sign payment messages, which can be associated with X.509 if desired. There was also a discussion about bounties and their signatories. John offered a 1BTC bounty and another for which he has offered 4BTC as a reward. Jeremy Spilman discussed the risks of reusing keys for signing and notes that it is bad form to do so. It was suggested that John should have gotten separate public keys for the purpose of his bounty. In addition, there was a discussion about disambiguating bounties, with suggestions such as creating a Python script for signature accumulation and signing.The payment protocol lacks the capability for a hidden service site to utilize its full authentication potential as they cannot obtain SSL certificates. However, a Tor hidden service (onion site) is managed by an RSA key which can be easily packed into a self-signed x509 certificate with the cn set to foooo.onion. If an additional validation procedure is added to the payment protocol for [base32].onion hosts to hash and base32 encode the pubkey, then the payment protocol could work flawlessly with tor hosts, displaying that the payment request came from "foooo.onion". Finally, it was noted that blockchain.info had previously added multisig escrow support but removed it due to near-zero usage. During a discussion about multisig wallets in 2015, the possibility of creating a Python script to do signature accumulation and signing was mentioned, but there hasn't been much demand yet to fully polish UIs. Peter Todd suggested keeping a copy of Tor URL bookmarks on your second signing computer to make it a genuinely useful tool for multisig wallets too. This would give us a fully supported option which is completely CA free, although it would only work for tor sites.
