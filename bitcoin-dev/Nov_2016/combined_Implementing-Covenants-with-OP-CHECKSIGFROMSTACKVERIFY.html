Summary:

Russell O'Connor has proposed a method for implementing covenants using two script extensions, OP_CAT and OP_CHECKSIGFROMSTACKVERIFY. This proposal was discussed on the Bitcoin-dev mailing list and scripts for the Moeser-Eyal-Sirer vault were provided as an example. Johnson Lau also shared his own implementation of OP_CHECKSIGFROMSTACKVERIFY from the Elements sidechain.The Moeser-Eyal-Sirer vault demonstrates how to create an output that is spendable by everyone if a user ever double-spends the output with two different transactions. Additionally, there was a discussion on using OP_CHECKSIGFROMSTACKVERIFY to implement "poison transactions".A business case was presented as an example of using multisig with an oracle determination to minimize counterparty risk in a transaction. O'Connor encouraged others to share their ideas for useful covenants and expressed interest in collecting and implementing them. For any questions regarding the implementation, Russell O'Connor can be contacted.
