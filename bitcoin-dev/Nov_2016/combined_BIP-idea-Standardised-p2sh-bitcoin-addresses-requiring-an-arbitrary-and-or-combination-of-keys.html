Summary:

Erland Lewin's proposal aims to create "Boolean addresses" that can be redeemed by a set of keys and multi-signatures combined with logical and/or operations. The proposal provides a consistent way to standardize these addresses, allowing wallet software interoperability to create, sign, and share signature requests for such addresses. A Boolean address is described as a tree starting at a root node, where a node can be an "and" operation, an "or" operation, a public key, a multisig operation n-of-m with a list of public keys or a CHECKLOCKTIMEVERIFY operation.The implementation will describe a single well-defined way to generate a P2SH script from a given boolean address tree. For instance, corporations can use this proposal to allow signatures by "A or (2 of 4 of B, C, D, E)" to spend accounts that can be spent by the CEO or board members using their keys. Similarly, Alice can redeem her address by "A or (B and 1-of-3 of C, D, E)" in case she loses her key, or her cloud wallet key is compromised or paper wallets C, D, or E are stolen.Reality Keys, a platform that verifies information against real-world events, confirms the proposal's aim to fix a painful point and sees it as a helpful solution if wallet authors adopt it. Edmund Edgar, founder of Social Minds Inc (KK), supports the proposal's sensible implementation. This proposal provides a well-defined way to create Boolean addresses that can be redeemed by an arbitrary set of keys and multi-signatures combined with logical and/or operations. Its implementation details how spending transactions are to be signed, providing a standardized method for wallet software interoperability.
