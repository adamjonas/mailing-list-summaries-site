Summary:

A proposal for a new Bitcoin Improvement Proposal (BIP), called bip48, has been put forward by Fontaine to document modern multi-sig derivations. The BIP is inspired by bip44 and aims to formalize existing tacit standards used in practice across multi-sig wallets. The proposed BIP removes all references to legacy p2sh derivations and keeps strictly to p2sh-p2wsh and p2wsh derivations as used in modern-day wallets. Some key points were brought up during the discussion on Twitter, such as not defining a `script_type` as a path level and including a defined "wild card" in the script_type level to define any future address types.Feedback from the community is welcome, and a PR has been created so that anyone can easily comment on it and raise any concerns. The proposal has garnered positive feedback and support from other developers. In an email thread, Fontaine shared the rough draft of the proposed BIP along with two attachments, one of which is an HTML file that has been scrubbed, and the other is a non-text attachment called bip-0048.mediawiki. Luke Dashjr reminded users not to self-assign BIP numbers and asked if the proposal was compatible with a previous pull request. Pavol Rusnak, co-founder and CTO of SatoshiLabs, applauded the effort and suggested adding script_type=0 for raw BIP-11 multisig.Fontaine provided an update stating that the proposal was still a work in progress and that they were using feedback from Pavol and others to update it. The proposal aims to provide more clarity on the usage of 48 paths and is seen as overdue. However, it is unclear whether the proposed BIP is intended to be compatible with another GitHub repository for BIP253.
