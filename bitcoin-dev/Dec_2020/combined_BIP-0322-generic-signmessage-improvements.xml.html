Summary:

A Bitcoin Improvement Proposal (BIP-0322) outlines a protocol for interoperable signed messages based on the Bitcoin Script format. The author has rewritten the proposal to simplify and clarify the rules for signing messages. The revised proposal merges "consensus" and "upgradeable" rules into one set of rules and specifies three formats for signing messages: legacy, simple, and full. Validators must output one of four states: valid, valid at time t and age s, consensus-valid, or inconclusive, and the signature must pass all checks described in the detailed specification.The BIP outlines additional states that can be output instead of "valid" or "invalid" to ease implementation. Pieter Wuille suggested modifying the BIP to allow incomplete validators to return "inconclusive" results while still detecting future features used in the signature. Andrew Poelstra supported this idea and added a commit to his PR at Github which reflects their thoughts.Karl-Johan Alm expressed his support for recent changes to the BIP but noted that this suggestion breaks the original use of inconclusive as a way to detect future features used in the signature. He suggested requiring reporting inconclusive instead of invalid whenever validation fails due to any standardness features unless something is actually invalid. Pieter Wuille also supported Karl-Johan's suggestion.The document also outlines a specification for a new message signing scheme called "signet," which is intended to serve as proof of funds. Validators require access to the current UTXO set and scriptPubKeys to confirm the claimed inputs exist on the blockchain. The BIP is backward-compatible with the legacy signmessage/verifymessage specification through a special case described in the document. Test vectors have not yet been provided, and there is currently no reference implementation. The author has shared the new text and welcomes comments on Github.
