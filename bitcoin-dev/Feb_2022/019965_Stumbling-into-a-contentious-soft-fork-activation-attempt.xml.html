Summary:

In a recent post, ZmnSCPxj has mentioned that if an improvement allows moving some activity off-chain and reduces activity on the blockchain, then the increase in functionality does not always require an increase in block size. The Taproot upgrade is one such example which offers improvements such as Schnorr multisignatures, MAST, `SIGHASH_ANYPREVOUT`, and `OP_CTV`. These features reduce the block size usage for complex contracts, enable offchain updateable multiparty cryptocurrency systems, and allow transactional cut-through without having to publish promised outputs immediately. ZmnSCPxj believes that these upgrades increase the functionality of Bitcoin while also providing opportunities to use the blockchain as judge+jury+executioner instead of a noisy marketplace.
