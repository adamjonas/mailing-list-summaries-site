Summary:

The Bitcoin network has a relay policy that indicates when transactions will "age out." However, users cannot be certain whether these transactions will actually age out as specified. This is because exceeding the specified timeout does not stop a miner from including it in the blockchain. Therefore, this relay policy doesn't provide any actionable information for users. Dan Bryant, in a message via bitcoin-dev on Dec 28, 2017, proposed a solution to make it easier for users to know when a transaction will age out with more certainty.
