Summary:

In the Bitcoin community, there is a discussion about whether or not to create a BIP (Bitcoin Improvement Proposal) for Sign/Verify message against a SegWit address. Damian Williamson suggests that signing a message is an important feature whether it is with Bitcoin Core or some other method. He believes that it would be worthwhile to update it for SegWit addresses. Mark Friedenbach proposes a solution that involves having the signature be an actual bitcoin transaction with inputs that have the script being signed. This approach has several benefits like enabling signing by any infrastructure out there, allowing multi-party signing setups, and unifying a single approach for message signing, proof of reserve and off-chain colored coins. However, there is an issue of size efficiency, which can be handled by a BIP specifying a template for constructing the pseudo-transaction and its inputs from a raw script. The proposal also suggests that this approach would unify a single approach for message signing, proof of reserve, and off-chain colored coins. Dan Bryant asks if anyone is writing a BIP for Sign/Verify message against a SegWit address, to which Pavol Rusnak asks if he is still planning to write it. Although there are some posts and an issue opened about it on GitHub, it seems that no one is currently working on creating the BIP. The conversation around this topic can be found on the GitHub issue #10542. The author suggests that signing a message is an important feature in scenarios where proof of assets is needed, or for cases where a person needs to prove that they are in charge of a large amount of funds. The post also mentions the possibility of signing a message to prove the ability to redeem some arbitrary branch in a MAST-like tree of scripts or being a minority part of a multisig. No BIP currently exists for the current message signing mechanism, and the author suggests a more generic implementation would be ideal to avoid exclusion of funds with complicated redeem scripts. It is unclear if the proposal will gain enough support to move forward, but it is clear that there are individuals in the community who see value in having this feature available. It remains to be seen whether or not a BIP will be created for Sign/Verify message against a SegWit address in the future.
