Summary:

Luke Dashjr, a member of the Bitcoin development team, has criticized a proposed expansion of capabilities into new address schemes that would allow for signature verification using a public key obtained from an address. Dashjr pointed out that this approach is not possible and instead, key recovery is performed using the signature and message to extract the public key of the signer, which is then hashed and compared to the address provided. The proposal aims to provide a reference point for future expansion of these capabilities into new address schemes. However, Dashjr argues that new schemes should not be based on the current one.Dan Bryant also criticized a proposed Bitcoin Improvement Proposal (BIP) by Brian Deery, which suggests adding a "verify messages" feature to Bitcoin wallets that would allow users to verify ownership of a particular Bitcoin address by signing a message with the associated private key. Bryant claimed that the proposal was incorrect because it suggests that a public key can be derived from a Bitcoin address, which is not possible. Instead, the proposed feature would use signature and message to perform key recovery and extract the public key of the signer to compare it to the address provided. In addition, Luke Dashjr inquired about the purpose of pubkey in sigscript for standard p2pkh transactions. He questioned the method used to extract the public key of the signer, which involves using the signature and message to perform key recovery, and then hashing it to compare it with the provided address. The discussion on bitcoin-dev mailing list revolves around the use of signatures and public keys in Bitcoin transactions.
