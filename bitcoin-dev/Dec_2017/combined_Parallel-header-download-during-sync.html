Summary:

Bitcoin developer Greg Maxwell has proposed a header sync mechanism that would enable parallel download from multiple peers, improving upon the current getheaders/headers semantics which require sequential downloads from genesis. In testing, it was found that syncing headers directly from a colocated node was quicker than downloading them from other nodes. Additionally, the idea of a compressed headers stream was discussed in the same thread. This proposal aims to improve the efficiency and speed of Bitcoin's syncing process.
