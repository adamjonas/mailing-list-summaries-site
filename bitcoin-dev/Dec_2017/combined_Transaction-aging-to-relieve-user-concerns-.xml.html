Summary:

Dan Bryant has proposed a solution to make it easier for Bitcoin users to know when a transaction will age out with more certainty. Currently, the relay policy indicates when transactions will age out, but exceeding the specified timeout does not stop a miner from including it in the blockchain. The proposed solution is to expand the transaction format to include chain-height (block number) in the TXN, which would allow nodes/miners to age the TXN and choose not to rebroadcast it after a certain age threshold. This would make it easier for users to know with better certainty when a TXN is going to age out.Currently, nodes/miners keep a "seen-list" of TXNs and age them based on when they were last seen. However, those TXNs could be broadcast forever, assuming the UTXO never got spent. The goal is not to enhance the protocol but rather make it easier for users to know when a TXN is going to age out. One possible abuse vector could be a bad party broadcasting a low fee TXN at the edge of the age-out threshold and trying to get goods, realizing the TXN will age out at the very next block.If this proposal might be something that core would entertain in a BIP proposal, the author will start drafting something. The proposed BIP would not enhance the protocol but would make it easier for users to know with better certainty when a TXN is going to age out.
