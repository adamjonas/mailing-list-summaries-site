Summary:

In February 2014, Tamas Blummer raised concerns about the spam limit of at least 5430 satoshis when the script is OP_RETURN. He was worried that it could lead to opening up the possibility for cheap spam. However, a response explained that OP_RETURN outputs are provably unspendable and not included in the UTXO set. The client checks and handles TX_NULL_DATA outputs separately, so they are not considered dust.According to Tamas Blummer, the current cost to create an output is 5430 satoshis. Wladimir suggested that if transactions like this are not abused and more space is needed, the limit can always be increased in a future version. It is unclear whether the same spam limit applies if the script is OP_RETURN.Overall, the discussion centered around the potential for spamming on the Bitcoin network and how the system handles different types of outputs. The responses provided reassurance that OP_RETURN outputs are not an avenue for cheap spam due to their unspendable nature and separate handling by the client.
