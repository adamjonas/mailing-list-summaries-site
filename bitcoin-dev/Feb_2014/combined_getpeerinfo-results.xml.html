Summary:

On February 22, 2014, Ronald Hoffman reported an issue with the 'services' field appearing garbled in the latest source code level from GitHub. He found that Bitcoind is connected to his Java node server at 127.0.0.1:8333 and thought he was sending a bad 'version' message, but he got the correct results using 0.8.6, indicating a change in the field between versions 0.8.6 and 0.9.0. The bitcoin-cli command to getpeerinfo was used to compare the 'services' field for both versions of Bitcoind, with the result that the 'services' field in version 0.9.0 appeared garbled with a value of "0000000164x", while in version 0.8.6 it was "00000001".Wladimir suggested creating an issue on Github and provided a link for testing purposes. He also asked what git revision Hoffman was testing with, built with what compiler on which platform, and whether all peers in getpeerinfo show up garbled or just Hoffman's. When Wladimir tried getpeerinfo locally, all his peers showed as ""services" : "00000001" as expected.The problem was later identified by Wladimir as a Windows inttypes.h border case, and he fixed it.
