Summary:

The author of an email is seeking clarification on the gettx / getrawtransaction (decoded) API call. They are questioning why the vout array in the scriptPubKey.addresses array is typed as an array when it usually contains only one address. The sender is uncertain whether there are any cases where this array contains more or fewer than one address or if it is structured this way for future extensibility. They believe owning the public/private key in the vout script is what matters for spending coins in a future transaction, and not the address. The author is working on a project to calculate unspent outputs and balances for all accounts and is unsure of any special cases that need to be considered to produce accurate data. The sender recommends checking out coinb.in/multisig and abis.io for additional information.
