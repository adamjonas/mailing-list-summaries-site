Summary:

On August 13, 2014, Richard Moore, the founder of Genetic Mistakes Software, posted a query about a multisig output script with a public key and whether invalid public keys are permitted or silently ignored. He also asked if the 0x00 prefix had any significance. In response, another user suggested looking at the Bitcoin Core script test cases and reading the source code. They explained that the Bitcoin protocol doesn't concern itself with whether public keys are valid or not. RicMoo, the same founder, discovered a transaction in the blockchain that contained an output script with a public key. He was unsure if invalid public keys were allowed and silently ignored, or if the 0x00 prefix carried meaning. The transaction was verified by the legit client, and RicMoo is verifying the blockchain one transaction after another using his own implementation. The email response to Richard's query was signed off with a PGP signature. Overall, these posts highlight the importance of understanding the technical details behind blockchain transactions, including how public keys are treated and what steps are taken to verify them.
