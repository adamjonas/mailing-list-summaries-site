Summary:

A proposed change in the BIP32 protocol has raised security concerns. The proposed change involves assigning I_L := I_L mod n when I_L >= n, which is considered less secure due to its potential to cause users to believe a key is worthless or lose funds when unable to spend them. Additionally, it would be inconsiderate to other parties who have done work to produce correct and compatible implementations. The proposed change would also make the selection of private keys uneven, which is objectively less secure and could create a devastating weakness if an incorrect implementation reflects a large class of keys to a small number of values.The email conversation highlights the importance of dealing with corner cases in cryptographic software such as handling the point at infinity cases, zero value signature, and adding colinear points. While some implementations do not check for these cases, it is important to handle them to ensure the security of the system. However, the bignum modulo operation requires complex additional logic, and there may not be a suitable modulo available due to layering issues.The author of the post proposes changing the BIP32 protocol regarding invalidation of nodes, suggesting that rather than proceeding with the next value for i when a resulting key is invalid, I_L should instead be assigned I_L := I_L mod n in all three cases (public CKD, private CKD, and master key generation). This change aims to simplify the process of marking a key as invalid and lead to more implementations actually checking for invalid keys. However, the author notes that there are still cases where a key may be invalid, such as when I_L = 0 or ki = 0 or ki = inf, but these have a low probability of occurring.In conclusion, the email conversation highlights the need to consider the security implications of proposed changes to the bitcoin specification and the importance of handling corner cases in cryptographic software. It suggests that those unprepared to deal with these complexities should leave writing this kind of software for other people.
