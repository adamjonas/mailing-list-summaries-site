Summary:

In an email thread, Pieter Wuille suggests that merges should be done through the github-merge tool, which allows for local testing and displays a resulting diff. Wladimir adds that he also uses this tool and recommends it to others. He encourages more people to review code changes, preferably through local tooling rather than the web page. However, he acknowledges that hosting the code on GitHub results in more eyes overall. For browsing git history locally, Wladimir finds "gitk" to be a useful tool.
