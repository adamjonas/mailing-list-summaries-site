Summary:

In an email thread from 2014, a user named Mem Wallet proposed a standardized PGP-style message for Bitcoin transactions. This would allow users to send signed encrypted messages using only public and private Bitcoin keys. However, using the same keys for signing and encryption is generally considered bad practice for security reasons. Additionally, implementing encryption in Bitcoin software is difficult and previous attempts have been severely flawed and insecure. The author of the email does not feel confident that a new effort would receive adequate review. In conclusion, while the proposal may have some potential benefits, there are significant security concerns and implementation challenges to be addressed.
