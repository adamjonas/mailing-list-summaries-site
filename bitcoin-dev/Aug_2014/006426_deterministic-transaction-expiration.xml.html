Summary:

On August 6, 2014, Christian Decker suggested adding a new field to improve the tx format without overloading existing fields. He proposed creating a new, parallel tx format where fields are committed by merkle radix tree in an extensible and provable way. This would be committed with a mandatory OP_RETURN output in the last txout or with a new merkle root. Decker believed that changing the tx format itself in a hard-fork is needlessly disruptive and wastes opportunities for improvement.
