Summary:

OpenSSL supports a non-standard encoding format for public keys, which is apparently useless. Although no hybrid keys are used in the main chain, they are supported by almost all fully validating clients on the network. A total of six encodings exist, with 0x06 and 0x07 being the hybrid formats for even and odd Y coordinates, respectively. Handling these keys is easy- if a public key starts with 0x06 or 0x07, use it as though it had started with 0x04 instead. Pieter suggests forbidding these encodings after a certain date or block height. Gavin Andresen advises treating any transactions that use them as 'non-standard' to make a rule change in the future easier.
