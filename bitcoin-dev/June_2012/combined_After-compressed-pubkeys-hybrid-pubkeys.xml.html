Summary:

There have been discussions within the Bitcoin development community about the use of 'hybrid' public keys with the prefix 0x06/0x07. On June 16, 2012, Gavin Andresen initiated a discussion on the Bitcoin development mailing list about these keys. The topic at hand was whether to forbid their usage or not. While forbidding them would make implementation easier in the future, it was unclear whether changing network rules would be worth the hassle. Gavin suggested treating any transactions using these keys as 'non-standard,' meaning they wouldn't be relayed or mined automatically but could still be accepted in a block.In another conversation between Pieter Wuille, Mike Hearn, and Wladimir, the focus was on the different formats used for representing elliptic curve points. These formats include the uncompressed format (0x04), hybrid format for even Y coords (0x06), and hybrid format for odd Y coords (0x07). The only difference between these formats is the first number. Pieter suggests treating transactions using these formats as non-standard by not relaying or mining them by default, but still accepting blocks that contain them to make it easier to implement a rule change in the future if necessary. The conversation also touches upon the different formats used for encryption keys in OpenSSL. To prevent any surprises if OpenSSL decides to add more key formats in the future, Wladimir suggests disabling all other key formats in advance.Furthermore, a message from 2012 discusses the different encoding formats for public keys used in OpenSSL. The various encodings include 0x00 for a point at infinity, 0x02 and 0x03 for compressed formats for even and odd Y coords respectively, 0x04 for uncompressed format, and 0x06 and 0x07 for hybrid formats for even and odd Y coords respectively. Handling these different encodings is deemed trivial, and the author suggests forbidding these formats after a certain date or block height, although the hassle of enforcing this through a network rule change may not be worth it.In summary, there have been discussions about the different formats of encryption keys in OpenSSL and the various encoding formats for public keys used in Bitcoin. The discussion around 'hybrid' public keys with the prefix 0x06/0x07 has also taken place, with some developers suggesting treating transactions using these keys as non-standard while others question whether forbidding their usage would be worth the hassle of a network rule change.
