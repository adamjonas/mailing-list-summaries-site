Summary:

In an email exchange from June 2012, Simon Barber suggested separating the filterinit and filterload functions in order to allow for a new filterload later on if a list changes, without the privacy implications of filteradd. However, Jeff Garzik disagreed with the idea stating that filterload loads a whole new bloom filter from scratch, in one atomic operation. Parameters are set, table sized, and data input into the table during this process. Therefore, a separate filterinit does not make sense for filterload.
