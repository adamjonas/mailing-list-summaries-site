Summary:

In a 2012 email exchange, Mike Hearn expressed his preference for someone other than Gavin to be responsible for OS-vendor signing, which refers to certifying software for use on a particular operating system. The reason for Hearn's concern is not given. The author of another email in the same discussion expresses concern over the security of binaries available on the bitcoin.org site and suggests signing all binaries with gpg to protect against potential hacks and wallet theft. Despite developers having gpg keys, detached gpg signatures are not being provided for binaries. MacOS X 10.8 requires application signing using a certificate issued by Apple's "identified developer" program to run unsigned apps, while Windows does not require signing but antiviruses tend to use signers with good reputations as a whitelisting signal. Signing Bitcoin releases may improve performance if antivirus engines ignore file reads/writes by Bitcoin and also shield from false positives. The email thread includes a suggestion that an unnamed individual could sign the Windows installer and binary, but Luke-Jr notes that Gavin would be the final signer, despite Hearn's preference for someone else to handle vendor signing. The email ends with a promotional message about a Live Security Virtual Conference covering the latest in malware threats, endpoint security, and mobile security.
