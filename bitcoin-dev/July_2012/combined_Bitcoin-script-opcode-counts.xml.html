Summary:

Jeff Garzik, an American software developer associated with exMULTI, Inc., conducted a query over the entire blockchain to inspect script opcode usage which counted the number of times each opcode was used, providing data for both the main blockchain and testnet3. The results of this analysis have been made available on GitHub in an organized chart which lists all opcodes and their corresponding count numbers. Overall, there were 26 different opcodes used in the blockchain. The highest frequency of use was for OP_CHECKSIG followed by OP_DUP. While some opcodes such as OP_4 and OP_NOP1 had minimal use, others like OP_PUSHDATA had a significant count. In a discussion thread on Bitcoin development from 2012, Mike Hearn expressed his curiosity about the use of OP_DEPTH and OP_CODESEPARATOR in Bitcoin scripts. He requested transaction IDs that were using these opcodes. However, it was later discovered that all instances of OP_DEPTH were errors in P2Pool blocks due to a bug in the software which went unnoticed for some time because it was assumed to be an intentional action by a user utilizing p2sh.
