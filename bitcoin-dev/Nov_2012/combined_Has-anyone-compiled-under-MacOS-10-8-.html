Summary:

There is an issue with Boost and the default build instructions, potentially related to the switch to clang++. The author of a message is investigating the issue and asking for help in finding a solution. They encourage anyone with information to reach out. One potential solution involves adding a magic incantation at the top, which may be related to how qmake is being used. To obtain a real makefile instead of an xcode project, qmake has to be run like "gmake -spec macx-g++". This might make gcc stricter than intended. Mike Hearn found that the issue was caused by using an older version of the Qt SDK with the new MacOS version. Reinstalling Qt fixed the problem.In a related email, Mike Hearn noted that there may be compatibility issues between Boost and the default build instructions, possibly due to a switch to clang++. He requested assistance in finding a solution, suggesting that someone share a recipe to fix the issue.
