Summary:

Bitcoin developer Luke Dashjr inquired about a serialization format to convey P2SH HD chains for recurring payments, suggesting a simple series of tokens to generate multiple P2SH addresses paying to a multisig script. Eric Lombrozo is working on a BIP to standardize a multisig application of BIP32 that would embed all necessary information for a "multisig xpub" into a string and pipe it through a BIP32-like algorithm to pop off P2SH addresses in a deterministic order. Gavin Andresen responded to Luke's inquiry by stating the importance of having an expiration date or some means of determining if the payment recipient is still operational.He referred to a January discussion on extending the payment protocol for recurring transactions and suggested offering a URL where the user can access PaymentRequests when they need to make a payment or possibly providing an array of PaymentRequests for the next X days/months/years of payments. The author has written a Javascript program to print minimal protobufs to base64 for multisig and regular pay to address outputs. However, it is recommended to have an expiry time for practical usage. An HD-iterating payment request would require additional flags and fields but should only add a few bytes to the protocol.Michael Perklin suggested connecting Eric and Luke to help each other if both are working on similar projects. Overall, there are ongoing efforts to develop standardized HD multisig chain-description formats and serialization formats to convey P2SH HD chains for recurring payments, with various proposals and ideas being discussed among Bitcoin developers. One individual expressed interest in creating a shared HD wallet format to allow different devices to generate new addresses using standardized HD multisig chain-description formats. Unfortunately, there are still many details to work through before having a concrete proposal. Another individual proposed a solution to convey P2SH HD chains by creating a simple series of tokens indicating either an HD chain or literal script content.The discussion revolves around finding ways to simplify the process of generating multiple P2SH addresses paying to a multisig script for recurring payments, with suggestions ranging from serialization formats to standardized HD multisig chain-description formats. The resultant multisig wallet would have a total keychains system, minimum number of keys required, and which derivation paths on each keychain are to be combined. The author suggests that the above strings are short enough to set as a username in a mining program to maintain Eligius's general UX.
