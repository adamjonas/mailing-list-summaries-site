Summary:

There is an ongoing discussion about adding additional inputs to the coinbase transaction, which would allow miners to voluntarily lock funds. Currently, only pseudo-inputs can be added to the coinbase transaction. This proposal would require a hard fork to allow for real inputs. The reason for this proposal is to ensure that in the future, when the blockchain is secured and the subsidy is lowered, Bitcoin can still compete with other payment networks without having to increase the number of confirmation blocks. There are concerns about this proposal, including the potential for changing incentives and making it easy for individuals to pay miners to mine specific blocks rather than transactions. Some argue that it could make transactions more efficient or lower fees, while others express concerns about changing incentives. It is also unclear what specific services could be enabled by adding inputs to the coinbase versus using pay-to-fee transactions.Various individuals, including Justus Ranvier, Mike Hearn, Sergio Lerner, and Gregory Maxwell, have suggested different ideas and proposals for optimizing transaction processing within the blockchain system. These include adding inputs to the coinbase, using pay-to-fee transactions, and creating outputs in the coinbase transaction with nValue == 0. However, many of these proposals would require either a hard fork or a soft fork to implement.Justus Ranvier suggests that if miners wanted to offer value-added services, the most natural way for their customers to pay them is to allow inputs to the generation transaction. He proposes a hard-fork to allow miners to voluntarily lock funds by adding additional inputs to the coinbase transaction. This proposal will not change incentives or security until a soft-fork is implemented later to enforce deposit bonds, proof-of-stake, and penalization of double-mining to prevent short-range attacks.In contrast to pay-to-fee transactions, inputs to a generation transaction cannot be poached, making some services possible that would not be safely achievable with pay-to-fee transactions. However, it is unclear what these services are. Sergio Lerner proposed an idea to allow miners to lock funds by adding additional inputs to the coinbase transaction. Gregory Maxwell provided a concrete example of how this could work by suggesting that anyone can spend output "PUSH OP_NOP3" would make the anyone can pay transaction simply a coinbase input. However, there was some concern about this proposal being an abuse of the VM. The ongoing debate and exploration around how best to structure and optimize blockchain transactions includes discussions on improving the functionality and scalability of these systems, including the use of coinbase inputs and pay-to-fee transactions. The writer expressed a desire for Bitcoin to remain simple and easy to understand for new programmers, rather than becoming overly complex.
