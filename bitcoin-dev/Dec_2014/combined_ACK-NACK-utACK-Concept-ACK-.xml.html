During code reviews for the Bitcoin software, various types of acknowledgments are used to indicate levels of review and approval. Wladimir introduced the terms 'Concept ACK', 'utACK', and 'Tested ACK' to describe different levels of review and testing completed. Additionally, he explained that 'NACK' is used to reject a code change with an explanation provided. However, he noted that outsiders may struggle to understand these terms without context and suggested creating a document explaining them.A user on the Bitcoin-development mailing list requested documentation explaining key terminology used within code reviews. This meant that he agreed with the overall direction of the proposal but acknowledged that the code changes had not been reviewed or tested.
