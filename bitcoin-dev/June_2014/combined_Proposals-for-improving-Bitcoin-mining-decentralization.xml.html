Summary:

Recently, there has been concern about the Ghash pool controlling 51% of the hashrate, which some consider a threat while others think it's not harmful. To prevent this from happening again, Raúl Martínez proposed a solution by creating a new mining protocol that does not rely on the pool sending the list of transactions to include in a block.The proposed protocol would require individual miners to collect transactions on their own, either by running a full node or an SPV-like node that asks other nodes for transactions. Once developed and standardized, the community could recommend all pools to use it. By using this protocol, certain 51% attacks can be avoided, such as selfish mining, double spending or reverse transactions, deciding which transactions to include, and avoiding other pools from mining blocks.Pool owners would send instructions using this protocol to the miner about how many transactions to include per block, how many zero-fee transactions to include, the minimum fee per KB to include transactions, and some info about the Coinbase field in the block. This way, the protocol can prevent some possible 51% attacks, including selfish mining, double spending or reverse transactions, deciding which transactions to include, and avoiding other pools from mining blocks.The proposal suggests treating miners who join a pool as independent miners rather than slave miners. The only thing a 51% pool owner can do is shut down their pool and drop the hashrate by 51% because they do not control the miners. However, if the pool owner owns all the hardware in the pool, the proposal is not valid, and if the pool clients do not use this protocol, the proposal is also not valid.While there is already a solution called Getblocktemplate (GBT), it is opt-in, and not all pools support it. If the mining community can find a way to make GBT more widely used, it could solve most of the pooling-centralization problems. Some believe there is no need to create anything new when GBT already exists, while others highlight that it is opt-in, and some pools do not support it. Additionally, it is believed that the stratum mining protocol improves some performance things that GBT lacks.In conclusion, the proposal aims to create a new mining protocol that does not rely on the pool sending the list of transactions to include in a block. It requires miners to collect transactions independently, and pool owners would send instructions using this protocol on how many transactions to include per block and some information about the Coinbase field in the block. The protocol can prevent possible 51% attacks, and miners who join a pool will be treated as independent miners. However, it is still unclear whether such a protocol exists or has been developed.
