Summary:

Alex Kotenko had set up DNS seeds at testnet-seed.alexykot.me and bitcoin-seed.alexykot.me, but there was a problem with the DNS settings that could not be resolved quickly by him and Andreas. The problem appeared to be that he could reach his nameserver and get dnsseed response if he queried it directly, but when he tried nslookup testnet-seed.alexykot.me, it failed, which led him to believe that the problem was with his DNS settings. Robert McKay replied that the problem might be due to the 'aa' flag missing in dig, and suggested suppressing authority records to see if it would work for some resolvers. McKay also mentioned that bind forwarding was stripping off the authoritative answer bit in the reply, causing the recursor to go into a loop chasing the authority server, which returned a non-authoritative answer with itself as the authority again. Alex Kotenko decided to switch to serving only testnet DNS on this server for now since having both DNS seeds running on the same IP address seemed impossible. He planned to later buy a second IP to set up mainnet DNS.
