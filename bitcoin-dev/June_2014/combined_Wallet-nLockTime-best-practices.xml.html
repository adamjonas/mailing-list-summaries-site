Summary:

Jeff Garzik, a Bitcoin core developer and open source evangelist, is considering implementing setting nLockTime on wallet-created transactions by default. This feature aims to discourage users from exploiting low fees by locking transactions for a certain period using nLockTime. Garzik has suggested considering the implementation of pull request #2340 from the Bitcoin GitHub repository and welcomes comments from other wallet implementors regarding this proposal.Aaron Voisine, the developer of breadwallet, an open-source SPV wallet, has expressed interest in implementing a new feature to discourage fee sniping using nLockTime. Jeff Garzik posted a message on the Bitcoin-development mailing list, seeking comments from other wallet implementors about pulling the feature into the codebase.Additionally, Garzik works as an open source evangelist at BitPay, Inc., a payment service provider that supports Bitcoin transactions. The proposed change can be found on Github as pull request #2340 in the Bitcoin repository. Lastly, the email includes a plug for NeoTech's "Graph Databases" book, which is available for free download.
