Summary:

On June 2nd, 2014, Toshi Morita reported an uninitialized memory problem in bitcoind using valgrind. The error message stated that a conditional jump or move in the program depends on uninitialised value(s). The error occurred in CWallet::LoadKeyMetadata function, which is responsible for loading key metadata from the wallet database. The error was traced back to ReadKeyValue and CWalletDB::LoadWallet functions, which are used to read data from the wallet database. This error was caught by the memory error detector, Valgrind-3.8.1 and LibVEX. Jeff Garzik, Bitcoin core developer and open-source evangelist, responded to Toshi's email about the issue. In the email conversation, Toshi confirmed that both sides of the comparison were uninitialized and he fixed one side but the other still has the same problem. He also said that he would investigate further once he gets out of meetings/meetings prep. Jeff Garzik believes he has identified the issue and ended the conversation by stating that he is a Bitcoin core developer and an open source evangelist at BitPay, Inc. A link to a pull request on Github was also provided for fixing the issue.The user is experiencing a memory-related issue in their bitcoind software when using valgrind. The valgrind output shows an uninitialized value error occurring in the CWallet::LoadKeyMetadata function in wallet.cpp, caused by a conditional jump or move that depends on uninitialised value(s). The function is called by ReadKeyValue in walletdb.cpp, which is in turn called by CWalletDB::LoadWallet. LoadWallet is called by CWallet::LoadWallet(bool&) in wallet.cpp, which is called by AppInit2 in init.cpp. Finally, AppInit2 is called by AppInit in bitcoind.cpp, which is the main function of the application.In addition to the issue at hand, there is information about a new guide to graph databases called "Graph Databases" that can be downloaded for free. Jeff Garzik is a Bitcoin core developer and open source evangelist at BitPay Inc.
