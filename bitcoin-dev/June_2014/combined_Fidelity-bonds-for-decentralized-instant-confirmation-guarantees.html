Summary:

The email conversation from 2014 discussed how to establish trust in the identity associated with an ECC key pair. Peter Todd suggested fidelity bonds or "trusted identities" as a solution, where a user could publicly throw away resources, such as transferring coins to a null address or creating transactions with large and equal transaction fees, to prove that they value their identity and have given up a significant amount of value to establish trust. This creates a distributed green address system where others can verify the trust purchase and accept zero-confirmation transactions at face value. If the user breaks their promise, others can publish their signed transaction to prove them as fraudulent and destroy the value needed to create trust.There are mechanisms in which a digital signature can reveal the private key and allow someone to steal coins, but they are not very safe to use due to the risk of theft from accidentally loading the same wallet on two machines or the wallet software crashing after signing and sending a transaction. A proposed solution was to sign a transaction that would pay a certain amount of bitcoins to an arbitrary address if someone presented proof of more than one transaction using the same output, which would serve as proof of double spending. This feature was previously used in Digicash by Chaum, but it did not automate punishment.
