Summary:

Luke Dashjr has proposed a new opcode, OP_CHECKBLOCKATHEIGHT, for the Bitcoin scripting system to deal with reissuing bitcoin transactions when the coins they spend have been double-spent or conflicted. He has shared the details of the proposal on GitHub and is seeking feedback on whether this is a good idea or not. Rusty, a developer, prefers a three-arg version of the proposal which allows playing gambling games on-chain but also raises concerns about it putting another nail in CBAH's coffin.
