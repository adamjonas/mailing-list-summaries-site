Summary:

Pavol Rusnak, the author of BIP44, suggested creating a new BIP number to manage GPG identity for encryption and signing with zero bytes of permanent storage instead of using BIP44. He proposed creating a GPG key hierarchy per device/master seed rather than per Bitcoin account. Pavol also shared his work on implementing a SignIdentity message for TREZOR to be used for HTTPS/SSH logins. His idea was to derive the BIP32 path from the URI and use it to derive the private key. The scheme might also work for GPG keys by using gpg://user@host.com for the URI.In an email thread, Pavol Rusnak discussed his implementation of a SignIdentity message for TREZOR which would be used for HTTPS/SSH logins. Another member mentioned FIDO's U2F protocol as well as Yubico's FIDO U2F security key to generate a unique keypair for each service for privacy purposes. A demonstration of using a bip44 Wallet to generate deterministic GPG identities has been implemented in javascript and is available on http://memwallet.info/bip44ext/test.html. This allows users to manage a GPG identity for encryption and signing with zero bytes of permanent storage, such as on Tails, for example.
