Summary:

The proposal suggests using the "Darknet" technique from Freenet to enhance privacy in Bitcoin by allowing nodes to establish authenticated connections with trusted peers via CurveCP. Transactions would only be broadcasted to these trusted peers, reducing privacy for those who are unaware of their peers' trustworthiness. To create a secure connection, both sides of the connection would need to know each other's long-term public key, which can be generated through an API command and copied to the ~/.bitcoin/curvecp directory with a .ref extension. This proposal can work independently of Tor usage, but separating transaction submission from normal node functionality would go a long way towards enhancing privacy. The proposal also suggests that CurveCP connections can be created between full nodes and SPV nodes, routing transactions originating from the SPV peers as if they originated from the full node. While this proposal may not necessarily help with privacy, it could be used for whitelisting to subject trusted nodes to fewer DoS constraints.
