Summary:

On March 11, 2015, Thomas Kerin proposed a new standard for multisignature P2SH addresses given 'm' and a set of public keys. He created a PR on bitcoin/bips and used BIP0090 as a placeholder but requested a BIP number for this purpose. The proposal can be found on GitHub at https://github.com/bitcoin/bips/pull/146. This method is currently used by CryptoCorp for their external signer service. In an email thread from the same day, Thomas Kerin requested a BIP number for his proposal. The email chain also discussed the issue of individuals squatting on BIP numbers and demanding access to them. As a result, it was suggested that proposals should not be referred to as "BIP[nn]" until they are actually a BIP. Instead, placeholder names can be used during the drafting process, and discussion must come before number assignment.The writer suggests the creation of a 'Bitcoin-Draft' (BD) status with an auto-expiry period similar to the Internet Engineering Task Force's (IETF) concept of 'Internet Drafts' (ID). However, the suggestion is met with caution due to past issues with people squatting on prior draft proposals and demanding access to the same numbers. Proposals should not be referred to as "BIP[nn]" until they are actually a BIP. Bitcoinj is a Java implementation of the Bitcoin protocol, which uses a convention where units of Bitcoin are represented in code as "satoshi", named after the pseudonymous creator of Bitcoin, Satoshi Nakamoto. This convention is also used by other Bitcoin implementations, including Bitcoin Core.In another email exchange on March 11, 2015, Pindar Wong suggested the introduction of a 'Bitcoin-Draft' (BD) status with an auto-expiry period similar to the IETF's "Internet Drafts" (ID), which have been successful thus far. However, another participant noted that posting to the list serves a similar purpose and creating a draft document requires no approval beyond filling out the appropriate form. The email conversation between Gregory Maxwell and Pindar Wong suggests adding a statement to indicate that after copy-editing and acceptance, a BIP number is assigned and it will be published on the Bitcoin Wiki page. Wong suggests introducing a 'Bitcoin-Draft' status with an auto-expiry period similar to the 'Internet Drafts' concept of IETF. Maxwell explains that posting to the list is already considered a draft document and requires no approval beyond filling out the right form. However, he agrees that calling it a distinct step might be better.
