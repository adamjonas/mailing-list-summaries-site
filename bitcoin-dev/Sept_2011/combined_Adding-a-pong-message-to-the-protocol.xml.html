Summary:

The focus should be on addressing the root issues of the Bitcoin network, according to a conversation between Mike Hearn and his correspondent in 2011. They suggested that nodes should determine their own suitability before becoming advertised listeners, and simulations should be set up to validate proposed changes. The speaker also suggests two solutions to address the issue of slow speeds due to link saturation: making it easy to determine latency and using that data to make better connection decisions. Adding a "pong" response to the existing "ping" message would allow useful stats about the node to be collected and let nodes be ordered by responsiveness. However, care must be taken to avoid sloshing load around too much when making nodes prefer responsive servers. A patch for this issue is being considered, and input on the proposed solution is being sought.
