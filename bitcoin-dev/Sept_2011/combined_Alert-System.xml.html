Summary:

In September 2011, a discussion thread raised the question of whether anyone besides Satoshi had the ability to send an alert regarding Bitcoin. Gavin Andresen confirmed that he did have this ability. David Perry suggested changing the alert system to give this privilege to current developers in case Satoshi's key was compromised and a malicious alert was sent. However, another user argued against this, stating that it would be unnecessary for the key-holder to send two alerts and expressing doubt that Satoshi would allow his key to be compromised. A similar discussion occurred on Bitcoin StackExchange around concerns of potential abuse of the alert protocol. David Perry forked bitcoin/bitcoin, removed questionable code from main.cpp and submitted a pull request. Gavin Andresen noted that it merited discussion and consensus before acting on the pull request. He argued that the feature was still useful rather than dangerous. The question posed was whether the alert system is valuable, an unnecessary risk, or just deprecated code. The alert system requires signature verification when receiving alerts, making it not a DoS target. While some may disagree with developers sending messages to everyone's computer, the alert system could prove valuable in financial software like Bitcoin. Urgent problems like the overflow bug require a way to notify people to upgrade immediately, which is what alerts provide. Since alerts no longer have the ability to put Bitcoin into RPC safe-mode, they are just a message, and there is no reason to remove them.In the discussions, it was mentioned that Satoshi had transferred his key to Gavin before leaving. Pieter agreed with David's suggestion and added that the alert system could also be used to ask people to upgrade to recent versions of the client when necessary. There may come a moment when the alert system needs to be used for that purpose. Ultimately, the conversations concluded with the reminder that these alerts are just messages and do not actually affect the functionality of Bitcoin.
