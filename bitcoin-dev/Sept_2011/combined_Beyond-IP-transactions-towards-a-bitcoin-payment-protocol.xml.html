Summary:

Pieter Wuille proposed an idea to move away from static bitcoin addresses as descriptions for transactions. He shared his idea in a Github gist and hoped it would start some discussion. He believes that current addresses are a reference to a public key and work well only if all other negotiation about the payment occurs out-of-band already. What he wants to accomplish is to standardize part of that out-of-band communication inside a protocol. By doing so, a lot of possibilities open up. He suggests adding signatures to authenticate the whole process and make sure the client is communicating with who he thinks he is.
