Summary:

Forrest Voight has submitted a pull request on Github that replaces the "getwork" RPC call for clients that generate their own transactions with his patch. Originally, the patch returned Bitcoin's memory pool as a list of transaction hashes and the total fees, but it evolved into a full alternative to "getwork." The diff in the pull request shows all the changes made to the code. Instead of producing its own merkle tree, the call now returns all the necessary information to create a generation transaction and the complete contents of other transactions that should be included.The getmemorypool [data] command is used to build a block to work on. If [data] is not specified, it returns data such as the block version, hash of the current highest block, non-coinbase transaction contents, maximum allowable input to coinbase transaction, timestamp appropriate for the next block, and the compressed target of the next block. If [data] is provided, it attempts to solve the block and returns true if successful.The new pull request was tested with p2pool and resulted in the first p2pool generated block with transactions. The post contains a link to the blockexplorer showing the results.
