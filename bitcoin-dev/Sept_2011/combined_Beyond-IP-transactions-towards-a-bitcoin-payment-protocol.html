Summary:

Pieter Wuille, a member of the Bitcoin community, has proposed a new idea for transactions that would move away from static Bitcoin addresses. He hopes to spark discussion and potentially lead to the development of a BIP/BEPS/BFC.However, some people have criticized this idea as being too complicated for addressing problems that don't exist. The main issue is that the scheme assumes there is no out-of-band channel, which is not true for most cases. Pieter clarifies that his scheme is not meant to solve anonymous donations, which work well with static addresses.Instead, he wants to standardize part of the out-of-band communication inside a protocol. By doing so, he aims to add support for anything that is possible through bitcoin scripts and use signatures to authenticate the whole process.In a discussion between Luke-Jr and Pieter Wuille, Pieter suggests standardizing out-of-band communication in a protocol. While Luke-Jr suggests using email or XMPP, Pieter disagrees and argues for using an HTTP connection instead.The discussion also highlights the need for bidirectional communication when generating freshly negotiated keys each time, and how deterministic keypairs remove the necessity of bidirectional communication.Overall, Pieter believes that addresses are a limited method for defining payments, and as soon as you move to a protocol instead of a static template, a lot of possibilities open up. Though, there is still uncertainty surrounding these protocols, so he has posted his proposal in a Gist for others to comment on.
