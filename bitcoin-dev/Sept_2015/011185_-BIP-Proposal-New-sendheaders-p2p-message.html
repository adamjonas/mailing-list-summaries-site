Summary:

On September 24, 2015, Suhas Daftuar proposed a new optional peer-to-peer message to improve the way blocks are announced on the network. This p2p message would allow nodes to announce blocks with headers messages instead of inv's, which would eliminate a round trip in network communication and facilitate more efficient propagation of reorgs. The draft BIP is available on GitHub, and there appears to be no requirement for the new message as new nodes could unilaterally switch to sending headers. The only DOS misbehaving penalty is if the header is invalid or if the headers don't form a chain.
