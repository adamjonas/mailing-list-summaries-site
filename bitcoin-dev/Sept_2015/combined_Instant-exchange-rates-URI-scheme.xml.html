Summary:

A proposal for a Bitcoin Improvement Proposal (BIP) has been made to create a standard URI scheme that would allow wallet operators to implement instant exchange or pegging to other currencies, cryptocurrencies or asset classes. The idea is to include parameters in the wallet address that would supplement the presentation of a proposed transaction. This would enable wallet operators to communicate with each other and abstract bitcoin rates from end users. Wallet operators would present a wallet address with encoded parameters such as currency, rate, and timestamp of when the rate loses validity. This would allow the sending wallet to present up-to-date exchange rates. When a wallet operator that pegs to the USD scans the address, it would be able to present the user with information regarding the USD to XAU rate, how much XAU will be received by the address, and how long before the rates expire. The intermediate currency used in this payment rails method is BTC. The proposal suggests that the sending wallet specify the amount of Bitcoin they wish to be paid rather than adding more things to consider at checkout time which confuses things.
