A user is trying to compile Bitcoin Core on their Ubuntu Linux machine using a custom lib directory, but they are encountering a build error with undefined references to Boost libs. The error message indicates that the Boost filesystem and program_options libraries are not found. They seek help to fix the build problem so that they can run the Bitcoin-core node on their machine. Sriram Karra, a member of the Bitcoin-dev mailing list, responds by pointing out that the Boost libs cannot be found and asks why the user is attempting to build with a custom lib directory. The conversation suggests that the user may have been using the wrong build option, which should have been "--with-gui=no."In this email thread, LinuXperia follows standard procedures by running "./autogen.sh" and "./configure", but they encounter a build error. They share their configuration command, which includes specifying CPPFLAGS and LDFLAGS for the custom lib directory. The error message shows that there are undefined references to various Boost functions. The email thread ends with links to the Bitcoin-dev mailing list and instructions to sign up for it.
