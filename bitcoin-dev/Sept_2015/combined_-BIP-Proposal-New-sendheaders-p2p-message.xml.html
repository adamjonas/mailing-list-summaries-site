Summary:

A proposal has been made to add a new optional p2p message for blocks announcement in the network. The aim of this p2p message is to make nodes able to announce blocks with headers messages rather than inv's, which will benefit efficient propagation of reorgs as it eliminates a round trip in network communication. Draft BIP text can be found here. This new p2p message is optional and nodes can ignore and continue to use inv messages to announce new blocks. The implementation of this BIP is available on Github. It includes code to directly fetch blocks based on announced headers. Old clients remain fully compatible after this change. The sendheaders message is defined as an empty message where pchCommand == "sendheaders". Upon receipt of a sendheaders message, the node will be permitted but not required to announce new blocks by sending the header of the new block (along with any other blocks that a node believes a peer might need in order for the block to connect). Feature discovery is enabled by checking protocol version >= 70012.
