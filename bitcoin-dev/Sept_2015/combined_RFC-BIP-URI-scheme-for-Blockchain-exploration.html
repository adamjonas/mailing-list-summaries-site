Summary:

A proposed new URI scheme for blockchain exploration has been submitted by Marco Pontello. The new scheme will allow users to handle requests for details about blocks, transactions, and addresses on a blockchain explorer with their preferred tool. Currently, Bitcoin clients point to arbitrary blockchain explorers when users search for transaction details. The proposed URI scheme aims to provide users with the convenience of always using their preferred explorer. Blockchain developers will simply offer to handle the blockchain: URI, and users will benefit from the ability to use their preferred explorer. Discussions revolve around the syntax of blockchain URLs, and there are suggestions to create a standard URI scheme that all explorers would understand, making blockchain exploration easier and more accessible for everyone.Suggestions have been made to address altcoins and genesis blocks, as well as to specify little endian searching and what the return results should look like. There were also discussions around standardizing HTTP queries for blockchain explorers. Some suggest that the authority component of a URL should identify the chain, while the path component should identify the particular block, transaction or address in that chain. Others proposed the idea of allowing well-known chains to register a name to be used as an alternative to the literal hash syntax.The email conversation discussed the structure of blockchain Uniform Resource Identifiers (URIs) and how to identify chains and specific blocks, transactions, or addresses within them. One participant suggested that the chain part should be optional since most references will be to Bitcoin's main net. Another participant argued that the "authority" component of the URL should identify the chain, while the "path" component should identify the specific block, transaction, or address in that chain.Matt Whitlock proposed a new format for blockchain URIs that would include the chain ID as the first element after the blockchain scheme, followed by the type of reference being made (e.g., tx for transaction), and finally the reference itself (e.g., the hash of the transaction). Danny Thorpe suggested using the network magic bytes to distinguish between mainnet and testnet instead of an extremely long hex string from the genesis hash. Jorge Timón expressed his preference for using "chainID" over "network" as a way to reference a transaction in the blockchain. He suggests using the hash of the genesis block as a rigorous means of referencing a transaction, with the option of using name constants as a simpler and more relaxed alternative. Marco Pontello agrees with the proposal but suggests making the chain part optional since most references will be to the Bitcoin main net. He also proposes letting users use name constants as a more relaxed alternative to using the genesis block hash.The proposed solution to the issue of identical genesis block hashes amongst altcoins is to move the genesis block to the next block or the first one that's not identical. The conversation ends with a mention of an online file identifier tool. Despite concerns over central authority to register unique and memorable chain name strings, this proposal could provide a standard URI scheme that all explorers would understand, making blockchain exploration easier and more accessible for everyone.
