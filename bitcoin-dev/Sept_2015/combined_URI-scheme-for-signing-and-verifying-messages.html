Summary:

Bitcoin developer Luke Dashjr expressed concern about the current signed message system and its insecure uses. He suggested a new method should be developed to avoid using the same key as for signing transactions and designing an entire authentication system. As addresses are losing their importance due to the payment protocol, it may be necessary to design an entire authentication system, which could be complex. Luke Dashjr believes that most of the common uses of the current signed messages are insecure and not working properly, especially in proving ownership or sending bitcoins. The signed message only proves that the person who received payment with the address agrees to a given message/contract, not that they presently still have the bitcoins received.Arthur Bouquet proposed a URI scheme that would enable users to access existing tools more easily, which could make signature verification easier and more accessible within the Bitcoin community. The proposed URI scheme could become available in bitcoin clients that support message signing/verifying and payment URLs or QR codes. Bouquet plans to post a topic in the "Development & Technical Discussion" section on Bitcointalk to discuss his idea further.The proposed URI scheme would require manual user approval similar to BIP0021 and could use the same URI scheme with an additional parameter or utilize a different scheme like BIP121. Another user in the thread expressed interest in a similar proposal for multi-signature wallets. Overall, Bouquet's suggestion aims to make signature verification easier and more accessible within the Bitcoin community.To gain proper consensus, Bouquet suggested going through a BIP and presented his idea publicly before proceeding with a draft BIP and reference implementation. In September 2015, Luke Dashjr suggested that the whole process of signing messages needs to be rethought since it is commonly used in insecure cases where it doesn't actually work.
