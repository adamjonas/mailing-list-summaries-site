Summary:

The Bitcoin protocol has recently introduced a new "pong" message that echoes back a 64-bit nonce contained in the ping if the protocol version is new enough. The purpose of this is to make it easier for clients, particularly mobile clients, to quickly determine whether a connection is stale or if a remote node is overloaded. Users can avoid talking to nodes that are busy with other tasks such as downloading the blockchain. This feature has been added to improve connectivity of clients and help users operate on the network. However, no concerns or objections about this update have been specified.TCP keep-alives are not always implemented reliably, leading to difficulties in keeping connections to the network, especially on mobile devices when roaming into areas with poor connectivity. Although the network stack does not entirely drop access, it is important to quickly check if the connection is really there with some kind of bounded, app layer deadline. A bounded, app layer deadline for checking connections would be useful as it is cheap and would help in cases where the network stack does not entirely drop access. The goal of this feature is to allow clients to check if a connection is stale with ease and avoid talking to an overloaded remote node. The implementation of this feature is aimed at improving the connectivity of clients, particularly mobile clients, to check if a connection is stale or if a remote node is overloaded.
