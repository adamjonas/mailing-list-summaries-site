Summary:

Eric Lombrozo, a Bitcoin developer, has been struggling to obtain real-time event notifications using two approaches. The first approach involves writing his own library and the second approach involves making custom builds of the Satoshi client with callback hooks. However, these methods involve issues such as code duplication and patching the satoshi client source. During a discussion on March 22, 2012, Eric Lombrozo proposed adding callback hooks to the code to help alleviate these issues. He also offered to assist in reorganizing the code and contributing source code if the proposal was accepted. Lombrozo believed that these callback hooks would benefit many developers and remove the need for workarounds.Lombrozo proposed implementing a callback architecture to handle events related to Bitcoin transactions without accessing wallet space. Additionally, he was interested in obtaining detailed information about message format errors that occur when nodes connect to bitcoind for debugging purposes. He proposed configuring callbacks to write an error log and inform connecting nodes about what went wrong.However, concerns were raised about allowing other code into the main wallet-handling process/memory space. As an alternative, it was suggested that a Wallet Protocol be designed for general use between the wallet and GUIs/applications. A draft for this protocol can be found at https://en.bitcoin.it/wiki/Wallet_protocol, but it was noted that it may be overcomplicated.
