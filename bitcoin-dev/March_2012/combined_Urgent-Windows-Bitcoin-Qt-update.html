Summary:

A critical security vulnerability has been identified in the Windows version of Bitcoin-Qt, affecting versions 0.5 through 0.6. Users are advised to shut down and upgrade immediately to either version 0.5.3.1 or 0.6rc4. The command-line bitcoin daemon (bitcoind), Mac and Linux versions of Bitcoin-Qt, and versions prior to 0.5 are not affected by this vulnerability. Although it is believed that an attacker would find it difficult to do anything more than crash the Bitcoin-Qt process due to the nature of the vulnerability, there is still a chance that a remote code execution could occur from such a crash. Binaries of the updated versions can be downloaded from SourceForge. Users with queries can visit the #bitcoin-dev channel on FreeNode IRC. The message is signed off by Gavin Andresen, Gregory Maxwell, Matt Corallo, Nils Schneider, and Wladimir J. van der Laan.
