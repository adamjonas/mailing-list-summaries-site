Summary:

There has been a proposal for fraud proofs in Bitcoin and how light clients can detect blockchains that are invalid due to excess size and/or weight. The proposal aims to protect Bitcoin holders from attacks, particularly oversized blocks that aim to force a hardfork. The draft BIP uses the observation that all you need to show the length and hash of a transaction is the final SHA256 midstate and chunk (max 64 bytes). The only issue is that since parts of the merkle tree are repeated, you need to ensure that the counting for the minimum number of transactions is accurate. This draft is probably ready for implementation but any improvement suggestions are welcome.
