Summary:

Pieter Wuille has proposed a new BIP (Bitcoin Improvement Proposal) for native SegWit addresses. These addresses are not mandatory but offer more efficiency, flexibility, and better user experience. The format is base 32 with a simple checksum algorithm that has strong error detection properties. The proposal has been submitted as BIP173 and reference code in several languages is available on GitHub. Pieter aims to address all the areas where humans interact with addresses, including voice communication. The proposal has received contributions from various people, including new reference implementations in Haskell and Rust, with C++ and Go ones underway.
