Summary:

In a thread of emails on the bitcoin-dev mailing list, Jorge Timón proposed using "chain=" instead of "network=" in blockchain URLs. He explained that by chainID he meant the hash of the genesis block and provided a link to his GitHub commit for more information. Timón suggested using an optional parameter to maintain backwards compatibility and expressed agreement with Andreas Schildbach's opinion that respecting commonly used schemes is desirable.In addition, Timón provided two examples of URL formats: one for a transaction in testnet and one for a block in bitcoin's mainnet. He also noted that some altcoins, such as LTC and FTC, have the same genesis block hash. Matt Whitlock had previously pointed out that "mainnet" and "testnet" are not host names and offered an alternative format for the blockchain URL using "network=".
