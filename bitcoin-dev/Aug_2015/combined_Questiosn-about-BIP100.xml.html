Summary:

A group of developers is discussing the implementation of BIP 100, a proposed Bitcoin Improvement Protocol for block size voting. The proposal suggests allowing miners to vote on the maximum block size they will accept, with the size increasing if 70% or more of the blocks mined in the previous two weeks have voted for a larger size. Jeff Garzik, one of the developers, suggests that the median is the most natural and logical option for determining the block size, as it would eliminate the incentive to 51% attack. He also believes that if something other than the median is used, it should be symmetrical. In an email thread, the author of one message asks whether any clients or codes currently implement BIP 100 and how it will be deployed. Jeff Garzik responds, stating that he is working on a technical draft and implementation for ScalingBitcoin.org, but only the PDF is publicly available at the moment. He confirms that the initial deployment will be in the same manner as size votes. Another developer, Andrew C, asks about the "most common floor," and Garzik responds that it could be interpreted as the 20th percentile, but there is some argument for taking the mode of several tranches. Overall, the email conversation provides insights into the ongoing block size debate in the Bitcoin community and the development of BIP 100.
