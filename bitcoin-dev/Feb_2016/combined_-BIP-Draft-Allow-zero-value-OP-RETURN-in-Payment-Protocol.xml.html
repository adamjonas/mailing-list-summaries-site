Summary:

The bitcoin-dev mailing list has been a platform for discussions on the use of coins in the Bitcoin network, support for OP_RETURN, and the importance of personal opinions in the acceptance of BIPs. Some members argue for full support of OP_RETURN and the need to support keyless environments, while others caution against the potential for spam and the detrimental effects of changing the current behavior. However, the criteria for acceptance should be met regardless of personal opinions, and wallets do not have to implement a BIP even if it is accepted.One point of contention is the uneven application of moderation on the mailing list. Some members claim that their posts took hours to go through while others were able to quickly post multiple heated arguments against them. Peter Todd explains that actual humans approve each post and that this is necessary to manage the large volume of traffic on the list.There is also debate over the use of OP_RETURN in Bitcoin transactions and whether burning coins or supporting zero value outputs is better for preventing spam. A proposal to modify BIP70 has been put forward that would allow for the use of OP_RETURN metadata in a keyless environment. The proposed change to BIP70 would see outputs with zero value being accepted by the wallet, and those with greater than zero rejected. This contrasts with current requirements under BIP70 where all zero value outputs are ignored. However, Luke Dashjr advises against implementing the proposal, stating his concern that it would encourage spam and make users unwilling (and possibly unaware) participants in it.Another BIP proposal suggests allowing merchants to add plain text invoice hashes to checkout transactions using an OP_RETURN. Prior to this proposal, transactions using OP_RETURN with zero value needed to be constructed and executed in the same software. The proposal aims to separate concerns and allow merchant software to create transactions with OP_RETURN metadata on a server without storing public or private Bitcoin keys. However, some members express concerns over spam transactions and the potential negative impact on Bitcoin.Overall, the discussions highlight the importance of considering all perspectives and potential outcomes before making changes to Bitcoin protocols and features. While personal opinions are important, the criteria for acceptance should still be met regardless of these opinions. Additionally, wallets do not have to implement a BIP even if it is accepted.
