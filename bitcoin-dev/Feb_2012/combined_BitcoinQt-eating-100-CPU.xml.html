Summary:

The email is a follow-up on an issue with Bitcoin-Qt, which was using 100% CPU after initial block chain download. The sender, Michael Grønager, built the latest program source using qt-creator on MacOSX 10.7.3, but it was still consuming CPU despite normal activity in debug.log. Sampling showed that most of the time in each thread was used for operations like __semwait_signal and kevent. The sender identified line 31 in qtipcserver.cpp as the culprit, which consumed CPU. After inserting `ipcShutdown()` into the ipcThread, the problem was fixed, and Bitcoin-Qt now runs at around 0.9% CPU. Gavin Andresen suggested a possible solution for the Mac version of Bitcoin-Qt that does not register itself as a bitcoin URL handler by not launching the ipcThread in the 0.6 release. However, it was unclear whether this would prevent users from registering it. The writer asked if anyone had experienced a similar issue.
