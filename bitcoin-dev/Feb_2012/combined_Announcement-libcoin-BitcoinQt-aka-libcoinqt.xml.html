Summary:

Michael has successfully ported BitcoinQt to use libcoin as its basic crypto library. The project is available on GitHub and is based on the latest 0.6 branch of BitcoinQt. However, there are a few things that have not yet been ported, including updating the splash text while loading, finding replacements for some of the globals (details in optionmodel), UPnP, and proxy support. To build libcoinqt, users must first build libcoin and then use qmake or Qt Creator. Although Michael has only tested the port on MacOSX, more tests and features will be added in the upcoming week.
