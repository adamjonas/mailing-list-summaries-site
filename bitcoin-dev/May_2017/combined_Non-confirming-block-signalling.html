Summary:

In May 2017, Tomas proposed a method to mark blocks that were generated without verifying the previous block by using a bit of the version field. Bryan Bishop responded to the proposal, acknowledging that it had been previously discussed in a Bitcoin-dev mailing list post from December 2015, which he provided a link to. The technique will help counteract the reduction of security caused by SPV-mining. The BIP for this proposal can be found on Tomas's GitHub page. In addition, contact information for Bryan at heybryan.org and his phone number was also included in the email exchange.
