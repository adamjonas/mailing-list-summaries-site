Summary:

Developer James Hilliard has proposed a method to activate segregated witness (segwit) at an 80% threshold and signal at bit 4. The aim is to minimize chain split risk and network disruption while maximizing backwards compatibility. The proposal specifies a way to activate the existing BIP9 segwit deployment with a majority hashpower less than 95%.Hilliard notes that segwit increases the blocksize, fixes transaction malleability, and makes scripting easier to upgrade, among other benefits. The proposed implementation is compatible with a "UASF movement" and turns all supporting miners into equivalent BIP148 enforcers. To avoid any subversion of a UASF, the proposed "agreement" would require a hard fork be locked in on the same bit (bit 4). Ideally, such a 2MB increase should be scheduled using BIP103-esque logic: gradually increasing from 1MB to 2MB over the course of at least a couple years, beginning 6 months from lock-in.All blocks must set the nVersion header top 3 bits to 001 together with bit field (1) while this BIP is active. This deployment is compatible with the existing "segwit" bit 1 deployment scheduled between midnight November 15th, 2016 and midnight November 15th, 2017. Users should either upgrade to segsignal or wait for additional confirmations when accepting payments.The proposal specifies a method to activate the existing BIP9 segwit deployment with a majority hashpower less than 95%. The proposal triggers a mandatory signaling period that will activate SegWit on existing nodes. However, the proposal is more risky than usual as only 31% of hashrate is required to false-signal the activation to fork-off honest miners. By orphaning non-signaling blocks during the BIP9 bit 1 "segwit" deployment, this BIP can cause the existing "segwit" deployment to activate without needing to release a new deployment.The proposal has received feedback from various participants, including Wang Chun and Andrew Chow, who raised concerns about the proposal's compatibility with the current SegWit implementation with regards to the NODE_WITNESS service bit. However, James Hilliard clarified that the proposal is compatible with the current SegWit implementation because it triggers a mandatory signaling period that will activate SegWit on existing nodes. Overall, the discussion provides insights into the technicalities of activating SegWit and the different perspectives that exist regarding the process.
