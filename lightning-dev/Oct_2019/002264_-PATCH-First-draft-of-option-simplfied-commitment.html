Summary:

The proposal entails adding `to_remote_delay OP_CHECKSEQUENCEVERIFY OP_DROP` to the `to_remote` output, unifying the `to_self_delay` values and introducing new addresses for `to_remote_anchor` and `to_local_anchor`. The anchor outputs need equal values and are paid for by the initiator. The value of the anchors is the dust limit negotiated in the funding flow. There are design constraints in the original watchtowers, which can lead to loss of privacy if keys are not rotated. Removing key rotation from `to-remote` output makes it easier to let those anchors go straight into the wallet, which may mitigate the dust utxo problem a bit.
