Summary:

Flare is a routing protocol designed for the Lightning Network, aimed at improving routing efficiency. It combines local neighborhood maps and paths to remote beacons to achieve this. A team of developers tested Flare on 2500 AWS nodes and discovered that it worked well, with an 80% chance of finding a route between any two nodes in about 500ms.The developers simplified the neighbor discovery/beacon selection process and the route selection process compared to the original paper. However, they warn that their implementation assumes an undirected graph and one-way channels, which may make finding routes difficult during bootstrapping. The authors of the modified version of Flare note that they envisioned a DHT acting as a substitute for a pure broadcast network to store information such as onion keys and identity keys. They also mention using something akin to HORNET for nodes to communicate with each other without necessarily knowing IP addresses or selected beacons. In conclusion, the developers of the modified version of Flare state that it works on a decent-sized network.
