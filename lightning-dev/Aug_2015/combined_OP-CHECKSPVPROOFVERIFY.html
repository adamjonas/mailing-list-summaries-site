Summary:

Bitcoin developer Eric Lombrozo has proposed a new idea to incentivize nodes to serve Simplified Payment Verification (SPV) clients. The proposal involves outsourcing SPV proofs from one node to another, creating a more concrete and quantifiable incentive system. The use of OP_CHECKSPVPROOFVERIFY in the Lightning Network was discussed to allow outsourced SPV proofs over LN but avoiding its use on the blockchain itself.To implement this proposal, Eric provided a script for the ProverSig + VerifierRevocation || ProverSig + SPV proof || VerifierSig + timeout output. Commitment transactions are created between two parties, Prover and Verifier, who then need to negotiate how they split the risk since sending the proof on the blockchain is expensive. After creating the commitment transactions, they would exchange revocations and negotiate settlement transactions.However, Rusty Russell suggested that a merkle tree calculation could be open-coded using OP_CAT and OP_HASH256 instead of a new OP. Eric Lombrozo was not entirely clear on how to use OP_CAT. It should also be noted that using P2SH has its limitations due to the disabled OP_CAT and the transaction length limit.
