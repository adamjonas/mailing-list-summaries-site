Summary:

The conversation revolves around the idea of making an exchange atomic in a commercial application. The advantage of the idea is that it prevents a malicious credential server from stealing money selectively while being honest with others. In this scheme, payment for a credential is anonymous and cannot be distinguished from other payments. A trustful exchange with LN eliminates the need for distinguishing users, and the server can choose one private channel to be dishonest with.
