Summary:

The Lightning Network protocol is a payment channel system that includes features such as exchanging version bits and using encryption protocols to ensure secure communication. One of the optimizations in the protocol is reserving before locking in payment channels to reduce the risk of funds being locked in a part of the route that no longer exists. The use of shachain and elkrem for generating revocation secrets has been discussed, with Rusty Russell proposing the use of binary tree method instead of shachain.There have been proposals for changes to the inter-node protocol, including the use of open-coded messaging and length prefix for initial key exchange and other packets. The Amiko Pay protocol is a decentralized payment network that uses TCP as its transport layer and JSON with added conventions for serialization. It also utilizes bi-directional routing, multiple channels per link, different channel classes for micro-transactions, and reserving before locking for optimization.Russell is concerned about the potential centralization pressure caused by higher latency and encourages people to sacrifice privacy. In an email exchange with the Chief Justice of the Supreme Court of Canada (CJP), they discussed the differences between their respective Lightning implementations. CJP acknowledged that C-Lightning's protocol description would be a good guide for developing a true Lightning style channel in Amiko Pay.Russell is currently reworking the update pipelining in Amiko Pay and has pushed his unworking code to the multi-htlc branch on Github. He hopes to build on his experience and expertise to make things faster in Amiko Pay, despite going on paternity leave for a few weeks.Overall, the Lightning Network protocol is being developed further to allow for new concepts and alternative designs. It is suggested that certain aspects be de-coupled and each subject have its sub-protocol with a version number/protocol identifier. The subjects with ongoing design or alternative concepts include micro-transaction channel design, commit conditions, and routing.
