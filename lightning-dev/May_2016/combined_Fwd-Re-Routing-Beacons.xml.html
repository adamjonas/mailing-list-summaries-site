Summary:

The Lightning Network's routing table update process, which involves landmark nodes periodically sending messages to their neighbors, is the topic of discussion in an email exchange between Rusty Russell and LaurentMT. The temporal framing technique used by landmark nodes is new to Rusty, but it enables every node to know the best route to/from a given landmark. This information is useful for onion routing when a node needs to send a payment. However, the "base+per-satoshi" fee model presents challenges, and the participants suggest nominal values and forwarding a set of best routes as a solution. They also discuss different fee models and suggest that fees should be associated with individual payment channels rather than nodes. Additionally, simulations could offer insight into routing table updates in the Lightning Network.
