Summary:

John Carvalho, CEO of Synonym.to, believes that the Lightning Network can scale tokens on Bitcoin if it can scale Bitcoin. He argues that supporting "tokens on Lightning" should be a separate concern from Taro and that Lightning Labs should create a separate BOLT proposal to allow for fair competition in the market and maximum interoperability between different token solutions. Laolu suggests creating a bLIP that fully specifies the optional series of TLV extensions needed to open channels using Taro assets and send them off-chain. This approach avoids the need to advertise exchange rates over the existing gossip protocol. In a recent email exchange between Lightning Network developers, it was revealed that there is a difference in vision between the parties involved. Instead of updating the core network to accommodate Taro assets, the plan is to leave the protocol unchanged and add new TLV extensions so that edges can be aware of and interact with Taro assets. Additionally, ZmnSCPxj discusses two topics: Advertising Exchange Rates and Routing Between Channels Of Different Asset Types, suggesting dropping the 'realm' byte and instead adding a new type for TLV which would have 20 bytes.
