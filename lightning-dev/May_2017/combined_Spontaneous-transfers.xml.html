Summary:

Bitcoin developers are discussing ways to expand the use cases of Lightning Network, a protocol for off-chain bitcoin payments. They want to find ways to facilitate donations without prior contact between the sender and recipient, creating recurring requirements such as the need for the payment preimage and signaling the sender's identity without out-of-band communication. Proposed solutions include creating a circular route or adding new information into the routing info. Another suggestion is to create a gambling service where the payout of the winning amount is the second half of the route. Developers have also anticipated a major problem in the implementation of the network, specifically probing attacks, and have devised a solution using XORing to make it harder to probe and turn the lightning node into a gambling casino. A prefix lookup in the routing table can be used to derive the payment preimage. The developers have discussed implementing the LightningDice service which would enable probabilistic payments via the LD server.
