Summary:

The context mainly revolves around discussions related to Simplified Payment Verification (SPV) and Lightning Network (LN). Eric Lombrozo and Joseph Poon discuss the possibility of outsourcing SPV proofs over LN. Lombrozo suggested using an OP_CHECKSPVPROOFVERIFY opcode to create a contract requesting payment from the SPV client, but clarified that it was not intended for providing better security than current SPV proofs, but rather enabling outsourceability. The article discusses how to incentivize nodes to construct SPV proofs and introduces scripts for Prover and Verifier parties to create commitment transactions and negotiate risk-sharing. Eric Lombrozo suggests using the OP_CHECKSPVPROOFVERIFY OP_DROP script command to incentivize nodes to construct SPV proofs. However, Rusty Russell notes that OP_CAT is disabled, and the txlen limit may pose a problem. Additionally, there are discussions about the usage of OP_CAT and how it can be used to open-code a merkle tree calculation along with OP_HASH256. Lombrozo raises concerns over incentivizing nodes to construct SPV proofs and notes that the issue can be partially solved through P2SH, but there are still obstacles.
