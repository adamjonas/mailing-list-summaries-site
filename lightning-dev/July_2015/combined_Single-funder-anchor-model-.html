Summary:

The Lightning Network is a proposed solution to the scalability problem of Bitcoin, and Rusty Russell has been discussing the possible implementation of a single-funder anchor as part of this network. Go1111111 responded with a post on Bitcointalk pointing out that an atomic swap tx can hit the Bitcoin blockchain around the same time as the anchor tx, so it does not necessarily take twice the time to get a full two-way channel. The asymmetric risk of dual-anchor proposal is discussed, but the non-cooperation risks in this model are not considered to be a big deal since the channel would be closed out anyway if they're not cooperative. There are likely some weird asymmetric economic incentive things going on, but if both channels are established with the same person, it should be fairly clean. Joseph Poon suggests using OP_CLTV instead of BIP62, but Rusty Russell is happy at this stage assuming BIP62.
