Summary:

A draft of Rusty Russell's latest project, ln-deploy, has been shared on ozlabs.org for review and feedback. The document provides instructions on how to deploy Lightning Network (LN) nodes using Rusty's ln-node software, which simplifies the process of deploying LN nodes. The LN is a layer-two scaling solution that allows for instant and low-cost transactions by creating payment channels between users. BIP62, an outdated Bitcoin Improvement Proposal, is not abandoned but requires time to implement significant changes and ensure their safety. Rules 2-7 in new blocks are debated as to whether they are necessary to close vulnerabilities or nonstandard. Mark Friedenbach explains that there is no deployment timeline for BIP62 due to a soft-fork logjam. However, he clarifies that the Bitcoin development team can push out more than one soft-fork upgrade signaling at a time. Rusty Russell also posted a draft proposal regarding ln-deploy for feedback from the Lightning-dev mailing list.
