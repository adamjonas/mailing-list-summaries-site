Summary:

The conversation is about the feasibility of cheating in a channel update by Bob, and how Alice can try stealing most of the channel's funds. The discussion revolves around the use of p2sh and HTLCs, where it is concluded that with p2sh, retrieving the hash values and timeouts is necessary to spend the output. However, since OP_RETURN is length-limited to 80 bytes, only two R hash values could be fitted in it. The solution is to have multiple OP_RETURN outputs, which would increase the transaction size by about 80%. In this case, the R values need to be remembered. If the number of updates per second is 100, and lasts for a month, that would be 259M updates, requiring 5GB of storage.
