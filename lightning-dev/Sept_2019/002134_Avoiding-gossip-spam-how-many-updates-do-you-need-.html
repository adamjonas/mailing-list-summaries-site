Summary:

The next release of c-lightning will implement a simple filter to suppress gossip that comes too fast. Each message (node_announcement or channel_update) will be allowed once per day on average, with a burst up to 4 times per day. Identical gossip messages will be discarded, except for the timestamp and signature. If people require more frequent updates, they are advised to convey it now. This will avoid scaling issues in the future. With 40,000 channels and 10,000 nodes, the limit is around 90,000 12 MB per day.Other implementations include using gossip_queries for backfilling, no longer asking for the last 24 hours of gossip on startup, suppressing duplicate gossip, issuing "refresh" updates only once every 13 days instead of the current seven, and suppressing reply gossip so that if someone sends a message, they will not receive it back. These measures are mainly motivated by a user who has an rPi on a 4G plan. Restarting their node is currently painful, hence the need for these new measures.
