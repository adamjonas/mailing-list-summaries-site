Summary:

The Lightning Network project utilizes random EC key pairs and ECDH to establish a shared secret with each hop, generating symmetric AES & HMAC keys based on the secret. Rusty Russell has shared a test onion routing code for review that includes basic encryption methods such as AES128 counter mode, HMAC using SHA256, and ECDH. In an email conversation in October 2015, Anthony Towns expressed frustration over implementing the program in Python but reported later that the decoding/forwarding was successfully working with pyelliptic. Rusty Russell made changes to the test_onion and Makefile test to separate generate and decode, with the inclusion of the Python version. Anthony Towns questioned the use of flip_key() in onion_key.c, so Russell disables it and makes other changes like making the 02/03 prefix explicit and tweaking onion_key to avoid the for loop and cut calls. The decoding process through Python is slow due to loading modules, so it may not be desirable to run decodes in a single Python process. Finally, the two discuss creating an RFC or wiki to document the Lightning protocol and technology decisions.
