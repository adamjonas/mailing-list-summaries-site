Summary:

In the context of the Lightning Network, Mats Jerratsch and Rusty Russell discuss a potential attack where an attacker stops one payment. Mats suggests that this is not very damaging as the system can self-correct by setting a timeout and issuing a refund. A GitHub repository has been set up to document the Lightning protocol, which allows developers to collaborate on the implementation of the protocol. In another email exchange, Anthony Towns seeks opinions on a written document related to the Lightning Network, with Joseph Poon responding positively. Additionally, Mats explains the recovery process in case one node does not relay the payment to the next node. Anthony Towns has proposed creating a repository for documenting the Lightning protocol, with plans to add early drafts such as Rusty's BOLT proposals and shachain design txt. Rusty plans to have BOLT #3 be the transaction formats, and BOLT #4 the onion and failure message formats. The author of the GitHub repository expresses concerns about the layout being too nested, but likes it as it separates different parts of the application and allows for different implementations side-by-side within that package.
