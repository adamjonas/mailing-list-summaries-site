Summary:

The context revolves around discussions among experts on designing a payment routing system that maintains privacy and reduces visibility of traces in the network. The proposed system uses a gossip protocol to distribute data about spending and receiving limits and fees for transactions. An anchoring mechanism is suggested to use the blockchain as a map of the network. However, challenges such as potential DDoS attacks, centralization issues with reputation systems, and the need for a priority mechanism for information packages are raised.Another topic discussed is the Lightning Network, which aims to solve scalability issues and enable instant, high-volume micropayments built on top of Bitcoin. However, being a node requires constant activity, which can burn battery and introduce latency. Privacy concerns are also raised, and there is a limit to the number of nodes that can serve customers profitably. Discussions were held on different aspects of the Lightning Network, including the use of beacon nodes, nanopayments to probe routes, and best-effort Distributed Hash Tables (DHT) to route to donation addresses.Experts also explored the design of a payment channel network, including the number of beacons required, efficiency of using beacon nodes for transactions, and the use of IRC channels for node discovery. Challenges include overload of beacons, bandwidth limitations, and the risk of funds being tied up in active HTLCs when making small purchases. Finally, the possibility of discovering the shape of a network by using k1/h1 probes without actually sending k2/h2 transactions was explored.In discussing the Lightning Network's routing schemes, concerns about overloading the network with too many beacons were raised, which can be mitigated by each user randomly choosing their own set of beacons. A flood-based route-finding scheme is proposed but has a trivial denial-of-service attack. The network requires a consensus protocol for choosing beacons, committed beacon IDs with spam/flooding protection, and a gossip protocol to announce potential beacons. Nodes must also use a DHT of routes-to-beacons and fees.Additionally, there is a discussion on privacy implications and the need for reputation/web-of-trust systems. The possibility of using anchor transactions in the blockchain as a map of the network is proposed. Lastly, Rusty Russell and Pieter suggest squeezing more bytes out of signatures and public keys.
