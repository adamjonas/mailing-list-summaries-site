Summary:

In a recent email, ZmnSCPxj discussed the importance of RBF-able punishment transactions in preventing theft attempts. While the commitment transaction being RBF-able is not crucial, having an RBF-able punishment transaction that spends the commitment transaction is essential. The punishment branch of the commitment script can be spent immediately to confirm the old commitment via CPFP, and it only requires a single signature from one node, making RBF trivial. ZmnSCPxj plans to review c-lightning code to see if onchaind does RBF for breach claim transactions. He vaguely recalls it being brought up some time ago for c-lightning.
