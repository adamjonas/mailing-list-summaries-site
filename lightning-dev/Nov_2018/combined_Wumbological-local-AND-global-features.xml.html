Summary:

ZmnSCPxj proposed to create a wumbo local and global feature bit. The local wumbo feature would mean "willingness to wumbo with you" and the global wumbo feature would mean "willingness to wumbo with anyone." A node must set the local feature bit if it sets the global feature bit in its announcement. It can clear the global feature bit even if it sends a set local feature bit to its peer. Different values for the local feature bit can be reported to different nodes. If a node did not set the local feature bit reported to its counterparty, it must respond with an error if it receives an open channel with funding_satoshis value beyond the indicated limit for the chain. Wumbo payments will have to be done by AMP if there is no direct path of wumborama nodes from payer to payee. Rusty Russell suggested naming global feature bit "option_wumborama." Laolu noted that there was an inconsistency in the spec concerning the optional max_htlc value. It states the value should be below the max capacity of the channel but makes no reference to the current pre-wumbo max HTLC limit.
