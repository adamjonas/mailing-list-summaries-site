Summary:

Margherita Favaretto, a member of the Lightning Network community, recently attended "LightningHackdayNYC" where she discussed and refined her project on creating a recovering protocol for Lightning Network. She has created a GitHub project where she welcomes feedback and comments on her draft research. Her aim is to solve the issue of false positive nodes in the Eltoo mechanism, which does not cover the case where the other node doesn't share the last transaction, but instead an older one that is more favorable for its balance.Her solution involves using other connected nodes as a backup for the most recent status. For each transaction between node A and B, A sends an encrypted information regarding the last commitment transaction with B to C and vice versa. This enables A to retrieve the information from the other connected node if it loses the last transactions, allowing it to update the status. Margherita believes this idea can solve the current lack in Eltoo and is keen to analyze it further in the next few days. Johan, another member of the community, suggested that Margherita should look into "data loss protection" as defined in the BOLTs, which is similar to her proposal, as well as watchtowers and static and dynamic channel backups.
