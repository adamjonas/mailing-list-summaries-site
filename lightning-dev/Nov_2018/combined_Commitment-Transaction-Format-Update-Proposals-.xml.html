Summary:

The Lightning Network developers proposed changes to the commitment transaction format to manage fees more efficiently. One proposal was to use an OP_TRUE-style output to enable Child Pays for Parent (CPFP) instead of agreeing on future fees. Another change was to make the `remotepubkey` a BIP-32-style to avoid the option_data_loss_protect problem. Additionally, it was suggested that the CLTV timeout be symmetrical to simplify game theory analysis.These changes will be combined into a single `commitment_style2` feature instead of having a testing matrix of all these disabled and enabled. If `commitment_style2` is negotiated, update_fee is a protocol error. The feerate for commitment transactions will always be 253 satoshi/Sipa, with fees and OP_TRUE paid by the initial funder in most cases.HTLC-timeout and HTLC-success txs sigs are SIGHASH_ANYONECANPAY|SIGHASH_SINGLE, and derivation now uses a two-stage unhardened BIP-32 derivation based on the commitment number. Furthermore, both parties can specify their own `to_self_delay`, and `to_remote` is now a P2WSH of `to_self_delay` OP_CSV OP_DROP OP_CHECKSIG.Overall, the conversations focused on finding ways to manage onchain fees in a more flexible and efficient manner, including using CPFP and symmetrical timeouts. The changes proposed aim to simplify the backup and recovery workflow with the current commitment design.
