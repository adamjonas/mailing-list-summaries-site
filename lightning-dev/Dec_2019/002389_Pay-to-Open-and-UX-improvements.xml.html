Summary:

David and ZmnSCPxj provide feedback on potential solutions for layer-2 constructions in Bitcoin. David suggests a solution involving single-show signatures, but notes that this method can be fragile and may prevent useful features like RBF fee bumping. ZmnSCPxj offers an alternative solution using the `OP_CAT` construction which allows for the enforcement of a particular `R` to implement single-show signatures. The `OP_SWAP OP_CAT OP_CHECKSIG` code feeds `s` only on the witness stack. Both solutions are deemed interesting and useful for future developments.
