One of the important aspects of the WatchTower protocol is accountability QoS. The encryption algorithm used by all clients and servers should be either ChaCha20 or AES-GCM-256.The context mentions the need to implement chacha20 for communicating with other peers. The payment modes for rewarding WatchTower are discussed; on-chain bounty, micropayments, and subscription. Both micropayments and subscriptions are more favorable for a WatchTower than an on-chain bounty approach. Additionally, the DoS attack is possible using the on-chain bounty method. Lastly, the storage requirements of WatchTower can be reduced by implementing shachain, but as it is simpler to store the full transaction, the hiring protocol has been kept simple.
