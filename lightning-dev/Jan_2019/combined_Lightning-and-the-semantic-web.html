Summary:

Melvin Carvalho, a developer working on the Solid project, is interested in integrating Lightning Network concepts into the semantic web. He proposes creating an ontology and asks if anyone has existing work or is interested in collaborating on this. René Pickhardt suggests using the vocabulary from BOLT 07 to create a consistent vocabulary across Lightning Network implementations. They discuss the differences in naming conventions used by different implementations. René also clarifies that the lightning: scheme should only apply to payments in the form of bolt11 strings, while TCP already has a scheme for identifying other types of addresses. They also discuss the possibility of using self-describing data in Lightning Network.Melvin shares two data structures in LND that describe channels and nodes in the Lightning Network graph. René suggests creating a vocabulary for Lightning Network terms and documenting their meaning to make it self-documenting. They also discuss the consistency of terms across different implementations. Melvin expresses interest in integrating Lightning Network with Solid and proposes creating an ontology for it. René suggests looking into the W3C Payment Group and LightningJoule as they are working on enabling low-level payment APIs. The conversation continues with discussions about creating ontologies, listing relevant concepts and attributes, and potential use cases for integration.
