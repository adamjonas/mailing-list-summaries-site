Summary:

The Lightning Network is a system that enables cross-currency exchange, but there are technical challenges to implementing this feature. The use of hashlocked timelocked contracts (HTLCs) in routing payments poses a risk to nodes advertising cross-asset exchange on the Lightning Network. HTLCs can be used to transfer value between cryptocurrency systems and construct American Call Options. However, the problem arises when these options are set up for free, as it allows individuals to earn money without any risk. This can lead to liquidity being tied up and potential monetary loss during volatile exchange rate fluctuations.American Call Options grant the right to purchase an asset at a specific price by a certain date. Some Lightning Network nodes advertise themselves as exchange providers, facilitating payment between different assets. For example, ZmnSCPxj can act as an intermediary node, enabling the exchange of bitcoins for WJT. However, this inadvertently creates premium-free American Call Options, which can cause liquidity issues for exchanges, especially during times of volatility.There are proposed solutions to address this problem, but each has drawbacks. One solution suggests adding a payment requirement to setting up HTLCs, which would introduce a premium to the American Call Options. However, this would eliminate the current benefit of free routing failure on the Lightning Network. Another solution proposes increasing fees to reduce risk, but this would still result in tying up most of the liquidity. The third solution suggests limiting the timelock of cross-asset swaps, but this would create friction when crossing assets.In conclusion, while HTLCs allow for the creation of American Call Options, their implementation across different assets becomes problematic. Exploiting these options can result in riskless earnings during periods of high exchange rate volatility. Therefore, a multi-asset Lightning Network may not be economically viable, and instead, a network with a single asset would be preferred.
