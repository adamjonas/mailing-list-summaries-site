Summary:

The email discusses the security vulnerabilities present in current exchange systems and proposes different solutions. The proposed solution involves trusting a third party to execute transactions instead of holding funds, as is currently done with centralized exchanges. However, it is acknowledged that this system is not perfect and there are still risks associated with it. The email also discusses potential mitigations for these risks, including reducing allowed forwarded CLTV distance and increasing bid-ask spread. The idea of punishing RMs for misbehavior is also explored, with suggestions made for collecting proof of misbehavior. Different routing options are discussed, including revealing the distance to RM but not S. The email then delves into more technical details regarding Poon-Dryja and Decker-Russell-Osuntokun networks, and how they can be used to prevent malfeasance by RM nodes. Overall, the email explores various potential solutions to improve the security of exchange systems.
