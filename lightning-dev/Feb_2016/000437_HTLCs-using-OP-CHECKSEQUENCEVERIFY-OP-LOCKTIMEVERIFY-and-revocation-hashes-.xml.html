Summary:

In a discussion about the options for closing a channel, Rusty Russell presents four possibilities: keeping it open, unilateral closure by either party, or cooperative closure. He believes that cooperative closure is the best option and that unilateral closure by the other party is better than doing it oneself. This is because of the OP_CSV delay on option (b), making it less preferable than (d) if one can use a lower transaction fee or spend directly to a useful output address. If trying to convince the other party to close unilaterally (b) to avoid doing it oneself (c), it is smarter to simply choose (d) as it is the preferred option anyway. Russell also notes that with the current arrangement, there aren't any realistic threats that the counterparty could make. However, if there were an OP_CSV on both sides of HTLCs, one could threaten to close unilaterally, locking up the counterparty's funds. Therefore, Russell concludes that changing the current system would make things marginally worse but not better.
