The Lightning Network community has been engaged in various discussions about the algorithm and fee structure for routing transactions. The idea of a "fee budget" has also been proposed, with research needed on the effect of base fees on computations. There are debates about whether base fees accurately reflect costs and whether ignoring nodes that don't set their fees to a particular structure is a good idea. Another conversation focused on the use of proportional fees and minimum payment values instead of base fees in routing algorithms for Lightning Network transactions. Therefore, it is necessary to explore these issues further and conduct more research before making any drastic changes to the system. In conclusion, the Lightning Network community is open to discussing and debating various points about the algorithm and fee structure for routing transactions. While some propose changing the default base fee, others believe it may be premature to reduce base fees. It is important to conduct more research before deploying new algorithms in production and to investigate whether it's possible to emulate the results of current routing algorithms in a faster algorithm without imposing a base fee.
