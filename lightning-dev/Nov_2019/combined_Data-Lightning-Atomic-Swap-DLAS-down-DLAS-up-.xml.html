Summary:

Zero-knowledge proofs (ZK proofs) are suggested as a method for ensuring the correctness of data transfer in Lightning payments. However, concerns have been raised that using ZK proofs may slow down the system and defeat the purpose of making transactions fast and scalable. It was noted that ZK proofs are fast for small programs and can be used in a two-party system for relatively simple programs. Zero Knowledge Contingent Payments (ZKCP) was mentioned as a way to detect if data is correct or not using ZK proofs. Although there is no proper academic paper on ZKCP, it was designed for on-chain applications but can fit into the Lightning Network. The discussion on the Lightning-dev mailing list revolves around finding a way to transfer data via Lightning Network without causing a denial-of-service attack. One possible solution proposed is ZKCP, which allows locking funds to a hash preimage and slots neatly into lightning. The conversation explores various potential solutions for transferring data securely over Lightning Network.
