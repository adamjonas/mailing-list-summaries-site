Summary:

The email suggests an idea to simplify the construction of WatchTowers. The current approach involves creating a single blob containing data for multiple justice transactions, but the proposed idea is to create one-blob-one-justice pairs for each outpoint of the revoked commitment transaction. Each blob would contain encrypted information such as the outnum on the watched transaction, the witness stack, and the scriptpubkey to pay out to. This simplifies the WatchTower construction process, as justice transactions will only be simple one-input, one-output transactions or two-output transactions with the other output being the WatchTower bounty. The proposed approach also allows for more insurance by providing the same blob to different WatchTowers and enables easy adaptation to changes in channel commitment structure without requiring changes to existing WatchTowers.
