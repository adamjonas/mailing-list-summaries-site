Summary:

The Lightning Network is a protocol layer that enables instant payments using Bitcoin's security through smart contracts. However, concerns about potential vulnerabilities have been raised regarding the finite lifespan of cryptographic primitives and Schor's algorithm. To address these concerns, scriptless scripts have been proposed, but they are currently not secure against Schor's algorithm. Lattice-based cryptography has also been suggested as a more secure option. Still, it is not yet widely available, and concerns remain about public key exposure and quantum computers stealing multisig funding outputs or using them to steal payments routed through them.Recently, various suggestions were made on the Lightning-dev mailing list to address these concerns. These include replacing regular multi-sig with a single p2wkh, which would reduce fees for opening/closing channels and make htlc timeout+success transactions smaller. Additionally, a scriptless version of the adaptor signatures and contracts required in the Lightning Network was proposed using only 2-party ECDSA signatures, which can be directly applied today without waiting for Schnorr signatures to be deployed in Bitcoin.To further address these concerns, it has been suggested that a new protocol be prepared for reanchoring transactions, a new "signing algo" concept be introduced, and a new `reopen_channel` be created to move a channel from one signing algorithm to another. Careful consideration and backup plans are emphasized, given the importance of the global economy.
