Summary:

In a mailing list discussion, Robert Olsson expressed his concern about the lack of standardization around QR codes for node at ip:port format in Lightning Network. He suggested using bech32 nodeid to keep QR small and proposed a format like lightning:ln1bech32nodeid/ipnumber/port which would avoid confusion of : in ipv6 and :portnumber. He mentioned that everything coded entirely in bech32 shouldn't need a protocol so the 'lightning:' part could possibly be omitted as well. Igor Cota replied that bech32 is designed to make small QR codes because of its ability to create text-mode QR-codes which are much smaller. Rusty Russell suggested using bech32 'ln1' encoding which would be 62 chars vs 66. Corné Plooy pointed out that reducing QR code size by thinking of a better way to encode the node ID would make more sense than using hexadecimal encoding.
