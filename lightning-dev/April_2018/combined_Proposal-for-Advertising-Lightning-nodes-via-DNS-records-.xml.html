Summary:

In a proposal to improve the security of Lightning Network, an additional BOLT (tentatively BOLT 12) has been suggested. This would allow domain name operators to create SRV records for their nodes, enabling users to connect to that domain's advertised nodes and open channels independently. Tyler and Robert Olsson have proposed this solution to make it easier for end-users to connect to nodes with enough liquidity to pay every merchant. However, there are concerns over selective attachment to a merchant's node and the creation of large hubs that are weakly interconnected.Tyler H has also proposed a new BOLT 10 for the Lightning Network, which aims to improve user experience and provide an additional means of advertising "official" nodes. The proposal suggests allowing cryptographic means of associating nodes and their associated domains to exist. But DNS hijacking is not an unlikely scenario, and caution would be necessary. Tyler believes that having private nodes with channels open to public nodes could help domain operators manage their risk.Tyler is working on finding a secure way to advertise human-understandable node names that retains the open and decentralized properties of Lightning technology and blockchain. A proposal called Numerifides has been developed which is committed directly to the Bitcoin blockchain and solves Zooko's triangle. In a conversation between Tyler and ZmnSCPxj, the latter suggests locking funds for a time may be enough without pulling in proof-of-work.The context discusses the use of Numerifides command, which is a scheme that allows for non-financial data to be stored on-chain. Tyler and Robert have proposed a new BOLT to allow domain operators to create SRV records for their nodes. Christian Decker has some issues with the proposal and suggests encouraging people to open random connections to strengthen the overall connectivity.Tyler and Christian discuss the proposed BOLT that allows payment routing through DNS records, which could serve as a fallback solution for improving user experience. They both agree that the solution should only be used as a fallback option, rather than replacing the decentralized nature of Lightning. In an email conversation between Tyler and ZmnSCPxj, the latter clarifies that a BOLT does not replace bootstrapping seed functionality for Bitcoin lightning network.Overall, the discussions revolve around finding solutions to improve the security and user experience of Lightning Network while retaining its decentralized properties.
