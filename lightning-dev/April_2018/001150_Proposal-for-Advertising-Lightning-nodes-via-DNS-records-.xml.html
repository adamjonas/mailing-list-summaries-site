The term "autoconnect" used in the mobile wallet is misleading. Instead, it should be "autochannel". ZmnSCPxj also explained the risk associated with public and private nodes owned by the same entity. If the public-facing node gets hacked, it cannot draw funds from the private node but can only send them out to the attacker on the network, or close channels and send the funds + wallet balance to an on-chain address.
