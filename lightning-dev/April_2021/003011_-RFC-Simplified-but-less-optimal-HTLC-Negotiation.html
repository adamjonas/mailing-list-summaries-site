Summary:

Rusty Russell prefers alternation over reflection, as it is more optimal than reflection for optimized implementations and fits better with the existing implementations. In particular, a simple implementation never sends a commitment_signed until it receives "giving way" so it doesn't have to deal with orphaned commitments. A more complex implementation sends opportunistically and then has to remember that it's committed if it loses the race. Such an implementation is only slower than the current system if that race happens.
