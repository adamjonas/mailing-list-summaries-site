Summary:

Rusty Russell and Lloyd Fournier discussed implementing a change to the DF spec to eliminate all "basepoints" using node pubkey and randomizing it using the shared secret for each purpose. However, Rusty notes that one issue with this method is that you can't sign the commitment_tx or htlc_tx without knowing the node's secret key or the tweaked key. Currently, c-lightning sends a round-trip to the signing daemon to handle this, but Rusty would prefer to avoid requiring this step. As a solution, he suggests adding commit_basepoint from which others are derived or just replacing funding_pubkey.
