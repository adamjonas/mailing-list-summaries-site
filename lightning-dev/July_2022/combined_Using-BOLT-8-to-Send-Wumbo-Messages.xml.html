Summary:

A developer from dlcspecs requested feedback on a PR that proposes sending messages larger than 65 KB using BOLT 8. During the discussion, Thibaut stated his motivation for having the segment explicit, including the ability to handle disconnects/reconnects without requiring access to the low-level network layer. Christian Decker mentioned the ability to send an urgent message in the middle of sending a large one. However, Thibaut has no strong opinion on what is best and thinks it would be nice to use something similar in LN to reduce the need for maintaining multiple things. Olaoluwa Osuntokun clarified that chunking messages and reading them into a read buffer over a stream-like interface is possible without adding any new application layer messages.This method is no different from sending a 65 KB message over TCP/IP, and the behavior was actually present in the original spec but removed to simplify things. To reintroduce this feature, a new wumbo message feature bit could be added to indicate that a peer can properly chunk and aggregate larger messages. The discussion involved dlcspecs developers and other contributors who provided comments and feedback. Relevant discussions and commits related to the topic were also linked.
