Summary:

The Lightning-dev mailing list has seen discussions on adapting the payment channel script to consume less space. The updated script saves around 32 bytes in the scriptpubkey and 70 bytes in the signature when Alice closes the channel, resulting in a total of 100 bytes saved between the anchor transaction and commitment broadcasted. Nicolas Dorier provides links to check the scripts for Alice closing the channel and Bob using revocation. The email thread also discusses using private keys for HD and disclosing temporary private keys instead of signatures for compact revocable examples. They propose OP_CODESEPARATOR as a potential space-saving solution. Another email conversation between Joseph Poon and Nicolas Dorier explores the use of SIGHASH_NOINPUT and OP_CODESEPARATOR to optimize Bitcoin transactions. Rusty commends Nicolas on his work with OP_CODESEPARATOR. Finally, the context describes the commitment revocation process and proposes modifying the escape redeem function using OP_CHECKSIG OP_IF to remove duplication of bytes.
