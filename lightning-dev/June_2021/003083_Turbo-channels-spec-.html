Summary:

In a message to Rusty, Bastien Teinturier discusses the `funding_locked` message in Eclair. They send it as soon as they see the funding transaction in the mempool, but Rusty suggests waiting for confirmation before sending it. Rusty proposes that this would still be backward compatible. The `announcement_signatures` message is also discussed, and Bastien asks if it should be deferred until after the funding tx has been confirmed, and what `min_depth` should be used. Rusty responds that it has to be at least 1, and that it is compliant with the spec to wait until `funding_locked` has been sent and received, and the funding transaction has at least six confirmations before sending `announcement_signatures`.
