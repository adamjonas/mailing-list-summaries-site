Summary:

In a recent email, g0b1el mentioned two additional improvements to the Lightning Network's payment protocol. The first improvement involves nodes on the route receiving the same amount of reservation without the need to put the amount inside the onion. This simplifies rendezvous routing and allows for quicker failure of reservation balances. The onion now only holds the next hop information, resulting in smaller onions. The second improvement eliminates the need for an HTLC onion in the second step as route hops already contain all the necessary information to create HTLCs. The sender only needs to sign the commitment and revoke the state with the first node in the route. This results in an increase in payment latency by just one more call between the route nodes. The proposal can also be found on the mailing list archive if ASCII graphics are not rendering correctly.
