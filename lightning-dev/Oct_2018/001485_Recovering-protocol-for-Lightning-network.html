Summary:

Over the weekend, Margherita Favaretto participated in "LightningHackdayNYC", where she discussed her project of realizing a recovering protocol for Lightning Network. She has created a project on GitHub to share her research and welcomes comments and feedback. Eltoo is already solving part of the problem by allowing nodes to share the last status of the channel; however, it does not cover the case where the other node shares an older transaction that is more favorable for their balance. Margherita's project aims to solve this specific issue and make the protocol more solid to face completely the case of a false positive node. Her idea is to use other connected nodes as a backup for own recent status. For each transaction between two nodes connected to a third node, A sends an encrypted information about the last commitment transaction with B to C, and for each commitment transaction with C, A sends an encrypted information about the last commitment transaction with C to B. This way, if A loses the last transactions, she may ask for the information from the other connected node and update the status. Margherita believes that this idea can solve the current lack in Eltoo and plans to analyze it further. She invites thoughts and suggestions for proceeding in the wisest way and finding a solution that can also cover all needs. If anyone is interested in this research, she is open to sharing more details about the design of her idea and collaborating to work together.
