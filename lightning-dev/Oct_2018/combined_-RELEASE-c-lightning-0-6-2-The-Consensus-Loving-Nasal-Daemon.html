Summary:

The c-lightning 0.6.2 release, named by @practicalswift and announced by Rusty Russell, is now available on Github at https://github.com/ElementsProject/lightning/releases/tag/v0.6.2. It will also be soon available on https://launchpad.net/~cdecker/+archive/ubuntu/clightning, although the correct link for now is https://launchpad.net/~lightningnetwork/+archive/ubuntu/ppa. Despite being labeled as -rc1, there are no differences from the previous release, except for the tag and CHANGELOG.md change. Rusty expresses gratitude to those who have already upgraded to the new release.This latest release of c-lightning 0.6.2 brings several highlights that aim to improve overall network health. Users are recommended to upgrade to take advantage of these improvements. One notable feature is the route boost in invoices, which enhances payment reliability. Additionally, the listforwards command has been introduced, allowing users to view all forwarded HTLCs. Furthermore, logging spam at the debug level has been reduced. Alongside these user-centric updates, numerous bug fixes have been implemented to enhance the overall user experience.In terms of network improvements, this release addresses issues such as spamming peers with dead channels and eliminating redundant gossip updates. It also incorporates the advertising of htlc_maximum_msat for channels, in accordance with a recent BOLT update. Internally, significant optimizations have been made in JSON command response. Moreover, connectd and hsmd are now part of the growing literate documentation effort.The latest release of c-lightning 0.6.2 recognizes contributions from 13 different authors, including three first-time c-lightning contributors. A total of 239 commits have been made since the last release, demonstrating the active development of the project. For a comprehensive list of changes, refer to the complete changelog available on the GitHub page.
