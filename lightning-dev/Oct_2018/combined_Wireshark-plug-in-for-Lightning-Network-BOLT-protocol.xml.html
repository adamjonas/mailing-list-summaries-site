Summary:

A team from Nayuta is developing a Wireshark plug-in for the Lightning Network (BOLT) protocol. The alpha version of the software can decode some BOLT messages and currently works with Nayuta's implementation (ptarmigan) and Ã‰clair. To make interoperability testing easier, the team proposes adding a compile option to every implementation that enables an output key information file. This file would allow communication logs of BOLT messages to be viewed in the same format (.pcap). Key rotation occurs whenever nonce equals zero, and lightning node software writes 16byteMAC and key of "first BOLT packet". Whenever a .pcap message starts with a message whose nonce is not equal to zero, the messages cannot be decrypted until the next key rotation. The team hopes their project will be useful to the Lightning Network ecosystem.
