Summary:

The context of the discussion is regarding DNS seeds described by BOLT10. The user is trying to understand how key-value pairs are used in a query and if they are encoded into the FQDN. Christian Decker explained that the flags are intended to be used as subdomains and gave an example. However, he also mentioned that query filtering has not yet been implemented. Thomas Steenholdt suggested that the spec needs further clarification on the direction of interpreting conditions and what happens if a condition is specified multiple times. He also proposed a solution where currently set conditions are always replaced when new conditions with the same key are met. He offered to help clarify the BOLT10 specification and made a pull request for review. Christian Decker welcomed the improvements and feedback.
