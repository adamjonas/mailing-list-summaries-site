Summary:

During a discussion among Lightning network developers, the issue of creating dust outputs on second-level transactions was raised. Rusty Russell pointed out that this should be avoided and Olaoluwa Osuntokun suggested adding a "don't create dust output" clause to the trim output spec portion. Additionally, Laolu suggested coalescing many HTLC transactions into a single one using SIGHASH_SINGLE|SIGHASH_ANYONECANPAY to save on-chain footprints and add more fees. Russell agreed that this could work but noted it is an obscure case where many HTLCs need to be closed at once or for fee bumping. The changes have been deferred to 1.1 as the team is in no changes except bug fixes mode.
