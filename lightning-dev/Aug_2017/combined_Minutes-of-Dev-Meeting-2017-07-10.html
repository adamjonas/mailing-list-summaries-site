During a discussion, Olaoluwa Osuntokun pointed out that the signing mode SIGHASH_SINGLE|SIGHASH_ANYONECANPAY is already in use. The code modifications required to switch to this mode are trivial, as per Rusty's suggestion. However, due to the organization being in a "NO CHANGES EXCEPT BUGFIXES" mode, this change has been deferred until version 1.1, which implies a new feature bit. Rusty has recorded this information on the project's Wiki page for future reference.In a discussion about the Lightning Network protocol, Rusty Russell provides feedback on a document discussing some of the technical details. One point addressed is the creation of a dust output in the process of setting HTLC floors in second-level transactions, which would render small amounts unusable on the chain. However, Laolu suggests that the trim output specification should also indicate that no dust output should be created on second-level transactions.Another topic discussed is the use of SIGHASH_SINGLE|SIGHASH_ANYONECANPAY instead of using sighash-all on second-level HTLC transactions. While this approach might allow for coalescing many HTLC transactions into a single one, it is considered to be an obscure case and is mostly intended for fee bumping. Additionally, the closing transaction and commitment tx with no HTLC outputs are two other cases where this approach would be viable. The use of this signing mode is not difficult to implement.Rusty Russell has provided feedback on a productive discussion that he missed. Pierre suggested choosing HTLC (Hash Time-Locked Contracts) min high enough to avoid issues, and Laolu pointed out that it would limit the ability to send and claim small amounts on the chain as it would create a dust output. Laolu suggested that the trim output spec portion should state not to create a dust output on second-level TX, although Rusty believes it is already mentioned in the document. Laolu also recommended against using sighash-all on second-level HTLC transactions, which would allow coalescing many HTLC transactions into a single one to save on-chain footprint and add more fees, similar to Lighthouse by Hearn.
