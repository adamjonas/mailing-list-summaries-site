Summary:

The Lightning Network team is exploring the implementation of a dynamic commitment update protocol to upgrade existing channels to taproot without on-chain transactions. The proposal includes the use of an adapter commitment and a protocol for updating commitments on-the-fly. This would allow all 80k+ SegWit v0 channels to be upgraded to the base level of Taprooty channels. The protocol introduces two new messages, 'commitment_update_propose' and 'commitment_update_apply', which enable parties to propose and apply commitment/channel parameter updates. The proposal aims to remove hard-coded limits and make channel upgrades more efficient and less disruptive. Johan Torås Halseth suggests dividing the taprootyness of a channel into funding and commitment outputs to enable upgrading channels only on the commitment level. The proposal has been submitted as a pull request on GitHub and is being discussed on the lightning-dev mailing list.
