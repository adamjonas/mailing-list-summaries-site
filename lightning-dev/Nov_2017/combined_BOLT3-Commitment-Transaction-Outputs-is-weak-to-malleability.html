Summary:

The Commitment Transaction Output script is vulnerable to malleability, which can delay the confirmation of revocation. The penalty transaction can be delayed by an attacker through malleating it, leading to bad outcomes as Lightning is dependent on time locks. This vulnerability can create a problem in which Eve can malleate OP_1 into a positive, huge number, leading to filling of mempool of nodes/miners with the malleated version, which has higher fee rates, delaying the confirmation of the penalty transaction. A policy rule called SCRIPT_VERIFY_MINIMALIF was merged into v0.15.1, fixing the situation with 2 bytes overhead. To fix this issue, RBF can be enabled for Penalty Transaction so that Eve's transaction would be replaced by the initial one. Laolu recommends implementing a scorched earth policy when responding to on-chain contract breach event, gradually bleeding away the funds of the violator while bumping up the fee on the transaction using RBF. This policy addresses Dorier's malleability concern and further stacks the incentives against attempting a contract breach.
