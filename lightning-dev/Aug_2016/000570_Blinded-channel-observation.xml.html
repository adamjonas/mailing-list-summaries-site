Summary:

The email conversation discusses different methods of outsourcing work and reducing computational cost when watching multiple channels. The method proposed involves using HMAC to check transactions against a key-value store and can be effective for a single channel, but it becomes more computationally expensive as the number of channels watched increases. To solve this problem, the observer can put all txids being watched for into the same tree and have seek access time independent of the number of separate channels being watched. Additionally, there is discussion about sending encrypted state information to the observer, but it is determined that most information needed by the observer is already available in the observed commitment tx, and reconstructing messages from hashes and/or signatures is not possible. There are also suggestions for using one's input signature as the key for the encrypted blob, which helps to encourage more nodes to download witness data.
