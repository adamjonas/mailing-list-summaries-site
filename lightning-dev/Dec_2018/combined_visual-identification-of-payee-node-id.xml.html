Summary:

In a recent Lightning-dev mailing list discussion, Pavol Rusnak proposed adding an identicon of the payee node id (public key) next to the QR code for payments in user-facing wallets to allow quick visual verification of the recipient's identity. This would eliminate the need to check lengthy hex values and make it easier for users to ensure they have the correct recipient. The suggestion was met with interest, and William Casarin suggested using cryptographic hash functions to generate different faces for small changes in inputs. However, jdenticon.com did not exhibit this property. Max proposed using a confirmation phrase of a few words instead, which could be more reliable if a pool of enough words was available. Overall, implementation across all user-facing wallets would be required for this feature to work effectively.
