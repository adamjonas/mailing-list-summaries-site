Summary:

A member of the W3C Web Payments Working Group, Christian Decker, has confirmed that Bitcoin and Lightning should be compatible with the group's candidate recommendation for a payment API. However, he added that special consideration is needed to assign a payment method ID to decentralised networks. The Payment Request API mentions XBT as a supported currency, in addition to ISO4217 codes. The proposal will be published no earlier than 31 October, according to the group's website. Feedback on the proposal can be left at the W3C's Github repository.
