Summary:

James Chiang inquired about sighash_noinput and its effect on privacy in the case of watchtowers. In response, Rusty Russell stated that while Eltoo, which requires SIGHASH_NOINPUT, reduces the number of penalty transactions for each state change, it does reveal related states. He suggests that encrypting using the txid could be used to minimize information disclosed to third parties and that watchtowers may need to consider transactions that have already been spent by SIGHASH_NOINPUT transactions. Furthermore, he clarifies that SIGHASH_NOINPUT should not be used by wallets to spend their own UTXOs but rather in smart contracts with unique conditions, such as Lightning's rotating keys.
