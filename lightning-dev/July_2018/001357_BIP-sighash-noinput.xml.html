Summary:

The email is a response to concerns about the potential dangers of using SIGHASH_NOINPUT in Bitcoin wallets. The author reassures that the name is intentionally scary to inform developers that it may be unsafe. This flag is intended for use in specialty protocols like LN and CoinSwap, not for general-purpose user wallets. It will only be used in exact specialty protocols and not for general wallet usage. By default, this flag is not set, and address reuse is still slightly safe for common usage, modulo other bugs in the implementation such as weak generation of random R. Luke Dash Jr.'s wallet explicitly rejects address reuse. The original email expressed concern that "SIGHASH_NOINPUT" could make reuse addresses more dangerous and put users at risk.
