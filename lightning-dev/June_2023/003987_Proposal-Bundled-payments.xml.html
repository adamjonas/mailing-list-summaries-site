Summary:

In an email conversation between Thomas Voegtlin and David A. Harding, they discuss the idea of bundling payments in BOLT-11 invoices. Thomas suggests that the invoice should have a feature bit to make prepayment optional or required. He explains that for the feature to be optional, the prepayment amount needs to be subtracted from the main payment amount. In this case, Alice would expect to receive either 1 BTC with the invoice payment hash or 1 BTC minus 10k sats with the invoice payment hash, along with a prepayment of 10k sats via keysend.David raises a question about whether a BOLT11 invoice is necessary for the prepayment or if it can be done using a keysend payment with the onion message payload. He suggests adding an extra_fee_via_keysend field to the BOLT11 invoice, which would be smaller and could be encoded in an existing field like description or metadata. This would allow for immediate implementation until an updated version of BOLT11 or an alternative using offers becomes widely deployed.The email conversation ends with the contact information for Electrum Technologies GmbH, including their address and legal registration details.
