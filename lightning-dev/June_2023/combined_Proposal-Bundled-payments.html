Summary:

Thomas Voegtlin has proposed an extension to BOLT-11, the Lightning Network protocol. The proposal suggests including bundled payments with distinct preimages and amounts in BOLT-11 invoices. This would benefit services that require prepayment of a mining fee for non-custodian exchanges like submarine swaps and JIT channels. Currently, dedicated client software can handle prepayments for submarine swaps, but competitors without a dedicated wallet face challenges in implementing this feature. Thomas suggests bundling the prepayment and main payment in the same BOLT-11 invoice to solve this issue.The proposed semantics of bundled payments involve the receiver waiting for both payments' HTLCs to arrive before fulfilling the prepayment's HTLCs. If the main payment fails to arrive, the prepayment should be failed with a MPP timeout. Once both payments' HTLCs have arrived, the receiver fulfills the prepayment's HTLCs and broadcasts the on-chain transaction. This change aims to make lightning service providers more competitive and enable non-custodian services to operate fully non-custodial.Matt Corallo responded to Thomas's proposal, expressing doubt about introducing new interoperable changes into BOLT-11. He suggests working towards specifying a "standard" way of doing swaps built on onion messages for enhanced privacy. In response to Thomas's question about the distinction between splice-out and submarine swaps, Matt explains that submarine swaps enable adding receiving capacity to a channel.The decision to make this feature mandatory or optional rests with the service provider. However, further discussions are needed to determine the best approach, considering the challenges of introducing new changes into BOLT-11 and exploring alternative options like BOLT-12 or onion messages.There are ongoing discussions about the proposed extension to BOLT-11 and considerations regarding its implementation and compatibility within the lightning network ecosystem. Some suggest focusing on BOLT-12 instead, as all implementations are actively working on it. However, concerns remain about the time it will take for BOLT-12 to become the dominant payment method on Lightning. Ultimately, the decision depends on technical needs and ecosystem support.
