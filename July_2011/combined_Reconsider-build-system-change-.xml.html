    Summary:

    The discussion on whether to use autotools or cmake as the future build system for Bitcoin began after someone submitted an autotools patch. While autotools is more standard than CMake, and because it's a shell script, there's no need to install CMake. However, the autotools stuff needs to be redone before it can be merged. CMake generates native build scripts for Make (all platforms), XCode, Visual Studio while Autoconf only does Make. In addition, CMake has a clearer syntax, and is very fast. It is supported natively in Qt Creator in the QT SDK, and is used by many other KDE/QT as well as other open source applications. Someone had already integrated a CMake build system for Bitcoin into rpcminer. Pieter Wuille expressed no preference for any particular project build system. He suggested that a build system should be easy enough to set up, allow building of the Bitcoin codebase on several Linux distros, do cross-compilation to Windows, support OSX, be easy to maintain, and not be too hard to adapt other GUIs to use it.
