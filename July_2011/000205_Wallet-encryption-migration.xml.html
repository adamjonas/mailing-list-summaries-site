Summary:

In a discussion, Luke-Jr suggested two ways to enhance the security of the Bitcoin wallet. The first suggestion was to write over the old unencrypted wallet.dat multiple times with pseudo-random data to secure-delete it. The second suggestion was to mark all keys imported from an unencrypted file as "potentially compromised" and never use them for new addresses. In response, Arthur Britto disagreed with the first suggestion, stating that writing zeros just once should be sufficient. He also pointed out that moving the keypool addresses aside so they won't be used sounds like a good idea. However, he lamented that there is currently no way for the wallet to be born-encrypted. The only way to prevent a leak is to build the wallet initially on a ramdisk or the like, then move it over after encrypting it. Finally, he agreed with Luke-Jr's second suggestion, as it would make the key leak on a new wallet inconsequential since all keys in it are keypool keys at that point.
