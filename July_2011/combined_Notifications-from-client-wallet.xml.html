    Summary:

    John Smith proposed to add notifications to the client and wallet, in order to decouple UI and core communication. He suggested using boost::signal mechanism for this purpose, which allows decoupled delivery of 'signals' from an object. Multiple other objects can listen in on an event without the emitting object having to care. Specific notifications that would be useful include balanceChanged(int64), transactionAdded(int256), transactionUpdated(int256), transactionRemoved(int256), addressAdded(int160), addressUpdated(int160), addressRemoved(int160), notification(std::string message, int severity), and numConnectionsChanged(int) and numBlocksChanged(int) for Network client.
