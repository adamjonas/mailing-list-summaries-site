    Summary:

    In 2011, Matt Corallo found a data structure in wallet settings that would cause all versions of Bitcoin to crash on load. He put an empty object in addrIncoming, which was not used anywhere in the code aside from its definition, to prevent creating an empty wallet and letting the user wonder where their wallet went. This way, the system fails soon and does not get a corrupt wallet. The commit for this solution can be found at https://github.com/TheBlueMatt/bitcoin/commit/2e8383469d7e12a495b3a1dbd41a8d211ff34fe8. Pieter provided an ACK on newenc, thanking Matt for all the work he had put into it already. The only remaining issue was the invalid mlock calculations, which Matt planned to fix.
