Summary:

Pieter Wuille sent an email to propose the standardization of version bytes used by Bitcoin. The purpose of this proposal is to adopt a convention before everyone creates their own. The proposed functions for non-alternate chains include four data classes: address, private key, master key, and extended. These functions are compatible with all currently-used version bytes such as testnet, realnet, addresses, private keys, namecoin, and multicoin. The EncodeVersionByte function takes three arguments: data class (address, privkey, masterkey, or extended), nVersion, and fTestNet. The output of the function is a version byte that can be used in base58-encoded data structures.
