    Summary:

    Bitcoin developers are considering a 0.3.24 release due to flood control limits becoming an issue again. The release will include sipa's flood limit fix, dnsseed on by default, and possibly UPnP enabled by default as well. A DNSSeed has been set up with reliable hosting which dynamically fills its hostname with random nodes known to be up and accepting connections. This was added to master in 44d16327. Since it is no longer a static list, it is now time to enable dnsseed by default. Additionally, enabling UPnP by default would increase the percentage of nodes that accept incoming connections. Bitcoin developers believe this is necessary for network stability over the next 90 days. The discussion revolves around the release of Bitcoin 0.3.24, which is intended to address flood control limits, DNS seed on by default and UPnP enabled by default. The flood control fix aims to improve bring-ups and prevent network partitioning caused by frequent flood disconnects. DNS seed has been enabled for a while, and there is no reason not to turn it on now. However, UPnP should be disabled by default on bitcoind but on by default on bitcoin. There is also a request to update the checkpoint in 0.3.24 since the difficulty has increased almost 17x since the highest one currently in there. Besides, some technical issues need fixing before release concerning build engineering. The release of 0.3.24 is seen as a minor update release, with most people testing either 0.3.23 or upstream git.
