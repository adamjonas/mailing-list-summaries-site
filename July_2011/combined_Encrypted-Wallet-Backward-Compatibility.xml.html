    Summary:

    There are backward-compatibility issues with Wallet encryption in Bitcoin, as pointed out by Joric. In version 0.3.23, Bitcoin dies with "ReserveKeyFromKeyPool() : unknown key in key pool" after writing one unencrypted private key to the (otherwise) encrypted wallet. In version 0.3.22, Bitcoin opens fine and displays transactions, however, shows a total balance of what is held only in unencrypted keys. Two options were suggested to work around this: renaming wallet.dat for encrypted wallets, which could cause issues with backup scripts and old clients creating new wallets instead of using the old one; or doing nothing and letting old clients become unable to use their wallet. The latter option was preferred, with comments suggesting that error messages are better than users believing their coins are gone. To prevent this issue in the future, it was suggested that Bitcoin should not continue on as if nothing had happened when faced with unknown keys. There were two proposed solutions, and agreement needed to be reached so that 0.3.24 could be released as soon as possible. The context discusses the intentional breaking of unsafe or badly designed backup mechanisms during the planned merging of import/export. Gavin Andresen proposed creating an encrypted wallet file, wallet_e.dat, on wallet encryption, truncating the original wallet.dat, and setting its file permissions to 000 to prevent old versions of Bitcoin or any dumb wallet backup scripts from reading it. However, renaming the file could cause issues with user restores, so Pieter suggested adding a nMinVersion that specifies the minimum version required to read the file. Matt Corallo found a data structure in wallet settings that could cause all versions since Bitcoin was in Github to crash on load, making a new wallet or opening in some bizarre half-state. He put an empty object in addrIncoming to cause this effect. This solution resolved all known issues and suggestions on newenc, except for the invalid mlock calculations, which he intended to fix.
