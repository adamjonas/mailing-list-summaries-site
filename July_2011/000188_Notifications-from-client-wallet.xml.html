    Summary:

    In an email thread from 2011, John Smith proposed the addition of notifications to the Bitcoin client and wallet. The purpose was to decouple UI and core communication and eliminate the need for polling for changes. The boost::signal mechanism was suggested as a way to achieve this, allowing for decoupled delivery of "signals" from an object. Multiple other objects could listen in on an event without the emitting object having to care. Specific notifications that would be useful were listed for the wallet and network client. For the wallet, examples included balanceChanged, transactionAdded, addressAdded, and notification. For the network client, examples included numConnectionsChanged, numBlocksChanged, and notification. Another significant idea mentioned in the email thread was the implementation of a central notification interface that all components such as the wallet(s), net, block store, etc., could communicate with. This would allow for clean separation of pieces and large-scale rewrites of one or another part without touching anything but the parts that are being redone. It would also allow for cool programs (like pushpoold/a block notification server/etc) to use Bitcoin as a library fairly efficiently.
