    Summary:

    A discussion on IRC has led to the decision to standardize the version bytes used by Bitcoin for various applications. The three components deemed meaningful are network, data class, and version. While there is no technical reason why different networks and data classes would require separate version bytes, it is beneficial to prevent collisions. The alternate chain uses version byte 52, and bit 16 will be reserved for "alternate chain." When bit 16 isn't set, testnet uses version 111. To avoid this conflict, bit 1 can be chosen for testnet, which will XOR the rest of the version number with 111 if set. Alternatively, testnet can be reset, and odd numbers will be designated as testnet while even numbers will be realnet. Six more bits can be utilized, with 128 already being used for private keys. Thus, (128,64,32) will define data class, (8,4,2) will define version number, and bit 1 will define testnet. Data class 192 is proposed for master keys, leaving lower numbers for basic data such as public keys.
