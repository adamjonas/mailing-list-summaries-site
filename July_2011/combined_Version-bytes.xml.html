    Summary:

    Bitcoin has decided to standardize the version bytes used for several applications. There are three components that seem meaningful: network, data class, and version. There is no technical reason why different networks and data classes would need separate version bytes, but it's a good thing to keep them from colliding. The proposal is to use bit 16 in the version byte for alternate chains, as Namecoin already uses version byte 52 for addresses. When bit 16 isn't set, the rest of the network can be defined. Testnet currently uses version 111, which is not a single bit. To solve this problem, they can choose bit 1 for testnet and XOR the rest of the version number with 111 if bit 1 is set. Otherwise, they could reset testnet and state odd=testnet and even=realnet. They can use bits 128, 64, and 32 for data classes and bits 8, 4, and 2 for version numbers. Bit 1 defines testnet, and bit 16 defines "private". Data class 192 is suggested for defining a format for master keys, leaving the lower numbers for more basic data such as public keys.
