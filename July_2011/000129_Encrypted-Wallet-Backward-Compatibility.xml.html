    Summary:

    The current version of Bitcoin has backward-compatibility issues with Wallet encryption. The error message "ReserveKeyFromKeyPool(): unknown key in key pool" occurs when trying to perform any action that reads the keypool. One suggestion for working around this problem is to rename wallet.dat for encrypted wallets; however, this has several disadvantages, such as breaking backup scripts and potentially causing confusion for users. Another option is to let old clients get mad and be unable to use their wallet, rather than accidentally creating multiple wallets or thinking their coins are gone. In the future, Bitcoin should not continue as if nothing happened when faced with unknown keys to prevent similar issues.
