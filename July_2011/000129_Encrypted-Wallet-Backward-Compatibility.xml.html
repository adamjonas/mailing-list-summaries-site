Encrypted Wallet Backward Compatibility

    Summary:

    Bitcoin has backward-compatibility issues with wallet encryption. In version 0.3.23, Bitcoin dies after writing one unencrypted private key to the otherwise encrypted wallet. In version 0.3.22, Bitcoin opens fine and displays transactions but shows a total balance of what is held only in unencrypted keys, and each transaction shows only confirmation count, date, no description, and a debit/credit of 0.00. When attempting to read keypool, users get the "ReserveKeyFromKeyPool() : unknown key in key pool" error. There are suggestions to work around this issue, such as renaming wallet.dat for encrypted wallets or doing nothing and letting old clients get mad. However, it is suggested that Bitcoin should handle multiple key types gracefully and ask for a password to encrypt the unencrypted ones. It is recommended that Bitcoin should not continue as if nothing had happened when faced with unknown keys in the future.
