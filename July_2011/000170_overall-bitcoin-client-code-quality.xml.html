Summary:

The author of the email is seeking a discussion about code quality and to create a cleanup plan for the Bitcoin codebase. The author points out several issues with the current code, including a lack of documentation and poor use of threads. The author suggests removing all threads and critical sections and building a sequential easy-to-follow model. Additionally, the author suggests using a platform-dependent start thread and mutex rather than Boost, which is seen as an unnecessary barrier for new developers. The author also calls for a style guide that includes readability guidelines such as having one file per class and no methods with single code lines longer than a screen page. Furthermore, there are tons of hardcoded values for the official and the testnet block chain, which the author suggests moving to a chain description file. Despite the need for code cleanup, the author acknowledges that the code is cleaner than it first appears, and most of it is fine.
