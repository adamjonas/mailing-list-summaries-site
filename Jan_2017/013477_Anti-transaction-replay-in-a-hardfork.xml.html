    Summary:

    The conversation is about the proposal by Johnson Lau regarding opt-in and opt-out bits for transaction replay protection in a new network. Tom Harding comments that while Lau's proposal relies on non-standardness for protection, a responsible new network should allow something invalid in the existing network for transactions where replay to the existing network is undesirable. Furthermore, Harding suggests that specifying changes to be included in the new network, such as the specific O(n^2) fix proposed by Lau, is an overreach and should be left up to the new network itself. Lau's proposal depends on the new network enforcing a new SignatureHash for transactions with an nVersion not used in the existing network. This is a hard fork and the existing network would never accept such transactions. If the new network wants to protect its users, it could include a SignatureHash hard fork like this, along with other hard fork changes.
