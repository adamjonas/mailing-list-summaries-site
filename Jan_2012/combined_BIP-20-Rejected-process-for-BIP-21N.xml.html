    Summary:

    The context is a discussion around BIP 21, which outlines the Bitcoin URI scheme. The main topic of discussion is how to handle backward compatibility. One suggestion is to use a version number, but this is deemed unsatisfactory due to the potential for old clients ignoring new variables. Instead, new variables will be added either as-is or with a mustimplement: prefix, and if a client does not know what the variable is after mustimplement:, it should consider the entire URI invalid. However, some are still concerned that depending on variable names for information is not ideal. Another point of discussion is whether to use satoshis instead of decimal bitcoin, but this is considered problematic since people working with COIN scale may end up confusing the two. Finally, there is a brief discussion about using doubles or floats for money, with some claiming that lots of people do.
