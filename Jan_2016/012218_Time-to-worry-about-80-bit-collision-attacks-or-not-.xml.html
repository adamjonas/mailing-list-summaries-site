    Summary:

    In this communication, Gavin Andresen is requesting clarification on a collision attack scenario involving a payment channel. The attacker creates two scripts with different values that hash to the same result and can use them to make payments without alerting the victim. Ethan Heilman provides a procedure for creating such scripts using a function of the attacker's private key. Using a weakened hash function, Heilman demonstrates how to find a "useful" collision in under 15*2**80 work.
