Time to worry about 80-bit collision attacks or not?

    Summary:

    The discussion is about using the RIPEMD160(SHA256()) hash function as a possible solution for segregated witness (segwit) in Bitcoin. The concern raised by Pieter is that collision attacks could be used to steal coins. Gavin argues that contract wallets can protect against this type of attack, and that adding an extra 12 bytes to the segwit to prevent an attack that takes 2^80 computation and storage is unlikely to be a problem in practice and is a wrong tradeoff to make. The general question raised is whether we should be worried about collision attacks against RIPEMD160, which has 160-bit hash. A successful brute-force collision attack would require at least O(2^80) CPU, which is feasible, but also requires O(2^80) storage, which is infeasible. Even assuming doubling every year, we are four decades away from an attacker with the entire world's storage capacity being able to mount a collision attack.
