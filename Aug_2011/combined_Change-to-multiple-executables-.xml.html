Summary:

In August 2011, there were discussions about the mainline client of Bitcoin and its single executable function. John Smith proposed a split into three separate executables: bitcoin daemon, bitcoin GUI executable, and bitcoin RPC command line. Pieter offered some reformulations and suggested that the GUI and RPC should not be split, but instead made optional for GUI users. Matt Corallo agreed with the idea of splitting up the source into separate library-ish components but disagreed on the idea of creating more executables. He suggested that if anyone wanted to distribute bitcoin-qt built off of bitcoind, they should pull-request it. The discussions touched upon issues such as the confusion of double-functioning as an RPC call and whether or not backward incompatible changes to the protocol would be worth the hassle. Finally, Pieter suggested creating a dev branch to make progress on internal changes.
