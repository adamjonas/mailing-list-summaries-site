    Summary:

    Pieter has created a pull request for BIP 32, which is a rewritten version of it to make it more readable. The changes made include not reusing the terminology 'public' vs 'private' for the alternate derivation scheme; instead, calling them "hardened". Additionally, explicit type conversion functions were added, and the ' suffix was not used to mean "alternate" in two different meanings. Instead, the 'h' suffix was utilized to mean hardened, and CKDpriv and CKDpub were used for the derivation functions that operate on private and public keys. Pieter clarifies that these are just documentation changes and that the semantics are unchanged. He welcomes comments, which can be found at https://github.com/bitcoin/bips/pull/12.
