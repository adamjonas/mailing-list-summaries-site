    Summary:

    OpenSSL 1.0.0p/1.0.1k has been released, but it is incompatible with the Bitcoin system and could lead to consensus forks. The incompatibility is due to the OpenSSL update changing the behavior of ECDSA validation, which can constitute a security vulnerability for the Bitcoin system. If you are running third-party or self-compiled Bitcoin Core or an alternative implementation using OpenSSL, you must not update OpenSSL or must run a Bitcoin software containing a workaround. The patches above only fix one symptom of the general problem, relying on software not designed or distributed for consensus use (in particular OpenSSL) for consensus-normative behavior. Therefore, as an incremental improvement, a targeted soft-fork to enforce strict DER compliance soon, utilizing a subset of BIP62, is proposed. Adding a blockchain rule for strict DER will reduce the risk of consensus inconsistencies from alternative implementations of signature parsing or signature verification, simplify BIP62, and better isolate the cryptographic validation code from the consensus algorithm.
