    Summary:

    The conversation is about atomic cross-chain trading between the Bitcoin and Dogecoin blockchains. Ross Nicoll has developed a mostly functional prototype, but there is an issue where if the refund transaction is relayed before the actual spend transaction, it blocks the legitimate spend transaction from being accepted into the memory pool. Gregory Maxwell suggests that this is not the case unless there is a serious bug that he is not aware of since the unlocked transaction is not relayable and will not be mempooled until right before it locks. Ross Nicoll has drafted a patch to solve the problem but has not yet raised a PR because historically, it has led to a lot of discussion in Github which is better suited to the mailing list. He continues to test the patch and comments are welcomed. However, according to Gregory Maxwell, the behavior on testnet is faulty because the non-mempooling is enforced by IsStandardTx, which is bypassed in testnet. They should enforce it elsewhere, but this is not the case on the real network. Additionally, Peter's patch for testnet is expected to fix the issue. Ross Nicoll thanks Gregory Maxwell for his suggestion and for presenting the solution as code, which is a useful way to communicate. Finally, there is a concern that Ross Nicoll's patch has no protection against denial of service since a single user can rapidly update their transaction and exhaust the relay bandwidth of the entire network.
